## 1. Purpose & scope

F0 stands up the empty Ensemble Go module so every later piece has a place to land and a
green build/CI from commit one. It creates `go.mod` (module path
`gitlab.rand0m.me/ruben/go/ensemble`, `go 1.26`, deps pinned exactly per A.11 + `go-mp3`),
the full `internal/` + `cmd/` + `web/` + `data/` + `scripts/` directory skeleton from
README §5 (each leaf package given a compile-clean placeholder so `go build ./...` and
`go vet ./...` pass before any real code exists), the four helper scripts
(`build`, `build-arm64`, `dev`, `make-test-clip`) re-homed from `../media/scripts`, the
toolchain-free GitLab pipeline `.gitlab-ci.yml` verbatim from doc 11 §5.2, and
`config.example.yaml` describing the operator config surface (ports/mDNS/audio defaults
from README §4 + doc 01 §5.1, with A.12 default ports). F0 writes **no** runtime logic —
it is pure scaffolding and the build/CI contract; it does **not** copy any algorithm code
(that is the job of the per-package pieces).

## 4. Public API

F0 exposes essentially no Go API (scaffolding). The two concrete declarations:

```go
// cmd/ensemble/main.go
package main
var version = "0.0.0-dev" // stamped via -ldflags "-X main.version=..."  (doc 11 §2)
func main()

// cmd/ensemble/run.go
func cmdRun(args []string) error // F0: parse flags (doc 01 §5.1), print resolved config
```

Each `internal/.../doc.go` is just `package <name>` with the responsibility comment from
the doc 01 §2.1 table. The canonical contracts (README §6: `AudioSink`, `ClockSource`,
`Timeline`, `Codec`, `FEC`, `ConfigDoc`, `web.Deps`, A.14 `PeerStore` etc.) are declared by
their **owning pieces**, not by F0 — F0 must not invent or pre-declare them.

- `mewkiz/flac` — FLAC **source decode** only (A.11; pure-Go).

writing `go.mod`, run `go mod tidy` (network) to generate `go.sum`.

## 5. Behavior & algorithms

F0 contains no runtime algorithm. The load-bearing build behaviors it must encode:

- **Pure-Go / `CGO_ENABLED=0` everywhere** (D24, doc 11 §1): all scripts and CI set it; no
  cgo, no build tags, no `-tags alsa`. Cross-compile to `linux/arm64` with no C toolchain.
- **go:embed ordering** (doc 11 §2): packages embed `internal/web/dist`, so the placeholder
  `internal/web/dist/index.html` MUST exist for `go build`/`vet`/`test` to compile before any
  Node build. F0 commits that placeholder and the `.gitignore` keeps it
  (`!internal/web/dist/index.html`).
- **Version stamping** (doc 11 §2): `-ldflags "-s -w -X main.version=<ver>"`; `-trimpath` for
  reproducibility.
- **`config.example.yaml`** documents the operator surface with A.12 defaults verbatim:
  control mTLS **8443**, clock UDP **9000**, audio UDP **9100** (A.12 Ports row); audio
  canonical **48000 / 2 / 480 (10 ms)** (A.12 Audio canonical) referenced as read-only
  context. It mirrors the doc 01 §5.1 flag set (flags override file). It must NOT contain
  PIN, admin password, or channel/gain/HWDelayUs (those are ConfigDoc/identity state, doc 01
  §5.1).
- **`.gitlab-ci.yml`** is reproduced **verbatim** from doc 11 §5.2: stages
  `[frontend, test, build, release]`; `frontend` (node:22-alpine, `npm ci && npm run build` →
  `internal/web/dist` artifact); `test` (golang:1.26, `go vet ./...`, `go test ./...`,
  `GOOS=linux GOARCH=arm64 go build ./...`); `build` matrix
  `{linux/amd64, linux/arm64, linux/arm GOARM=7}`; `release` on tag. `CGO_ENABLED: "0"` in
  global `variables`.

## 6. Import constraints

F0 creates the package directories that the doc 01 §2.2 layering rules govern but writes no
imports inside them (the `doc.go` files import nothing). Constraints F0 must respect so the
skeleton can later be filled without rework:
- `cmd/ensemble` is the only package allowed to import the full set (doc 01 rule 6); F0's
  `main.go`/`run.go` stub imports only stdlib (`flag`, `fmt`, `os`).
- The empty `internal/web` package must remain free of `group`/`stream/*`/`audio/*` imports
  (doc 01 §2.2 rule 1, the `Deps` seam). F0's `web/doc.go` imports nothing, preserving this.
- `state`, `clock`, `config` are leaves (rules 2–4); their `doc.go` imports nothing.

## 7. Test plan

F0's correctness IS the build/CI contract; there is little to unit-test, but the gates must
be proven locally and in CI.

| Check | Command | Asserts |
|---|---|---|
| Module resolves | `go mod download && go mod verify` | go.mod parses; all A.11 deps + go-mp3 fetch; `go.sum` consistent |
| Compiles | `CGO_ENABLED=0 go build ./...` | every placeholder package + `cmd/ensemble` compile; go:embed of `internal/web/dist` succeeds (placeholder present) |
| Vet clean | `go vet ./...` | no vet errors (CI `test` stage gate) |
| Tests pass | `go test ./...` | trivially green (no tests yet); proves test harness wiring |
| arm64 cross-build | `CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go build ./...` | D24 / doc 11 §3: cross-compiles with no C toolchain |
| Binary builds + stamps | `scripts/build.sh` then `bin/ensemble version` | prints the `-X main.version` value; `scripts/build-arm64.sh` emits an arm64 ELF (`file bin/ensemble-arm64`) |
| Run stub | `bin/ensemble run --data ./tmp` | parses flags, prints resolved config (ports default 8443/9000/9100), exits 0 |
| Scripts executable | `test -x scripts/build.sh …` | exec bits set |
| Test-clip generator | `scripts/make-test-clip.sh /tmp/clip.flac 5` | emits a 5 s 48 kHz stereo FLAC with the A.10b click+tone pattern (audio-only, no video stream — `ffprobe` shows 0 video streams) |

One tiny table-driven unit test belongs to F0 to anchor the test harness:
- `cmd/ensemble/run_test.go` — `TestFlagDefaults`: table of `{args, wantWebPort, wantClockPort, wantAudioPort, wantData}` asserting the doc 01 §5.1 defaults (8443/9000/9100, `./data`) and that explicit flags override them.

**Acceptance (A.13 P0 skeleton):** "cmd builds, data dir + identity persist" → for F0 the
**build half** is in scope: `go build` succeeds and produces a runnable binary, arm64 cross
builds clean, CI pipeline is green. (The "identity persists across restart" half is delivered
by the `internal/config` piece, F-config, which F0 unblocks.)

## 8. Dependencies

**Consumes (upstream pieces):** none. F0 is the root piece — nothing precedes it.

**Exposes for downstream pieces (everything else depends on F0):**
- The Go module + pinned dep set (`go.mod`/`go.sum`) — every Go piece builds against it.
- The package directory skeleton — each owning piece fills its `internal/...` dir.
- `cmd/ensemble/main.go` subcommand dispatch + `version` var — the wiring piece extends
  `cmdRun`; `pki`/`group`/`web` pieces hang off the daemon it launches.
- `cmd/ensemble/run.go` flag set (doc 01 §5.1) — `config` piece consumes the parsed flags;
  socket-binding pieces (`web`, `clock`, `stream`) consume `--web-port/--clock-port/--audio-port`.
- `scripts/*` + `.gitlab-ci.yml` — the build/CI contract every piece's tests run under.
- `config.example.yaml` — the `config` piece's parser target (it must round-trip this file).
- `internal/web/dist/index.html` placeholder — unblocks the `web` embed piece and any
  `go build` before the frontend exists.

Immediate downstream: **F-config** (`internal/config`: `Paths`, `Identity`,
`OpenDataDir`, `config.yaml` parse) is the next piece and the second half of A.13 P0.

## 9. Risks / open questions

- **Q1 (BLOCKING for `go mod tidy`): `go-mp3` version pin.** A.11 names
  `github.com/hajimehoshi/go-mp3` but pins no version, and it is absent from `media/go.sum`,
  so there is no in-repo source of truth. Proposed `v0.3.4` (latest stable as of the cutoff);
  the orchestrator must confirm the exact tag (or let `go mod tidy` select latest and record
  it). Must be CGO-free at that version (it is, but verify) to preserve D24.
- **Q2 (non-blocking): commit `go.sum`?** It cannot be hand-authored; it requires a network
  `go mod tidy`/`download`. If CI/dev has no network at module-init time, F0 must run tidy in
  a connected environment and commit the resulting `go.sum`. Flag whether the build env is
  online.
- **Q3 (non-blocking): `analyze-sync.py` + ffmpeg dependency.** `make-test-clip.sh` and the
  analyzer require `ffmpeg`/Python+numpy at *dev* time only (never runtime/build). Confirm
  these are acceptable dev-host tools (they are not in doc 11 §1 build prereqs, and must not
  be — they are P4 test aids only).
- **Q4 (non-blocking): `cluster.yaml` vs `cluster.json`.** media's `.gitignore` lists
  `cluster.json`; doc 01 §5.2 specifies `cluster.yaml`. F0 follows the doc (`cluster.yaml`);
  noting the divergence so the `config` piece stays consistent.
- **Risk: empty-package vet noise.** A `doc.go` with only a package clause is fine; but if any
  later-added placeholder imports an unused dep, `go vet`/build breaks CI. Keep `doc.go` files
  import-free until their owning piece adds real code.
