# P0.1 — `internal/config` (Paths + Identity + config.yaml)

> Track: **BACKEND (Go)**. Module `gitlab.rand0m.me/ruben/go/ensemble`, Go 1.26,
> `CGO_ENABLED=0`, cross-compiles to `linux/arm64` with no C toolchain (D24).
> This piece is the **pure leaf** of the import DAG (doc 01 §2.1/§2.3): filesystem
> + data types only, no networking, imports nothing internal.

---

## 1. Purpose & scope

`internal/config` owns the node's on-disk **data directory**: it resolves and creates
the canonical data-dir layout (doc 01 §5.2 — `config.yaml`, `node.json`, `cluster.yaml`,
`certs/`, `doc.json`, `data/`, `run/`), loads/saves the **stable node `Identity`** (node
id that survives restart, name, `HWDelayUs`, audio device), and parses the per-node
**`config.yaml`** including the ports, mDNS, and the **audio capability-masking** keys
(doc 07 §2.4.2: `audio.render`, `audio.backends.{disable,prefer}`, `audio.codecs.disable`).
It is the pure data + filesystem leaf (doc 01 §2.1) that `pki`, `auth`, `discovery`,
`cluster`, `state`, `group`, `stream/source`, `audio/render`, `web`, and `cmd` all read;
it imports no other internal package. It does **not** own the replicated `ConfigDoc`
(that is `internal/state`, P-state) — it only resolves `doc.json`'s path.

    type AudioConfig struct {
        Render   *bool          `yaml:"render"`   // nil => probe-driven; false => force sink-less (control-only)
        Backends BackendsConfig `yaml:"backends"`
        Codecs   CodecsConfig   `yaml:"codecs"`
        Device   string         `yaml:"device"`   // optional default sink device; --device overrides
    }
    type BackendsConfig struct {
        Disable []string `yaml:"disable"` // probed backends to never use (e.g. ["pipewire"])
        Prefer  []string `yaml:"prefer"`  // ordered preference among remaining (Open(preferred,…), README §6.1)

    > "Force `Render=false` … regardless of probe; `Sinks` emptied"); `true` = allow
    > render if a backend probes. These keys are consumed at startup by the effective-caps
    > computation (doc 07 §2.4.1) which lives in `audio/sink`+`group`, **not** here —
    > `config` only parses/validates them.

## 4. Public API (exact Go signatures)

```go
package config

// ---- paths.go ----

const DefaultDataDir = "./data"

type Paths struct {
    Root       string // <data>
    Config     string // <data>/config.yaml
    NodeConfig string // <data>/node.json   (Identity, 0644)
    Cluster    string // <data>/cluster.yaml (0600)
    Doc        string // <data>/doc.json    (persisted ConfigDoc, 0644)
    Peers      string // <data>/peers.json  (gossip seed cache, 0644, A.14.1)
    Certs      string // <data>/certs/      (0700)
    CACert     string // <data>/certs/ca.crt
    NodeKey    string // <data>/certs/node.key (0600)
    NodeCert   string // <data>/certs/node.crt
    Data       string // <data>/data/       (media folder)
    Run        string // <data>/run/
}

func OpenDataDir(dir string) (Paths, error)

// ---- identity.go ----

type Identity struct {
    NodeID    string `json:"node_id"`             // stable; generated once; election tiebreak
    Name      string `json:"name,omitempty"`      // editable friendly name
    HWDelayUs int    `json:"hw_delay_us,omitempty"` // per-node output latency trim, µs (D13)
    Device    string `json:"device,omitempty"`    // audio sink device (e.g. "hw:0"); "" => sink default
}

func LoadOrCreateIdentity(paths Paths) (Identity, error)
func SaveIdentity(paths Paths, id Identity) error

// ---- config.go ----

const DefaultWebPort = 8443 // control mTLS base port (A.12)

type Config struct {
    NodeName string        `yaml:"node_name"`
    Cluster  ClusterConfig `yaml:"cluster"`
    Web      WebConfig     `yaml:"web"`
    Audio    AudioConfig   `yaml:"audio"`
}

type ClusterConfig struct {
    BindPort  int      `yaml:"bind_port"`  // gossip (memberlist), default 7946
    ClockPort int      `yaml:"clock_port"` // clock UDP, default 9000
    AudioPort int      `yaml:"audio_port"` // audio UDP, default 9100
    MDNS      bool     `yaml:"mdns"`       // default true
    Seeds     []string `yaml:"seeds"`      // explicit gossip seeds host:port
}

type WebConfig struct {
    Listen bool `yaml:"listen"` // default true (wizard surface always serves)
    Port   int  `yaml:"port"`   // control mTLS base, default 8443, +1-on-conflict at bind
}

type AudioConfig struct {
    Render   *bool          `yaml:"render"`
    Backends BackendsConfig `yaml:"backends"`
    Codecs   CodecsConfig   `yaml:"codecs"`
    Device   string         `yaml:"device"`
}
type BackendsConfig struct {
    Disable []string `yaml:"disable"`
    Prefer  []string `yaml:"prefer"`
}
type CodecsConfig struct {
    Disable []string `yaml:"disable"`
}

func Defaults() Config
func Load(path string, required bool) (Config, error)
```

> **Canonical-name note.** README §6.5 / doc 07 use `hwDelayUs` (camelCase) in the
> replicated `ConfigDoc.NodeRecord`; the *local persisted* `Identity` uses snake_case
> JSON (`hw_delay_us`) to match the mpvsync `node.json` convention (doc 07 §2: "matching
> mpvsync convention"). The Go field is `HWDelayUs` in both; mapping `Identity →
> NodeRecord` is done by `cmd`/`group`, not here.

## 5. Behavior & algorithms (cited, with verbatim A.12 numbers)

- **Data-dir layout** is doc 01 §5.2 exactly. `OpenDataDir` resolves `--data` to an
  absolute path and creates `{Root, Certs, Data, Run}`. Per doc 01 §5.2: `node.key` and
  `cluster.yaml` are mode **0600** (written by their owners — `pki`, `state`/`cmd`); the
  `certs/` directory itself is created **0700** here. `node.json` and `doc.json` are 0644.
- **Node id stability** (doc 01 §5.2, A.13 P0): "The node id in `node.json` is generated
  once and **must be stable** — it is the election tiebreak (a fresh id each boot would
  reshuffle the master)." `LoadOrCreateIdentity` returns the existing id if present and
  non-empty; only generates (128-bit random, 32-char hex) on first run / empty id.
- **Acceptance P0** (A.13): "`go build`, **restart keeps node id**." The identity tests
  assert a second `LoadOrCreateIdentity` returns the same `NodeID`.
- **Config precedence** (doc 01 §5.1): defaults → `config.yaml` → set flags (`fs.Visit`).
  `config` provides defaults + YAML; flag overlay happens in `cmd` (F0). A missing
  `config.yaml` is non-fatal unless the path came from an explicit `--config` (`required`).
- **Default ports — verbatim A.12 ("Ports | control mTLS / clock UDP / audio UDP |
  8443 / 9000 / 9100")** and doc 01 §5.1 (`--bind-port` default `7946`):
  - `Web.Port = 8443`, `Web.Listen = true`
  - `Cluster.BindPort = 7946`, `Cluster.ClockPort = 9000`, `Cluster.AudioPort = 9100`
  - `Cluster.MDNS = true`
- **Audio masking keys** (doc 07 §2.4.2) — `config` parses and validates names but applies
  **no** intersection; the effective-caps computation (`detected ∩ enabled`, doc 07
  §2.4.1) is owned by `audio/sink` + `group` at startup. `normalize` validates that any
  `audio.backends.disable/prefer` names are syntactically plausible backend tokens
  (`"alsa"`, `"exec:aplay"`, `"pipewire"`, README §6.1) and `audio.codecs.disable` names
  are in `{"pcm","opus"}` (README §6.5), erroring on unknown tokens so a typo cannot
  silently fail to mask. (PCM is the mandatory baseline, D3 — masking it is allowed in
  config but the negotiator still falls back to PCM; not enforced here.)
- **`normalize`** runs after overlay and backfills any port/bool zeroed by a partial
  YAML block (same rationale as mpvsync `normalize`): if `Web.Port <= 0` → 8443; if
  `Cluster.BindPort/ClockPort/AudioPort <= 0` → 7946/9000/9100. `MDNS`/`Listen` default
  via `Defaults()` (a YAML that explicitly sets `false` is honored — booleans are not
  backfilled).
- **No tunable invented.** Every number above is quoted from A.12 / doc 01 §5.1. The drift
  PI gains, FEC params, `LeadMs`, clock windows, etc. are **not** in `config.yaml` for the
  MVP — they live in A.12 as code constants consumed by their owning packages (R-note in
  README §6.4: "Appendix A.12 is the single source of truth for all tunables").

## 6. Import constraints (doc 01 §2.1/§2.2)

- **Allowed internal imports: none.** `config` is the pure leaf (doc 01 §2.1 row 1:
  "May import (internal): *(none)*"; §2.3 graph: bottom leaf). It must not import
  `state`, `cluster`, `pki`, `audio/*`, etc. — they all import *it*. Making it depend on
  any of them would create a cycle (doc 01 §2.2 rule 3).
- **Allowed external:** stdlib (`os`, `path/filepath`, `crypto/rand`, `encoding/hex`,
  `encoding/json`, `fmt`, `strings`) and `gopkg.in/yaml.v3` (A.11 pinned YAML dep).
- **No networking** (doc 01 §2.1: "Pure data + filesystem; no networking").

All unit tests are table-driven, using `t.TempDir()` for the data dir (no fixtures on the
real FS). No integration test is required for P0.1 (it is a leaf); its behavior is covered
by the P0 skeleton acceptance (A.13).

### `paths_test.go`
| Case | Asserts |
|---|---|
| `OpenDataDir("./relative")` | `Paths.Root` is absolute; equals `filepath.Abs`. |
| layout fields | every `Paths` field equals `filepath.Join(Root, expected)` per §4. |
| dirs created | `Root`, `Certs`, `Data`, `Run` exist after the call; leaf files (`node.json`, `doc.json`, `ca.crt`) do **not**. |
| `certs/` perm | `Certs` dir mode is `0700` (mask-checked; skip on non-POSIX via `runtime.GOOS`). |
| idempotent | calling `OpenDataDir` twice on the same dir is a no-op success (no error on existing dirs). |
| bad path | a path under a read-only parent returns a wrapped `create … :` error. |

### `identity_test.go`
| Case | Asserts |
|---|---|
| first run | `LoadOrCreateIdentity` on an empty dir writes `node.json`, returns a 32-char hex `NodeID`. |
| **stability (A.13 P0)** | a second call returns the **same** `NodeID`; file unchanged. |
| empty-id regen | a `node.json` with `"node_id":""` is rewritten with a fresh id, other fields (`name`) preserved. |
| corrupt json | invalid JSON returns a wrapped `parse … :` error (does not silently overwrite). |
| `SaveIdentity` round-trip | save `{NodeID,Name,HWDelayUs:4200,Device:"hw:0"}`, reload, deep-equal; `node.json` mode `0644`; atomic (`.tmp` gone). |
| omitempty | zero `HWDelayUs`/empty `Device`/`Name` are absent from the marshalled JSON. |

### `config_test.go`
| Case | Asserts |
|---|---|
| `Defaults()` | ports = `8443/7946/9000/9100`, `MDNS=true`, `Web.Listen=true`, `Audio.Render==nil`. |
| missing file, `required=false` | returns `Defaults()`, no error. |
| missing file, `required=true` | returns a wrapped `read config … :` error. |
| full YAML overlay | every field from a complete YAML round-trips into `Config`. |
| **partial block backfill** | YAML setting only `cluster.bind_port: 7000` keeps `clock_port=9000`, `audio_port=9100` (normalize backfills zeroed siblings). |
| `web.listen: false` honored | explicit `false` is **not** backfilled to `true`. |
| audio masking parse | `audio.{render:false, backends.disable:[pipewire], backends.prefer:[alsa,exec:aplay], codecs.disable:[opus]}` parses into the structs; `Render` is non-nil `false`. |
| `render` tri-state | absent `render` → `Render==nil`; `render:true` → non-nil `true`; `render:false` → non-nil `false`. |
| unknown backend token | `audio.backends.disable:[bogus]` → wrapped validation error from `normalize`. |
| unknown codec token | `audio.codecs.disable:[mp3]` → wrapped validation error (`mp3` is a *source* decoder, not a wire codec, R2). |
| malformed YAML | returns a wrapped `parse config … :` error. |

**Acceptance (A.13 P0 + doc 01 §5.2):** `go build ./...` succeeds; a node's id persists
across a simulated restart (two `LoadOrCreateIdentity` calls); `OpenDataDir` produces the
exact doc 01 §5.2 layout with correct perms.

## 8. Dependencies

**Consumes (upstream): F0** — the module/skeleton bootstrap. P0.1 assumes F0 has created
`go.mod` (module `gitlab.rand0m.me/ruben/go/ensemble`, Go 1.26) and added the
`gopkg.in/yaml.v3` require (A.11). If F0 does not add `yaml.v3`, P0.1 must add it. P0.1
needs **no exported symbol** from F0 beyond the module path existing — it is the pure leaf.

**Exposes (downstream): every package depends on `internal/config`.** The load-bearing
symbols other pieces consume:
- `config.Paths` + `config.OpenDataDir` — consumed by `cmd` (wiring), `pki` (cert paths
  `Certs/CACert/NodeKey/NodeCert`), `state` (`Doc` path for the persisted `ConfigDoc`,
  doc 07 §5.4), `cluster` (`Peers` path, A.14.1).
- `config.Identity` + `LoadOrCreateIdentity`/`SaveIdentity` — consumed by `cmd`
  (`NodeID` → memberlist `Name` / election tiebreak; `Name`→`NodeRecord.Name`;
  `HWDelayUs`→`audio/render`; `Device`→`audio/sink`).
- `config.Config` (and `ClusterConfig`/`WebConfig`/`AudioConfig`) — consumed by `cmd`
  (port binding for all three planes; flag overlay) and the startup effective-caps
  computation in `audio/sink`+`group` (the `Audio.*` masking keys, doc 07 §2.4.1).

This piece does **not** consume `internal/state` (`ConfigDoc`) — it only resolves the
`doc.json` path; the `ConfigDoc` type and its `Store` are P-state's deliverable.

## 9. Risks / open questions

1. **`HWDelayUs` lives in two places.** It is in the local `Identity` (`node.json`) **and**
   in the replicated `NodeRecord` (`ConfigDoc`, README §6.5 / doc 07 §2.4). Doc 01 §5.2
   lists it in `node.json`; doc 07 makes the `NodeRecord` authoritative & replicated.
   **Resolution adopted here:** `node.json` is the *local* persisted hint (survives a
   pre-cluster boot and is the seed when the node first writes its own `NodeRecord`); the
   `ConfigDoc` is authoritative once adopted. The `cmd`/`group` layer reconciles them
   (write-back on change). **Open Q for the orchestrator:** confirm the direction of truth
   (local→doc on boot, doc→local on merge) so P0.1 and P-state/P-group agree; P0.1 only
   stores the field and does not reconcile.
2. **`Device` overlap.** `Identity.Device` (node.json) and `AudioConfig.Device`
   (config.yaml) both name the sink device, and `--device` (doc 01 §5.1) overrides. The
   precedence (`--device` > `config.yaml` > `node.json`) must be fixed in `cmd` (F0); P0.1
   simply exposes both. Flagged so F0 doesn't pick a different order.
3. **`Peers` path ownership.** A.14.1 puts `peers.json` logic in `internal/cluster`, but
   the *path* is resolved in `Paths` here for layout consistency. No behavior risk; noted
   so `cluster` (P-cluster) reads `Paths.Peers` rather than re-deriving it.
4. **No `go.mod` yet.** The ensemble repo currently has only `README.md` + `doc/`. P0.1
   cannot compile until F0 lands `go.mod`. If F0 is not scheduled before P0.1, P0.1's
   first commit should also create `go.mod` (module path + Go 1.26 + `yaml.v3`). The
   orchestrator must confirm F0 owns this, else P0.1 absorbs it.
