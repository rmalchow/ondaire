## 1. Purpose & scope

Stand up `internal/web` as the control-plane HTTP/WebSocket server skeleton: a
`Server` constructed from a **`web.Deps` function-value struct** ([A.14.3](../A-appendix-algorithms-and-pinned-choices.md)),
serving the **`go:embed`-ed** Svelte build under `internal/web/dist`, a `/healthz`
liveness probe, the SPA-fallback asset handler, and a **3 Hz WebSocket push hub**
(`coder/websocket`). The defining net-new addition versus mpvsync is the **TLS-listener
seam**: the server accepts a *pre-bound* `net.Listener` from `cmd` and wraps it in an
mTLS `tls.Config` supplied through `Deps` (built by `pki` in a later piece), per
[01 §3.1](../01-architecture-and-packages.md). This piece ships **no API handlers and no
engine logic** — it must not import `group`, `stream/*`, or `audio/*` ([01 §2 rule 1](../01-architecture-and-packages.md));
it only proves the seam, the embed, the listener wrap, `/healthz`, and the WS hub
compile, run, and serve on the reference Pi-class hardware.

## 4. Public API (canonical names — A.14.3 + README §6)

```go
package web

// Server hosts the control-plane HTTP/1.1 + WebSocket surface over (m)TLS.
// It reads node state and triggers operations only through Deps (no engine import).
type Server struct { /* deps, devDir, mux, hub (unexported) */ }

// New builds a Server. devDir!="" serves assets from disk (on-device iteration);
// else from the embedded dist build.
func New(deps Deps, devDir string) *Server

// Serve runs on a PRE-BOUND listener until ctx is cancelled, then graceful-shuts.
// cmd owns port selection (so it can advertise the bound port before anything
// else starts). If deps.TLSConfig != nil the listener is wrapped via tls.NewListener.
func (s *Server) Serve(ctx context.Context, ln net.Listener) error
```

```go
// Deps is the function-value seam (A.14.3). web depends only on this struct of
// closures + view types; cmd supplies implementations bound to cluster/group/
// state/pki. web imports NONE of group, stream/*, audio/* (01 §2 rule 1).
type Deps struct {
    // --- transport (net-new TLS seam, 01 §3.1) ---
    TLSConfig func() *tls.Config            // mTLS server config from pki; nil => plain TLS-less serve (dev)

    // --- read-mostly snapshots (A.14.3) ---
    State      func() ConfigView            // current replicated ConfigDoc (web view projection)
    Transcodes func() []TranscodeStatus     // per-group stream/transcode status snapshots
    Discovery  func() []Discovered          // discovered-but-unadopted nodes

    // --- cluster mutations (proxied node->node over mTLS as needed) ---
    Adopt  func(addr, pin string) error     // run A.9 adoption against a target
    Forget func(nodeID string) error        // revoke + drop from ConfigDoc
    Leave  func() error                      // coordinated self-forget (POST /cluster/leave)

    // --- node/group config ---
    SetNodeConfig func(nodeID string, patch NodePatch) error // channel/gain/HWDelayUs/name

    // --- calibration ---
    CalibratePlay func(sel CalibrateSel, durationSec int) error // play A.10b signal synchronously

    // --- liveness/UI plumbing (skeleton needs these; not in A.14.3 verbatim) ---
    Changed     func() <-chan struct{}      // coalesced change signal -> immediate WS push; nil-safe
    Status      func() NodeStatus           // this node's role/sync snapshot; nil => zero value
    SetupStatus func() SetupStatus          // GET /api/v1/setup gate; nil => {Configured:false}
    NodeID      string                      // this node's stable id (P0.1 Identity)
    Paths       config.Paths                // resolved data-dir layout (P0.1)
}
```

## 5. Behavior & algorithms

- **TLS-listener seam** ([01 §3.1](../01-architecture-and-packages.md)): `cmd` pre-binds a
  plain `net.Listener` (so it knows the bound port before mDNS/Meta advertise it — the
  mpvsync `listenWeb` +1-on-conflict pattern), then calls `Serve(ctx, ln)`. `Serve` wraps
  it: `ln = tls.NewListener(ln, deps.TLSConfig())`. The `tls.Config` (built by `pki` in a
  later piece) sets `MinVersion: tls.VersionTLS13`, `ClientAuth: tls.VerifyClientCertIfGiven`,
  `ClientCAs`/`RootCAs` = cluster CA, leaf = node cert. **P0.2 does not build the
  `tls.Config`** — it only consumes `deps.TLSConfig`; when nil (dev/test) it serves the
  bare listener so the skeleton is runnable before `pki` lands.
- **Embed** ([README §5](../README.md)): `//go:embed all:dist`; `assetHandler` does
  `fs.Sub(distFS,"dist")`, SPA-fallback to `index.html` for unknown non-`/api/`,
  non-`/ws` paths; `/api/` + `/ws` that fall through return `404` (their real handlers
  arrive in later pieces).
- **`/healthz`**: `GET /healthz` → `200 "ok"`. Unauthenticated, no TLS-client-cert
  requirement (it is a liveness probe, served below the auth middleware that later pieces
  add). Not under `/api/v1`, so the §6.6 auth rules do not gate it.
- **WebSocket push hub** (`ws.go`, dep `github.com/coder/websocket` pinned in
  [A.11](../A-appendix-algorithms-and-pinned-choices.md)): one `hub` fans `StateSnapshot`
  to all conns; per-conn buffered `send` channel (cap 8) with **drop-on-full** so a slow
  client cannot stall the hub. `runHub` ticks at **3 Hz (333 ms)** (copied from mpvsync;
  not an A.12 tunable) and also pushes immediately on `deps.Changed()`. `handleWS` uses
  `OriginPatterns:["*"]` (LAN appliance, no cookie ambient authority — keep the rationale
  comment). `wsWriter` writes with a 5 s deadline and pings every 15 s. A fresh client
  gets a snapshot on connect.
- **Snapshot** `BuildSnapshot()`: assembles `StateSnapshot` from `deps.State()`,
  `deps.Status()`, `deps.Transcodes()`, `deps.NodeID`. Every `deps.*` read is **nil-safe**
  (return zero values) so the server runs with a partially-wired `Deps` in tests and
  during early bring-up.
- **No A.12 numbers are load-bearing in this piece.** Default control port `:8443` and
  the clock/audio ports ([A.12 "Ports … 8443 / 9000 / 9100"](../A-appendix-algorithms-and-pinned-choices.md))
  are owned by `cmd`'s flag parsing, not by `web`. `protocolEpoch = 1` (fixed this build)
  is surfaced by the bootstrap piece, not here.

## 6. Import constraints (01 §2)

Per [01 §2 table](../01-architecture-and-packages.md), `web` **may import (internal):**
`auth`, `pki`, `cluster`, `state`, `config`. In **this skeleton** it imports only:

- `gitlab.rand0m.me/ruben/go/ensemble/internal/config` (for `config.Paths` in `Deps`).
- stdlib: `context`, `crypto/tls`, `errors`, `io`, `io/fs`, `net`, `net/http`, `os`,
  `path`, `strings`, `sync`, `time`, `encoding/json`.
- third-party: `github.com/coder/websocket` (+ `/wsjson`).

It **MUST NOT import** `group`, `stream/*`, `audio/*` ([01 §2 rule 1](../01-architecture-and-packages.md)) — the
hard invariant. It does **not** import `auth`/`pki`/`cluster`/`state` yet (the `tls.Config`
and `ConfigView` arrive through `Deps` closures, not imports), keeping the skeleton's
compile-time graph minimal. A unit test (§7) greps the import set to enforce the
forbidden edges.

## 9. Risks / open questions

1. **`Deps.State` return type — `ConfigView` vs `state.ConfigDoc`.** A.14.3 literally writes
   `State func() ConfigDoc`. I propose a **web-owned `ConfigView` projection** (matching the
   mpvsync `TranscodeRow` "surface-but-don't-import" pattern, [01 §2 rule 1 note](../01-architecture-and-packages.md))
   so `web` stays decoupled from `state`'s concrete type. **Blocking-ish:** if the
   orchestrator wants the literal `state.ConfigDoc` (web is *permitted* to import `state`
   per [01 §2 table](../01-architecture-and-packages.md)), the seam type changes and the
   `cmd` adapter is dropped. **Recommend ConfigView; confirm.**
2. **Where does `listenControl` (port pre-bind + +1 retry) live?** mpvsync's `listenWeb`
   sits in `internal/node` (= ensemble `cmd`). I place it in `cmd`, not `web`, so `web`
   stays listener-agnostic. Confirm `cmd` owns it (this piece assumes so).
3. **`/healthz` auth posture.** Placed outside `/api/v1` and below auth, unauthenticated,
   no client-cert requirement — consistent with a liveness probe but [08](../08-http-api-reference.md)
   does not enumerate `/healthz`. Assumed intentional (probe endpoint); flag if it must be
   mTLS-gated.
4. **WS `OriginPatterns:["*"]` under mTLS control plane.** mpvsync justified `*` for a
   cookie-less LAN appliance. Ensemble's control plane is mTLS + admin-session; once the
   auth middleware lands (later piece) the WS upgrade should sit behind it. P0.2 keeps `*`
   (no auth yet) — revisit when `auth` middleware is wired so the WS endpoint is not an
   unauthenticated state firehose.
5. **3 Hz snapshot cadence is not an A.12 tunable** (copied from mpvsync). Fine for the Pi
   budget (tiny JSON), but if a future piece adds per-group sync telemetry to the snapshot
   it may warrant promotion to A.12. Non-blocking.
