## 1. Purpose & scope

This piece is the **executable entry point and daemon wiring** for an Ensemble node:
`package main` under `cmd/ensemble`, the subcommand dispatcher + `run` flag set (01 §5.1),
the version stamp, and the **wiring skeleton** that constructs the always-on subsystems
(`config`, `pki`, `auth`, `state`, the web listener) and builds the `web.Deps` function-value
seam (A.14.3) so `web` never imports the engine. It also lays down the **activate/deactivate
lifecycle skeleton** (01 §4.4): an unconfigured node serves only the wizard + mDNS; a
configured node `activate()`s the full session in-process and `deactivate()`/`forget()` returns
to unconfigured — **no restart**. P0.3 is *wiring only*: it defines the orchestrator type, its
lifecycle methods, and the seams, with the realtime subsystems (`group`, `clock`, `stream/*`,
`audio/*`, `cluster`) referenced through interfaces/closures and left as `TODO(P1+)` stubs so
the binary builds and the lifecycle is exercisable per acceptance A.13 row **P0** (`go build`,
restart keeps node id).

## 5. Behavior & algorithms (cite docs; A.12 numbers verbatim)

**B1 — Subcommand dispatch (main.go).** Mirror mpvsync `main`: `os.Args[1]` switch over
`run` / `version`(`-v`/`--version`) / `help`(`-h`/`--help`); unknown → usage to stderr, exit 2;
`cmdRun` error → `fmt.Fprintln(os.Stderr, "ensemble:", err)` + exit 1. `version` prints the
`version` const. (`protocolEpoch` is fixed at **1** for this build — see invariants; surfaced
later via `softwareVersion`/probe, not by P0.3.)

**B2 — Flag overlay precedence (run.go).** Built-in defaults → `config.yaml` → **only the
explicitly set flags** override, via `fs.Visit` populating `set[name]=true` then conditional
assignment (verbatim mpvsync pattern, 01 §5.1: "only the *set* flags override"). `--config`
when set is *required* (load error is fatal); unset → `<data>/config.yaml` optional.

**B3 — Identity & data dir (01 §5.2).** `config.OpenDataDir(--data)` resolves/creates the
layout; `config.LoadOrCreateIdentity` yields the **stable** node id (generated once, persisted
in `node.json`; a fresh id per boot would reshuffle election — same invariant as mpvsync).
Name precedence: explicit flag/config → persisted `id.Name` → `id.NodeID[:8]`.

**B4 — Listener pre-bind (01 §3.1).** `cmd`/`daemon` pre-binds the control TCP listener via
`listenWeb(base)` **before** mDNS/meta advertise, retrying base+1…base+63 on `EADDRINUSE`
(copied verbatim; 01 §5.3 multi-node-on-one-box depends on it), so the actual port is known to
advertise. The TLS wrap (mTLS `tls.Config` from `pki`) is applied by `web.Server.Serve`
(P1/P2 own the `tls.Config`; P0.3 hands the raw listener).

**B5 — Activation lifecycle (01 §4.4).** State machine:
`UNCONFIGURED` (idle: web wizard + mDNS announce only — *no* membership/clock/group/render) →
on configured-at-boot **or** `Configure()` → `activate()` (full session in-process, no restart)
→ on `forget()` → `deactivate()` → `UNCONFIGURED`. The **configured** predicate is
"has cluster CA + `cluster.yaml`" (P1 `pki.IsConfigured`), replacing mpvsync's group+password.
`activate` is idempotent: same group → no-op; different → `deactivate` then `activate`.
`Configure` **activates first, persists/advertises only on success** (so a failed activate
leaves the node cleanly UNCONFIGURED, never writing `cluster.yaml` — mpvsync's exact rationale).

**B6 — Role loop skeleton (A.14.4).** `loop(ctx)` runs per the active session; on each
membership/config tick it calls `applyRole(electedMaster, generation, streamGen)` which
idempotently starts/stops master-vs-follower subsystems **fenced by `generation`/`streamGen`**
so a superseded master cannot emit on the planes. In P0.3 the start/stop bodies are
`TODO(P3 clock+group / P4 sync)` stubs that only flip `Status.Role`; the **fence + cancel-old-
ctx-before-start-new** discipline is wired now (A.14.4) so the realtime pieces drop in without
restructuring. The generation fence and `streamGen` bump-on-master-change are governed by
A.5 (election) — not invented here.

**B7 — `web.Deps` seam (A.14.3, 01 §2 rule 1).** `buildDeps` returns a `web.Deps` whose
closures bind to the subsystems; `web` receives only function values + the `*state.Store`
pointer, never a `group`/`stream`/`audio` import. P0.3 wires `State`/`Discovery`/`Status`-class
reads to live data where available and stubs mutating closures (`Adopt`,`Forget`,`Leave`,
`SetNodeConfig`,`CalibratePlay`) to `errNotImplemented` until their owning pieces land.

**B8 — Tunable constants (A.12, quoted verbatim).** Default ports: **control mTLS `8443` /
clock UDP `9000` / audio UDP `9100`**; memberlist gossip **`7946`** (A.14.2). The canonical
audio rate/channels/`FramesPerChunk` = **48000 / 2 / 480 (10 ms)** and `LeadMs` = **300 ms**
are *referenced* by the engine pieces, not consumed by P0.3 wiring — P0.3 only passes
`ClockPort`/`AudioPort`/`Device` through `Options`. **Do not invent any other numbers.**

## 6. Import constraints (01 §2 layering)

- `cmd/ensemble` (rule 6): **may import the full set** — `internal/daemon`, `config`, plus
  whatever `daemon.Run` needs are reached *through* `internal/daemon`. `main`/`run` themselves
  import only `internal/{daemon,config}` + stdlib (`context`,`flag`,`fmt`,`io`,`os`,`os/signal`,
  `path/filepath`,`syscall`).
- `internal/daemon` (the orchestrator below `cmd`, rule 5/6): may import
  `config`, `pki`, `auth`, `state`, `web`, `cluster`, `discovery`, `clock`, `group`,
  `allowlist`, `stream/*`, `audio/*`. It is **not** imported by anyone except `cmd`.
- **Hard seam (rule 1, invariant):** `internal/web` must **not** import `group`, `stream/*`,
  or `audio/*`. P0.3 enforces this by only ever handing `web` a `web.Deps` value + `*state.Store`
  — verified by a test that greps web's import set (P0.2 owns the assertion; P0.3 must not
  regress it). `daemon` is the bridge; `web`→engine flows exclusively through `Deps` closures.
- No realtime package (`stream/*`,`audio/*`) imports `web` or `cmd`/`daemon` (rule 7).

## 7. Test plan

All table-driven (`go test ./cmd/ensemble/... ./internal/daemon/...`), pure-Go, no hardware.

**T1 `run_test.go` — flag/config precedence (B2).** Table of `{argv, configYAML, wantOpts}`:
- defaults only → ports `8443/9000/9100/7946`, mDNS on, data `./data`.
- config.yaml sets ports → reflected when no flag.
- flag overrides config (only the *set* flag; an unset flag does **not** clobber a config value).
- `--no-mdns` flips `UseMDNS=false`.
- `--join a --join b` accumulates `["a","b"]` (stringSlice).
- `--name` precedence: flag > config > `id.NodeID[:8]`.
  Asserts the resolved `daemon.Options` (factor the overlay into a pure
  `buildOptions(set, cfg, flags, id) Options` so it is testable without binding sockets).

**T2 `run_test.go` — dispatch (B1).** `version` prints `version`; unknown subcommand → exit-2
path (test via a dispatch func returning an exit code, not calling `os.Exit`).

**T3 `daemon_test.go` — lifecycle (B5, A.13 P0).** Table over a fake `pki.IsConfigured`:
- boot **unconfigured** → `Status.Configured=false,Active=false,Role="idle"`; web+mDNS only
  (assert no clock/group constructed: stub subsystem factories record calls).
- `Configure()` (fake CA present) → `activate` runs, `Status.Active=true`; **second** `Configure`
  same group → no-op (idempotent).
- `forget()` → `deactivate`, back to `Configured=false`; safe to call twice.
- `activate` failure path → `Configure` returns the wrapped error AND leaves node UNCONFIGURED
  (no `cluster.yaml` write) — assert the persist closure was **not** called.

**T4 `daemon_test.go` — `listenWeb` +1 retry (B4).** Bind a listener on port P, then
`listenWeb(P)` returns P+1; non-EADDRINUSE error fails fast. (Verbatim-behavior guard.)

**T5 `daemon_test.go` — `applyRole` fence skeleton (B6).** Drive `applyRole` with a fake
membership sequence; assert: master→follower transition cancels the prior role ctx before
starting the next; a stale generation is ignored. (Bodies stubbed; the *control flow* is tested.)

**T6 `deps_test.go` (in daemon) — seam (B7).** `buildDeps(...)` returns a `web.Deps` whose
stubbed mutators return `errNotImplemented`, and `State()` returns the live `*state.Store`
snapshot. Compile-level guard that `daemon` is the only constructor of `web.Deps`.

**Integration / acceptance (A.13 row P0):** `go build ./...` cross-compiles
`GOOS=linux GOARCH=arm64 CGO_ENABLED=0 go build ./cmd/ensemble` with **no C toolchain** (D24);
running `ensemble run --data <tmp>` twice keeps the same `node.json` node id across restarts
(stable-id invariant, 01 §5.2). `ensemble version` prints the stamp; `ensemble --help` prints
usage. These satisfy: *"cmd builds, data dir + identity persist"* / *"`go build`, restart keeps
node id"* (A.13 P0).

## 9. Risks / open questions

1. **BLOCKING — "configured" predicate ownership.** 01 §4.4 says configured ⇔ "has cluster CA +
   `cluster.yaml`". The exact symbol P0.3 calls (e.g. `pki.IsConfigured(paths) bool` vs.
   `state`/`config` exposing it) is **owned by P1 (pki)**, which is not yet specced. P0.3 must
   code against a small interface (`type configuredFn func() bool`) and let `cmd`/P1 supply it.
   **Orchestrator must confirm:** does P1 expose `pki.IsConfigured(config.Paths) (bool, error)`,
   or should `internal/config` own a `cluster.yaml` presence check? P0.3 picks the interface
   either way, but the wiring call site needs the answer.
2. **`internal/daemon` vs. `internal/node` name.** This spec chose `internal/daemon` to avoid
   the "node" overload (NodeRecord/Identity). If the orchestrator prefers `internal/node` for
   mpvsync parity, it is a pure rename — flag before P0.1/P0.2 cross-reference it.
3. **Gossip secret model.** mpvsync derived a memberlist `SecretKey` from a group password;
   Ensemble's control trust is mTLS (D10) and gossip auth/encryption is **not** P0.3's concern.
   Confirm P2 (cluster/membership) owns whether memberlist gossip is encrypted at all under the
   mTLS model (A.14.2 sets no `SecretKey`); P0.3 deliberately drops the password plumbing.
4. **WebSocket dep.** A.11 pins `github.com/coder/websocket` for UI push; whether the live-status
   push is wired in P0.2 `web` or surfaced as a `Deps` field is a P0.2 call — P0.3's `buildDeps`
   must match whatever `web.Deps` shape P0.2 freezes (A.14.3 is the reference).
```
