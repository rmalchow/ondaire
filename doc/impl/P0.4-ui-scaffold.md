## 1. Purpose & scope

Stand up the **Svelte 5 + Vite** single-page application that every Ensemble node embeds and serves (README §6.7, 09 §0): the app shell with a **persistent left-nav** over the seven operational screens, a **client-side SPA router with an auth guard** (uninitialized → Setup Wizard; unauthenticated → Login; otherwise the shell), a **typed API client** for the `/api/v1` surface (08) with `If-Match`/`ETag` optimistic-concurrency handling and the canonical error-envelope decode, a **dark-mode design system** (CSS-custom-property token set + base element styling) lifted from mpvsync's dark palette, and the **shared state-machine components** (`loading` / `empty` / `error` / `offline`) that every data-backed screen composes per 09 §0. This piece builds **only the scaffold**: shell, routing, client, tokens, and reusable state components — the individual screens (Setup, Login, Dashboard, Cluster, Groups, Node detail, Media, Settings) are downstream pieces that consume these primitives. The Vite build emits to `internal/web/dist` so the Go `web` package (a separate piece) can `go:embed` it.

**`web/src/lib/api.ts`**
```ts
export class ApiError extends Error {
  code: string          // envelope code, e.g. "version_conflict"
  status: number        // HTTP status
  constructor(status: number, code: string, message: string)
}

// Result of a read; `version` is the ETag (ConfigDoc.Version) to seed a follow-up If-Match.
export interface Read<T> { data: T; version?: number }

// Core fetch: decodes the §0.4 error envelope, surfaces ETag as `version`.
// `ifMatch` sets the If-Match header for config-mutating calls (08 §0.5);
// a 409 surfaces as ApiError{code:"version_conflict"}, 412 as "precondition_required".
export function apiFetch<T>(
  path: string,                                   // e.g. "/api/v1/groups"
  opts?: { method?: string; body?: unknown; ifMatch?: number; signal?: AbortSignal },
): Promise<Read<T>>

// Auth / boot
export function getStatus(): Promise<Read<{ initialized: boolean; nodeId: string; fingerprint: string; clusterName?: string }>>
export function getSession(): Promise<Read<{ authenticated: boolean; method: string; nodeId: string; configVersion: number }>>
export function login(password: string, keep: boolean): Promise<void>
export function logout(): Promise<void>

// Read-mostly ConfigDoc projections (08 C.1/D.1/E.1)
export function getClusterInfo(): Promise<Read<{ cluster: { name: string; caFingerprint: string; created: string }; counts: { nodes: number; groups: number } }>>
export function listNodes(): Promise<Read<{ nodes: NodeRecord[] }>>
export function listGroups(): Promise<Read<{ groups: GroupRecord[] }>>

// Live (08 G.2)
export function getGroupStatus(id: string, signal?: AbortSignal): Promise<GroupStatus>
```

## 5. Behavior & algorithms

- **Boot / first-run detection (09 screen 1):** on app load `App.svelte` probes `GET /api/v1/status`; `{initialized:false}` → force `/setup` and block all other routes (the **uninitialized** shared state, 09 §0). `{initialized:true}` → probe `GET /api/v1/auth/session` (08 B.4); unauthenticated → `/login`; authenticated → seed `configVersion` from `session.configVersion` and render the shell at the requested route (default `/`). If the **probe itself errors**, render the **error** state with Retry — never guess (09 screen 1 "Detection must be resilient").
- **Auth guard (router):** the guard runs on every navigation. Uninitialized blocks everything except `/setup`. Initialized-but-unauthenticated allows only `/login`. Session expiry mid-session (any `401 unauthenticated` from `apiFetch`) clears `session` and redirects to `/login`, preserving the requested route for post-login return (09 screen 2).
- **Persistent left-nav (09 §0):** `NavRail` exposes the five flat operational screens — **Dashboard (`/`) · Cluster (`/cluster`) · Groups (`/groups`) · Media (`/media`) · Settings (`/settings`)**. **Node detail (`/nodes/{id}`) is reached contextually, not a flat nav item.** `Header` shows cluster name, serving-node id, a connection/health chip, and a user menu with logout.
- **Error envelope (08 §0.4):** `apiFetch` parses `{"error":{"code","message"}}` on every non-2xx and throws `ApiError{status,code,message}`. The `code`/`message` are shown verbatim by `ErrorBanner` (09 §0 error state).
- **Optimistic concurrency (08 §0.5, 09 §0):** read calls capture the `ETag` header as `Read.version`; the UI seeds `If-Match` from the last-seen version on mutating calls. A `409 version_conflict` surfaces the **error** state with a **Reload & reapply** affordance (re-`GET`, show what changed, ask to reapply — never silently overwrite). A `412 precondition_required` is treated as a programming error (a mutating call sent without `If-Match`) and logged.
- **Offline / proxy handling (09 §0, §3):** a `502 proxy_failed` or a timed-out proxied call renders the **offline** treatment — region greyed with an "offline" chip, last-known ConfigDoc value shown dimmed with a "last known" tag — and surfaces an error **toast** on action failure. Distinguish offline (dimmed + chip) from a sink-less node's `—` (normal weight + "⊘ no sink" badge); that distinction is a screen-piece concern but the `OfflineChip`/`Chip` primitives support both.
- **Live status (09 §0):** `live.ts` polls `GET /api/v1/groups/{id}/status` at **~1 Hz** per visible group (08 G.2); WebSocket is optional and only used if the backend exposes a stream. The reconnect loop uses capped exponential backoff (base 500 ms, max 10 s) reused from the mpvsync `ws.ts` shape.
- **Dark-mode design system:** `tokens.css` defines the palette under `:root` / `[data-theme="dark"]`. Seed values are the **mpvsync dark palette already proven in the copied components** so the look is consistent: surface `#0d1117`, raised `#010409`, border `#30363d` / subtle `#21262d`, text `#e6edf3` / muted `#8b949e` / dim `#c9d1d9`, accent/primary `#1f6feb`+`#58a6ff`, success `#238636`/`#2ea043`, danger `#da3633`/`#f85149`, warn (sync ⚠) amber. Plus `--space-*`, `--radius-*` (0.35/0.5/0.6rem in use), `--shadow-modal` (`0 8–10px 32–40px rgba(0,0,0,.5)`), `--z-modal: 60`. `base.css` wires body/input/button/focus to the tokens. (These are UI palette values, not protocol tunables — Appendix A imposes no UI numbers.)
- **A.12 tunables surfaced in the UI** (quoted verbatim, the single source of truth — used for display defaults/thresholds, not redefined): canonical **rate `48000`**, **channels `2`**, **FramesPerChunk `480` (10 ms)**; **playout lead `LeadMs` 300 ms**; **MaxPPM 200 ppm**; **HardErrSamp 2400 samples (50 ms)**; FEC **k `8` / interleave depth D `4`**; **Ddup `5` packets**; ports **control mTLS `8443` / clock UDP `9000` / audio UDP `9100`**; adoption PIN placeholder **`"0000"`** (D9, treated as a real secret). The scaffold hard-codes only the dev-proxy port (`8443`) and the adoption-PIN default (`"0000"`) for the Setup/Cluster screens to consume; sync-error display formats µs values from `GroupStatus` (the ⚠ threshold is owned by [06](../06-audio-output-scheduling.md), referenced not invented).

## 6. Import constraints

This piece is almost entirely the `web/` frontend tree (TypeScript/Svelte) — **no Go package-layering concern applies to it**. The only Go file is `internal/web/embed.go`, which imports the stdlib `embed` only.

Per [doc 01 §2 / README §5 + §6.6]: `internal/web` reaches the engine **only** through the `web.Deps` function-value seam and **MUST NOT import `group`, `stream/*`, or `audio/*`** (A.14.3). This piece does **not** create any `internal/web` handler or `Deps` consumer, so it introduces no engine imports; it only provides the embeddable asset FS. The downstream `internal/web` server piece that consumes `DistFS` is bound by the Deps-seam rule. The frontend talks to the node **exclusively over HTTP `/api/v1`** (08) — never any other surface.

## 9. Risks / open questions
