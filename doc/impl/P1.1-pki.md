> Track: BACKEND (Go) · Status: implementation-ready · Depends on: **P0.1**
> Canonical sources: [README §5/§6](../README.md), [03 §1/§5/§8](../03-adoption-takeover-security-pki.md),
> [Appendix A.11/A.12](../A-appendix-algorithms-and-pinned-choices.md).

## 1. Purpose & scope

`internal/pki` is the **trust origin** of the cluster's control plane (03 §0.2). It mints and
operates the Ed25519 cluster CA (create + self-sign + leaf signing), generates per-node
keypairs and CSRs, marshals/parses the plaintext-replicated CA private key
(`ClusterSecrets.caKeyPEM`, D18), mints the cluster gossip key, builds the TLS 1.3 mTLS
`*tls.Config` for **both** server and client roles, and supplies the `PeerVerifier`
that enforces the grow-only `RevokedSet` (SHA-256-fingerprint) revocation check on every
handshake (03 §5.4). It is **pure library** code: it does no I/O, owns no persistence, holds
no network sockets, and imports no other `internal/*` package — callers (P0.x wiring, P2
state/cluster, P1.x web/auth) hand it bytes and live `ConfigDoc` snapshots and consume its
artifacts. This piece is **all NEW code**: there is no PKI/x509 package in `../media` to copy;
only the HKDF / constant-time-compare *idioms* from media `internal/auth` are referenced.

## 4. Public API (exact signatures — canonical names from README §6 / 03)

All names are taken verbatim from 03 §1.1, §1.2, §1.3, §1.6, §5.4, §8. Do not rename.

```go
package pki

import (
    "crypto"
    "crypto/tls"
    "crypto/x509"
    "net"
    "time"
)

// ---- CA (03 §1.1, §1.2) -------------------------------------------------

// CA is the cluster certificate authority. Cert is public (mirrored into
// ConfigDoc.Cluster); key is the PRIVATE Ed25519 signer (plaintext-replicated to
// full nodes as ClusterSecrets.caKeyPEM, D18 §1.2).
type CA struct {
    Cert    *x509.Certificate // public; mirrored into ConfigDoc.Cluster
    certPEM []byte            // public CA cert PEM (publish into ConfigDoc.Cluster)
    key     crypto.Signer     // PRIVATE Ed25519 signer
}

// CertPEM returns the public CA certificate PEM to publish into ConfigDoc.Cluster.
func (ca *CA) CertPEM() []byte

// CreateCA mints a fresh cluster CA on first-init (03 §1.1). Ed25519 keypair,
// self-signed CA cert: CN="ensemble-ca/<clusterName>", IsCA=true,
// BasicConstraintsValid=true, MaxPathLen=0, KeyUsage=CertSign|CRLSign,
// validity 10 years from now.
func CreateCA(clusterName string, now time.Time) (*CA, error)

// Sign issues a leaf certificate for a CSR (03 §1.1, §1.3). SANs are set from
// AUTHENTICATED inputs (nodeID, addrs) — the CSR's own SANs are ignored. The CSR
// signature (proof of possession) IS verified. validity is the leaf lifetime
// (30 days, A.12); NotBefore is now-5min (skew backdate, 03 §1.3).
func (ca *CA) Sign(csr *x509.CertificateRequest, nodeID string, addrs []net.IP, validity time.Duration, now time.Time) (certPEM []byte, err error)

// MarshalCAKey / ParseCAKey move the CA private key as plaintext PEM (D18 §1.2:
// no sealing, no AEAD-at-rest). Round-trip target: ClusterSecrets.caKeyPEM.
func MarshalCAKey(caKey crypto.Signer) (pemBytes []byte, err error)
func ParseCAKey(pemBytes []byte) (crypto.Signer, error)

// ParseCA reconstructs a usable CA from its public cert PEM + private key PEM
// (used when a full node loads ClusterSecrets at boot and must sign).
func ParseCA(certPEM, keyPEM []byte) (*CA, error)

// ---- Node identity (03 §1.3) -------------------------------------------

// NewIdentity generates a node's Ed25519 keypair. The private key NEVER leaves
// the node.
func NewIdentity() (crypto.Signer, error)

// NewCSR builds a self-signed CSR: Subject.CN = nodeID, PublicKey = key's public,
// signature proves possession. SANs are deliberately NOT set (the CA ignores CSR
// SANs and sets them from authenticated inputs, 03 §1.3).
func NewCSR(key crypto.Signer, nodeID string) (csrPEM []byte, err error)

// ---- TLS construction (03 §8) ------------------------------------------

// ServerTLS / ClientTLS build the mTLS config for the control plane (03 §8).
// TLS 1.3 only; both sides present the same leaf (ExtKeyUsage Server+Client) and
// verify the peer against caPool PLUS the PeerVerifier revoked-set check (§5.4).
func ServerTLS(leaf tls.Certificate, caPool *x509.CertPool, v *PeerVerifier) *tls.Config
func ClientTLS(leaf tls.Certificate, caPool *x509.CertPool, v *PeerVerifier) *tls.Config

// CAPoolFromPEM builds a *x509.CertPool from ConfigDoc.Cluster's CA cert PEM.
func CAPoolFromPEM(caCertPEM []byte) (*x509.CertPool, error)

// LeafFromPEM assembles a tls.Certificate from a node's signed cert PEM + its
// private key (held only in memory).
func LeafFromPEM(certPEM []byte, key crypto.Signer) (tls.Certificate, error)

// ---- Peer verification / revocation (03 §5.4) --------------------------

// PeerVerifier runs the extra post-chain check installed as
// tls.Config.VerifyPeerCertificate on BOTH sides. It reads a LIVE ConfigDoc
// snapshot via the revoked closure so a gossiped revocation takes effect without
// a restart (03 §8, §5.3).
type PeerVerifier struct { /* revoked func(fp string) bool; optional knownID func(string) bool */ }

// NewPeerVerifier wraps a live revoked-set predicate. revoked(fp) reports whether
// the lowercase-hex SHA-256 fingerprint is in ConfigDoc.RevokedSet. knownID may be
// nil (the optional CN/URI-SAN cross-check, 03 §5.4 step 3).
func NewPeerVerifier(revoked func(fingerprint string) bool, knownID func(nodeID string) bool) *PeerVerifier

// Verify is the VerifyPeerCertificate callback (rawCerts[0] is the peer leaf).
// Chain-to-CA is already enforced by ClientCAs/RootCAs+ClientAuth; this adds
// the revoked-set reject (and optional id cross-check).
func (v *PeerVerifier) Verify(rawCerts [][]byte, verifiedChains [][]*x509.Certificate) error

// Fingerprint is the canonical revocation key: lowercase hex of SHA-256 over the
// leaf DER (03 §5.2 / §5.4). Used by forget (to add) and Verify (to test).
func Fingerprint(der []byte) string

// ErrRevoked is returned by Verify when the peer leaf's fingerprint is revoked.
var ErrRevoked = errors.New("pki: peer certificate revoked")

// ---- Gossip key (03 §1.6) ----------------------------------------------

// NewGossipKey returns 32 cryptographically-random bytes for memberlist's
// SecretKey (AES-256-GCM). D18: random, NOT derived, NOT sealed.
func NewGossipKey() []byte
```

## 5. Behavior & algorithms (cite + quote)

**CA creation (`CreateCA`) — 03 §1.1 + §1.5.**
- Ed25519 keypair (`crypto/ed25519` via `crypto.Signer`); 03 §1.5: "small keys/sigs … fast
  verify, no parameter foot-guns".
- Self-signed cert: `Subject.CN = "ensemble-ca/" + clusterName`, `IsCA=true`,
  `BasicConstraintsValid=true`, `MaxPathLen=0`, `MaxPathLenZero=true` (signs leaves only,
  never intermediates), `KeyUsage = x509.KeyUsageCertSign | x509.KeyUsageCRLSign`.
- Validity: `NotBefore = now`, `NotAfter = now.AddDate(10, 0, 0)` — 03 §1.1 "validity 10 years".
- `SerialNumber`: random 128-bit (`crypto/rand` + `(*big.Int)` over 2^128).
- Returns `*CA` (private key in memory) + `CertPEM()` to publish into `ConfigDoc.Cluster`.

**Leaf signing (`Sign`) — 03 §1.3.** Fields set by the CA (NOT copied from the CSR):

| Field | Value | Source |
|---|---|---|
| `Subject.CN` | `nodeID` | authenticated (assigned by controller) |
| SAN URI | `ensemble://node/<nodeID>` | 03 §1.3 — canonical address-independent id |
| SAN IP | every IP in `addrs` | 03 §1.3 — keeps cert ↔ allowlist consistent |
| SAN DNS | `<nodeID>.ensemble.local` | 03 §1.3 — optional mDNS match |
| `KeyUsage` | `DigitalSignature` | leaf, not a CA |
| `ExtKeyUsage` | `ServerAuth, ClientAuth` | same leaf both directions (03 §8) |
| `NotBefore` | `now.Add(-5*time.Minute)` | 03 §1.3 skew backdate |
| `NotAfter` | `now.Add(validity)` | `validity` = **30 days** (A.12 "leaf lifetime / renew-at | 30 days / ⅓ life (day 10)") |
| `SerialNumber` | random 128-bit | 03 §1.3 |

The CSR signature **must** be verified (`csr.CheckSignature()`); the public key carried in
the CSR becomes the leaf public key. The CSR's own SAN/extension fields are **ignored**
(03 §1.3: "a node cannot request a cert that impersonates another's id or address").

**`MarshalCAKey` / `ParseCAKey` — 03 §1.2.** PKCS#8 DER wrapped in a `PEM` block
(`type "PRIVATE KEY"`); plaintext, no encryption (D18 §1.2: "There is no seal key, no AEAD
blob, no `caKeySealed`"). `ParseCAKey` returns the `crypto.Signer` (Ed25519). Callers persist
the result at mode `0600` (persistence owned by P2 state, A.6 "mode 0600 (carries the CA key
+ hashes)") — `pki` itself does no file I/O.

**`NewIdentity` / `NewCSR` — 03 §1.3.** Ed25519 keypair; CSR `Subject.CN = nodeID`,
self-signed for proof of possession. Private key never serialized by this package.

**mTLS (`ServerTLS` / `ClientTLS`) — 03 §8 (verbatim construction).**
- Both: `MinVersion: tls.VersionTLS13` (03 §1.5 "TLS 1.3 only … no downgrade").
- `ServerTLS`: `Certificates: []tls.Certificate{leaf}`, `ClientAuth:
  tls.RequireAndVerifyClientCert`, `ClientCAs: caPool`, `VerifyPeerCertificate: v.Verify`.
- `ClientTLS`: `Certificates: []tls.Certificate{leaf}`, `RootCAs: caPool`,
  `VerifyPeerCertificate: v.Verify`. The caller sets `ServerName` to the target nodeId/IP so
  SAN matching holds (03 §8 comment); `pki` exposes the config, caller fills `ServerName` per
  dial (document this in `tlsconfig.go`).
- The bootstrap surface is the **only** non-mTLS TLS endpoint (`ClientAuth: NoClientCert`,
  self-signed) and is **out of scope** for P1.1 (owned by the adoption piece, 03 §9).

**Peer verification (`PeerVerifier.Verify`) — 03 §5.4 (quoted hook).** After chain validation
(already done by `ClientCAs`/`RootCAs` + `ClientAuth`):
1. parse `rawCerts[0]` (peer leaf DER) → `*x509.Certificate`.
2. `if v.revoked(Fingerprint(rawCerts[0])) { return ErrRevoked }` — fingerprint =
   lowercase-hex `SHA-256(leaf DER)` (03 §5.2: "key on the cert's SHA-256 fingerprint rather
   than its serial so takeover … only revokes the *specific* old certificate").
3. optional: `if v.knownID != nil && !v.knownID(cnOrURISAN) { return errUnknownPeer }`
   (03 §5.4 step 3 "confirm CN/URI-SAN matches a current NodeRecord id").
The `revoked` closure reads a **live** `ConfigDoc` snapshot (03 §8: "rebuilt whenever the
`ConfigDoc` changes … without restart"). `RevokedSet` is grow-only union-merged (A.6:
"`body.RevokedSet = union(doc.RevokedSet, remote.RevokedSet)` … ALWAYS grow-only union") — but
that merge is owned by P2 state; `pki` only *reads* via the predicate.

**`NewGossipKey` — 03 §1.6 + A.11.** `crypto/rand.Read` into a 32-byte slice; AES-256-GCM
memberlist `SecretKey` (03 §1.5 "AES-256-GCM"). 03 §1.6: "generated and replicated to full
nodes in plaintext … There is no derived seal key". **Do not** HKDF-derive it.

**Pinned numbers used here (A.12 — single source of truth, quoted):**
- Leaf lifetime: "leaf lifetime / renew-at | 30 days / ⅓ life (day 10)" → `validity = 30*24h`
  default at call sites; `Sign` takes `validity` as a parameter, the caller passes `30 days`.
- CA validity 10 years (03 §1.1 — not in A.12, taken from the owning doc).
- Cert skew backdate 5 min (03 §1.3).
Renewal cadence (day 10) is a *caller* concern (the `/renew` scheduler, 03 §1.4) — `pki` only
re-signs via `Sign`; it holds no timer.

## 6. Import constraints (01 §2 layering)

`internal/pki` is a **leaf library** in the dependency graph. It may import **only**:
- Go stdlib: `crypto`, `crypto/ed25519`, `crypto/rand`, `crypto/sha256`, `crypto/subtle`,
  `crypto/tls`, `crypto/x509`, `crypto/x509/pkix`, `encoding/pem`, `encoding/hex`,
  `math/big`, `net`, `net/url`, `time`, `errors`, `fmt`.
- **No `golang.org/x/crypto`** is required by P1.1 itself (HKDF/ChaCha belong to the adoption
  piece; gossip key is plain `crypto/rand`).
- **No `internal/*` imports.** In particular `pki` MUST NOT import `state`, `cluster`, `group`,
  `web`, `auth`, `config`, `stream/*`, or `audio/*`. The live-`ConfigDoc` coupling is inverted
  through the `func(fingerprint string) bool` closure passed to `NewPeerVerifier` (so `state`
  depends on nothing here and `pki` depends on nothing there). `web` reaches the engine only
  through the `web.Deps` seam (README invariant / 01 §2) and never imports `pki` directly for
  engine state; `cmd` wires `pki` artifacts into the TLS server.

## 7. Test plan

All table-driven where natural; `go test ./internal/pki/...` must pass with
`CGO_ENABLED=0 GOARCH=arm64 go build` succeeding. Acceptance gate: **A.13 P1**
("CA self-init, mTLS peer dial … two procs mTLS-handshake; bad cert rejected") and 03 §8.

**`ca_test.go`**
- `CreateCA` → assert: cert `IsCA`, `MaxPathLen==0` & `MaxPathLenZero`, `KeyUsage` =
  `CertSign|CRLSign`, `CN=="ensemble-ca/"+name`, `NotAfter≈now+10y`, public-key algorithm
  Ed25519, serial non-zero & ≤128-bit.
- `Sign` table: inputs {single IPv4, IPv4+IPv6, no addrs, multi-addr}. Assert leaf
  `CN==nodeID`, URI SAN `ensemble://node/<id>`, IP SANs == `addrs`, DNS SAN
  `<id>.ensemble.local`, `ExtKeyUsage` contains both Server+Client, `KeyUsage ==
  DigitalSignature`, `NotBefore == now-5m`, `NotAfter == now+validity`, **and the leaf chains
  to the CA** (`x509.Verify` against a pool of the CA cert).
- `Sign` ignores CSR SANs: build a CSR requesting `IP 10.0.0.99`/`CN evil`; assert the issued
  leaf contains neither — only the authenticated `nodeID`/`addrs`.
- `Sign` rejects a CSR with a broken self-signature (mutate a byte) → error.
- `MarshalCAKey`→`ParseCAKey` round-trip: signer signs a message, parsed key's public verifies
  it; PEM block type is `PRIVATE KEY`; output contains no `ENCRYPTED`.
- `ParseCA(certPEM, keyPEM)` reconstructs a CA that can `Sign` a leaf chaining to the original
  CA cert.

**`identity_test.go`**
- `NewIdentity` returns an Ed25519 `crypto.Signer`; two calls differ.
- `NewCSR`: parse the PEM, `csr.CheckSignature()==nil`, `csr.Subject.CN==nodeID`, public key
  matches the signer; no SANs present.

**`tlsconfig_test.go`**
- `ServerTLS`/`ClientTLS`: assert `MinVersion==VersionTLS13`, server `ClientAuth==
  RequireAndVerifyClientCert`, server `ClientCAs` & client `RootCAs` set, both
  `VerifyPeerCertificate != nil`, both carry the leaf.
- `CAPoolFromPEM` / `LeafFromPEM`: round-trip from `CreateCA`+`Sign`.

**`verifier_test.go`**
- `Fingerprint` == known lowercase-hex SHA-256 of a fixed DER (golden).
- `Verify`: leaf NOT in revoked set → nil; leaf IN revoked set → `ErrRevoked` (constant-time
  predicate). `knownID==nil` skips step 3; `knownID` returning false → unknown-peer error.
- `Verify` reads live snapshot: revoke the fingerprint between two calls (flip the closure) →
  first nil, second `ErrRevoked` (no restart).

**`gossip_test.go`**
- `NewGossipKey` returns 32 bytes; two calls differ; high-entropy sanity (not all-zero).

**`integration_test.go` (A.13 P1 acceptance, in-proc two-party mTLS):**
- Build CA; sign two leaves (`n-aaaa` @127.0.0.1, `n-bbbb` @127.0.0.1). Stand up an
  `httptest`/`tls.Listen` server with `ServerTLS`; dial with `ClientTLS` (ServerName set to the
  peer id/IP). Assert: **handshake succeeds**, server sees client CN `n-bbbb`.
- **Bad cert rejected:** dial with a leaf signed by a *different* (foreign) CA → handshake
  fails (chain error). 03 §8 / A.13 P1 "bad cert rejected".
- **Revoked rejected:** add `n-bbbb`'s fingerprint to the revoked predicate → handshake fails
  with the verifier error surfaced (03 §5.4). Confirms forget closes the control plane.
- **Epoch/SAN:** dial by IP with the IP in the leaf SANs → SAN match passes; dial with a
  ServerName/IP absent from SANs → fails (proves §1.3 SAN binding).

## 9. Risks / open questions

1. **`crypto.Signer` for Ed25519 CA signing.** `x509.CreateCertificate` accepts an
   `ed25519.PrivateKey` (which satisfies `crypto.Signer`) and signs with `SignatureAlgorithm
   = PureEd25519` when `priv` is Ed25519 and the template leaves `SignatureAlgorithm` zero.
   Keep `SignatureAlgorithm` unset so stdlib selects PureEd25519 — do **not** hand-set it.
   Low risk, but assert the issued cert's `SignatureAlgorithm == x509.PureEd25519` in tests.
2. **CA cert validity unit (10 years).** 03 §1.1 says "10 years" but it is **not** in the A.12
   table (which only pins the 30-day *leaf*). Treated as owned by 03 §1.1. *Open: confirm 10y
   is intended vs. an A.12 omission.* — **non-blocking** (use 10y per 03 §1.1).
3. **Live-`ConfigDoc` snapshot freshness.** `PeerVerifier` reads via the `revoked` closure; the
   contract requires that closure to be cheap and lock-free-ish (called per handshake, possibly
   per-connection burst). Spec'd as a `func(string) bool` so P2 can back it with an atomic
   snapshot (mirrors `allowlist.Set`'s atomic-snapshot pattern, 03 §6.3). No blocking concern.
4. **`knownID` cross-check optionality.** 03 §5.4 marks step 3 "(optional)". P1.1 ships it as
   a nilable closure (off by default); enabling it cluster-wide is a P2 policy call. Non-blocking.
5. **No blocking open question for the orchestrator.** P1.1 is stdlib-only and fully specified
   by 03 §1/§5/§8 + A.11/A.12; it can be built ahead of P2 against the closure seam, with the
   integration test standing in a fake `revoked` predicate.
