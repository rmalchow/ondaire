# P1.2 — `internal/auth` (admin password, sessions, API keys, adoption PIN, HTTP auth middleware)

> Track: BACKEND (Go) · Phase: P1 (PKI+auth, A.13) · Module `gitlab.rand0m.me/ruben/go/ensemble`, Go 1.26, `CGO_ENABLED=0`, cross-compiles linux/arm64 with no C toolchain (D24).
> Owning docs: [03 §7](../03-adoption-takeover-security-pki.md) (UI/API auth), [03 §3.4 + A.9](../03-adoption-takeover-security-pki.md) (PIN), [07 §2.3](../07-config-and-replication.md) (`AuthConfig`/`Argon2id`/`APIKey`), [A.11/A.12](../A-appendix-algorithms-and-pinned-choices.md) (pinned deps + tunables).

## 1. Purpose & scope

`internal/auth` is the **human-path authentication** layer of the control plane (D11): it hashes and verifies the single cluster admin password with **argon2id**, mints and validates per-node in-memory **sessions** (HttpOnly/Secure/SameSite=Strict cookies, sliding+absolute TTL), mints/hashes/verifies revocable **API keys** with salted **SHA-256**, verifies the adoption **PIN** (argon2id hash stored in `ConfigDoc.Auth.PINHash`, plus the bootstrap online-guess **guard** of A.9/A.12 §3.4), and exposes the composable **HTTP auth middleware chain** that wraps every `/api/v1` handler in the exact ordering of [03 §7.4]. It is pure verifier/credential logic: it owns no schema (that is `internal/state`, P-state) and reaches the engine through nothing — it is consumed by `internal/web` and `cmd`. The node (mTLS) path is *authenticated by `internal/pki`'s `PeerVerifier`*; this package only orchestrates the chain step that calls into it via an injected func.

## 4. Public API (canonical names from README §6 / 03 §7 / 07 §2.3 — do not invent alternatives)

### 4.1 Admin password (argon2id) — `password.go` (03 §7.1)

```go
// HashPassword hashes pw with argon2id using the given cost params and a fresh
// random salt, returning a PHC-encoded string ($argon2id$v=19$m=...,t=...,p=...$salt$hash).
func HashPassword(pw string, p Argon2id) (phc string, err error)

// VerifyPassword constant-time-compares pw against a PHC string produced by
// HashPassword. Cost params are read FROM the PHC string (self-describing), not args.
func VerifyPassword(pw, phc string) bool

// Argon2id mirrors state.Argon2id (07 §2.3) — the cost params for the NEXT write.
// auth does not own this type; it is re-declared here ONLY if state cannot be imported
// without a cycle. Preferred: import the canonical state.Argon2id. (See §6.)
type Argon2id = state.Argon2id
```

Default cost params (03 §7.1, A.11 row "Argon2id"): `m=64MiB (65536 KiB), t=3, p=2|4, keyLen=32, saltLen=16`. The example doc in 07 §6 uses `p=4`; **use the params carried in `ConfigDoc.Auth.Argon` for the next write, and the params embedded in each PHC string for verification** (07 §2.3 note).

### 4.2 Sessions (cookies, in-memory, per-node) — `session.go` + `cookie.go` (03 §7.2)

```go
// Sessions is the per-node, in-memory session store. NOT replicated (03 §7.2):
// a cookie issued by N1 is invalid at N2. Safe for concurrent use.
type Sessions struct{ /* id-hash -> sessionEntry{issuedAt, lastSeen}; mu; clock func() time.Time */ }

func NewSessions() *Sessions

// Issue returns the plaintext cookie value (32 random bytes, base64url); the store
// keeps only its SHA-256 hash.
func (s *Sessions) Issue() (cookieValue string)

// Validate constant-time-looks-up the hash of cookieValue; on hit it slides the
// 12h idle TTL (and enforces the 7d absolute cap) and returns true. Expired/absent => false.
func (s *Sessions) Validate(cookieValue string) bool

// Revoke drops the server-side entry (logout, 03 §7.2 / 08 §B.3).
func (s *Sessions) Revoke(cookieValue string)

// Sweep evicts expired entries; call from a background ticker (e.g. every 1m).
func (s *Sessions) Sweep()
```

Cookie constants/helpers (`cookie.go`, 03 §7.2 table):

```go
const SessionCookieName = "ensemble_session"

// SetSessionCookie writes the session cookie with the 03 §7.2 attributes:
// HttpOnly=true, Secure=true, SameSite=http.SameSiteStrictMode, Path="/", MaxAge from idle TTL.
func SetSessionCookie(w http.ResponseWriter, value string)
func ClearSessionCookie(w http.ResponseWriter) // MaxAge=-1 on logout
```

### 4.3 API keys (revocable, SHA-256) — `apikey.go` (03 §7.3)

```go
// NewAPIKey mints a key: id is an opaque handle, plaintext is shown exactly once
// to the operator (08 §B.6) and never persisted. plaintext = "ek_live_" + base64url(32B).
func NewAPIKey() (id, plaintext string)

// HashAPIKey returns hex(SHA-256(salt || plaintext)). API keys are high-entropy
// (>=128-bit), so a fast salted hash + constant-time compare suffices (03 §7.3) —
// argon2id is reserved for the low-entropy human password.
func HashAPIKey(plaintext, salt string) string

// VerifyAPIKey constant-time-matches a presented bearer key against the stored
// keys (state.APIKey carries id+Hash; the per-key salt is embedded in/alongside Hash).
// Returns the matching key id on success. Iterates all keys with a constant-time
// compare to avoid leaking which key (if any) matched via timing.
func VerifyAPIKey(plaintext string, keys []state.APIKey) (id string, ok bool)
```

> README §6.5 / 03 §7.3 signature is `VerifyAPIKey(plaintext string, stored []KeyRecord) (id string, ok bool)`. The canonical persisted type is `state.APIKey` (07 §2.3); use it directly rather than introducing a parallel `KeyRecord`. Salt handling: persist `salt` either as a field on the stored record or as a prefix segment of `Hash` (e.g. `salt$hex`); **resolve in §9**.

### 4.4 Adoption PIN — `pin.go` (03 §3.4, A.9, A.12)

```go
// VerifyPIN constant-time-checks a candidate PIN against ConfigDoc.Auth.PINHash
// (argon2id PHC). An empty PINHash means the placeholder default "0000" (07 §2.3, D9).
func VerifyPIN(pin, pinHash string) bool

// HashPIN argon2id-hashes a PIN (used by setup / change-PIN to populate Auth.PINHash).
func HashPIN(pin string, p Argon2id) (phc string, err error)

// AdoptionGuard enforces the A.9/A.12 §3.4 online-guess hardening on the bootstrap
// surface: soft backoff after 3 consecutive fails, hard 15-min lockout after 10 fails
// per 5 min, single-use nonces with 30s TTL. Per-source AND global counters, in memory.
type AdoptionGuard struct{ /* per-source + global windows; nonce set; mu; clock */ }

func NewAdoptionGuard() *AdoptionGuard

// Allow reports whether a /bootstrap/adopt attempt from src may proceed now.
// reason is "" when allowed, or "rate_limited"/"locked_out" (08 §0.4 code) when not.
func (g *AdoptionGuard) Allow(src netip.Addr, now time.Time) (ok bool, reason string)

// RecordFail registers a failed PIN proof (drives backoff/lockout); logs src (audit, 03 §3.4).
func (g *AdoptionGuard) RecordFail(src netip.Addr, now time.Time)

// RecordSuccess resets the per-source counters after a good proof.
func (g *AdoptionGuard) RecordSuccess(src netip.Addr)

// IssueNonce returns a single-use nonce valid for NonceTTL (30s).
func (g *AdoptionGuard) IssueNonce(now time.Time) (nonce string)

// ConsumeNonce validates+burns a nonce; false if unknown/expired/already-used.
func (g *AdoptionGuard) ConsumeNonce(nonce string, now time.Time) bool
```

> The **A.9 cryptographic handshake** (ECDH `k`, PIN-keyed HMAC `kp`, ChaCha20-Poly1305 AEAD) is *not* in this piece — it lives with P2 adoption + `internal/pki`. `auth` owns only the PIN **verifier** + the **guard** (throttle/lockout/nonce), which the bootstrap handler in P2/web calls.

### 4.5 Middleware chain — `middleware.go` (03 §7.4, README §6.6)

```go
// AuthMethod is the authenticated principal class attached to the request context.
type AuthMethod string
const ( MethodNode AuthMethod = "node"; MethodSession AuthMethod = "session"; MethodAPIKey AuthMethod = "apiKey" )

// Deps are the injected seams the chain needs (no imports of pki/state/web concretes):
type Deps struct {
    Initialized func() bool                                   // false => uninitialized gate [1] (no CA / no admin hash)
    NodeAuth    func(r *http.Request) (nodeID string, ok bool) // [2] mTLS leaf chains to CA AND fp not revoked (pki.PeerVerifier-backed)
    Sessions    *Sessions                                     // [3] human path
    VerifyKey   func(plaintext string) (id string, ok bool)   // [3] bound to state.APIKeys via auth.VerifyAPIKey
}

// Authenticate runs steps [1]-[3] of 03 §7.4: uninitialized gate, node path, human path.
// On success it stores the AuthMethod (and node/key id) in the request context; on failure
// it writes the JSON error envelope with the correct status and returns without calling next.
func (d Deps) Authenticate(next http.Handler) http.Handler

// RequireAdminSession is step [4] AuthZ: gates admin-session-only endpoints (mint/list API
// keys, 08 §B.5/B.6) — a node cert authenticates but does NOT authorize these.
func RequireAdminSession(next http.Handler) http.Handler

// Chain composes [0] recover+log, [1-3] Authenticate, then the per-endpoint [4]/[5] wrappers.
func Chain(d Deps, mw ...func(http.Handler) http.Handler) func(http.Handler) http.Handler

// MethodFromContext / NodeIDFromContext / KeyIDFromContext read what Authenticate stored.
func MethodFromContext(ctx context.Context) (AuthMethod, bool)

// WriteError emits the canonical envelope {"error":{"code","message"}} (README §6.6, 08 §0.4).
func WriteError(w http.ResponseWriter, status int, code, message string)
```

State-gated exceptions wired by the caller (README §6.6, 03 §7.5): `POST /api/v1/setup` and `POST /api/v1/login` are reachable without a session/cert (they mint credentials); `/bootstrap/*` lives **outside** this chain entirely (PIN-gated, self-signed TLS, no mTLS). `If-Match` precondition [5] is owned by `internal/web`/`state`, not `auth` (referenced here for chain ordering completeness).

## 5. Behavior & algorithms (cite docs; A.12 numbers verbatim)

- **Admin password (03 §7.1):** argon2id via `golang.org/x/crypto/argon2` `IDKey`. PHC encode `$argon2id$v=19$m=<KiB>,t=<t>,p=<p>$<b64salt>$<b64hash>`. Verify re-derives with the **params parsed from the stored PHC string** and compares with `subtle.ConstantTimeCompare`. Default cost (03 §7.1): `m=64MiB, t=3, p=2`; `keyLen=32, saltLen=16` (07 §2.3). Generation params come from `ConfigDoc.Auth.Argon`.
- **Sessions (03 §7.2):** value = 32 random bytes (`crypto/rand`) base64url; store only `SHA-256` of the value (never plaintext). Cookie attributes verbatim from the 03 §7.2 table: `HttpOnly=yes`, `SameSite=Strict`, `Secure=yes`, `Path=/`. **TTL: 12 h sliding (idle), absolute 7 d.** `Validate` slides `lastSeen` on hit and rejects if `now-issuedAt > 7d` or `now-lastSeen > 12h`. Per-node, in-memory, not replicated.
- **API keys (03 §7.3):** plaintext `ek_live_` + base64url(32 random bytes). Hash = `hex(SHA-256(salt‖plaintext))`, per-key random salt. Verify with constant-time compare over all stored keys (avoid early-exit timing leak). Revocation = drop from `ConfigDoc.Auth.APIKeys` (07 §2.3); no `RevokedSet` (keys are not certs).
- **PIN (03 §3.4, A.9):** verifier = argon2id over `ConfigDoc.Auth.PINHash`; `""` ⇒ default `"0000"` (D9, treated as a real secret in the protocol). The PIN never crosses the wire — only HMAC tags (A.9), built in P2.
- **Adoption guard (A.12 "Adoption guard" row, verbatim):** *"backoff after 3 consecutive fails / 15-min lockout after 10 fails per 5 min / nonce TTL 30 s"*. Soft backoff → `429 {code:"rate_limited"}` (03 §3.4, 08 §0.4); hard lockout → refuse `adopt` for **15 min** while `/bootstrap/info` still answers (03 §3.4). Counters per **source IP** and **global**, in memory; every failed proof logged with source IP (audit, 03 §3.4).
- **Middleware ordering (03 §7.4 — authoritative):** `[0]` panic-recover + request log → `[1]` uninitialized gate (no CA / no admin hash ⇒ allow only `setup` + `/bootstrap/*`, else `503 not_ready`) → `[2]` NODE path (mTLS client cert chains to cluster CA AND fingerprint ∉ `RevokedSet` ⇒ `method:"node"`) → `[3]` HUMAN path (valid session cookie ⇒ `"session"`, or `Bearer` API key ⇒ `"apiKey"`; `/login` exempt — it mints) → `[4]` per-endpoint AuthZ (some calls admin-session-only even when authenticated, e.g. API-key mgmt) → `[5]` `If-Match` precondition → handler. Node-before-human (a peer call short-circuits session/key checks). `[2]` delegates the actual cert/chain/revoked check to `Deps.NodeAuth` (backed by `pki.PeerVerifier`), keeping `auth` free of a `pki` import.
- **Error envelope (README §6.6, 08 §0.4):** `{"error":{"code":"string","message":"..."}}` with proper status: `503 not_ready` (uninitialized gate), `401` (no valid human credential), `403` (authenticated but not authorized at [4]), `429 rate_limited` (guard backoff).

## 6. Import constraints (01 §2 layering)

`internal/auth` is a **leaf-ish credential package**; per the layering rules it may import:

- Go stdlib: `crypto/rand`, `crypto/sha256`, `crypto/subtle`, `encoding/base64`, `encoding/hex`, `net/http`, `net/netip`, `context`, `sync`, `time`, `strings`.
- Third-party (A.11): `golang.org/x/crypto/argon2`. (`golang.org/x/crypto/hkdf` only if the guard needs domain-separated nonce derivation — likely not; HKDF for A.9 is in P2.)
- `internal/state` — **only** for the canonical credential types it consumes (`state.Argon2id`, `state.APIKey`, and read-only `ConfigDoc.Auth` access via the values passed in). If this creates an import cycle (`state` importing `auth`), invert it: keep `auth` free of `state` and have `state`/`web` pass primitives (`[]APIKey`, cost ints) into `auth` functions. **Resolve in §9.**

It MUST NOT import: `internal/web`, `internal/group`, `internal/stream/*`, `internal/audio/*`, `internal/cluster`, `internal/clock`, `internal/pki` (the node/cert check is injected as `Deps.NodeAuth`, not imported — preserves the web→engine seam discipline of 01 §2 and keeps `auth` testable without TLS). It is consumed by `internal/web` and `cmd`.

## 9. Risks / open questions

1. **`state.APIKey` salt field (BLOCKING for `apikey.go` signature).** 07 §2.3 `APIKey` has `Hash` (PHC string) but **no explicit `Salt` field**, and it says argon2id for the *example* — yet 03 §7.3 mandates **salted SHA-256** (not argon2id) for keys. Resolve: (a) does the per-key SHA-256 salt live in a new `APIKey.Salt` field, or be embedded as a `salt$hexhash` segment in `Hash`? and (b) confirm SHA-256 (03 §7.3) overrides the argon2id-looking example hash in 07 §6. **This determines `HashAPIKey`/`VerifyAPIKey` exactly and may require a one-field edit to the `state.APIKey` schema (07).**
2. **`auth` ↔ `state` import direction.** Preferred is `auth` importing `state` for `Argon2id`/`APIKey`. If `state` ever needs `auth` (it should not — `state` is pure schema/merge), there's no cycle; but to be safe, the fallback is keeping `auth` dependency-free and passing primitives. Confirm `state` will not import `auth`.
3. **`Argon2id` ownership.** README/03 §7.1 show a local `AdminCred{Argon2id string}`; 07 §2.3 defines the cost struct `state.Argon2id`. Spec uses the **07** cost struct as canonical and stores the PHC string in `Auth.AdminHash`. Confirm no second `Argon2id` type is wanted in `auth`.
4. **Session store growth / eviction policy.** Spec adds `Sweep` (not in 03 §7.2) to bound memory on a 512 MB Pi; confirm a background sweep ticker (and whether a hard cap on concurrent sessions is desired) is acceptable vs. lazy expiry only.
5. **Guard nonce store coordination with P2.** `AdoptionGuard` owns nonces (A.9 single-use, 30 s TTL), but the A.9 handshake (`nonceA`/`nonceB` feeding the transcript) is implemented in P2. Confirm the nonce lifecycle boundary: does `auth.AdoptionGuard` mint/burn the bootstrap nonces, or only throttle while P2 owns nonces? Spec assumes `auth` owns them (keeps the guard self-contained); P2 must consume `IssueNonce`/`ConsumeNonce`.
```
