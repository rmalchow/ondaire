## 1. Purpose & scope

This piece implements the **credential surface and the gate that protects every other
API surface on a node**: the genesis `POST /api/v1/setup` (mint cluster CA → set admin
password → self-sign this node's leaf → write `ConfigDoc` v1 → log the operator in),
the full auth endpoint set from [08 §B](../08-http-api-reference.md)
(`login`/`logout`/`password`/`session`/`keys`), the `GET /api/v1/status` init probe used
by the SPA to choose wizard-vs-app, and — most load-bearing — the **`[1]` UNINITIALIZED
GATE** plus the **`[2]` node / `[3]` human / `[4]` per-endpoint-authz** middleware chain
([03 §7.4](../03-adoption-takeover-security-pki.md)). It lives entirely in `internal/auth`
(trust primitives) and `internal/web` (handlers + middleware), reaching the engine only
through the `web.Deps` function-value seam (01 §2 rule 1). It does **not** implement the
crypto of adoption (A.9 / `/bootstrap/*` — that is P2) nor the `pki.CA` itself (P1.2);
it *consumes* both. PCM/audio, clock, and group code are out of scope.

## 4. Public API (canonical names — do not invent alternatives)

### 4.1 `internal/auth` — password (03 §7.1)

```go
// PHC string form: $argon2id$v=19$m=65536,t=3,p=2$<salt>$<hash>
func HashPassword(pw string) (phc string, err error) // per-hash random salt, argon2id
func VerifyPassword(pw, phc string) bool             // constant-time compare of derived tag
```

Params (A.12 / 03 §7.1): **`m=64MiB, t=3, p=2`** (i.e. `m=65536` KiB, `t=3`, `p=2`),
32-byte salt, 32-byte tag, via `golang.org/x/crypto/argon2.IDKey`.

### 4.2 `internal/auth` — sessions (03 §7.2)

```go
type Sessions struct{ /* id-hash -> {issuedAt, lastSeen}; sync.Mutex */ }
func NewSessions() *Sessions
func (s *Sessions) Issue() (cookieValue string)      // 32 random bytes base64url; stores SHA-256(hash) only
func (s *Sessions) Validate(cookieValue string) bool // constant-time; slides TTL; enforces absolute cap
func (s *Sessions) Revoke(cookieValue string)
func (s *Sessions) GC()                              // drop expired entries (called lazily on Validate)
```

Cookie attributes (03 §7.2), emitted by the handlers: name `ensemble_session`, value =
base64url(32 random bytes), `HttpOnly`, `SameSite=Strict`, `Secure`, `Path=/`,
**TTL 12 h sliding; absolute 7 d**. Sessions are **per-node, in-memory, NOT replicated**.

### 4.3 `internal/auth` — API keys (03 §7.3)

```go
type KeyRecord struct {
    ID         string `json:"id"`
    Label      string `json:"label"`
    Salt       string `json:"salt"`       // per-key, hex
    Hash       string `json:"hash"`       // SHA-256(salt||plaintext), hex
    CreatedAt  string `json:"createdAt"`  // RFC3339
    LastUsedAt string `json:"lastUsedAt,omitempty"`
}
func NewAPIKey() (id, plaintext string)                            // plaintext = "ek_live_" + base64url(32B)
func HashAPIKey(plaintext, salt string) string                     // SHA-256(salt||plaintext), hex
func VerifyAPIKey(plaintext string, stored []KeyRecord) (id string, ok bool) // constant-time over all
```

API keys are stored as **hashes in `ConfigDoc.Auth`** (replicated). SHA-256 + per-key
salt suffices (keys are ≥128-bit random — no memory-hardness needed, 03 §7.3). The
plaintext secret is returned **exactly once** (08 §B.6).

### 4.5 `internal/auth` — middleware (03 §7.4, the authoritative ordering)

```go
// Gate is the boundary object the web layer holds; cmd constructs it with closures
// that read live trust state from state/pki without auth importing them (01 §2 rule 2).
type Gate struct{ /* sessions *Sessions + closures below */ }

type GateDeps struct {
    Initialized func() bool                              // [1] node has CA + admin hash?
    AdminHash   func() (phc string, ok bool)             // [3] argon2id PHC from ConfigDoc.Auth
    APIKeys     func() []KeyRecord                        // [3] hashed keys from ConfigDoc.Auth
    NodeID      func() string                             // self id (for session/apiKey AuthInfo)
    // mTLS [2]: the verified client cert (CN, fingerprint) is surfaced by pki's
    // VerifyPeerCertificate via *tls.ConnectionState; the gate reads r.TLS.PeerCertificates.
    // RevokedSet membership is enforced in pki.PeerVerifier (03 §5.4) BEFORE the handler,
    // so [2] here only needs: a verified chain present ⇒ method:"node", CN ⇒ NodeID.
}
func NewGate(s *Sessions, d GateDeps) *Gate

// [1] UNINITIALIZED GATE + [0] recover/log. Wraps the whole /api/v1 mux.
func (g *Gate) AuthGate(next http.Handler) http.Handler

// [2]+[3] authentication. On success injects AuthInfo into the request context; on
// failure writes 401 unauthenticated (or 503 not_ready when [1] tripped upstream).
func (g *Gate) RequireAuth(next http.Handler) http.Handler

// [4] per-endpoint authz: admin-session ONLY (rejects node cert / apiKey where the
// endpoint demands the interactive admin — e.g. key mgmt, password change).
func (g *Gate) RequireAdmin(next http.Handler) http.Handler
```

## 5. Behavior & algorithms

### 5.1 `[1]` UNINITIALIZED GATE (03 §7.4 / §7.5) — the load-bearing piece

`AuthGate` runs **before [2]/[3]**. If `deps.Initialized()==false` (node has no cluster
CA **and** no admin hash), it allows **only**:

- `GET /api/v1/status` and `POST /api/v1/setup` (the wizard), and
- `/bootstrap/*` (served on a *different* mux/listener by P2 — outside this gate; noted
  for completeness, not wired here).

Everything else under `/api/v1` → **`503 not_ready`** with envelope
`{"error":{"code":"not_ready",...}}` (08 §0.4). When initialized, the gate falls through
to `[2]/[3]`. Conversely `POST /api/v1/setup` on an **initialized** node → **`409 conflict`**
(self-adoption is a once-only genesis act, 03 §2.6). Reachability table (03 §7.5):

| Surface | Uninitialized | Initialized |
|---|---|---|
| `GET /api/v1/status`, `POST /api/v1/setup` | yes (wizard) | status: yes · setup: `409 conflict` |
| everything else `/api/v1` | `503 not_ready` | per auth class |

`[0]` panic-recover + request log wraps the outermost layer (03 §7.4 step [0]).

### 5.2 `[2]` node / `[3]` human / `[4]` authz (03 §7.4)

- **[2] node path:** if `r.TLS` has a verified peer cert chained to the cluster CA (chain
  + revoked-set already enforced by `pki.PeerVerifier`, 03 §5.4/§8), authenticate as
  `method:"node"`, `NodeID = CN`, skip [3]. Browser requests carry no client cert
  (`tls.VerifyClientCertIfGiven`, 01 §3.1) → fall to [3].
- **[3] human path:** require ONE of a valid `ensemble_session` cookie (`Sessions.Validate`,
  slides TTL) → `method:"session"`, or `Authorization: Bearer <key>` matching a stored
  `KeyRecord` (`VerifyAPIKey`, constant-time) → `method:"apiKey"`. Neither → `401 unauthenticated`.
  **Exceptions** (the human-path bootstrap, never reach `RequireAuth`): `/auth/login` mints
  a session, `/api/v1/setup`/`/api/v1/status` are pre-init. `/auth/session` accepts session
  *or* api-key (B.4).
- **[4] authz:** `RequireAdmin` wraps the admin-only endpoints — `auth/logout`,
  `auth/password`, and **all** `auth/keys` mutations + the list (08 §B.3/B.3a/B.5/B.6/B.7
  are admin-session). A request authenticated as `method:"node"` or `"apiKey"` is rejected
  `403 forbidden` on these (03 §7.4 rationale: a node cert authenticates but does not
  authorize key management).

### 5.3 `POST /api/v1/setup` genesis sequence (03 §1.1, 01 §6a)

The handler validates body (`clusterName` non-empty → else `422 unprocessable`;
`adminPassword` meets the 03 password policy → else `400 invalid_request`), confirms
`!deps.Initialized()` (else `409 conflict`), then calls `deps.Setup(...)`. The closure
(built in `cmd`) performs, in order: argon2id-hash the admin pw (`HashPassword`); `pki.CreateCA`
(Ed25519, `CN=ensemble-ca/<clusterName>`, `IsCA`, `MaxPathLen=0`, 10 y — 03 §1.1);
`pki.NewIdentity`+`NewCSR`+`CA.Sign` to self-issue this node's leaf (30 d, 03 §1.3/§2.6);
mint `ClusterSecrets{caKeyPEM,gossipKey}` (`NewGossipKey`); write genesis `ConfigDoc`
with `Version=1`, `Cluster{name,caCert,created}`, `Auth{adminHash}`, `Nodes:[self]`,
`Groups:[solo]`; persist (`certs/`, `doc.json`, `cluster.yaml` 0600 — 01 §5.2); `activate()`
in-process (no restart, 01 §4.4). On success the handler issues a session
(`Sessions.Issue` → `Set-Cookie`) so the operator is logged in immediately (B.1), sets
`ETag: 1`, and returns the B.1 body. protocolEpoch is fixed **1** (it does not appear in
the setup body but is asserted by `/bootstrap/info`, owned by P2).

### 5.4 Auth endpoints (08 §B) — concise behavior

- **B.2 login:** `503 not_ready` if uninitialized; `VerifyAdminPassword` const-time; on ok
  `Sessions.Issue` + `Set-Cookie` + `{session:{expiresAt}}`; wrong pw `401`; brute-force
  throttle is reused conceptually but the **adoption guard numbers in A.12 are for
  `/bootstrap/adopt`, not login** — login throttling envelope is `code:"rate_limited"`
  (08 §B.2); see §9 open question on exact login limits.
- **B.3 logout:** `Sessions.Revoke` + clear cookie → `204`.
- **B.3a password:** `If-Match` **required** (`412` if absent, `409 version_conflict` on
  mismatch); verify `currentPassword` (else `401`); validate `newPassword` policy (else
  `400`); `ChangeAdminPassword` writes new PHC to `ConfigDoc.Auth`, bumps version, gossips;
  `{version}` + `ETag`.
- **B.4 session:** session **or** api-key; returns `{authenticated,method,nodeId,configVersion}`
  + `ETag: <version>`.
- **B.5 keys (list):** admin-session; `{version,keys:[{id,label,createdAt,lastUsedAt}]}` —
  metadata only, never the secret; `ETag`.
- **B.6 keys (create):** admin-session; `If-Match` required; `NewAPIKey`+`HashAPIKey`,
  store `KeyRecord` in `ConfigDoc.Auth`, return secret **once**, `201` + `ETag`.
- **B.7 keys (delete):** admin-session; `If-Match` required; drop the hash, `200`/`204` + `ETag`.
- **G.1 status:** read-only; returns `NodeStatus`; `Initialized=false` drives the SPA to
  the wizard; `503 not_ready` only if a started-sink probe is requested and unavailable.

### 5.5 Quoted tunables (A.12, verbatim — never invent)

- argon2id admin password: **`m=64MiB, t=3, p=2`** (03 §7.1; A.11 pins `golang.org/x/crypto/argon2`).
- Session TTL: **12 h sliding; absolute 7 d** (03 §7.2). Session id: **32 random bytes** base64url.
- API key plaintext: **`ek_live_` + base64url(32B)** (≥128-bit) (03 §7.3).
- PKI (consumed via `deps.Setup`/pki): **leaf lifetime / renew-at = 30 days / ⅓ life (day 10)**;
  ports **control mTLS 8443 / clock UDP 9000 / audio UDP 9100** (A.12).
- **Not used by this piece but adjacent (A.12 adoption guard, owned by P2):**
  *"backoff after 3 consecutive fails / 15-min lockout after 10 fails per 5 min / nonce TTL 30 s"*.

## 6. Import constraints (01 §2)

- `internal/auth` may import **`config` only** (01 §2 table + rule 2). It must **NOT**
  import `cluster`, `group`, `state`, `pki`, or `web`. Trust state (admin hash, key list,
  initialized flag, self id) is handed in as **closures** via `GateDeps`/`Config`, so
  `auth` stays a unit-testable leaf with no cluster.
- `internal/web` may import **`auth`, `pki`, `cluster`, `state`, `config`** (01 §2 table).
  It must **NOT** import `group`, `stream/*`, or `audio/*` (01 §2 rule 1). All engine
  reach-through (the `Setup`/`Status`/key-mutation closures) flows through `web.Deps`,
  whose implementations are built in `cmd/ensemble` (01 §2 rule 6). `web` may import the
  *value types* `state.ConfigDoc` and `auth.KeyRecord` (they are data, not engine).
- `cmd/ensemble` (P0.2 / downstream) wires `auth.Gate`, `auth.Sessions`, and the `web.Deps`
  closures over `pki`+`state`+`group`.

**Exposes (downstream):**
- `auth.Gate`, `auth.Sessions`, `auth.{HashPassword,VerifyPassword,NewAPIKey,HashAPIKey,
  VerifyAPIKey,KeyRecord}`, `auth.AuthInfo`/`FromContext` — consumed by **P2** (adoption/
  cluster ops endpoints, which run *inside* `RequireAuth`), **P3+** (every authenticated
  `/api/v1` handler wraps with `RequireAuth`; admin-only ones add `RequireAdmin`), and the
  UI (`GET /status` init probe, `GET /auth/session` auth-guard).
- The extended `web.Deps` field set (§4.7) is the contract `cmd/ensemble` must satisfy.

## 9. Risks / open questions

1. **Login throttling values (BLOCKING-ish).** A.12 defines the brute-force guard numbers
   **only for `/bootstrap/adopt`** (3-fail soft backoff / 10-per-5-min → 15-min lockout).
   08 §B.2 says `/auth/login` must throttle (`code:"rate_limited"`) but neither 08 nor
   A.12 pins login-specific numbers. **Open question for the orchestrator:** reuse the
   A.12 adoption-guard numbers for login, or define a separate row? I recommend reusing
   them verbatim (per-source + global in-memory counter, identical to 03 §3.4) to avoid
   inventing values — confirm.
2. **`ClusterSecrets` ownership boundary.** The genesis write mints `ClusterSecrets`
   (caKeyPEM+gossipKey, 03 §1.1/§1.6). Whether the *struct* and its 0600 persistence live
   in P1.1 (`state`) or P1.2 (`pki`) is not nailed down here; this piece only calls
   `deps.Setup`. Confirm the home so the `cmd` closure compiles against the right package.
3. **`If-Match` source in `Deps` closures.** §4.7 threads `ifMatch uint64` through the
   key/password mutation closures. If P1.1's `state.Store.Apply` already takes the
   If-Match version, the web handler just forwards the parsed header; confirm the exact
   `Apply` signature to avoid a double-read race.
4. **`Initialized()` definition.** Defined as "has CA **and** admin hash". During the
   genesis write there is a brief window where one exists and not the other — `deps.Setup`
   must be atomic (write everything, then flip a single `initialized` flag) so the gate
   never sees a half-initialized node. This is a `cmd`/P1.1 responsibility flagged here.
5. **Password policy.** 03 references a "password policy" for `400 invalid_request` but
   does not quote a minimum length/charset. Pinning a concrete minimum (e.g. ≥12 chars)
   is needed for the `400` test; currently treated as "non-empty + length ≥ N" with N TBD
   — orchestrator to confirm N.
