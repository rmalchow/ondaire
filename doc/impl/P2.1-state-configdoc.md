## 1. Purpose & scope

Implement `internal/state`: the single replicated, version-stamped `ConfigDoc` and
its concurrency-safe `Store`. This piece owns the **complete typed schema** of the
ConfigDoc (07 §2: `ClusterInfo`, `ClusterSecrets`, `AuthConfig`/`Argon2id`/`APIKey`,
`NodeRecord`/`Capabilities`, `GroupRecord`/`MediaSelection`/`TransportProfile`,
`RevokedSet`/`RevokedCert`, plus `Version`/`UpdatedBy`/`UpdatedAt`), and the
mechanics lifted near-verbatim from mpvsync `internal/state/state.go`: optimistic
`Apply` (bump `Version`, `ErrConflict` on stale), gossip `Merge` (LWW by
`Version` with `UpdatedBy`/id tiebreak **plus** grow-only `Revoked` **union**
regardless of who wins the doc — A.6), `Get` deep-copy, the `{nodeId, doc}`
`MarshalGossip`/`MergeGossip` envelope, a coalesced `Changed()` signal, and
**`0600`** atomic temp+rename persistence (07 §5.3 — tightened from mpvsync's
`0644` because the doc carries the plaintext CA private key and credential
verifiers). It is the source of truth consumed by `cluster` gossip (02 §4),
`allowlist` (07 §3), `group` election/negotiation (02 §5, 04), `auth`, `pki`, and
`web`.

## 4. Public API (canonical names — 07 §2 / §5.1, README §6.5)

### 4.1 Schema (`internal/state/doc.go`)

Field order below is the canonical marshal order (07 §2).

```go
package state

type ConfigDoc struct {
	Version   uint64         `json:"version"`
	Cluster   ClusterInfo    `json:"cluster"`
	Secrets   ClusterSecrets `json:"secrets"`
	Auth      AuthConfig     `json:"auth"`
	Nodes     []NodeRecord   `json:"nodes"`
	Groups    []GroupRecord  `json:"groups"`
	Revoked   RevokedSet     `json:"revoked"`
	UpdatedBy string         `json:"updatedBy"`
	UpdatedAt string         `json:"updatedAt,omitempty"`
}

type ClusterInfo struct {
	Name        string `json:"name"`
	CACertPEM   string `json:"caCertPem"`
	Created     string `json:"created"`
	Fingerprint string `json:"fingerprint"`
}

type ClusterSecrets struct {
	CAKeyPEM     string `json:"caKeyPem"`
	SharedSecret string `json:"sharedSecret,omitempty"`
}

type AuthConfig struct {
	AdminHash string   `json:"adminHash"`
	Argon     Argon2id `json:"argon"`
	APIKeys   []APIKey `json:"apiKeys"`
	PINHash   string   `json:"pinHash,omitempty"`
}

type Argon2id struct {
	MemKiB  uint32 `json:"memKiB"`
	Time    uint32 `json:"time"`
	Threads uint8  `json:"threads"`
	KeyLen  uint32 `json:"keyLen"`
	SaltLen uint32 `json:"saltLen"`
}

type APIKey struct {
	ID       string `json:"id"`
	Name     string `json:"name"`
	Hash     string `json:"hash"`
	Created  string `json:"created"`
	LastUsed string `json:"lastUsed,omitempty"`
}

type NodeRecord struct {
	ID        string       `json:"id"`
	Name      string       `json:"name"`
	CertPEM   string       `json:"certPem"`
	Addrs     []string     `json:"addrs"`
	HWDelayUs int          `json:"hwDelayUs"`
	Channel   string       `json:"channel"` // "stereo"|"left"|"right"
	GainDB    float64      `json:"gainDb"`
	Caps      Capabilities `json:"caps"`
	LastSeen  string       `json:"lastSeen,omitempty"`
}

type Capabilities struct {
	Render       bool     `json:"render"`
	Sinks        []string `json:"sinks"`   // "alsa","exec:aplay",...
	EncodeCodecs []string `json:"encode"`  // "pcm","opus"
	DecodeCodecs []string `json:"decode"`  // "pcm","opus"
	FEC          []string `json:"fec"`     // "none","xorParity","duplicate"
	MaxRate      int      `json:"maxRate"`
}

type GroupRecord struct {
	ID            string           `json:"id"`
	Name          string           `json:"name"`
	MemberNodeIDs []string         `json:"memberNodeIds"`
	Media         MediaSelection   `json:"media"`
	Playing       bool             `json:"playing"`
	Profile       TransportProfile `json:"profile"`
	MasterHint    string           `json:"masterHint,omitempty"`
}

type MediaSelection struct {
	File string `json:"file"`
	Loop bool   `json:"loop"`
}

type TransportProfile struct {
	Codec          string `json:"codec"`          // "pcm"|"opus"
	FEC            string `json:"fec"`            // "none"|"xorParity"|"duplicate"
	Rate           int    `json:"rate"`           // Hz
	FramesPerChunk int    `json:"framesPerChunk"` // default 480
	FECK           int    `json:"fecK"`           // 0 if FEC "none"
	Interleave     int    `json:"interleave"`     // 0/1 = none
	Transport      string `json:"transport"`      // "udp"|"tcp"
}

type RevokedSet struct {
	Entries []RevokedCert `json:"entries"`
}

type RevokedCert struct {
	Fingerprint string `json:"fingerprint"` // SHA-256 of cert DER, hex (match key)
	NodeID      string `json:"nodeId,omitempty"`
	Reason      string `json:"reason"` // "forget"|"takeover"|"rotate"
	At          string `json:"at"`     // RFC3339
}
```

> JSON uses **string enums** for codec/FEC/transport/channel; integer `CodecID`/`FECID`
> exist only at the §6.4 wire layer and **never** appear in this package (07 §2.4 note).

### 4.2 Store (`internal/state/store.go`)

```go
var ErrConflict = errors.New("config version conflict")

type Store struct {
	mu      sync.Mutex
	selfID  string
	doc     ConfigDoc
	changed chan struct{}
	path    string
	saveMu  sync.Mutex
}

func New(selfID string) *Store                              // empty, non-persistent (tests)
func Load(selfID, path string) *Store                       // seed from path; missing/corrupt => empty
func (s *Store) Get() ConfigDoc                             // deep copy; caller may mutate
func (s *Store) Apply(update ConfigDoc) (ConfigDoc, error)  // optimistic; ErrConflict on stale Version
func (s *Store) Merge(remote ConfigDoc, remoteID string)    // LWW + grow-only Revoked union
func (s *Store) Changed() <-chan struct{}                   // coalesced signal
func (s *Store) MarshalGossip() []byte                      // {nodeId, doc} envelope
func (s *Store) MergeGossip(b []byte)                       // parse envelope -> Merge
```

Unexported helpers (`internal/state/clone.go`): `cloneConfigDoc(ConfigDoc) ConfigDoc`,
`unionRevoked(a, b RevokedSet) RevokedSet`. Envelope type (`store.go`):
`type gossipState struct { NodeID string ` + "`json:\"nodeId\"`" + `; Doc ConfigDoc ` + "`json:\"doc\"`" + ` }`.

## 5. Behavior & algorithms

### 5.1 `Apply` (optimistic concurrency — 07 §4.5, A.6)
Reject when `update.Version != s.doc.Version` → return `(cloneConfigDoc(s.doc), ErrConflict)`,
store unchanged (mpvsync `state.go:123` verbatim). On match:
`next = cloneConfigDoc(update); next.Version = s.doc.Version + 1; next.UpdatedBy = s.selfID;
next.UpdatedAt = time.Now().UTC().Format(time.RFC3339)`; store; `save(out)` outside `mu`;
`signal()`; return `(out, nil)`. (`UpdatedBy`/`UpdatedAt` set is the only delta vs mpvsync;
A.6 line `next.UpdatedBy = self.id`. 07 §2.1: `UpdatedAt` is human/debug only, never a merge input.)

### 5.2 `Merge` (LWW by Version + tiebreak; grow-only Revoked union — A.6, 07 §4.3, §5.2)
```
take = remote.Version > doc.Version
     || (remote.Version == doc.Version && remote.Version != 0 && tiebreak)
merged = take ? cloneConfigDoc(remote) : doc
merged.Revoked = unionRevoked(doc.Revoked, remote.Revoked)   // ALWAYS, regardless of take
changed = take || len(merged.Revoked.Entries) != len(doc.Revoked.Entries)
if changed { doc = merged; save; signal }
```
**Tiebreak source (07 §2.1 resolution):** the envelope `remoteID` is the actual sender
and **wins** the tiebreak; compare `remoteID > s.selfID`. (07 §2.1: when `UpdatedBy` and
the envelope id differ, the envelope id wins for the tiebreak; `UpdatedBy` is advisory
provenance only. Use `remoteID`, matching mpvsync `state.go:151`.) `signal()` fires **only**
when the doc actually advanced or the Revoked set grew (07 §5.2).

### 5.3 `unionRevoked` (monotonic, dedup by fingerprint — 07 §4.3)
Build a `map[string]struct{}` of `Fingerprint`s; append every entry from `a` then from `b`
whose fingerprint is unseen; preserve insertion order (resident first, then incoming) so
output is deterministic. Never removes entries (compaction is an out-of-band admin op, 07 §4.3).
A fingerprint present in both keeps the **first-seen** `RevokedCert` value (resident wins
metadata; fingerprint is the identity).

### 5.4 `cloneConfigDoc` (deep copy — 07 §5.1)
Copy scalars by value (`Version`, `UpdatedBy`, `UpdatedAt`, `Cluster`, `Secrets`, `Auth`
scalars + `Argon`). Deep-copy every slice so a caller mutating `Get()`'s result cannot
race the store: `Auth.APIKeys`; `Nodes` and per-node `Addrs`, `Caps.Sinks`,
`Caps.EncodeCodecs`, `Caps.DecodeCodecs`, `Caps.FEC`; `Groups` and per-group
`MemberNodeIDs` (`Media`, `Profile` are value structs — copied by assignment);
`Revoked.Entries`. Preserve `nil` vs empty-slice distinction (mpvsync clones only when
`!= nil`) so JSON round-trips are byte-stable. The `Secrets` value is copied with the rest
(07 §5.1: "The `Secrets` value is copied with the rest of the doc").

### 5.5 Persistence (`save` — 07 §5.3, A.6)
`json.MarshalIndent(doc, "", "  ")`; under `saveMu` (outside `mu`, on the snapshot passed in):
`MkdirAll(dir, 0o700)`, write `path+".tmp"` at `0o600`, `Chmod(tmp, 0o600)` (defeat umask),
`Rename(tmp, path)`, `Chmod(path, 0o600)`. Best-effort: any error is logged to stderr and
**never** fails `Apply`/`Merge`. No-op when `path == ""`. `Load` seeds the in-memory doc from
a well-formed file; missing/corrupt is non-fatal → empty doc (gossip reconciles highest
`Version` on rejoin).

### 5.6 `Changed`/`signal` & gossip envelope
`signal()` is a non-blocking send into a `chan struct{}` of cap 1 (coalesced; mpvsync
`state.go:238`). `MarshalGossip` emits `{nodeId: selfID, doc}`; `MergeGossip` unmarshals and
calls `Merge(env.Doc, env.NodeID)`; malformed/empty payloads ignored (02 §4.2, A.14.2 bridge).

### 5.7 Tunables referenced (verbatim from A.12)
This package stores but does not generate tunables; the only A.12 values that appear as
defaults in fixtures/tests are: canonical **rate 48000**, **FramesPerChunk 480 (10 ms)**,
FEC XOR **k = 8 / interleave depth D = 4**, FEC dup offset **Ddup = 5**, ports **control
mTLS 8443 / clock UDP 9000 / audio UDP 9100**, argon2id cost example **memKiB 65536, time 3,
threads 4, keyLen 32, saltLen 16** (07 §2.3 example / A.12). Tests must not invent other
numbers — quote A.12.

## 6. Import constraints (01 §2 layering)

`internal/state` is a **leaf** in the dependency graph: it imports **only the Go stdlib**.
It must NOT import `cluster`, `group`, `allowlist`, `auth`, `pki`, `web`, `stream/*`, or
`audio/*` — those all depend on `state`, never the reverse. `web` never imports `state`
directly; it sees the doc only through the `web.Deps.State func() ConfigDoc` seam (A.14.3,
spine §6.6 layering rule). The `cluster` delegate (A.14.2) bridges gossip into the store via
the public `MarshalGossip`/`MergeGossip`, so `state` does not import `cluster` either. No
third-party deps (no `netip`, no memberlist) enter this package.

### 7.2 `store_test.go`
| Test | Asserts |
|---|---|
| `Apply` happy path | `Version` bumps by 1; returned doc has `UpdatedBy==selfID` and non-empty `UpdatedAt`; `Changed()` fires once. |
| `Apply` conflict | stale `update.Version` → `(currentDoc, ErrConflict)`; store unchanged; `Changed()` does NOT fire. (A.13 P2 / 07 §4.5.) |
| `Apply` then `Get` isolation | mutate the `Get()` result; a fresh `Get()` is unaffected. |
| `Merge` higher version wins | remote `Version` > local → local adopts remote; `Changed()` fires; `save` runs. |
| `Merge` lower/equal-loses | remote `Version` < local, or equal with `remoteID < selfID` → doc unchanged (but Revoked still unioned — next row). |
| `Merge` tiebreak by id | equal non-zero `Version`, `remoteID > selfID` → remote taken; `remoteID < selfID` → kept. Deterministic both directions. |
| `Merge` Revoked union on losing doc | remote LOSES the LWW doc but carries a new `RevokedCert` → that fingerprint is present after merge; `Changed()` fires because Revoked grew. (07 §4.3 headline.) |
| `Merge` Revoked dedup | overlapping fingerprints in both docs → no duplicates; count == size of the union. |
| `Merge` resurrection guard | local forgets X (removes `NodeRecord`, adds X cert to `Revoked`, `Version=50`); remote stale doc still lists X at `Version=51` → after merge X's `NodeRecord` may be back (LWW) BUT X's fingerprint is still in `Revoked` (proves the §4.1 mitigation). |
| `Merge` Version 0 never tiebreaks | both `Version==0` → no take regardless of id (mpvsync `remote.Version != 0` guard). |
| Persistence round-trip | `Load(self, path)` on empty dir → empty doc; `Apply`; new `Load` from same path sees the bumped doc with the persisted `Version`. |
| Persistence mode | after a `save`, `os.Stat(path).Mode().Perm() == 0o600` and the temp file is gone. (07 §5.3 / A.6.) |
| Persistence corrupt file | seed `path` with garbage bytes → `Load` returns empty doc, no panic. |
| Persistence missing dir | `path` under a non-existent subdir → `save` creates it `0o700`, write succeeds. |
| `MarshalGossip`/`MergeGossip` | marshal from store A, merge into store B → B converges to A's doc; `env.NodeID == A.selfID`. Malformed/empty bytes → no-op, no panic. |
| `Changed` coalescing | two rapid `Apply`s with a full channel → at most one pending signal (non-blocking send). |

### 7.3 Integration / acceptance (A.13 P2, 07 acceptance)
The store's *contribution* to P2's acceptance ("node B adopted from A; doc converges;
forgotten node's packets dropped + cert rejected") is exercised here at the unit level via
the `Merge`/`MarshalGossip` convergence + resurrection-guard tests; the full two-process
gossip convergence test belongs to the `cluster`/`allowlist` pieces that consume this store
(P2.x). This piece's DoR: `go test ./internal/state/...` green and `go build` clean under
`CGO_ENABLED=0 GOOS=linux GOARCH=arm64`.

## 9. Risks / open questions

1. **`Secrets` placement (07 §2.8 alternative).** 07 §2.8 permits carrying `ClusterSecrets`
   either inline in `ConfigDoc` (this plan) **or** as a sibling replicated blob to keep the
   CA key out of every doc snapshot served to the UI. This plan embeds it inline (simpler,
   single gossip envelope) and relies on `web` **never** serializing `Secrets` to any read
   endpoint (the `Deps.State` consumer must redact). **Open:** confirm with the P-owner of
   `web`/`pki` that inline-with-redaction is acceptable vs. a sibling blob. Recommendation:
   inline now; revisit only if a UI snapshot leak is found.
2. **`UpdatedBy` vs envelope id tiebreak.** 07 §2.1 says the envelope id wins the tiebreak
   and `UpdatedBy` is advisory; A.6 pseudocode tiebreaks on `UpdatedBy`. This plan follows
   07 §2.1 (use `remoteID`, matching mpvsync `state.go:151`). Flagging the appendix↔07 wording
   gap so the orchestrator can confirm 07 §2.1 is authoritative. **Low risk** — both converge;
   only matters when a doc's `UpdatedBy` disagrees with its actual gossip sender.
3. **`Argon2id`/`Created`/`At` defaults.** Whether the empty `ConfigDoc` from `New`/first-init
   carries zero-value argon params or P1.x (auth) writes sane defaults on setup is an `auth`
   concern, not `state`. `state` stores whatever it is handed. Noted so a downstream piece owns
   seeding A.12 argon costs.
4. **`UpdatedAt` clock.** Set from `time.Now().UTC()` in `Apply` only; `Merge` carries whatever
   the winning doc had (never regenerated). Confirmed non-merge-input (07 §2.1) — no risk, but
   stated to avoid a reviewer "wall-clock in merge" objection.
