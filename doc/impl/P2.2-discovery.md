## 1. Purpose & scope

`internal/discovery` is the mDNS bootstrap layer: it advertises this node over DNS-SD
(`_ensemble._udp`) carrying the Ensemble TXT key set (`id/name/cf/gid/init/ctrl/clk/aud/wp`,
doc 02 §2.2), and provides two browse modes — `Browse` (keyed by **cluster fingerprint**,
returning memberlist join seeds for *our* cluster) and `BrowseAll` (a survey of every
advertised node as `DiscoveredNode`, for the UI). It also supplies the **5-state device
classification** (doc 02 §2.4) that extends mpvsync's 4-state `buildDevices` with the
`uninitialized` state. Discovery's job ends at *surfacing* peers and adoptable nodes; trust
(adoption/PKI) and liveness (memberlist gossip, election) are out of scope and live in
`internal/cluster` and downstream pieces. This is a direct re-home + rewrite of the proven
mpvsync `internal/cluster/discovery.go`, not new architecture.

## 4. Public API (canonical names — doc 02 §2.2–2.4)

```go
package discovery

const (
    mdnsService = "_ensemble._udp" // doc 02 §2.2
    mdnsDomain  = "local."
)

// Discovery holds the live mDNS advertisement (wraps *zeroconf.Server).
type Discovery struct { /* server *zeroconf.Server */ }

// Announce describes everything advertised over mDNS for this node.
// The memberlist gossip port is the SRV/A-record port, passed separately to Register.
type Announce struct {
    NodeID      string // TXT "id"
    Name        string // TXT "name"
    ClusterFP   string // TXT "cf"  — "" ⇒ uninitialized
    GroupID     string // TXT "gid"
    Initialized bool   // TXT "init" == "1"
    ControlPort int    // TXT "ctrl" — mTLS API (default 8443, A.12)
    ClockPort   int    // TXT "clk"  — clock UDP (default 9000, A.12)
    AudioPort   int    // TXT "aud"  — audio UDP (default 9100, A.12)
    WebPort     int    // TXT "wp"   — browser deep-link
}

// Register advertises this node. gossipPort is the memberlist gossip port (the SRV/A
// record port peers Join); all other fields ride TXT. (cf. mpvsync discovery.go:31)
func Register(a Announce, gossipPort int) (*Discovery, error)

// Close withdraws the mDNS advertisement.
func (d *Discovery) Close()

// Browse returns memberlist seed addrs ("ip:port", gossip port) for peers in OUR cluster
// (TXT cf == clusterFP), excluding selfID, for the duration of ctx.
// (mpvsync discovery.go:55 Browse — rekeyed group→cf.)
func Browse(ctx context.Context, clusterFP, selfID string) ([]string, error)

// BrowseAll surfaces EVERY advertised node (all clusters, uninitialized included) for the
// UI's device classification. Result is meant to be cached (do NOT browse per WS tick).
// (mpvsync discovery.go:106 BrowseAll.)
func BrowseAll(ctx context.Context) ([]DiscoveredNode, error)

// DiscoveredNode is one mDNS-advertised peer (doc 02 §2.3, verbatim).
type DiscoveredNode struct {
    NodeID      string
    Name        string
    ClusterFP   string // TXT "cf" — "" means uninitialized
    GroupID     string // TXT "gid"
    Initialized bool   // TXT "init" == "1"
    Addr        string // first IPv4
    Port        int    // memberlist gossip port
    ControlPort int    // TXT "ctrl" — mTLS API
    ClockPort   int    // TXT "clk"  — clock UDP
    AudioPort   int    // TXT "aud"  — audio UDP
    WebPort     int    // TXT "wp"
}
```

### Classification (doc 02 §2.4 — 5 states)

```go
package discovery

// NodeState is the UI device classification (doc 02 §2.4, table).
type NodeState string

const (
    StatePresent       NodeState = "present"       // same cluster, alive gossip peer
    StateOffline       NodeState = "offline"       // known member, not currently alive
    StateUninitialized NodeState = "uninitialized" // init=0 / cf=="" ⇒ "Adopt"
    StateForeign       NodeState = "foreign"       // cf != "" and cf != ourCF ⇒ "Takeover"
    StateDiscovered    NodeState = "discovered"    // same cf, advertised, not yet a gossip member
)

// ClassifyInput is the read-only view Classify needs. The caller (cmd/web) supplies these
// sets from cluster.Members() (alive), the ConfigDoc/peer-cache (known members), and
// BrowseAll() (the mDNS survey). Discovery owns the RULE, not the data sources.
type ClassifyInput struct {
    OurClusterFP string                    // "" if this node is itself uninitialized
    AliveIDs     map[string]bool           // ids alive in gossip (same cluster) — cluster.Members()
    KnownIDs     map[string]bool           // ids that are adopted members (ConfigDoc.Nodes ∪ peer cache)
    Discovered   []DiscoveredNode          // BrowseAll() survey (cached)
}

// Device is one classified row for the UI (id + chosen state + the survey record if any).
type Device struct {
    NodeID string
    State  NodeState
    Node   *DiscoveredNode // nil for an offline known-but-not-currently-advertised member
}

// Classify folds alive/known/discovered into one row per node id, applying the precedence
// present > offline > {discovered|foreign|uninitialized} (mpvsync api_cluster.go:60).
func Classify(in ClassifyInput) []Device
```

**Classification rule (doc 02 §2.4 table + precedence):**

1. Every id in `KnownIDs`: `present` if also in `AliveIDs`, else `offline`.
2. Every `DiscoveredNode` in the survey, for ids **not** already resolved to present/offline:
   - `!Initialized || ClusterFP == ""` → `uninitialized`.
   - `ClusterFP != "" && ClusterFP != OurClusterFP` → `foreign`.
   - `ClusterFP == OurClusterFP` (same cluster, advertised, not yet a gossip member) →
     `discovered` (transient; should converge to `present`).
3. Skip the survey row whose `NodeID == "" ` and dedup by id (a present/offline row wins
   over a survey row for the same id — the precedence).

## 5. Behavior & algorithms

- **mDNS is bootstrap-only** (doc 02 §2.1). `Browse` exists to seed `memberlist.Join`; it
  may be slow/lossy/wrong — gossip is the source of truth for membership, never mDNS.
- **What is advertised** (doc 02 §2.2): the positional `Register` port is the **memberlist
  gossip port** (SRV/A); the nine TXT keys carry identity/cluster/group/init-flag/ports.
  `cf=""` + `init=0` marks an uninitialized node (the adoption hook).
- **Two browse modes** (doc 02 §2.3): `Browse` filters `txt["cf"] != clusterFP` and excludes
  self; `BrowseAll` surfaces all clusters and dedups by id, selecting the first IPv4.
  `BrowseAll` is **meant to be cached** by the caller on the 5 s `rebrowse` ticker — the web
  layer reads the cache and never browses per WS tick (doc 02 §2.3, §3.5).
- **5-state classification** (doc 02 §2.4): see §4 rule above; precedence mirrors mpvsync.
- **Tunables (A.12, quoted verbatim):**
  - Ports — `control mTLS / clock UDP / audio UDP = 8443 / 9000 / 9100`. These are the
    defaults the caller passes into `Announce.{ControlPort,ClockPort,AudioPort}`; discovery
    does not invent them, it only transcribes.
  - Rebrowse cadence — `mDNS browse / survey: 5 s ticker, 1 s ctx` (doc 02 §2.3, §3.5;
    mpvsync `node.go:705`). This is the **caller's** loop policy, not enforced inside the
    package; callers pass a 1 s-deadline `ctx` to each browse call and tick every 5 s.
- **`protocolEpoch` is fixed at 1** (project invariant). Not a discovery TXT key in this
  build (epoch policy is enforced at adoption, README §6.6); do **not** add an epoch TXT key.

## 9. Risks / open questions

1. **TXT size budget.** Nine TXT key=value pairs vs. mpvsync's four. DNS-SD TXT records have
   practical size limits (each string ≤255 bytes; total ideally within one MTU). The keys are
   short and values small (ids, names, decimal ports), so this is well within budget — but
   confirm `zeroconf` does not silently truncate when `Name` is long. Low risk; note for the
   live-mDNS spike.
2. **`ClassifyInput` ownership boundary.** This spec puts the classification *rule* in
   `discovery` but the *data adaptation* in `cmd`/`web`, to keep `discovery` a leaf package and
   honor the `web.Deps` seam (doc 01 §2). **Open question for the orchestrator:** confirm this
   split is acceptable vs. putting `Classify` in `internal/web`. If `web` is preferred, P2.2
   ships only `discovery.go` (mDNS) and the 5-state classifier moves to the web piece — the
   doc 02 §8 citation index lists `buildDevices` under `internal/web` for mpvsync, which argues
   for the web home. **This is the one decision that changes P2.2's file list.**
3. **`zeroconf` maintenance / arm64.** `grandcat/zeroconf v1.0.0` is the pinned, mpvsync-proven
   lib; it is pure-Go (cross-compiles clean). No action unless the live spike shows multicast
   issues on the Pi's wlan interface.
4. **`init` TXT vs. gossip `Meta`.** A node's `cf`/`gid`/`init` appear both in mDNS TXT and in
   gossip `Meta` (doc 02 §3.1); they can briefly disagree mid-move (doc 02 §7 table). Discovery
   only reports what mDNS says; reconciliation is the cluster/election piece's job — documented,
   not a discovery bug.
