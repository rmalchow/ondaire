## 1. Purpose & scope

`internal/cluster` is the LAN membership and leadership layer of an Ensemble node. It
wraps `hashicorp/memberlist` (SWIM gossip) behind an Ensemble-shaped `Membership` type
that gossips a per-node `Meta` blob and bridges the replicated `ConfigDoc` into
memberlist push/pull anti-entropy (A.14.2) via a `delegate`; it persists previously-seen
gossip seeds to `peers.json` for fast rejoin (`PeerStore`, A.14.1); and it runs **one
master election per group** (`GroupElections` over `cluster.Election`) using the soft
`MasterHint` + lowest-stable-id + generation rule (02 §5 / A.5). It is the steady-state
machinery a node enters after discovery (`internal/discovery`, P2.2) bootstraps a join —
discovery only *surfaces* peers; membership/election/replication are owned here. This
package is the input to the per-group clock and group engine ([04](../04-clock-and-groups.md)),
and to the realtime allowlist ([03](../03-adoption-takeover-security-pki.md)/A.8): both
consume `Members()` and the per-group `Outcome` produced here.

## 4. Public API (canonical names — README §6, 02 §3/§5, A.14)

```go
package cluster

// Meta is the per-node metadata gossiped via memberlist Node.Meta (02 §3.1).
type Meta struct {
    NodeID      string `json:"id"`
    Name        string `json:"name"`
    GroupID     string `json:"gid"`
    ClusterFP   string `json:"cf"`   // cluster CA fingerprint; "" => uninitialized
    ControlPort int    `json:"ctrl"` // mTLS HTTP API
    ClockPort   int    `json:"clk"`  // clock UDP
    AudioPort   int    `json:"aud"`  // audio UDP
    WebPort     int    `json:"wp"`
}

// Member is an alive peer in the cluster (self included).
type Member struct {
    Addr net.IP
    Port uint16 // memberlist gossip port (from the live memberlist Node)
    Meta Meta
}
func (m Member) GossipAddr() string   // ip:port — the form memberlist.Join accepts
func (m Member) ControlAddr() string  // ip:Meta.ControlPort
func (m Member) ClockAddr() string    // ip:Meta.ClockPort
func (m Member) AudioAddr() string    // ip:Meta.AudioPort
func (m Member) WebAddr() string      // ip:Meta.WebPort

type Config struct {
    NodeID      string
    Name        string
    GroupID     string
    ClusterFP   string
    BindAddr    string       // default "0.0.0.0"
    BindPort    int          // memberlist gossip port (required) — A.12: 7946
    ControlPort int          // A.12: 8443
    ClockPort   int          // A.12: 9000
    AudioPort   int          // A.12: 9100
    WebPort     int
    SecretKey   []byte       // gossipKey (32 random bytes) from ClusterSecrets — NOT HKDF(password)
    Seeds       []string     // explicit join addresses (host or host:port)
    LogOutput   io.Writer    // nil => io.Discard
    State       *state.Store // shared ConfigDoc store; bridged into anti-entropy (A.14.2)
}

type Membership struct{ /* unexported */ }

func New(cfg Config) (*Membership, error)
func (m *Membership) Members() []Member            // alive peers incl. self, decoded Meta
func (m *Membership) Self() Meta
func (m *Membership) Join(addrs []string) (int, error)
func (m *Membership) Changed() <-chan struct{}     // coalesced membership-change signal
func (m *Membership) NumMembers() int
func (m *Membership) BindPort() int
func (m *Membership) Leave() error                 // ml.Leave(0) + Shutdown()
func (m *Membership) UpdateName(name string) error // re-broadcasts renamed Meta (UpdateNode 5s)
func (m *Membership) Rekey(key []byte) error       // runtime primary-key rotation (encrypted-start only)

// ── Election (one instance per group; reused mpvsync election.go + A.5 hint) ──
type Election struct{ /* selfID, masterID, generation */ }
func NewElection(selfID string) *Election
// Update: soft MasterHint preference, else lowest stable id among alive members.
func (e *Election) Update(members []Member, hint string) (masterID string, changed bool)
func (e *Election) Master() string
func (e *Election) Generation() uint64  // ++ on every master change
func (e *Election) IsMaster() bool

// ── Per-group elections (02 §5.2) ──
type Outcome struct {
    MasterID   string
    Generation uint64
    IsSelf     bool
}
type GroupElections struct{ /* mu; e map[string]*Election; selfID */ }
func NewGroupElections(selfID string) *GroupElections
// Recompute runs each group's election over alive members assigned to that group
// (ConfigDoc.Groups[].MemberNodeIDs ∩ alive ∩ Meta.GroupID match), honoring the
// group's MasterHint; returns only the groups whose master CHANGED.
func (g *GroupElections) Recompute(doc state.ConfigDoc, alive []Member) (changed map[string]Outcome)
func (g *GroupElections) Master(groupID string) string
func (g *GroupElections) Generation(groupID string) uint64
func (g *GroupElections) Drop(groupID string)   // forget a deleted group's Election

// ── PeerStore / peers.json (A.14.1) ──
type Peer struct {
    ID         string `json:"id"`
    Name       string `json:"name"`
    GossipAddr string `json:"gossipAddr"`
    LastSeen   int64  `json:"lastSeen"` // unix seconds
}
type PeerStore struct{ /* path; mu; peers map[string]Peer */ }
func LoadPeerStore(path string) *PeerStore   // <data>/peers.json; missing/corrupt => empty
func (s *PeerStore) Upsert(members []Member) // refresh + persist (0644, atomic)
func (s *PeerStore) JoinSeeds() []string     // stored gossip addrs to seed memberlist.Join
func (s *PeerStore) Snapshot() []Peer
func (s *PeerStore) Remove(id string)
func (s *PeerStore) Clear()                  // wipe on leave/forget (02 §3.3)
```

> `state.Store`, `state.ConfigDoc` come from `internal/state` (P2.1). `MasterHint` is a
> field of `state.GroupRecord` (README §6.5 + D23); this package reads it, never writes it.

## 5. Behavior & algorithms

**Membership wrapper (A.14.2, 02 §3).** Base profile `memberlist.DefaultLANConfig()`.
A.14.2 fields-we-set: `Name = cfg.NodeID`; `BindAddr/BindPort = 0.0.0.0 / 7946`;
`AdvertiseAddr = observed LAN ip`, `AdvertisePort = BindPort`; LAN preset timings
**`GossipInterval = 200 ms`, `ProbeInterval = 1 s`, `ProbeTimeout = 500 ms`** (verbatim
A.14.2); `Delegate = &delegate{meta, state}`; `Events` coalesced into one `changed`
signal (02 §3.1). `SecretKey`, when non-empty, enables gossip encryption keyed by
`gossipKey` (02 §3.2); `Rekey` rotates the primary key at runtime **only if started
encrypted** (open→encrypted needs a coordinated restart). Join is non-fatal on error (we
may be the first node; anti-entropy reconciles).

**ConfigDoc bridge (02 §4, A.14.2).** `delegate.LocalState(bool)` ships
`state.MarshalGossip()` (doc + self id); `MergeRemoteState(buf,_)` calls
`state.MergeGossip(buf)`, which applies **LWW by `Version`, `UpdatedBy` tiebreak, and
grow-only `RevokedSet` union** (A.6) — that logic lives in `internal/state` (P2.1), not
here. `NotifyMsg`/`GetBroadcasts` stay unused; periodic push/pull at LAN
`PushPullInterval` (~30 s) suffices for an infrequently-edited doc (02 §4.1).

**Per-group election (02 §5, A.5).** For each `GroupRecord`, `Recompute`:
1. `candidates = { alive members whose id ∈ GroupRecord.MemberNodeIDs AND whose gossip
   Meta.GroupID == GroupRecord.ID }` (02 §5.2 step 1 / §7 intersection — excludes a node
   mid-move advertising a stale `gid`).
2. `master = (GroupRecord.MasterHint ∈ candidates) ? MasterHint : min(candidates by id)`
   — A.5 soft preference; hint honored **only while alive & member**, else lowest stable
   id. A group with zero candidates ⇒ `master == ""` (NO-MASTER / `starting`).
3. Call the group's `Election.Update(candidates, hint)`; on change read `Generation()`
   and emit `Outcome{MasterID, Generation, IsSelf: master==selfID}`.
The rule is a **pure, stateless function of the agreed member set**, so all nodes
converge on the same master per group once they share `(ConfigDoc, alive-set)`
(02 §5.1 / §6). `Generation` bumps on every master change and fences stale masters; a
master change is also a stream-origin change ⇒ the group engine bumps `streamGen`
(02 §5.3 / A.5 rejoin-flip cost; consumed downstream in P3/P4, not here).

**Failover & split-brain (02 §6).** SWIM suspect→dead ⇒ `Changed()` ⇒ `Recompute` ⇒
next-lowest (or hinted) live id masters. Partitions may transiently run two masters per
group; on heal the union elects the single global winner and the loser demotes
(`IsMaster()` flips false). No quorum/Raft (spine choice). Triggers: every `Changed()`
**and** a `membershipTick` safety re-eval (02 §3.5 table: **2 s**) **and** on
`ConfigDoc.Groups` change (A.5).

**Timing/tunables quoted verbatim (A.12).** Gossip port `7946`; control mTLS `8443`,
clock UDP `9000`, audio UDP `9100`. SWIM (A.14.2): `GossipInterval 200 ms`,
`ProbeInterval 1 s`, `ProbeTimeout 500 ms`. Suspicion window ≈ `SuspicionMult × log(N) ×
ProbeInterval` (≈ 4–5 s small N, 02 §3.4) — from `DefaultLANConfig()`, not overridden.
Safety re-eval tick `2 s`, mDNS rebrowse `5 s` (owned by P2.2/wiring). `peers.json` mode
`0644`. **No tunable in this package is invented; all come from A.12 / A.14.2 /
`DefaultLANConfig()`.**

**Resource note (low-end target).** `Members()` re-decodes Meta on each call — callers
re-read on `Changed()`/2 s tick, not per packet (02 §3.5). `PeerStore` is ≤8 entries,
never pruned, best-effort disk I/O off the hot path. Election is O(candidates) per group,
trivial. No per-packet crypto here (gossip encryption is memberlist's, low-volume).

reaches up.

## 9. Risks / open questions

- **Q1 (BLOCKING-ish, schema mismatch).** A.14.1 specifies `PeerStore` as
  `Upsert(addr string)` / `JoinSeeds() []string` / `Clear()` with disk shape
  `{"peers":["host:7946",...]}`, but mpvsync's richer `Peer{ID,Name,GossipAddr,LastSeen}`
  array is what the `offline` UI state (02 §2.4/§3.3) and `Snapshot()` need. This spec
  keeps the **richer** mpvsync form (`Upsert(members []Member)`, `[]Peer` JSON, +`Remove`/
  `Snapshot`). **Orchestrator must confirm** we may diverge from A.14.1's flat-string
  contract (recommended), or whether A.14.1 should be amended to the richer shape.
- **Q2.** `Election.Update` signature gains a `hint string` param (A.5) vs. mpvsync's
  `Update(members)`. README §6 and 02 §5.1 show the no-hint form; A.5 mandates the hint.
  This spec adds the param (canonical-by-A.5). Confirm callers (P3 group engine) expect
  the 2-arg form.
