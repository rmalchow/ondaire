---

## 1. Purpose & scope

`internal/allowlist` is the **source-IP gate** that is the *entire* defense on the two
unauthenticated realtime planes ‚Äî clock UDP (`:9000`) and audio UDP/TCP (`:9100`) ‚Äî per
D2/D6/D5 and **03 ¬ß0/¬ß6**. It derives the set of allowed source IPs as the **union** of
every `NodeRecord.Addrs` in the replicated `ConfigDoc` (durable, from `internal/state`,
P2.3) and the currently-alive gossip members' observed addresses (live, from
`internal/cluster`, P2.1) ‚Äî exactly **07 ¬ß3.1 / A.8**. The set is held in an atomically
swapped snapshot so the per-packet check is lock-free; it is **recomputed** whenever
`state.Store.Changed()` or `cluster.Membership.Changed()` fires (**07 ¬ß3.2**). A
`GateUDP` wrapper drains a `*net.UDPConn` and drops (silently, no reply) any datagram
whose source IP is not in the snapshot **before** the bytes reach the (untrusted-input)
clock/audio decode/FEC code (**03 ¬ß6.3**). Nothing in this package authenticates,
encrypts, or parses payloads; it is a coarse, fail-safe filter.

## 4. Public API (canonical names ‚Äî 03 ¬ß6.3, 07 ¬ß3.1, A.8)

```go
// Package allowlist is the source-IP gate guarding the unauthenticated realtime
// planes (clock UDP, audio UDP/TCP). It is the ONLY protection on those planes (03 ¬ß6).
package allowlist

import (
    "net"
    "net/netip"

    "gitlab.rand0m.me/ruben/go/ensemble/internal/state"
)

// MemberAddr is one live gossip member's observed source address (07 ¬ß3.1).
// The cluster layer (P2.1) supplies these; only the IP is load-bearing for the gate
// (realtime ports are fixed per plane, 04/05), so Port is informational.
type MemberAddr struct {
    Addr netip.Addr
}

// Set is the source-IP gate. It holds an atomically swapped snapshot of the allowed
// IP set so the UDP read path does a lock-free, per-packet lookup (07 ¬ß3.2, 03 ¬ß6.3).
// The zero value denies everything until the first Update.
type Set struct {
    // unexported: current atomic.Pointer[map[netip.Addr]struct{}]
}

// New returns an empty Set (denies all until Update). Used by cmd wiring and tests.
func New() *Set

// Allowed reports whether a source IP may send on the realtime planes (03 ¬ß6.3).
// Lock-free: reads the current atomic snapshot. An un-parseable/zero ip is denied.
func (s *Set) Allowed(ip net.IP) bool

// AllowedAddr is the netip.Addr fast path (no net.IP alloc on the hot recv path).
func (s *Set) AllowedAddr(ip netip.Addr) bool

// Update atomically swaps in a freshly derived snapshot. Called whenever
// ConfigDoc.Nodes OR the live member set changes (07 ¬ß3.2). Idempotent.
func (s *Set) Update(doc state.ConfigDoc, live []MemberAddr)

// AllowedSources is the pure derivation function (07 ¬ß3.1, A.8): the union of every
// node's ConfigDoc Addrs and the live members' observed addrs. Exported + side-effect
// free so it is unit-testable in isolation. A forgotten node has no NodeRecord and so
// contributes nothing (07 ¬ß3.1, ¬ß4.2).
func AllowedSources(doc state.ConfigDoc, live []MemberAddr) map[netip.Addr]struct{}

// GateUDP runs a blocking read loop over conn: it reads datagrams and, for each whose
// source IP is in set, calls deliver(src, payload); packets from disallowed sources are
// dropped silently (no reply) BEFORE deliver, so untrusted bytes never reach the
// clock/audio decoders (03 ¬ß6.3, A.8). It returns when the conn is closed or ctx is
// done. The []byte handed to deliver is only valid for the duration of the call
// (reused read buffer); deliver must copy if it retains it.
func GateUDP(ctx context.Context, conn *net.UDPConn, set *Set, deliver func(src netip.AddrPort, b []byte)) error

// Run is the recompute loop (07 ¬ß3.2): on every state.Store.Changed() or
// membership-changed signal it rebuilds the Set from store.Get() + liveFn(). It blocks
// until ctx is done. liveFn returns the current alive member addrs (cluster.Members()
// adapted to []MemberAddr). The two channels are the SAME coalesced Changed() signals
// used elsewhere (state P2.3, cluster P2.1).
func Run(ctx context.Context, set *Set, store *state.Store,
    membershipChanged <-chan struct{}, liveFn func() []MemberAddr) error
```

> Naming note: 03 ¬ß6.3 sketches `Update(configAddrs, liveMemberAddrs []net.IP)` and
> `GateUDP(conn, set, deliver func(src *net.UDPAddr, b []byte))`; 07 ¬ß3.1 specifies the
> richer `AllowedSources(doc ConfigDoc, live []MemberAddr) map[netip.Addr]struct{}`.
> **07 ¬ß3.1 is authoritative for the derivation signature** (it carries the doc, so the
> "forgotten node contributes nothing" property is structural). The API above adopts the
> 07 forms and adds the lock-free `netip.Addr` recv-path variants for the Pi hot path
> (¬ß9 Q2 records the `net.UDPAddr`‚Üí`netip.AddrPort` choice).

## 5. Behavior & algorithms

### 5.1 Derivation (07 ¬ß3.1, A.8)

```
allowed = ‚à™ { netip.ParseAddr(a) : a ‚àà n.Addrs, n ‚àà doc.Nodes }   // durable, survives restart
        ‚à™ { m.Addr               : m ‚àà live }                      // live, covers DHCP/multi-home lag
```
- **Union, not intersection** (07 ¬ß3.1): a node may be a valid source before its
  `NodeRecord.Addrs` catches up (fresh DHCP lease) or while memberlist briefly suspects
  it; the union avoids dropping legitimate realtime packets in those transients.
- **Host-only match.** `Addrs` are stored without ports; the gate matches source **IP**
  only ‚Äî realtime ports are fixed per plane (control mTLS `:8443`, clock UDP `:9000`,
  audio UDP `:9100`, A.12). Multi-homed nodes list every reachable address; all enter the
  set (07 ¬ß3.3).
- **Un-parseable `Addrs` are skipped silently** (07 ¬ß3.1 example uses
  `netip.ParseAddr(a); err == nil`), never failing the rebuild.
- A **forgotten** node has no `NodeRecord` (07 ¬ß4.2) so it contributes nothing ‚Äî the
  whole point of deriving from the doc rather than a separate list.

### 5.2 Atomic snapshot + recompute (07 ¬ß3.2)

The set is swapped with `atomic.Pointer[map[netip.Addr]struct{}]`; the UDP read path does
a lock-free lookup. `Run` blocks on `select` over the two coalesced channels and rebuilds
on either:

```
for {
    select {
    case <-ctx.Done():            return ctx.Err()
    case <-store.Changed():       // ConfigDoc advanced (Apply or Merge), README ¬ß6.5 / A.6
    case <-membershipChanged:     // live peer set changed (P2.1 / 02 ¬ß3.1)
    }
    set.Update(store.Get(), liveFn())   // store.Get() returns a deep copy (07 ¬ß5.1)
}
```
Recompute is **cheap and rare** (a few dozen IPs; human-driven config + occasional
membership flaps), so coalescing through the single `Changed()` signal is sufficient
(07 ¬ß3.2 ‚Äî same rationale mpvsync gives). Do a one-shot `set.Update(...)` before entering
the loop so the gate is populated at startup (otherwise the zero `Set` denies all until
the first signal).

### 5.3 The gate (03 ¬ß6.3, A.8)

```
GateUDP: buf := make([]byte, maxPkt)
  loop:
    n, src, err := conn.ReadFromUDPAddrPort(buf)   // netip.AddrPort, no alloc
    if err != nil { return err (closed/ctx) }
    if !set.AllowedAddr(src.Addr()) { continue }   // DROP silently, no reply (A.8)
    deliver(src, buf[:n])                           // only now do bytes reach decoders
```
- The gate runs **in the recv path** so a rejected packet never touches the codec/FEC
  code (03 ¬ß6.3) ‚Äî the codecs treat input as untrusted, and this keeps malformed/foreign
  packets from ever reaching them.
- `maxPkt`: size the read buffer to the max realtime datagram. Audio: ¬ß6.4 header (44 B)
  + payload; for the canonical 480-frame/2-ch **S16LE** wire PCM (A.10 "m5", A.12) that is
  `44 + 480*2*2 = 1964 B`; clock packets are the fixed **40-byte** mpvsync format (A.1).
  Use a single safe constant (e.g. 2048) for the shared buffer; exact sizing is the
  clock/audio pieces' concern, P2.4 just must not truncate them.

### 5.4 Honest limits (carry into doc comments ‚Äî 03 ¬ß6.4, 07 ¬ß3.4)

This gate stops off-path / non-member strangers, **not** an on-link source-IP spoofer
(D2/D5/D6 accept unauthenticated realtime). Stale-doc over-permission (a not-yet-merged
forget still lists the IP) is bounded by gossip convergence; control-plane access is cut
immediately by cert revocation (03 ¬ß5, the `RevokedSet`). State these as comments, not
behavior ‚Äî P2.4 implements the gate, not these mitigations.

### 5.5 Quoted tunables (A.12, verbatim ‚Äî never invent)

| Domain | Parameter | Start value |
|---|---|---|
| Ports | control mTLS / clock UDP / audio UDP | `8443 / 9000 / 9100` |
| Audio canonical | rate / channels / FramesPerChunk | `48000 / 2 / 480 (10 ms)` |

(P2.4 reads ports/frame sizes only to size its read buffer; it owns none of these
numbers ‚Äî A.12 is the single source of truth.)

## 6. Import constraints (01 ¬ß2 layering)

`internal/allowlist` is a **derived runtime** package below the engine. It MAY import:

- `internal/state` (P2.3) ‚Äî for `state.ConfigDoc` and `*state.Store` (`.Get()`,
  `.Changed()`). This is the durable derivation input.
- Go stdlib: `context`, `net`, `net/netip`, `sync`, `sync/atomic`.

It MUST NOT import:

- `internal/cluster` (P2.1) ‚Äî to avoid a heavy/cyclic dependency on memberlist and keep
  the package unit-testable without a live gossip stack. The live half is injected as a
  `liveFn func() []MemberAddr` / a `<-chan struct{}` (a thin adapter in `cmd` maps
  `cluster.Members()`‚Üí`[]MemberAddr` and `cluster.Membership.Changed()`‚Üíthe channel).
  This also keeps `MemberAddr` a small local type rather than re-exporting cluster's
  `Member` (which carries `Meta`, ports, etc.).
- `internal/group`, `internal/stream/*`, `internal/audio/*` ‚Äî those are *consumers* of
  the gate, never dependencies (no upward imports).
- `internal/web` ‚Äî the `web.Deps` seam never touches allowlist (web must not import the
  engine; 01 ¬ß2 / A.14.3).

> Rationale for injecting cluster rather than importing it: it preserves the
> layering (allowlist sits beside `state`, both below `cluster`'s wiring) and lets
> `allowlist_test.go` run with a plain `state.Store` and a synthetic `liveFn`.

## 7. Test plan

All tests are `package allowlist` (white-box where needed for the atomic snapshot), Go
1.26 `testing`, table-driven, no network except the loopback gate test. Acceptance ties
to **A.13 P2** ("forgotten node's packets dropped") and **03 ¬ß6 / 07 ¬ß3** behavior.

### 7.1 `allowlist_test.go` ‚Äî derivation & lookup (unit)

| Case | Input | Asserts |
|---|---|---|
| empty doc, no live | `ConfigDoc{}`, `nil` | empty set; `Allowed(anything)==false` |
| doc-only | 2 nodes, 3 addrs (incl. an IPv6 + a dup) | set has exactly the 3 unique IPs; dup deduped |
| live-only | empty doc, 2 live members | both live IPs allowed |
| union | doc node `.21` + live member `.99` (DHCP-fresh, not yet in doc) | both allowed (07 ¬ß3.1 union) |
| multi-homed | one node with `["192.168.1.23","192.168.1.24"]` | both allowed (07 ¬ß3.3) |
| bad addr skipped | node `Addrs=["not-an-ip","192.168.1.5"]` | only `.5` allowed; no panic/error (07 ¬ß3.1) |
| forgotten node | doc *without* the node's record, member gone | its old IP **not** allowed (A.13 P2; 07 ¬ß4.2) |
| IPv4/IPv6 forms | `192.168.1.21` vs `::ffff:192.168.1.21`, `fe80::21` | canonical `netip.Addr` match; mapped-v4 normalized consistently (Q1) |
| `Allowed` vs `AllowedAddr` | same IP via both | identical result; `net.IP{}`/invalid ‚áí false |

### 7.2 `allowlist_test.go` ‚Äî `Update` / `Run` recompute (unit)

| Case | Asserts |
|---|---|
| `Update` atomic swap | after `Update(doc,live)`, snapshot reflects new set; an in-flight `Allowed` reader sees either old or new, never a torn map (run under `-race`) |
| `Run` on `store.Changed()` | apply a doc adding an IP via `store.Apply`; `Run` rebuilds; the new IP becomes `Allowed` without restart (07 ¬ß3.2) |
| `Run` on membership signal | push the injected `<-chan struct{}`; `liveFn` now returns a new IP; it becomes allowed |
| `Run` startup priming | `Run` does one rebuild before first signal so the gate is non-empty at start |
| `Run` ctx cancel | cancel ‚áí `Run` returns promptly (`ctx.Err()`); no goroutine leak (goleak or a WaitGroup join) |
| coalescing | rapid double-signal ‚áí at least one rebuild, no deadlock (matches coalesced `Changed()`) |

### 7.3 `gate_test.go` ‚Äî `GateUDP` (integration, loopback UDP)

Bind a `*net.UDPConn` on `127.0.0.1:0`; run `GateUDP` in a goroutine with a `Set` allowing
only `127.0.0.1` (then a variant denying it).

| Case | Asserts |
|---|---|
| allowed source delivered | a datagram from `127.0.0.1` reaches `deliver` with exact bytes & correct `src.Addr()` |
| disallowed source dropped | with `127.0.0.1` **not** in the set, send a datagram ‚áí `deliver` is **never** called (drop, no reply ‚Äî A.8); assert via timeout + counter |
| no reply on drop | the gate sends nothing back to the source (passive sink confirms silence) |
| buffer not truncated | a max-size audio-shaped datagram (1964 B, ¬ß5.3) is delivered whole; `len(b)==1964` |
| live re-derive mid-stream | flip the `Set` via `Update` to allow a previously-denied source ‚áí subsequent packets now delivered (no socket restart ‚Äî 03 ¬ß6.3) |
| conn close / ctx done | closing the conn or cancelling ctx makes `GateUDP` return without panic |

## 9. Risks / open questions

- **Q1 (blocking-ish, needs a one-line ruling): IPv4-mapped-IPv6 normalization.**
  memberlist / `net.IP` may surface a v4 source as `::ffff:a.b.c.d` while `NodeRecord.Addrs`
  stores `a.b.c.d` (or vice-versa). If the set and the lookup don't canonicalize
  identically, a legitimate member's packets get dropped. **Proposed rule:** call
  `netip.Addr.Unmap()` on *both* sides (derivation and per-packet lookup) so mapped-v4 and
  native-v4 compare equal. Confirm this matches what `internal/cluster` (P2.1) actually
  reports for `Member.Addr` and what nodes write into their own `Addrs` (07 ¬ß3.2 self-write).
- **Q2 (resolved here, flag if the realtime pieces disagree): `deliver` address type.**
  03 ¬ß6.3 sketches `func(src *net.UDPAddr, b []byte)`; this spec uses
  `func(src netip.AddrPort, b []byte)` to avoid a per-packet `*net.UDPAddr` allocation on
  the Pi hot path (matches `ReadFromUDPAddrPort`). If the clock/audio pieces (04/05) need
  `*net.UDPAddr` for their reply path, they can `net.UDPAddrFromAddrPort(src)` once per
  packet ‚Äî but the clock *reply* path is the master side, which reads via its own
  socket; confirm no consumer needs the `net.UDPAddr` form in the gate signature.
- **Q3 (non-blocking): TCP-fallback audio.** D2/03 ¬ß6.1 mention a TCP-fallback audio
  listener also guarded by the allowlist. `GateUDP` only covers UDP. The TCP path is a
  per-connection `RemoteAddr().Addr()` check at `Accept` time ‚Äî a tiny helper
  (`AllowedAddr` already suffices). Decide whether that `GateTCP`/accept-filter helper
  lives here or in the audio piece (doc 05); recommend a thin `set.AllowedAddr` call at
  the audio listener's `Accept` rather than adding API surface to P2.4 now.
- **R (low): buffer sizing coupling.** The `GateUDP` read buffer must not truncate the
  largest realtime datagram (¬ß5.3, 1964 B audio / 40 B clock). If a future profile raises
  `FramesPerChunk` or rate beyond A.12 defaults, the constant must grow. Keep it a single
  named constant referencing A.12, not a magic literal.
