## 1. Purpose & scope

This piece implements the **bootstrap surface that lives outside mTLS** and the
**controller-side cluster-membership operations** that hang off the cluster CA. Concretely:
(a) the uninitialized node's `/bootstrap/*` server — `GET /bootstrap/info` plus the
three-phase `POST /bootstrap/adopt?phase={key,csr,complete}` realizing the A.9 6-step
handshake (X25519 ECDH for confidentiality + HKDF-SHA256 + PIN-keyed HMAC-SHA256 for
authentication + ChaCha20-Poly1305 AEAD for the secret payload, [03 §2.3/§9](../03-adoption-takeover-security-pki.md));
(b) the controller-side driver for `POST /api/v1/cluster/adopt`, `POST /api/v1/cluster/takeover`,
`POST /api/v1/nodes/{id}/forget`, and `POST /api/v1/cluster/leave` ([08 §C.3–C.6](../08-http-api-reference.md)),
including CA signing on the controller (Model B), the `RevokedSet` write + **gossip rekey on
forget** ([03 §5](../03-adoption-takeover-security-pki.md)); and (c) the **online brute-force
guard** (A.12 adoption guard: soft backoff after 3 consecutive fails, hard 15-min lockout
after 10 fails per 5 min, single-use nonces with 30 s TTL). It does **not** implement the CA
primitives themselves (P1.1), the auth/session model (P1.2), the ConfigDoc/state store (P2.3),
or discovery/membership (P2.1) — it *consumes* them.

## 4. Public API (canonical names)

### 4.1 `internal/adopt` — the handshake engine

```go
package adopt

// Phase identifies the bootstrap sub-exchange (03 §9: key | csr | complete).
type Phase string
const (
    PhaseKey      Phase = "key"      // A.9 steps 2-3: swap ECDH publics + nonces
    PhaseCSR      Phase = "csr"      // A.9 step 5: PIN-tagged csr_request -> AEAD_k(csrPem)
    PhaseComplete Phase = "complete" // A.9 step 6: AEAD_k(leaf‖caBundle‖clusterSecrets) + tag2
)

// HKDF info labels — pinned, normative (A.9 step 4).
const (
    InfoAdopt = "ensemble-adopt-v1" // k  = HKDF(Z, nonceA‖nonceB, InfoAdopt)
    InfoPIN   = "ensemble-pin-v1"   // kp = HKDF(PIN, nonceA‖nonceB, InfoPIN)
    tagReq    = "req"               // HMAC(kp, transcript‖tagReq)
    tagDone   = "done"              // HMAC(kp, transcript‖tagDone)
)

// ProtocolEpoch is fixed at 1 for this build (README invariant).
const ProtocolEpoch = 1

// ---- wire bodies (JSON; base64 for byte fields) ----

// KeyReq is POST /bootstrap/adopt?phase=key body (controller -> node). A.9 step 2.
type KeyReq struct {
    PubB   []byte `json:"NB"`     // controller X25519 ephemeral public (32 B)
    NonceB []byte `json:"nonceB"` // 16 B random, single-use
    Epoch  int    `json:"protocolEpoch"`
}
// KeyResp is the node's reply. A.9 step 3.
type KeyResp struct {
    PubA   []byte `json:"NA"`     // node X25519 ephemeral public (32 B)
    NonceA []byte `json:"nonceA"` // 16 B random, single-use
}

// CSRReq is POST …?phase=csr body (controller -> node). A.9 step 5.
type CSRReq struct {
    Tag []byte `json:"tag"` // HMAC(kp, transcript‖"req")
}
// CSRResp returns the AEAD-wrapped CSR (node -> controller).
type CSRResp struct {
    EncCSR []byte `json:"encCsr"` // ChaCha20-Poly1305 seal under k of csrPem
}

// CompleteReq is POST …?phase=complete body (controller -> node). A.9 step 6.
type CompleteReq struct {
    EncPayload     []byte `json:"encPayload"` // AEAD_k(leafPem‖caBundlePem‖clusterSecretsJSON)
    Tag2           []byte `json:"tag2"`       // HMAC(kp, transcript‖"done")
    AssignedNodeID string `json:"assignedNodeId"`
    ClusterName    string `json:"clusterName"`
    SeedPeers      []string `json:"seedPeers"`
}
// CompleteResp is the node's final ack (-> controller). 08 §A.2 200 body.
type CompleteResp struct {
    NodeID string `json:"nodeId"`
    State  string `json:"state"` // "member"
}

// ClusterSecrets is the plaintext-replicated secret bundle delivered in step 6
// (03 §1.6). Schema/persistence owned by P2.3/state; mirrored here for AEAD payload.
type ClusterSecrets struct {
    CAKeyPEM  []byte `json:"caKeyPem"`  // CA private key PEM (D18: plaintext)
    GossipKey []byte `json:"gossipKey"` // 32 B memberlist SecretKey
}

// ---- node side (the adoptee) ----

// NodeSession holds the per-handshake state on the uninitialized node across the
// three phases, keyed by nonce. Single-use; expires after NonceTTL (A.12: 30 s).
type NodeSession struct { /* priv *ecdh.PrivateKey, k, kp []byte, transcript []byte, born time.Time */ }

// Node is the adoptee's handshake half. PIN is the local secret (default "0000").
type Node struct { /* pin string, key crypto.Signer (node leaf key, P1.1), nodeID string, guard *Guard */ }

func NewNode(nodeID, pin string, leafKey crypto.Signer, g *Guard) *Node

// BeginKey runs A.9 steps 3-4 for the node: validates epoch, mints (nA,NA)+nonceA,
// derives k,kp once both nonces are known, stores a NodeSession. Returns KeyResp.
func (n *Node) BeginKey(src string, req KeyReq) (KeyResp, *NodeSession, error)

// AcceptCSR runs A.9 step 5: verify tag (const-time) -> build+seal its own CSR.
// A bad tag returns ErrBadPIN and is reported to the guard by the caller.
func (n *Node) AcceptCSR(sess *NodeSession, req CSRReq) (CSRResp, error)

// Complete runs A.9 step 6: verify tag2 (const-time), open AEAD_k(payload),
// split leaf‖caBundle‖clusterSecrets, hand them to install. ATOMIC (takeover §4).
func (n *Node) Complete(sess *NodeSession, req CompleteReq) (Installed, CompleteResp, error)

// Installed is the verified, decrypted result for the node to persist (P2.3 writes it).
type Installed struct {
    LeafPEM     []byte
    CABundlePEM []byte
    Secrets     ClusterSecrets
    SeedPeers   []string
    ClusterName string
    NodeID      string
}

// ---- controller side (holds the CA, Model B) ----

// Controller drives the three phases against a target over a fingerprint-pinned
// self-signed TLS channel. Sign is the CA signing callback (pki.CA.Sign, P1.1).
type Controller struct { /* httpClient *http.Client, sign SignFunc, caBundle []byte, secrets ClusterSecrets, clusterName string */ }

// SignFunc signs a CSR with the cluster CA (pki.CA.Sign bound by cmd; 30-day leaf).
type SignFunc func(csrPEM []byte, nodeID string, addrs []net.IP) (leafPEM []byte, err error)

// Run executes the full A.9 exchange against target `addr`, pinning its self-signed
// cert to `fingerprint` (sha256:…), authenticating with `pin`. force=true allows a
// "foreign" target (takeover, 03 §4); force=false aborts a foreign target with ErrForeign.
func (c *Controller) Run(ctx context.Context, addr, fingerprint, pin, assignedNodeID, name string, force bool) (NodeRecordSeed, error)

// NodeRecordSeed is what the controller learned to write into ConfigDoc.Nodes
// (the public projection; the controller then does the If-Match config write via Deps/state).
type NodeRecordSeed struct {
    ID      string
    Name    string
    Addrs   []net.IP
    CertPEM []byte // the leaf the controller just signed
}
```

### 4.2 `internal/adopt` — the online brute-force guard (A.12)

```go
// Guard enforces the A.12 adoption guard on /bootstrap/adopt. In-memory, per-source
// + global. Safe for concurrent use.
type Guard struct { /* … */ }

// GuardParams — defaults are the A.12 verbatim values (see §5); exposed for tests.
type GuardParams struct {
    SoftBackoffAfter int           // 3 consecutive fails
    HardFailCount    int           // 10 fails ...
    HardFailWindow   time.Duration // ... per 5 min
    HardLockout      time.Duration // 15 min
    NonceTTL         time.Duration // 30 s
}
func DefaultGuardParams() GuardParams // returns the A.12 values

func NewGuard(p GuardParams, now func() time.Time) *Guard

// Allow reports whether a fresh adopt attempt from src may proceed, or the backoff/
// lockout remaining. Called BEFORE running a phase that consumes a PIN proof.
func (g *Guard) Allow(src string) (ok bool, retryAfter time.Duration)

// RecordFail logs a failed PIN proof for src (and globally); drives soft backoff + lockout.
func (g *Guard) RecordFail(src string)

// RecordSuccess clears the per-source counters on a completed adoption.
func (g *Guard) RecordSuccess(src string)

// Nonce lifecycle (single-use, NonceTTL): IssueNonce mints+stores; ConsumeNonce
// validates-and-removes (ErrNonceExpired / ErrNonceUnknown).
func (g *Guard) IssueNonce() (nonce []byte)
func (g *Guard) ConsumeNonce(nonce []byte) error
```

## 5. Behavior & algorithms

### 5.1 The A.9 6-step handshake, mapped to the 3 phases ([03 §2.3, §9](../03-adoption-takeover-security-pki.md))

- **`phase=key`** (A.9 steps 2–3, 4-prep). Controller POSTs `{NB,nonceB,protocolEpoch}`.
  Node: **refuse epoch mismatch** (`ProtocolEpoch != req.Epoch` → `422 unprocessable`,
  `ErrEpochMismatch`, [03 §2.7](../03-adoption-takeover-security-pki.md) m7) *before any PIN
  work*; mint `(nA,NA)` via `crypto/ecdh.X25519()`, mint `nonceA` via `guard.IssueNonce()`;
  compute `Z = X25519(nA, NB)`, `k = HKDF-SHA256(Z, salt=nonceA‖nonceB, info=InfoAdopt)`,
  `kp = HKDF-SHA256(PIN, salt=nonceA‖nonceB, info=InfoPIN)`, `transcript = NA‖NB‖nonceA‖nonceB`;
  store a `NodeSession` keyed by `nonceA`; reply `{NA,nonceA}`. Controller computes the
  mirror `Z`/`k`/`kp`/`transcript`.
- **`phase=csr`** (A.9 step 5). Controller POSTs `{tag = HMAC(kp, transcript‖"req")}`. Node
  **verifies tag with `subtle.ConstantTimeCompare`** (the media `controller.go:116` idiom). On
  mismatch → `ErrBadPIN`, handler calls `guard.RecordFail(src)` and returns `401 unauthenticated`.
  On match the node builds its CSR (`pki.NewCSR`, P1.1) and returns `AEAD_k(csrPem)` (ChaCha20-Poly1305).
- **`phase=complete`** (A.9 step 6). Controller signs the CSR (`SignFunc` → `pki.CA.Sign`,
  30-day leaf, SANs set from authenticated addrs — [03 §1.3](../03-adoption-takeover-security-pki.md)),
  AEAD-seals `leafPem‖caBundlePem‖clusterSecretsJSON` under `k`, computes `tag2 = HMAC(kp,
  transcript‖"done")`, POSTs `{encPayload,tag2,assignedNodeId,clusterName,seedPeers}`. Node
  verifies `tag2` (const-time), opens the AEAD, splits the payload, **atomically** installs
  leaf+CA+`ClusterSecrets` (P2.3 write, 0600), closes `/bootstrap/*`, joins gossip with
  `seedPeers`, flips its mDNS TXT `init=1`+`cf` (P2.1). Replies `{nodeId,state:"member"}`.

Framing detail for the AEAD payload split: prefix each PEM/JSON segment with a 4-byte
big-endian length (`leafLen‖leaf‖caLen‖ca‖secretsLen‖secretsJSON`) so `Complete` parses
unambiguously. ChaCha20-Poly1305 nonce = 12 zero bytes is acceptable here because each `k` is
single-use per handshake (fresh ECDH ephemerals + dual random nonces feed `k`); document this
in code. Confidentiality of the bundle rests on `Z`/`k`, **not** the PIN ([03 §2.5/§3.2](../03-adoption-takeover-security-pki.md)).

### 5.2 Online brute-force guard — A.12 verbatim

From [A.12](../A-appendix-algorithms-and-pinned-choices.md) "Adoption guard" row:

> **soft backoff after 3 consecutive fails / 15-min lockout after 10 fails per 5 min / nonce TTL 30 s**

`DefaultGuardParams()` therefore returns exactly: `SoftBackoffAfter = 3`,
`HardFailCount = 10`, `HardFailWindow = 5 * time.Minute`, `HardLockout = 15 * time.Minute`,
`NonceTTL = 30 * time.Second`. Counters are per-source-IP **and** global ([03 §3.4](../03-adoption-takeover-security-pki.md):
"per *source* + global, in memory"). `phase=key` (no PIN) is rate-limited only by lockout state;
`phase=csr`/`phase=complete` (PIN-bearing) call `Allow` first and `RecordFail`/`RecordSuccess`
on outcome. Soft backoff after 3 consecutive fails returns `429` envelope `code:"rate_limited"`
([03 §3.4](../03-adoption-takeover-security-pki.md), [08 §0.4](../08-http-api-reference.md));
hard lockout refuses `adopt` for 15 min while `/bootstrap/info` keeps answering
([03 §3.4](../03-adoption-takeover-security-pki.md)). Every failed proof is logged with the
source IP (audit). Nonces are single-use (`ConsumeNonce` removes them) and expire in 30 s.

### 5.3 Controller endpoints ([08 §C.3–C.6](../08-http-api-reference.md))

- **`/cluster/adopt` (C.3)** — auth `admin-session`/`api-key`. Body
  `{nodeId,addr,fingerprint,pin,name?}`. **If-Match required** (adds a `NodeRecord`). Runs
  `Deps.Adopt(addr, fingerprint, pin, nodeId, name, force=false)`; `Deps.Adopt` pins the cert,
  drives the three phases (CA signing **on this controller**, Model B,
  [03 §1.2](../03-adoption-takeover-security-pki.md)), then does the local `ConfigDoc` write
  (add `NodeRecord{CertPEM,Addrs,…}`) → gossip (P2.3). A **foreign** target → `403 forbidden`
  (use takeover). Fingerprint or epoch mismatch → `422`. Target unreachable → `502 proxy_failed`.
- **`/cluster/takeover` (C.4)** — identical shape + `force:true`; `Deps.Adopt(…, force=true)`.
  Still PIN-gated. The node performs the **atomic identity swap** (drop old leaf/CA/secrets,
  install ours, rekey gossip to ours) — [03 §4](../03-adoption-takeover-security-pki.md).
- **`/nodes/{id}/forget` (C.5)** — auth node/session/api-key. **If-Match required.** Single
  `ConfigDoc` write via `Deps.Forget(id)`: add the leaf's **SHA-256 fingerprint** to the
  grow-only `RevokedSet` (P2.3), drop the `NodeRecord`, pull the id from every
  `GroupRecord.MemberNodeIDs`; then **trigger a gossip rekey** (`cluster.Membership.Rekey`,
  P2.1 — [03 §5.3](../03-adoption-takeover-security-pki.md)) so the forgotten node cannot
  re-add its IP; best-effort mTLS revoke notice to the target (non-fatal). Returns
  `{version,removedNodeId,affectedGroups}`. Cannot forget the last node / sole playing master
  → `409 conflict`.
- **`/cluster/leave` (C.6)** — coordinated self-forget. `Deps.Leave()` proxies the
  revoke+drop to a peer over mTLS (own fingerprint → `RevokedSet`, drop own `NodeRecord`),
  **then** wipes local identity/config and reboots into the Setup Wizard. Unreachable cluster →
  local wipe anyway, return `coordinated:false`. Cannot leave as last node / sole playing master
  → `409 conflict`.

### 5.4 CA signing on the controller, leaf fields

`SignFunc` is bound in `cmd` to `pki.CA.Sign(csr, nodeID, addrs, 30*24*time.Hour, time.Now())`
(P1.1; leaf validity **30 days**, renew at ⅓ life — A.12). SANs are set by the CA from
**authenticated** inputs (nodeID assigned by the controller, addrs observed), never from the
CSR ([03 §1.3](../03-adoption-takeover-security-pki.md)). `caBundle` is the public CA cert
from `ConfigDoc.Cluster`; `ClusterSecrets` (`caKeyPEM`+`gossipKey`) comes from the replicated
secret projection (P2.3). Epoch is fixed at **1**.

## 6. Import constraints ([01 §2](../01-architecture-and-packages.md) layering)

- `internal/web` reaches the engine **only** through the `web.Deps` function-value seam. It
  **must not** import `group`, `stream/*`, or `audio/*`. The `bootstrap.go`/`api_cluster.go`
  handlers therefore use only: stdlib (`net/http`, `crypto/*`, `encoding/json`),
  `internal/adopt`, and—for *types passed through Deps*—`internal/state` (the `ConfigDoc`
  projection) and `internal/auth` (the auth middleware classes from P1.2). They do **not**
  import `internal/pki`, `internal/cluster`, or `internal/group` directly; CA signing, gossip
  rekey, and ConfigDoc writes are reached via the `Deps.Adopt/Forget/Leave` closures, which
  `cmd` binds.
- `internal/adopt` imports **only** stdlib (`crypto/ecdh`, `crypto/hmac`, `crypto/sha256`,
  `crypto/subtle`, `crypto/rand`, `net`, `time`) and `golang.org/x/crypto/{hkdf,chacha20poly1305}`
  (A.11). It must not import `web`, `state`, `cluster`, `pki`, `group`. It accepts a
  `crypto.Signer` (the node leaf key) and a `SignFunc` callback so it stays decoupled from `pki`.
- `cmd/ensemble` is the only package allowed to wire `adopt` ↔ `pki` ↔ `state` ↔ `cluster`.

### 7.4 Integration test (P2-level, in `cmd` or a `_integration_test.go`)

Two in-process nodes (A = controller with a real `pki.CA` from P1.1, B = uninitialized):
A adopts B over a real self-signed-pinned `httptest.NewTLSServer`; assert B ends up with a
CA-signed leaf that chains to A's CA, the `ConfigDoc` converges with B's `NodeRecord`, B's IP
enters the allowlist; then A forgets B and assert B's fingerprint is in `RevokedSet`, B's
packets are dropped by the allowlist gate (P2.1/allowlist), and B's cert is rejected by the
mTLS verify hook (P1.1 `PeerVerifier`). This is the **A.13 P2 acceptance**: *"node B adopted
from A; doc converges; forgotten node's packets dropped + cert rejected."* Bad-PIN and
takeover-of-foreign paths are also asserted at this level (A.13 P7 "takeover supersedes old
cert").

**Acceptance (A.13):** P2 row — adopt(PIN) + ConfigDoc replicate + allowlist + forget all
green; the bad-PIN rejection and forgotten-node packet-drop + cert-reject are the explicit
gates. P7 row (later) — takeover supersedes the old cert.

## 9. Risks / open questions

1. **`/bootstrap/*` is served on a *separate* listener (self-signed, `ClientAuth:
   NoClientCert`) from the mTLS `/api/v1` listener** ([03 §8](../03-adoption-takeover-security-pki.md):
   "the only TLS endpoint that is not mTLS"). This piece assumes P1.2 stands up that second
   listener (or a SNI/port split). **Open: which port?** README §4 names control mTLS `:8443`;
   the bootstrap listener's port is **not pinned in A.12** — orchestrator must confirm whether
   bootstrap shares `:8443` (different TLS config by ServerName) or takes a dedicated port.
   **This is the one blocking question.**
2. **ChaCha20-Poly1305 nonce reuse safety** rests on `k` being unique per handshake (fresh
   ephemerals + dual random nonces). The spec uses a zero AEAD nonce; if a single `k` ever
   encrypted two payloads (it does not in A.9 — CSR uses `k`, complete uses `k`, *two*
   messages under one `k`), we must use **distinct AEAD nonces** for the csr-direction vs
   complete-direction ciphertexts. **Resolution (in-spec):** use AEAD nonce `0x00…01` for
   `EncCSR` and `0x00…02` for `EncPayload` to guarantee non-reuse under the same `k`. Flag for
   crypto review.
3. **Takeover atomicity** ([03 §4](../03-adoption-takeover-security-pki.md)): the node's
   "drop old identity → install new" must be crash-safe (temp-write+rename of the whole
   identity set, owned by P2.3). If P2.3's secret persistence is not transactional across
   leaf+CA+secrets, a crash mid-swap could leave a half-adopted node. Confirm P2.3 provides an
   atomic multi-file install.
4. **`affectedGroups`/sole-master `409`** on forget/leave needs group state the `web` layer
   may only read via `Deps.State()`; the "sole group master mid-play" check (per
   [04](../04-clock-and-groups.md)) may need a `Deps` helper not yet in A.14.3. Likely a small
   read-only addition rather than a new mutation seam.
5. **`/cluster/adopt` `If-Match`**: the controller drives bootstrap phases *then* writes the
   `ConfigDoc`. If gossip bumps the version mid-handshake, the post-sign write can `409`. The
   node is already adopted (cert signed) at that point — handler must retry the config write
   (re-GET version, re-apply the `NodeRecord` add) rather than failing the whole adoption.
