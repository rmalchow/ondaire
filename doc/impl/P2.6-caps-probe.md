## 1. Purpose & scope

Implement the **effective-capability computation and self-write** for a node: at startup
(and on a reload of the masking keys) compute `Caps = detected(runtime probe) ‚à©
enabled(per-node config)` (07 ¬ß2.4.1, 06 ¬ß1.5, D16) and publish it by writing the node's
**own** `NodeRecord.Caps` into the replicated `ConfigDoc` via the optimistic `If-Match`
self-write (07 ¬ß2.4.1 / ¬ß3.2 / ¬ß4.5), after which gossip distributes it and profile
negotiation (04) consumes it. This piece owns the **detect ‚Üí mask ‚Üí intersect ‚Üí self-write**
pipeline and the codec/FEC/maxRate detectors; it does **not** own the `AudioSink` registry's
`Probe()` (that sink-half ‚Äî `Render`/`Sinks` ‚Äî is produced by `internal/audio/sink`, the
**P4.6** sink registry, 06 ¬ß1.5) nor the rendering pipeline. P2.6 defines the **seam**
through which it consumes the sink probe (a `SinkProber` function value) so it can be built
and unit-tested before P4.6 lands, with a stub that reports a deterministic backend set.

## 4. Public API

Canonical type names from README ¬ß6.5 / 07 ¬ß2.4 (`Capabilities`, the `render/sinks/encode/

### 4.1 Detected, Mask, effective (`internal/caps/caps.go`)

```go
package caps

import (
    "gitlab.rand0m.me/ruben/go/ensemble/internal/config"
    "gitlab.rand0m.me/ruben/go/ensemble/internal/state"
)

// Detected is the raw runtime probe result for this machine, BEFORE config masking
// (07 ¬ß2.4.1 "detected"). It is the union of the sink registry probe (P4.6) and the
// local codec/FEC/rate detectors. All slices are sorted, deduplicated, canonical-string.
type Detected struct {
    Sinks        []string // usable output backend Names from the sink Probe() (06 ¬ß1.5): "alsa","exec:aplay","exec:pw-play"
    EncodeCodecs []string // wire codecs this node can ORIGINATE: "pcm" always; "opus" iff the encoder is available
    DecodeCodecs []string // wire codecs this node can PLAY: "pcm" always; "opus" iff the decoder is available
    FEC          []string // FEC schemes: "none","xorParity","duplicate" (all pure-Go, always present)
    MaxRate      int      // highest sample rate the (best precise) device accepts, Hz; 0 if no sink
}

// Mask is the per-node "enabled" intent parsed from config.AudioConfig (07 ¬ß2.4.2, P0.1).
// A nil ForceRender means "probe-driven"; non-nil false forces a sink-less node.
type Mask struct {
    ForceRender     *bool    // audio.render: nil => probe-driven; &false => force Render=false
    DisableBackends []string // audio.backends.disable: backends removed from detected before Sinks
    DisableCodecs   []string // audio.codecs.disable: codecs removed from Encode/DecodeCodecs
    PreferBackends  []string // audio.backends.prefer: ORDER only ‚Äî does NOT add/remove caps (07 ¬ß2.4.2)
}

// MaskFromConfig projects the parsed per-node audio config (P0.1 config.AudioConfig) into a
// Mask. Pure; no I/O. PreferBackends is carried through but never affects the effective set.
func MaskFromConfig(a config.AudioConfig) Mask

// Compute applies the masking config to the detected set to yield the EFFECTIVE Capabilities
// written into NodeRecord.Caps (07 ¬ß2.4.1: effective = detected ‚à© enabled). Pure; deterministic
// (output slices sorted, deduplicated). This is THE intersection function (D16).
func Compute(d Detected, m Mask) state.Capabilities
```

### 4.2 Probe seam + assembly (`internal/caps/probe.go`)

```go
// SinkProber is the seam to the P4.6 sink registry (06 ¬ß1.5 Probe). It returns the backends
// that actually work on this machine, in registry preference order. P2.6 consumes ONLY this
// function value; P4.6 supplies the real implementation (internal/audio/sink.Probe wrapped),
// and tests/cmd-before-P4.6 supply a stub. Defining it here lets P2.6 build + test standalone.
type SinkProber func() []Backend

// Backend mirrors README ¬ß6.1 / 06 ¬ß1.1 (audio/sink.Backend) by VALUE so internal/caps need
// not import internal/audio/sink (avoids the dependency until P4.6 exists). P4.6's adapter
// maps sink.Backend -> caps.Backend 1:1.
type Backend struct {
    Name    string // "alsa" | "exec:aplay" | "exec:pw-play"
    Precise bool   // true => precise (kernel-ioctl ALSA) Delay()
}

// MaxRateProber reports the highest sample rate the chosen device will run (06 ¬ß1.1). Seam to
// P4.6 (the precise backend can query the device; the coarse exec path returns the canonical
// 48000). Returns 0 when no sink is usable.
type MaxRateProber func() int

// ProbeDeps bundles the seams + the local detectors so Probe is fully injectable for tests.
type ProbeDeps struct {
    Sinks   SinkProber    // REQUIRED; from P4.6 (or stub)
    MaxRate MaxRateProber // REQUIRED; from P4.6 (or stub)
    // Codec/FEC detection is local (detect_codec.go) and needs no seam for the MVP.
}

// Probe runs the seams + local detectors and assembles the raw Detected (pre-mask).
func Probe(d ProbeDeps) Detected
```

### 4.3 Publisher self-write (`internal/caps/publish.go`)

```go
// Publisher computes this node's effective Caps and self-writes them into the node's own
// NodeRecord.Caps in the replicated ConfigDoc (07 ¬ß2.4.1). One per node, owned by cmd/group.
type Publisher struct {
    // unexported: store *state.Store, selfID string, detected Detected, mask Mask
}

// NewPublisher binds the store, this node's id, the (already-run) Probe result, and the Mask.
// detected+mask are captured once; Compute is re-evaluated on each Publish so a config reload
// can change the mask and re-publish (07 ¬ß2.4.1 "on a config reload of the masking keys").
func NewPublisher(store *state.Store, selfID string, detected Detected, mask Mask) *Publisher

// SetMask swaps the masking intent (config reload) and triggers a re-publish on next Publish.
func (p *Publisher) SetMask(m Mask)

// Publish computes effective caps and writes them to this node's NodeRecord.Caps via the
// optimistic If-Match self-write (07 ¬ß3.2/¬ß4.5): Get -> if this node's Caps already equal the
// target, NO-OP (no version bump); else edit the node's record -> Apply -> on ErrConflict
// refetch+retry (bounded, jittered). Idempotent. Returns the effective caps it published (or
// that were already current) and any terminal error after retries exhausted.
func (p *Publisher) Publish(ctx context.Context) (state.Capabilities, error)

// Effective returns Compute(p.detected, p.mask) without writing (for cmd logging / the UI).
func (p *Publisher) Effective() state.Capabilities
```

> The node's `NodeRecord` must already exist in the doc (created at adoption, P2.x). If this
> node's id has no record yet, `Publish` is a **no-op that returns no error** (logs at debug):
> caps are published once the adoption write has seeded the record and `Publish` is re-driven
> on the next `Store.Changed()` tick. This avoids P2.6 racing adoption.

## 5. Behavior & algorithms

### 5.1 detected ‚à© enabled ‚Äî `Compute` (07 ¬ß2.4.1, D16)

The effective set is the intersection of the runtime probe with the per-node enabled mask:

1. **Sinks** = `Detected.Sinks` minus `Mask.DisableBackends` (07 ¬ß2.4.2 table: "remove the
   named backends from detected before they enter `Sinks`"). `PreferBackends` does **not**
   add/remove ‚Äî ordering only, consumed later by `Open(preferred, ‚Ä¶)` (README ¬ß6.1).
2. **Render** = `len(Sinks) > 0` (06 ¬ß1.5: "`Render=false` when there is no usable+enabled
   sink"), **unless** `Mask.ForceRender` is non-nil:
   - `ForceRender == &false` ‚áí `Render=false` **and `Sinks` emptied** (07 ¬ß2.4.2 table:
     "Force `Render=false` regardless of probe; `Sinks` emptied"; D17).
   - `ForceRender == &true` is **not** a force-on: a node with no usable sink cannot render
     by fiat. Treat `&true` as "probe-driven" (i.e. equivalent to nil) ‚Äî you cannot conjure a
     working sink. (Documented so the operator can't mis-set `render: true` into a broken
     sink.)
3. **EncodeCodecs / DecodeCodecs** = the detected codec lists minus `Mask.DisableCodecs`
   (07 ¬ß2.4.2: "remove the named codecs from `EncodeCodecs` and `DecodeCodecs`"). `"pcm"`
   is the mandatory baseline (D3) and is **always** detected; masking it in config is
   *allowed* here (the spec permits it) but the negotiator (04) still falls back to PCM ‚Äî
   P2.6 does not re-add it (consistent with P0.1's note that PCM masking is "not enforced").
4. **FEC** = `Detected.FEC` (no FEC masking key exists in 07 ¬ß2.4.2 ‚Äî all three schemes are
   pure-Go and always advertised); passed through, sorted.
5. **MaxRate** = `Detected.MaxRate` when `Render`; `0` when sink-less (no device to clamp).

All output slices are **sorted ascending and deduplicated** so the gossiped `Caps` is
byte-stable (avoids spurious version bumps from slice-order churn). A sink-less node
(`Render=false`) emits `Sinks: []` and `MaxRate: 0`, but may still keep non-empty
`EncodeCodecs` (it can be a master/origin) and empty-or-kept `DecodeCodecs` per the mask ‚Äî
mirroring the `nas` node in the 07 ¬ß6 worked example (`render:false, sinks:[], decode:[],
encode:["pcm","opus"], maxRate:48000`‚Ä¶ note the example keeps a non-zero maxRate for a
control node; P2.6 zeroes it for a node with no sink ‚Äî see ¬ß9 open question 1).

### 5.2 Detectors (`detect_codec.go`)

- **Codecs (A.11 R2, D3).** Wire codec set is `{PCM, Opus}` only. `detectCodecs` returns
  `EncodeCodecs`/`DecodeCodecs` = `["pcm"]` **always** (PCM baseline, no encoder needed). Opus
  is added to both **iff** the Opus binding is available at runtime. Per A.11 the Opus binding
  is **deferred** (purego/dlopen, off the critical path); for the MVP `detectCodecs` reports
  Opus **absent** (returns `false` from an `opusAvailable()` that is a single `return false`
  stub today, to be replaced by the dlopen probe when Opus lands). **No FLAC** ‚Äî FLAC is a
  source decoder only, never a wire codec (A.11 R2).
- **FEC (A.10, D4).** `detectFEC` returns `["none","xorParity","duplicate"]` unconditionally ‚Äî
  all three are pure-Go and always supported.
- **MaxRate (06 ¬ß1.1).** Comes from the `MaxRateProber` seam (P4.6 can ask the precise device;
  coarse exec path reports the canonical **48000**, A.12). `detect_codec.go` holds the
  canonical fallback constant `canonicalRate = 48000` (A.12 "Audio canonical rate 48000") used
  when the prober yields ‚â§ 0 but a sink exists.

### 5.3 Self-write (`publish.go`, 07 ¬ß2.4.1 / ¬ß3.2 / ¬ß4.5, A.6)

The publish loop is the same optimistic-concurrency idiom as a node maintaining its own
`Addrs` (07 ¬ß3.2 ‚Äî "a node only edits *its own* record ‚Ä¶ `Apply` resolves by refetch+retry,
as in mpvsync `state.go`"):

```
target := Compute(detected, mask)
for attempt := 0; attempt < maxAttempts; attempt++ {
    doc := store.Get()
    rec, idx := findNode(doc, selfID)
    if idx < 0 { return zero, nil }                 // record not seeded yet (adoption pending) ‚Äî no-op
    if capsEqual(rec.Caps, target) { return target, nil }   // idempotent: no version bump
    doc.Nodes[idx].Caps = target                    // edit ONLY this node's record
    if _, err := store.Apply(doc); err == nil { return target, nil }
    else if errors.Is(err, state.ErrConflict) { sleep(jitter()); continue }  // refetch+retry
    else { return zero, err }
}
return zero, errTooManyConflicts
```

- **`store.Apply` bumps `Version`, sets `UpdatedBy`, persists, fires `Changed()`** (P2.1 ¬ß5.1);
  gossip then distributes the new `Caps` to all full nodes (07 ¬ß2.4.1; A.6).
- **Idempotent.** `capsEqual` compares the effective struct field-by-field (slices order-
  insensitive ‚Äî both are sorted by `Compute`, so a direct `slices.Equal` after the sort holds).
  No write ‚áí no version bump ‚áí no gossip churn. This is essential because `Publish` is re-driven
  on every `Store.Changed()` (the doc advances for unrelated reasons) and must not write-storm.
- **Retry bound.** `maxAttempts` small (e.g. 8) with jittered backoff (07 ¬ß4.5 m6: "jittered
  retry-on-409"); on exhaustion return `errTooManyConflicts` (cmd logs + retries on the next
  `Changed()`). No A.12 number governs this; it is a local robustness constant, not a tunable.
- **Re-publish triggers.** `cmd`/`group` drives `Publish` (a) once at startup after the sink
  probe + adoption, and (b) on `SetMask` after a config reload of the `audio.*` keys
  (07 ¬ß2.4.1: "and on a config reload of the masking keys"). It does **not** poll.

### 5.4 Tunables referenced (verbatim from A.12)

The only A.12 numbers this piece embeds are the **canonical rate 48000** (max-rate fallback /
sink-less default, ¬ß5.2) and the retry uses **07 ¬ß4.5 m6** jittered-retry policy (no numeric
tunable). Everything else (drift gains, FEC k/D, LeadMs, ports) is **not** referenced here ‚Äî
P2.6 produces capability strings, not realtime parameters. Tests must not invent numbers;
quote A.12 (`48000`).

## 6. Import constraints (doc 01 ¬ß2 layering)

`internal/caps` sits **above** `state`/`config` and **below** `group`/`cmd`:

- **May import (internal):** `internal/state` (for `Capabilities`/`NodeRecord`/`ConfigDoc`/
  `Store`/`ErrConflict`) and `internal/config` (for `AudioConfig`/`BackendsConfig`/
  `CodecsConfig` ‚Äî the masking-key types, P0.1). Both are leaves (doc 01 ¬ß2.3 graph), so no
  cycle.
- **Must NOT import:** `internal/audio/sink` or any `audio/*` (the sink half is consumed only
  through the **`SinkProber`/`MaxRateProber` function-value seam**, ¬ß4.2 ‚Äî this keeps P2.6
  buildable before P4.6 and preserves the layering: `audio/sink` is a leaf that must not pull
  in `caps`). Also must not import `group`, `cluster`, `stream/*`, `web`, `pki`, `auth`.
- **`group` (the orchestrator, doc 01 ¬ß2 rule 5) imports `caps`**, supplies the real
  `SinkProber` (an adapter over `internal/audio/sink.Probe`, mapping `sink.Backend` ‚Üí
  `caps.Backend`), and drives `Publisher.Publish`. `cmd` wires it (doc 01 ¬ß2 rule on `cmd`
  building the engine).
- **`web` never imports `caps`** ‚Äî the UI sees effective caps only as `NodeRecord.Caps` in the
  `Deps.State() ConfigDoc` snapshot (A.14.3, spine ¬ß6.6).

### 7.3 Integration / acceptance (A.13 P2)

P2.6's contribution to **A.13 P2** ("ConfigDoc replicate ‚Ä¶ doc converges") and to the D16
capability story is: after a node probes, its effective `Caps` appear in its own
`NodeRecord.Caps` and gossip-converge to peers. The end-to-end two-process convergence is
owned by the `cluster`/`state` gossip pieces (P2.1/P2.2); P2.6 asserts at unit level that the
self-write lands in the store and is picked up by `MarshalGossip` (write via `Publisher`, then
`store.MarshalGossip()` carries the updated `Caps`). DoR: `go test ./internal/caps/...` green;
`CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go build ./...` clean (the binding-target acceptance is
satisfied because P2.6 adds **no** cgo and the sink seam is a function value, not a sink
import).

- **`group`** (04 profile negotiation) consumes the *result* indirectly ‚Äî it reads

## 9. Risks / open questions

1. **`MaxRate` for a sink-less node ‚Äî spec disagreement.** The 07 ¬ß6 worked example shows the
   `nas` node (`render:false, sinks:[]`) **with `maxRate:48000`**, but 07 ¬ß2.4.1's logic
   ("effective = detected ‚à© enabled", and a node with no usable sink) implies a node with no
   device has no meaningful max output rate. This plan **zeroes `MaxRate` when `Render==false`**
   (no device to clamp). **Open:** confirm whether a sink-less origin should advertise a
   non-zero `MaxRate` (it can still *encode* at a rate even without playing). If the negotiator
   (04) needs an origin's encode rate, P2.6 should instead keep `MaxRate = canonicalRate`
   (48000) for a sink-less node. **Recommendation:** keep `MaxRate` = max(deviceRate, and if
   `EncodeCodecs` non-empty then canonicalRate) so an origin contributes a usable rate;
   flagging for the 04/group P-owner. Low effort to flip once decided.
2. **`audio.render: true` semantics.** This plan treats `&true` as "probe-driven" (cannot
   force a working sink into existence). 07 ¬ß2.4.2 only documents `render: false`. Confirm no
   operator expects `render: true` to *re-enable* a probe-disabled sink (it can't). Documented
   in ¬ß5.1; no code risk, only a UX expectation.
3. **PCM masking.** 07 ¬ß2.4.2 permits `audio.codecs.disable:["pcm"]`; D3 makes PCM the
   mandatory baseline. P2.6 honors the mask literally (drops PCM from the advertised set) and
   relies on the negotiator (04) to fall back to PCM regardless (P0.1 note). **Open:** confirm
   the negotiator truly ignores a missing-PCM advertisement; if not, P2.6 must refuse to mask
   PCM. **Recommendation:** leave as-is (honor the mask), since 04 owns the baseline fallback.
4. **Opus probe shape (future).** When Opus lands (A.11, purego/dlopen, off critical path),
   `opusAvailable()` becomes a dlopen liveness check. It must stay **off the default/critical
   path** (R7) ‚Äî i.e. a failed dlopen reports Opus absent and never aborts the node. Noted so
   the Opus piece does not turn this into a hard dependency.
5. **Re-publish cadence ownership.** P2.6 exposes `Publish`/`SetMask` but does **not** own the
   loop that calls them on `Store.Changed()` / config-reload ‚Äî that is `group`/`cmd` (doc 01
   ¬ß2 rule 5). Confirm the orchestrator drives `Publish` after adoption seeds the `NodeRecord`
   (else the ¬ß4.3 no-op path silently never publishes until the next `Changed()`).
