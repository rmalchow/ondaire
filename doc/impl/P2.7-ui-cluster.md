## 1. Purpose & scope

Implement **screen 4, "Cluster"** ([09 §4](../09-ui-screens.md)): the operator surface for
cluster membership. It renders (a) a **CA-fingerprint header** (copyable trust anchor from
`Cluster.CACert`), (b) a **Discovered list** of uninitialized/foreign nodes each with a
**PIN field defaulting to `0000`** and an **Adopt** action plus a **Rescan** control, and
(c) a **Members list** with online/offline status, a **"control / media only"** tag for
sink-less (`Caps.Render == false`) members, a **"master (no local audio)"** note where
applicable, and per-row **Node / Takeover / Forget** actions. Disruptive actions
(Takeover, Forget) go through the shared **confirm dialog**; all data-backed regions
implement the shared **loading / empty / error / offline** state machine and surface the
**proxy** "via serving → target" hint. This piece is pure presentation + action wiring on
top of P0.4's shell and P2.5's API; it owns **no** network protocol or config-write logic
of its own (those live behind the P2.5 endpoints).

## 4. Public API (the screen's TypeScript surface)

Names follow the canonical JSON shapes in [08 §0.7](../08-http-api-reference.md) and the
`Capabilities` object in [README §6.5](../README.md). Codec/FEC are **string enums**; the
UI never touches wire integer ids.

```ts
// web/src/lib/cluster.ts

// ---- View types (mirror 08 §0.7 / C.2 / README §6.5) ----
export interface Capabilities {
  render: boolean
  sinks: string[]            // "alsa" | "exec:aplay" | ...
  encode: string[]           // "pcm" | "opus"
  decode: string[]           // "pcm" | "opus"  ("—" when not a listener)
  fec: string[]              // "none" | "xorParity" | "duplicate"
  maxRate: number
}

// A current cluster member (ConfigDoc Nodes[] projection + gossip liveness).
export interface MemberNode {
  id: string
  name: string
  addrs: string[]
  online: boolean            // from gossip/health (C.1/C.2 members[].online)
  caps: Capabilities
  fingerprint?: string       // node cert fingerprint (for the member row footer)
  groupId?: string           // current group, if any
  isMaster?: boolean         // elected master of its group (for the "master (no local audio)" note)
}

// A node seen on the LAN but not yet in this cluster (C.2 discovered[]).
export interface DiscoveredNode {
  nodeId: string
  name: string
  addrs: string[]
  fingerprint: string        // the node's self-signed CSR fingerprint (pin before sending PIN)
  state: 'uninitialized' | 'foreign'
  softwareVersion?: string
}

export interface DiscoverySnapshot {
  members: { id: string; name: string; addrs: string[]; state: string; online: boolean }[]
  discovered: DiscoveredNode[]
}

export interface ClusterInfo {
  version: number
  cluster: { name: string; caFingerprint: string; created: string }
  counts: { nodes: number; groups: number }
}

// Normalized API error from the {error:{code,message}} envelope (08 §0.4).
export interface ApiError extends Error { code: string; status: number }

// ---- Reads ----
export function getClusterInfo(): Promise<ClusterInfo>                  // GET /api/v1/cluster/info (C.1)
export function getDiscovery(): Promise<DiscoverySnapshot>             // GET /api/v1/discovery   (C.2)
export function getNodes(): Promise<{ version: number; nodes: MemberNode[] }> // GET /api/v1/nodes (D.1)
export function rescan(): Promise<DiscoverySnapshot>                   // re-issues GET /api/v1/discovery (Rescan)

// ---- Writes (all require If-Match: <version>; threaded by apiClient) ----
export interface AdoptReq { nodeId: string; addr: string; fingerprint: string; pin: string; name?: string }
export function adopt(req: AdoptReq, ifMatch: number):    Promise<{ version: number; node: MemberNode }> // C.3
export function takeover(req: AdoptReq & { force: true }, ifMatch: number): Promise<{ version: number; node: MemberNode }> // C.4
export function forget(nodeId: string, ifMatch: number):  Promise<{ version: number; removedNodeId: string; affectedGroups: string[] }> // C.5
```

## 5. Behavior & algorithms (cite docs; quote tunables verbatim)

**Layout & components** — exactly the [09 §4 wireframe](../09-ui-screens.md): header with
`CA fingerprint: SHA256:…` + `[copy]`; "Discovered — not yet in this cluster" table
(`node | addr | fingerprint | action`) with a `[ rescan ]` button; "Cluster members" table
(`node | addrs | status | actions`); a confirm row for Forget/Takeover.

**Discovered list & Adopt** ([09 §4 components/flows](../09-ui-screens.md), [08 §C.2/§C.3](../08-http-api-reference.md)):
- Data from `GET /api/v1/discovery` (C.2); each row shows `nodeId`, source `addrs[0]`, the
  node's **CSR `fingerprint`**, a **PIN input defaulting to `"0000"`** ([D9](../README.md),
  09 §4), and **Adopt**.
- The PIN placeholder `0000` is **treated as a real secret** ([D9](../README.md)): the UI
  shows it pre-filled but allows edit; it is sent verbatim in `POST /api/v1/cluster/adopt`
  `{nodeId, addr, fingerprint, pin, name?}` with **`If-Match: <configVersion>`** (C.3).
- **Adopt flow** (09 §4 flow 1): operator confirms the displayed fingerprint matches the
  target node's Setup Wizard (screen 1b) out-of-band, enters PIN, clicks Adopt → on `200`
  the row's busy spinner clears and the node migrates from `discovered[]` to `members[]`
  once gossip converges (re-fetch via `refreshCluster()` and/or the live `state` push).
- **Rescan** re-issues `GET /api/v1/discovery` (a fresh sweep; C.2 is fan-out-read on the
  serving node). No config write, no `If-Match`.
- A `state: "foreign"` discovered node is shown with a hint that it needs **Takeover**, not
  Adopt (09 §4 empty-state copy + flow 2).

**Members list, sink-less tag, master note** ([09 §4](../09-ui-screens.md), [D16/D17](../README.md)):
- Members from `GET /api/v1/nodes` (D.1) joined with `online` liveness from
  `GET /api/v1/cluster/info`/discovery `members[]` (C.1/C.2).
- A member with **`caps.render === false`** renders the **"control / media only"** tag
  (`ControlMediaOnlyTag`), linking to its Node-detail Capabilities panel (screen 6). It is
  **not** an error state — it is configuration ([D17](../README.md)); rendered normal-weight,
  distinct from the dimmed **offline** treatment (09 §4 states).
- When a sink-less member is also its group's elected master (`isMaster && !caps.render`),
  append the **"master (no local audio)"** badge (09 §3/§4/§5 cross-refs).

**Online / offline** ([09 §0 shared states, §4](../09-ui-screens.md)):
- `online: true` → `● online` chip; `false` → `⌁ offline` chip, row dimmed, values shown as
  "last known" from the gossiped ConfigDoc. **Takeover/Forget remain available** while
  offline: Forget of an offline node still revokes+removes it; Takeover may need the node
  reachable to complete the handshake — show "node must be reachable" copy if the call
  returns `502 proxy_failed` (09 §4 offline note).

**Takeover** ([09 §4 flow 2](../09-ui-screens.md), [08 §C.4](../08-http-api-reference.md)):
- Gate behind `confirmAction({ type:'takeover', danger:true, confirmLabel:'Take over',
  message:'Force re-issue this node\'s identity into this cluster?' })`.
- On confirm, `POST /api/v1/cluster/takeover` with the C.3 body **plus `force:true`** and
  **`If-Match`** (C.4). Still PIN-gated (reuse the row's PIN; default `0000`).

**Forget** ([09 §4 flow 3](../09-ui-screens.md), [08 §C.5](../08-http-api-reference.md)):
- Gate behind `confirmAction({ type:'forget', danger:true, confirmLabel:'Forget',
  message:'Forget <name>? Revokes its cert and drops it from config + allowlist.' })`
  — copy taken verbatim from the [09 §4 wireframe](../09-ui-screens.md) confirm row.
- On confirm, `POST /api/v1/nodes/{id}/forget` with **`If-Match`** (C.5). On `200`, surface
  `affectedGroups[]` (e.g. "removed from group 'Outdoor'; a new master was elected"); the
  member disappears once the doc gossips.

**Optimistic concurrency** ([09 §0, §4](../09-ui-screens.md), [08 §0.5](../08-http-api-reference.md)):
- Every write sends `If-Match: <configVersion>` (seeded from the latest read's `ETag` via
  P0.4's `apiClient`). On **`409 version_conflict`** the screen **reloads** the doc
  (`refreshCluster()`), shows what changed, and asks the operator to **reapply** — never a
  silent overwrite (09 §0 "409" rule). Missing `If-Match` would be `412`; `apiClient` always
  attaches it for mutating calls, so `412` is a programming-error guard only.

**Error envelope & bad PIN** ([09 §4 states](../09-ui-screens.md), [08 §0.4](../08-http-api-reference.md)):
- All non-2xx render the envelope `code` + `message` inline (per row for Adopt/Takeover/Forget,
  banner for list-level fetch). A **bad PIN** surfaces as `401 unauthenticated` (often inside
  `502 proxy_failed` detail per C.3) — keep the message **generic**, no PIN oracle beyond the
  protocol's own gating ([03](../03-adoption-takeover-security-pki.md), 09 §4 error note).

**Proxy hint** ([09 §0 proxying, §4](../09-ui-screens.md)): Adopt/Takeover/Forget may run
against a node other than the serving node; show the subtle `via <servingNode> → <target>`
hint (`ProxyHint`), and on a `502 proxy_failed` switch the affected region to the **offline**
treatment plus an error toast.

**Tunable numbers** (quoted verbatim from [A.12](../A-appendix-algorithms-and-pinned-choices.md)):
- Adoption PIN placeholder is **`"0000"`** ([D9](../README.md)); it is the only literal this
  screen hard-codes, and only as the **default value of the PIN input**.
- Adoption guard (informational, drives the rate-limit/lockout messaging this screen may
  display from a `code:"rate_limited"`/`403` response — A.12 verbatim): **"soft backoff /
  hard lockout / nonce TTL | backoff after 3 consecutive fails / 15-min lockout after 10
  fails per 5 min / nonce TTL 30 s"**. The UI does **not** implement these limits; it only
  renders the server's envelope when they trigger.
- This screen sets **no audio/clock/FEC tunables** — those (`48000/2/480`, `LeadMs 300 ms`,
  drift `Kp 0.05 / Ki 0.005 / MaxPPM 200`, FEC `k 8 / D 4`, ports `8443/9000/9100`, etc.,
  per A.12) are out of scope here.
- `protocolEpoch` is fixed at **1**; a `422 unprocessable` from C.3/C.4 may carry a
  protocol-epoch-mismatch message ("upgrade together") that this screen renders verbatim.

## 6. Import constraints

This is a **frontend** piece; the Go layering rule ([01 §2](../01-architecture-and-packages.md):
`internal/web` reaches the engine only through the `web.Deps` seam) is satisfied **upstream**
by P2.5's handlers — P2.7 never touches Go packages. Frontend import discipline:

- May import: `svelte`, `svelte/store`, P0.4's `web/src/lib/{apiClient,confirm,format,stores}`
  and `web/src/components/ConfirmModal.svelte`, and this piece's own `lib/cluster.ts` /
  `lib/clusterStore.ts`.
- Must **not** import other screens' route components or their private stores (screens
  compose only via the P0.4 router and shared `lib`).
- Must **not** add new runtime npm dependencies (the screen is dependency-free TS/Svelte,
  matching the `../media` "kept dependency-free" discipline). Test-only devDeps
  (`vitest`, `@testing-library/svelte`) are added by P0.4; if absent, add them as
  **devDependencies only** (see §9).
- All server access goes through `lib/cluster.ts` → P0.4 `apiClient`; components never call
  `fetch` directly (mirrors the mpvsync "components never touch the socket directly"
  discipline in `ws.ts`/`stores.ts`).

  + component suite above (the cross-process convergence is exercised by P2.5/integration).

## 9. Risks / open questions

1. **(Blocking) P0.4 surface area.** This spec assumes P0.4 provides: a client-side
   **router** (route key `/cluster`), an **`apiClient`** that injects auth + threads
   `If-Match` from a tracked `ConfigDoc.version` and exposes `ETag`, the
   `confirm.ts`/`ConfirmModal` pair, and the **shared screen-state** primitives
   (skeleton/empty/error/offline). **The orchestrator must confirm the exact P0.4 export
   names and the `If-Match` threading mechanism** — if `apiClient` does *not* auto-attach
   `If-Match`, `lib/cluster.ts` must accept and send it explicitly (already modeled via the
   `ifMatch` params in §4). If P0.4 already ships `ProxyHint`/`StatusChip`/empty-error
   primitives, delete the local duplicates listed in §2.
2. **No router library in `../media`.** mpvsync used tabs, not routes; the ensemble screen
   index ([09](../09-ui-screens.md)) is route-based (`/cluster`). The router is **P0.4's**
   responsibility; if P0.4 chose a specific lib (e.g. `svelte-spa-router`) vs. a hand-rolled
   hash router, `Cluster.svelte`'s registration/`onOpenNode` navigation must match it.
   Resolve P0.4's navigation API before wiring `onOpenNode`/`ControlMediaOnlyTag` links.
3. **Live updates vs. polling.** [09 §4](../09-ui-screens.md) does not mandate SSE/WS for the
   Cluster screen (unlike `/status` polling on Dashboard). Plan: load via `refreshCluster()`
   on mount + after each write, and additionally re-derive from P0.4's live `state` push if
   it carries ConfigDoc deltas. **Open: does P2.5/P0.2's WS snapshot include
   discovered[]/members liveness, or must this screen poll `GET /api/v1/discovery`?** If
   poll-only, add a ~2–5 s interval for the discovered table while the screen is mounted.
4. **Discovery enrichment latency.** `GET /api/v1/discovery` is **fan-out (read)** (C.2): the
   serving node probes each raw node's `/bootstrap/info`. Slow/partial probes mean the
   discovered table may populate incrementally — the loading state should be per-table, not
   block the members table.
5. **Takeover reachability.** C.4 may need the target reachable; an offline `foreign` node
   can be listed but not taken over until it returns. The UI shows "node must be reachable"
   on `502 proxy_failed`, but there is **no** discovery signal distinguishing "foreign +
   reachable" from "foreign + gone" beyond a probe attempt — acceptable for MVP.
