## 1. Purpose & scope

`internal/clock` establishes the per-group shared timebase that every audio consumer relies
on: `master_mono ≈ follower_mono + Offset` (README §6.2, doc 04 §4.1.1). It is the
NTP-style four-timestamp UDP exchange reused **unchanged** from mpvsync (spine D6): a
`Follower` pings its group's master, derives `(offset, delay)` per sample, runs them through
a minimum-delay filter + EWMA `Estimator` (A.1), and exposes a slewed offset; a `Server`
(the master) answers any allowlisted requester by stamping `t2`/`t3` and echoing `t1`. P3.1
re-homes that code into the ensemble module (import rewrites only — same module-internal
stdlib), adds the three **`ClockSource` adapters** (master self-reference / follower wrapper
/ orphan zero-value) that satisfy the canonical §6.2 interface per doc 04 §4.1.3, and
**allowlist-gates the server read loop** so the unauthenticated clock plane drops packets
from non-member source IPs (spine §4 clock plane, A.8, P2.4). It does **not** own the
`Timeline` (that is `internal/group`, doc 04 §4.4), the per-group election that decides who
is master (`internal/cluster`, doc 02), or the ping cadence policy beyond exposing the
`WithInterval/WithTimeout` knobs that `cmd`/`group` set from A.12.

member (spine §4 clock plane, A.8, doc 04 §4.1.2 "the master's server accepts requests only
from current cluster member IPs"). Two coupled adaptations:

- Add an injected per-packet source-IP predicate. `Listen` gains a variant
  `ListenGated(addr string, allowed func(netip.Addr) bool)` (and keep `Listen(addr)` as
  `ListenGated(addr, allowAll)` for tests/solo). In `serve()`, after
  `ReadFromUDPAddrPort` (use the `netip.AddrPort` form, not `ReadFromUDP`, to avoid a
  per-packet `*net.UDPAddr` alloc on the Pi hot path), call `allowed(src.Addr())`; if false
  **`continue` (drop, no reply)** *before* `unmarshal`. This matches P2.4's `AllowedAddr`
  fast-path and the "untrusted bytes never reach the decoder" rule (P2.4 §5.3).
- The predicate is supplied by `cmd` wiring as `set.AllowedAddr` from the shared
  `*allowlist.Set` (P2.4). The server reply path still uses `WriteToUDP(reply, from)` where
  `from` is recovered via `net.UDPAddrFromAddrPort(src)` once per *accepted* packet.

> Why gate the **server** and not the follower: the follower dials a single master and only
> reads that socket; the server is the open listener that any host can reach, so it is the
> attack surface (P2.4 §5.3, doc 04 §4.1.2).

## 4. Public API (canonical names — README §6.2, doc 04 §4.1.3)

```go
// Package clock establishes a shared timebase across nodes using an NTP-style
// four-timestamp UDP exchange (spine D6, README §6.2). master_mono ≈ follower_mono + Offset.
package clock

// ---- reused from media (clock.go), copied verbatim ----

const PacketSize = 40 // fixed 40-byte 'MPVC' clock packet (README §6.4)

func NowMono() int64 // ns since this process's monotonic epoch (robust to wall-clock steps)

type Sample struct {
    Offset time.Duration // master_mono - follower_mono
    Delay  time.Duration // round-trip delay (>= 0)
}

type Estimator struct{ /* unexported */ }
func NewEstimator(window int, alpha float64) *Estimator
func (e *Estimator) Add(s Sample) time.Duration
func (e *Estimator) Offset() (time.Duration, bool)
func (e *Estimator) MinDelay() (time.Duration, bool)
func (e *Estimator) Samples() int

// ---- reused from media (follower.go) ----

type Follower struct{ /* unexported */ }
type FollowerOption func(*Follower)
func WithInterval(d time.Duration) FollowerOption // A.12: 1 s steady (500 ms initial lock)
func WithTimeout(d time.Duration) FollowerOption  // A.12: 200 ms RPC timeout
func WithSampleHook(fn func(Sample, time.Duration)) FollowerOption
func WithEstimator(window int, alpha float64) FollowerOption // NEW opt: A.12 window/alpha
func NewFollower(opts ...FollowerOption) *Follower
func (f *Follower) Run(ctx context.Context, masterAddr string) error
func (f *Follower) Offset() (time.Duration, bool)   // EWMA offset (master - follower)
func (f *Follower) MinDelay() (time.Duration, bool) // sync-quality proxy
func (f *Follower) MasterNow() (int64, bool)        // follower_mono + Offset (doc 04 §4.4.3)
func (f *Follower) SamplesSeen() int

// ---- reused + gated (server.go) ----

type Server struct{ /* unexported */ }
func Listen(addr string) (*Server, error) // ungated convenience = ListenGated(addr, allowAll)
func ListenGated(addr string, allowed func(netip.Addr) bool) (*Server, error) // NEW: allowlist gate
func (s *Server) Addr() net.Addr
func (s *Server) Close() error

// ---- NEW: ClockSource adapters (source.go) — implement README §6.2, doc 04 §4.1.3 ----

// ClockSource is the canonical README §6.2 contract. DO NOT redefine; it lives in the spine.
// Reproduced here only to show the three adapters satisfy it:
//   type ClockSource interface {
//       Offset() (d time.Duration, ok bool)
//       MinDelay() (time.Duration, bool)
//   }

// MasterSource is the master/solo self-reference: the master IS the timebase, so its offset
// is exactly zero and its delay exactly zero (doc 04 §4.1.3 table row `master`).
type MasterSource struct{}
func (MasterSource) Offset() (time.Duration, bool)   { return 0, true }
func (MasterSource) MinDelay() (time.Duration, bool) { return 0, true }

// FollowerSource adapts a running *Follower to ClockSource (doc 04 §4.1.3 row `follower`).
// It is a thin pass-through; *Follower already has the Offset/MinDelay shape.
type FollowerSource struct{ F *Follower }
func (s FollowerSource) Offset() (time.Duration, bool)   { return s.F.Offset() }
func (s FollowerSource) MinDelay() (time.Duration, bool) { return s.F.MinDelay() }

// OrphanSource is the no-estimate-yet zero value: ok=false on both (doc 04 §4.1.3 row
// `orphan`). A consumer seeing ok=false holds render (doc 04 §4.2.3); Timeline returns
// ok=false (A.1: "A follower with have=false does not render").
type OrphanSource struct{}
func (OrphanSource) Offset() (time.Duration, bool)   { return 0, false }
func (OrphanSource) MinDelay() (time.Duration, bool) { return 0, false }
```

> The `*Follower` already satisfies `ClockSource` structurally; `FollowerSource` exists as
> the explicit named adapter doc 04 §4.1.3 calls for (and so `cmd`/`group` can hold the
> interface uniformly across the three roles). `WithEstimator` is the only *added* public
> symbol on the reused types — it lets the caller pass A.12's `(window, alpha)` instead of
> the mpvsync `(8, 0.15)` default without exposing the `Estimator` construction.

## 5. Behavior & algorithms

### 5.1 Four-timestamp exchange (doc 04 §4.1.1, A.1)

Follower stamps `t1` (send), master stamps `t2` (receive ASAP) and `t3` (send as late as
possible) echoing `t1`, follower stamps `t4` (reply receipt). Per-sample math (A.1, verbatim
from `computeSample`):

```
offset = ((t2 - t1) + (t3 - t4)) / 2      // master_mono - follower_mono
delay  = (t4 - t1) - (t3 - t2)            // round trip minus server processing
if delay < 0 { delay = 0 }
```

### 5.2 Estimator: min-delay filter + EWMA (A.1, doc 04 §4.1.1)

Keep a sliding **window** of `Sample{Offset, Delay}`; pick the **minimum-delay** sample (high
delay = one-directional queuing that biases the offset); EWMA that best offset so the applied
value **slews, never steps**:

```
best = argmin_{w in window} w.delay
if !have { offset = best.offset; have = true }
else      { offset += alpha * (best.offset - offset) }
```

`Offset() -> (offset, have)`; `MinDelay() -> min delay in window` (sync-quality proxy for the
UI, doc 04 §4.1.1 "Quality metric"). A follower with `have=false` does not render (A.1) →
`OrphanSource` / Timeline `ok=false`.

### 5.3 Tunables — **A.12 verbatim, never invent** (doc 04 §4.1.4 gives rationale only)

| Domain | Parameter | Start value (A.12) |
|---|---|---|
| Clock | window N (wired·WiFi) | **8·16** |
| Clock | alpha (wired·WiFi) | **0.15·0.10** |
| Clock | ping | **1 s (500 ms initial lock)** |
| Clock RPC | RPC timeout | **200 ms** |

These are *parameters to the unchanged code* (doc 04 §4.1.4) — set by `cmd`/`group` via
`WithEstimator(window, alpha)`, `WithInterval`, `WithTimeout`. doc 04 §4.1.4 rationale: wider
window → cleaner minimum on WiFi; lower alpha rides over RTT bursts; `200 ms` timeout
tolerates WiFi retry latency. The "500 ms during the first ~5 s of a lock, then 1 s" cadence
(A.1) is a **caller-side** schedule: the engine starts the follower with `WithInterval(500ms)`
and switches to `1s` after lock (≈5 s / first stable `MinDelay`). This package exposes the
knob; it does not own the schedule policy (that is `internal/group`, doc 04 §4.2). Quote A.12
in code comments; do not hard-code a second copy of the numbers anywhere.

### 5.4 One server per master; each follower pings its own master (doc 04 §4.1.2)

The master runs exactly one `ListenGated` (clock UDP `:9000`, A.12); a solo master runs one
too but no one pings it (harmless). Each follower runs one `Follower.Run(ctx, masterAddr)`
where `masterAddr` = the elected master's `NodeRecord.Addrs` (doc 02/07) + `:9000`. A node is
in exactly one group (README §6.2) so it runs **either** a server **or** a follower, never
both. The server socket is allowlist-gated (§5.5); the follower needs no gate (dials one
master, rejects foreign/stale `seq` in `exchange`).

### 5.5 Allowlist gate on the server read loop (A.8, P2.4 §5.3, doc 04 §4.1.2)

```
serve():
  buf := make([]byte, PacketSize)
  for {
    n, src, err := conn.ReadFromUDPAddrPort(buf)   // netip.AddrPort, no per-pkt alloc
    t2 := NowMono()                                 // stamp receive ASAP, before any branch
    if err != nil { if errClosed { return }; continue }
    if !allowed(src.Addr()) { continue }            // DROP, no reply (A.8) — before unmarshal
    req, err := unmarshal(buf[:n]); if err || req.kind != request { continue }
    reply := packet{kind: reply, seq, t1: req.t1, t2, t3: NowMono()}
    conn.WriteToUDP(reply.marshal(), net.UDPAddrFromAddrPort(src))
  }
```

Stamp `t2` immediately on read return (before the allowlist branch) to keep master service
time minimal and honest; the drop happens before `unmarshal` so foreign/malformed bytes never
reach the codec. `allowed` is `set.AllowedAddr` from the shared `*allowlist.Set` (P2.4),
recomputed on `ConfigDoc`/membership change by `allowlist.Run` — the gate updates live with no
socket restart (P2.4 §5.2). The cross-group safety property (doc 04 §4.1.2) holds: identical
wire format, but a cross-group/forgotten source IP is dropped by the gate.

### 5.6 `ClockSource` role mapping (doc 04 §4.1.3)

| Role | impl | `Offset()` | `MinDelay()` |
|---|---|---|---|
| `master`/`solo` | `MasterSource{}` | `0, true` | `0, true` (reference is exact) |
| `follower` | `FollowerSource{F}` | `F.Offset()` (EWMA) | `F.MinDelay()` |
| `orphan` | `OrphanSource{}` | `0, false` | `0, false` |

The engine (doc 04 §4.2) selects the adapter per role and feeds it to the `Timeline`
projection (doc 04 §4.4.3): `master_now = NowMono() + Offset`. `orphan` ⇒ `ok=false` ⇒ render
held (doc 04 §4.2.3). `MinDelay()` is the §4.2 quality gate (hysteretic enter/exit so a WiFi
spike never flaps the state, doc 04 §4.1.4/§4.2.3) — but the hysteresis lives in `group`, not
here; this package only reports the metric.

## 6. Import constraints (01 §2 layering)

`internal/clock` is a **low-level realtime package** below the engine. It MAY import:

- Go stdlib only for the reused core: `context`, `encoding/binary`, `errors`, `net`,
  `net/netip`, `sync`, `time`.

It MUST NOT import:

- `internal/group`, `internal/stream/*`, `internal/audio/*` — those are **consumers** of
  `ClockSource`/`Follower`/`Server`, never dependencies (no upward imports; doc 04 §4.0 owns
  the engine wiring).
- `internal/cluster`, `internal/state`, `internal/allowlist` — the allowlist is **injected**
  as a `func(netip.Addr) bool` predicate (§5.5), *not* imported. This keeps `clock` testable
  with a synthetic predicate and avoids pulling memberlist/state into the realtime package
  (mirrors P2.4 §6's injection rule — the gate is supplied, not depended on). `cmd` binds
  `set.AllowedAddr` into `ListenGated`.
- `internal/web` — the `web.Deps` seam never touches clock (web must not import the engine,
  01 §2 / A.14.3).

> Rationale: injecting the allowlist predicate (rather than importing `internal/allowlist`)
> preserves layering and lets `server_test.go` run with a plain in-test predicate. The §6.2
> `ClockSource` interface is satisfied structurally — `clock` does not import the spine
> definition (it lives wherever §6.2 places it / is duck-typed by consumers), it only
> provides conforming types.

## 9. Risks / open questions

- **Q1 (resolved, flag if P2.4 disagrees): gate predicate type.** P3.1 uses
  `func(netip.Addr) bool` (= `allowlist.Set.AllowedAddr`, P2.4 §4) on `ListenGated`, reading
  the socket with `ReadFromUDPAddrPort` to avoid a per-packet `*net.UDPAddr` alloc on the Pi
  hot path. This matches P2.4 §5.3 / Q2. The reply path recovers `*net.UDPAddr` via
  `net.UDPAddrFromAddrPort(src)` once per *accepted* packet only. If P2.4 instead standardizes
  on a `GateUDP(conn, set, deliver)` *read-loop* wrapper, the clock server's `serve()` could
  alternatively be expressed as a `GateUDP` `deliver` callback — but the server **must reply**
  (master side), and `GateUDP`'s `deliver(src, b)` gives the address, so it is workable.
  **Decision needed from the orchestrator:** does the clock server gate inline (predicate, as
  specified here) or route through P2.4's `GateUDP` wrapper? Specified inline because the
  server owns the reply socket; confirm P2.4 is content to expose `AllowedAddr` for this.
