## 1. Purpose & scope

`internal/group` is the per-node **group engine**: the deterministic role state machine (`solo` / `master` / `follower` / `orphan`, plus the render-orthogonal "master, no local render" and control-only `member` postures — 04 §4.2/§4.2.4), the **sample-domain `Timeline`** that projects the shared clock into a canonical-rate stream sample index (04 §4.4, A.2, README §6.2), the master-side **`NegotiateProfile`** least-capable-listener selection (04 §4.3), the **`ClockSource` adapters** (master self-reference / follower wrap / orphan zero-value, 04 §4.1.3), and the idempotent **`applyRole`** loop that starts/stops role subsystems fenced by `generation`/`streamGen` (A.14.4). It is a pure function of `(electionResult, myGroupMembership, profile, myCaps, clockHealth)` recomputed on every gossip/election event (04 §4.2.2 "Determinism"); it never infers role from packets. It owns the in-process Timeline and the role lifecycle; it consumes — but does not implement — the clock (P3.1), election/membership (P2.3), the stream origin/receiver (stream/*), and the render loop (audio/render).

## 4. Public API (canonical names — README §6, 04 §4.7)

```go
package group

// ── Sample-domain timeline (README §6.2; 04 §4.4) ───────────────────────────

// Timeline is the per-group synchronized stream sample index (canonical-rate frames).
// On the master it is authoritative; a follower projects via a ChunkMetaSource + ClockSource.
type Timeline interface { // README §6.2 — do not redefine
    NowSample() (sample int64, playing bool, ok bool)
}

// MasterTimeline is the authoritative timeline (04 §4.4.2). Implements Timeline.
type MasterTimeline struct { /* mu; baseSample int64; baseMono int64; rate int; playing bool */ }

func NewMasterTimeline(rate int) *MasterTimeline       // paused at sample 0 (T1)
func (t *MasterTimeline) Play(fromSample int64)        // transport: play from sample
func (t *MasterTimeline) Pause()                        // freeze at current sample
func (t *MasterTimeline) Seek(posSample int64)          // re-base, preserve playing
func (t *MasterTimeline) Seed(baseSample int64, playing bool) // T4/T8 failover continuity (04 §4.4.4)
func (t *MasterTimeline) NowSample() (int64, bool, bool)      // ok always true (it is the reference)
func (t *MasterTimeline) Rate() int

// ChunkMeta is the newest-received chunk's anchor (README §6.4 header fields).
type ChunkMeta struct {
    SampleIndex int64  // first canonical-rate frame on the group timeline
    MasterMono  int64  // master monotonic ns when chunk was sourced
    StreamGen   uint64 // group stream generation
    Playing     bool   // carried from master transport state
}

// ChunkMetaSource is supplied by the stream receiver (P3.1 / stream/sink_net).
type ChunkMetaSource interface { LatestChunkMeta() (ChunkMeta, bool) }

// FollowerTimeline projects the master timeline to "now" (04 §4.4.3 derivation A, A.2).
type FollowerTimeline struct { /* chunks ChunkMetaSource; clk ClockSource; rate int; gen uint64 */ }

func NewFollowerTimeline(chunks ChunkMetaSource, clk ClockSource, rate int) *FollowerTimeline
func (f *FollowerTimeline) SetStreamGen(gen uint64)         // current generation gate (§4.4.3)
func (f *FollowerTimeline) NowSample() (int64, bool, bool)  // ok=false until Offset ok && chunk for gen

// ── ClockSource adapters (README §6.2; 04 §4.1.3) ───────────────────────────

type ClockSource interface { // README §6.2 — do not redefine
    Offset() (d time.Duration, ok bool)
    MinDelay() (time.Duration, bool)
}

func MasterClock() ClockSource                 // self-reference: (0,true)/(0,true)
func FollowerClock(f *clock.Follower) ClockSource // wraps clock.Follower (P3.1)
func OrphanClock() ClockSource                  // (0,false)/(0,false)

// ── Profile negotiation (04 §4.3.2) ─────────────────────────────────────────

type Profile struct {
    Codec          string // "pcm"|"opus"   (§6.3 string enum; FLAC is source-only)
    FEC            string // "none"|"xorParity"|"duplicate" (§6.3 string enum)
    Rate           int    // canonical Hz, default 48000 (A.12)
    FramesPerChunk int    // §6.4, default 480 (A.12)
}

// NegotiateProfile picks the richest codec/FEC/rate every Render=true LISTENER can decode,
// constrained to what the MASTER can encode. Render=false members are ignored. (04 §4.3.1/§4.3.2)
func NegotiateProfile(members []state.NodeRecord, master state.NodeRecord) Profile

// ── Engine state machine (04 §4.2) ──────────────────────────────────────────

type Role int
const ( RoleStarting Role = iota; RoleSolo; RoleMaster; RoleFollower; RoleOrphan )

// Posture overlays Role for the render-orthogonal sink-less variants (04 §4.2 modifiers).
type Posture int
const ( PostureNormal Posture = iota; PostureNoLocalRender; PostureControlOnlyMember )

// Inputs is the complete pure-function input (04 §4.2.2 "Determinism").
type Inputs struct {
    SelfID         string
    GroupID        string
    Members        []state.NodeRecord   // this group's members (from ConfigDoc; P2.3 state)
    MasterID       string               // election outcome (P2.3 cluster.GroupElections)
    Generation     uint64               // per-group election generation (P2.3; A.5)
    Profile        Profile              // published GroupRecord.profile
    Playing        bool                 // replicated GroupRecord.Playing (R4)
    SampleIndex    int64                // replicated/last-projected sample for Seed continuity
    MyCaps         state.Capabilities   // self effective caps (README §6.5)
    ClockOK        bool                 // Offset().ok
    MinDelayOK     bool                 // MinDelay under the orphan exit/enter threshold (orphan.go)
}

// Decision is the resolved role/posture + the subsystems applyRole must run.
type Decision struct {
    Role        Role
    Posture     Posture
    IsMaster    bool
    RunOrigin   bool   // start stream origin + clock server
    RunReceiver bool   // start stream receiver
    RunRender   bool   // start local render (gated on MyCaps.Render)
    RunClockSrv bool
    RunClockFol bool
    StreamGen   uint64 // bumped on master change / profile change (A.5/§4.3.3)
}

// Recompute is the pure transition function; safe to call on every gossip/election tick.
func Recompute(prev Decision, in Inputs) Decision

// ── applyRole loop (A.14.4) ─────────────────────────────────────────────────

// Hooks is the side-effecting seam (origin/receiver/clock/render lifecycle). cmd wires it
// to stream/* + audio/render + clock; tests use a fake. The engine itself imports none of them.
type Hooks struct {
    StartClockServer   func(group string) error
    StopClockServer    func()
    StartOrigin        func(group string, streamGen uint64) error
    StopOrigin         func()
    OriginResumeAt     func(sampleIndex int64, playing bool)        // A.14.4 / R4
    StartClockFollower func(group, masterAddr string) error
    StopClockFollower  func()
    StartReceiver      func(group string) error
    StopReceiver       func()
    ReceiverFlushReprime func()                                      // R11
    StartRender        func() error
    StopRender         func()
}

type Engine struct { /* selfID; hooks Hooks; mu; running Decision; orphan *orphanGate */ }

func NewEngine(selfID string, hooks Hooks) *Engine
// Apply recomputes the decision from in and idempotently reconciles subsystems,
// fenced by Generation/StreamGen (A.14.4). Returns the applied Decision.
func (e *Engine) Apply(in Inputs) Decision
```

## 5. Behavior & algorithms

**Timeline math (04 §4.4, A.2).** Master authoritative: `atLocked(mono) = playing ? baseSample + round((mono−baseMono)·rate/1e9) : baseSample` (04 §4.4.2); `rate` is the profile rate (A.12 default **48000**). Follower projection (derivation A, the implemented one): `master_now = clock.NowMono() + Offset; NowSample = chunk.SampleIndex + round((master_now − chunk.MasterMono)·rate/1e9)` (04 §4.4.3, A.2). `ok = Offset().ok && have a chunk for the current streamGen`; a `streamGen` mismatch is treated as not-synced until a current-generation chunk arrives (A.2, 04 §4.4.3). `playing` is carried from the master via `ChunkMeta.Playing`; a paused follower freezes `NowSample` at the last sample. The worked example (04 §4.5) is a unit-test vector: anchor `(96000, masterMono=3_000_000ns)`, `Offset=+812_000ns`, `clock.NowMono()=2_400_000ns`, `rate=48000` ⇒ `NowSample = 96000 + round(212_000·48000/1e9) = 96000 + round(10.176) = 96010`.

**ClockSource adapters (04 §4.1.3).** master ⇒ `(0,true)/(0,true)`; follower ⇒ delegates to `clock.Follower.Offset()/MinDelay()` (EWMA + min-delay filter; A.1; P3.1); orphan ⇒ `(0,false)/(0,false)`.

**NegotiateProfile (04 §4.3.1/§4.3.2).** `listeners = members where Caps.Render==true`. codec: `"opus" > "pcm"` over `∩ listeners' Caps.DecodeCodecs`, then `∩ master.Caps.EncodeCodecs` — **PCM is the mandatory universal floor** (D15-safe). FEC: `"xorParity" > "duplicate" > "none"` over `∩ listeners' Caps.FEC` (XOR is the default ceiling, D4). rate: `min over listeners' Caps.MaxRate` (default **48000**, A.12); `FramesPerChunk` fixed at **480** (A.12). Render=false members (incl. a sink-less master) never constrain codec/FEC/rate — only the master's `EncodeCodecs` acts as a ceiling. **Zero-listener group** (04 §4.3.4): empty listener set yields no playback constraint; leave `Profile` at default/last bounded by master `EncodeCodecs` (not an error/orphan). Worked example (04 §4.3.2): N1 sink-less master `{pcm,opus}` + N2 `{pcm,opus}/{xorParity}` + N3 `{pcm,opus}/{xorParity,duplicate}` + dumb N4 `{pcm}/{duplicate}` ⇒ `Profile{"pcm","duplicate",48000,480}`; drop N4 ⇒ `Profile{"opus","xorParity",48000,480}`.

**Engine / transitions (04 §4.2.1/§4.2.2).** States `solo/master/follower/orphan`; `solo` = `master` with empty follower set (promotion needs no origin/clock restart — T2/T10). Render-orthogonal postures (D17, §6.5 `Caps.Render`): a `Caps.Render=false` master/solo runs `PostureNoLocalRender` (origin+clock, no sink); a non-master `Caps.Render=false` member runs `PostureControlOnlyMember` (clock follower for liveness only — no receiver/decode/ring/render — 04 §4.2.4) and is **never** `follower`/`orphan`. Implement the full T1–T11 matrix (04 §4.2.2): T2 solo→master (negotiate+publish+unicast), T3 →follower (render-capable) else control-only member, T4 follower→master (read replicated `Playing`, `Timeline.Seed(lastSample, Playing)`, bump `streamGen` — 04 §4.4.4 / R4), T5 re-point follower on master change, T6/T7 orphan enter/exit, T8 orphan→master, T9 →solo on removal, T10 master→solo, T11 render toggle (start/stop render sub-system only; `follower`↔control-only member; may trigger master renegotiation). **Determinism:** `Recompute` is a pure function of `Inputs`; never infers role from packets (control plane authoritative). Group-move transient (R13 m1 / 02 §7): on observing own `GroupID` change, **orphan (silence)** until the new group's master+clock+stream are in hand.

**orphan policy (04 §4.2.3).** Silence > skew. Hysteretic on `MinDelay()`: a **high enter threshold** and a **lower exit threshold** plus a consecutive-window count (T6 requires `MinDelay` over threshold for N consecutive windows), so a single WiFi spike never flaps state (04 §4.1.4). `solo`/`master` and `Caps.Render=false` members are never `orphan`. (A.12 fixes clock window/alpha; the orphan thresholds are not pinned in A.12 — see Risks.)

**applyRole (A.14.4).** Idempotent: if master and not already running master, stop follower parts, start clock server + stream origin under `streamGen`, `OriginResumeAt(SampleIndex, Playing)`; else stop master parts, start clock follower (against elected master addr) + receiver, `ReceiverFlushReprime()` (~one buffer, R11). **Generation fence:** any subsystem started under an older `generation`/`streamGen` is torn down before the new one starts, so a superseded master cannot keep emitting on the group's planes. On master change the new master bumps `streamGen` (A.5/R11).

## 8. Dependencies

**Consumes (upstream pieces):**
- **P3.1 (`internal/clock`)** — `clock.Follower` (`Offset()`, `MinDelay()`, `MasterNow()`, `Run(ctx, masterAddr)`), `clock.Server`/`clock.Listen`, package func `clock.NowMono()`. Used by `FollowerClock`, the Timeline mono stamp, and (via Hooks, in cmd) the clock server/follower lifecycle.
- **P2.3 (`internal/cluster` + `internal/state`)** — `cluster.GroupElections.Recompute` outcome (`MasterID`, `Generation`, `IsSelf`) fed into `Inputs`; `state.NodeRecord`/`state.Capabilities`/`ConfigDoc`/`GroupRecord.Profile`/`GroupRecord.Playing` (README §6.5). The engine reads these; it does not mutate the ConfigDoc (writes go through `state.Apply` in the master's publication path, wired in cmd).

**Exposes (for downstream pieces):**
- `group.Timeline` (interface) + `MasterTimeline`/`FollowerTimeline` — consumed by **audio/render** (06) `NowSample()` to pick the next ring sample, and by **stream/origin** (master stamps chunks from `MasterTimeline`).
- `group.Profile` + `NegotiateProfile` — consumed by the master publication path (writes `GroupRecord.profile`) and by **stream/codec**+**stream/fec** config (followers configure decoders from the published profile).
- `group.ChunkMetaSource` (interface) — implemented by **stream/sink_net** receiver (P3.1/stream); the receiver pushes `ChunkMeta` so the FollowerTimeline can anchor.
- `group.Engine` / `Hooks` — wired in **cmd/ensemble** to stream/origin, stream/sink_net, audio/render, and clock; also feeds **web** group/role status indirectly (via cmd, never a direct web→group import — A.14.3).

## 9. Risks / open questions

1. **Orphan hysteresis thresholds are NOT in A.12.** 04 §4.2.3 mandates a high enter / low exit `MinDelay()` threshold and "N consecutive windows," but Appendix A.12 pins clock `window`/`alpha`/`ping` only — not the orphan enter/exit ms or N. **Blocking-ish:** values must be invented (violating "never invent tunables") unless the orchestrator pins them. Proposed defaults to ratify and add to A.12: enter > 15 ms, exit < 8 ms, N = 3 windows (consistent with the WiFi 10–50 ms spike envelope, 04 §4.1.4). **Needs orchestrator confirmation.**
2. **`ChunkMeta` provider ownership.** The FollowerTimeline needs `LatestChunkMeta()` from the receive path. Whether `ChunkMetaSource` is implemented in P3.1 or in the stream/sink_net piece (later phase) determines wiring order; the interface lives here so P3.2 can be built and tested standalone with a fake. Confirm the receiver piece will implement this interface.
3. **`clock.NowMono()` injection for tests.** media's `Timeline` calls `clock.NowMono()` directly (unmockable). For deterministic Timeline tests, either expose a `nowMono func() int64` field on `MasterTimeline`/`FollowerTimeline` (defaulting to `clock.NowMono`) or test via offset deltas.
