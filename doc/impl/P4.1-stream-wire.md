## 1. Purpose & scope

`internal/stream/wire` is the lowest layer of the audio plane: it defines the on-wire
**ESND** packet — the fixed-size **44-byte big-endian header** (README §6.4) followed by
an opaque payload — and provides allocation-conscious `Marshal`/`Unmarshal`, the `Header`
and `Packet` value types, the `Flags` bitset (`bit0 = repair`, `bit1 = keyframe`), and the
**name↔id registry** that maps the ConfigDoc/API **string** codec/FEC enums (`"pcm"|"opus"`,
`"none"|"xorParity"|"duplicate"`) to/from the integer `CodecID`/`FECID` that appear *only*
on the wire (README §6.5, 07 §6). It owns no audio logic: it never decodes, encodes, FECs,
paces, or touches a socket — it is a pure, dependency-light codec for bytes ⇄ structs that
both the origin (master, 05 §5.2) and the receiver (follower, 05 §5.6) build on. It is
allocation-free on the hot path (caller-supplied buffers), strictly validates untrusted
input on `Unmarshal` (the receiver treats every datagram as hostile, 03 §6.3), and is the
single place the 44-byte layout is encoded so the rest of `stream/*` never hand-rolls
offsets.

## 4. Public API (canonical names — README §6.3 / §6.4 / §6.5, 05, 07 §6)

```go
// Package wire defines the ESND audio packet: a fixed 44-byte big-endian header
// (README §6.4) plus an opaque payload, with Marshal/Unmarshal and the name↔id
// registry mapping the ConfigDoc/API string codec/FEC enums to the integer ids that
// appear only on the wire (README §6.5, 07 §6). Pure bytes<->structs; no audio logic.
package wire

import "encoding/binary"

// --- Layout constants (README §6.4 — the ONLY normative source; do not restate elsewhere) ---

const (
    Magic       uint32 = 0x45534E44 // 'E''S''N''D' (big-endian on the wire, offset 0)
    Version     uint8  = 1          // header version, fixed =1 this build (README §6.4)
    HeaderSize         = 44         // fixed header size in bytes (README §6.4)
)

// Field byte offsets within the header (README §6.4). Exported for golden tests / readers.
const (
    offMagic       = 0  // 4  uint32  'ESND'
    offVersion     = 4  // 1  uint8   =1
    offFlags       = 5  // 1  uint8   bit0 repair, bit1 keyframe
    offCodecID     = 6  // 1  uint8   CodecID
    offFECID       = 7  // 1  uint8   FECID
    offStreamGen   = 8  // 8  uint64  group stream generation (§5.8)
    offSeq         = 16 // 8  uint64  monotonic source packet sequence
    offSampleIndex = 24 // 8  int64   first sample's canonical-rate frame index
    offMasterMono  = 32 // 8  int64   master monotonic ns when sourced
    offPayloadLen  = 40 // 2  uint16  payload length in bytes
    offRate100     = 42 // 2  uint16  rate/100 (480 => 48000 Hz)
)

// Flags is the header flags bitset (README §6.4, offset 5).
type Flags uint8

const (
    FlagRepair   Flags = 1 << 0 // bit0: this is an FEC repair packet (§5.5.1)
    FlagKeyframe Flags = 1 << 1 // bit1: decodable cold (always set for PCM; Opus keyframes) (§5.4)
)

func (f Flags) Repair() bool   { return f&FlagRepair != 0 }
func (f Flags) Keyframe() bool { return f&FlagKeyframe != 0 }

// Header is the parsed ESND header (README §6.4). Field order/types mirror the wire
// layout. SampleIndex/MasterMono are signed (timeline frame index / monotonic ns,
// both fit int64 and may be diffed). StreamGen/Seq are unsigned monotonic counters.
type Header struct {
    Flags       Flags  // offset 5
    CodecID     CodecID // offset 6
    FECID       FECID   // offset 7
    StreamGen   uint64 // offset 8  (§5.8)
    Seq         uint64 // offset 16
    SampleIndex int64  // offset 24 (canonical-rate frame index on the group timeline)
    MasterMono  int64  // offset 32 (master monotonic ns at source, §5.2.2)
    PayloadLen  uint16 // offset 40 (length of the payload that follows)
    Rate100     uint16 // offset 42 (rate/100; 480 => 48000 Hz; sanity/redundancy)
}
// Magic and Version are NOT stored on Header: they are constants this build always
// writes and always validates; carrying them as fields would invite drift.

// Marshal returns a freshly-allocated []byte of HeaderSize+len(payload) bytes:
// the big-endian header followed by payload verbatim. h.PayloadLen MUST equal
// len(payload) (Marshal sets it from len(payload) defensively and returns an error
// on uint16 overflow). Convenience wrapper over MarshalInto for non-hot-path callers.
func Marshal(h Header, payload []byte) ([]byte, error)

// MarshalInto writes the header into dst[:HeaderSize] and is the allocation-free hot
// path for the origin (05 §5.2.3 encodes once, fans out): the caller owns the buffer
// and appends/copies the payload itself, or passes a dst already sized to
// HeaderSize+payloadLen. It writes ONLY the 44 header bytes; it does not touch the
// payload region. Returns an error if len(dst) < HeaderSize or payloadLen overflows
// uint16. h.PayloadLen is taken as authoritative (caller sets it = len(payload)).
func MarshalInto(dst []byte, h Header) error

// Unmarshal parses one received datagram (05 §5.6.2 step 3). It validates magic,
// version, len(buf) >= HeaderSize, and HeaderSize+PayloadLen <= len(buf); on any
// failure it returns a non-nil error and the zero Header/nil payload. The returned
// payload is a SUBSLICE of buf (no copy) valid only while buf is unmodified — the
// receiver's reorder window must Clone (Packet.Clone) before retaining it.
func Unmarshal(buf []byte) (Header, payload []byte, err error)

// --- packet.go ---

// Packet is a parsed ESND packet: the header plus its payload bytes. It is the unit
// the FEC interface trades in (README §6.3: Recover(p Packet) / the receiver's
// Packet{hdr,payload} at 05 §5.6.2 step 5). The codec/fec packages (P3.2) import this
// type by its canonical name.
type Packet struct {
    Header  Header
    Payload []byte // codec frame, or FEC repair data (when Header.Flags.Repair())
}

func (p Packet) Repair() bool   { return p.Header.Flags.Repair() }   // README §6.4 bit0
func (p Packet) Keyframe() bool { return p.Header.Flags.Keyframe() } // README §6.4 bit1

// Clone returns a deep copy whose Payload is a freshly-allocated slice, so the result
// is safe to retain past the read buffer's lifetime (the reorder/FEC window keeps
// packets across reads — 05 §5.6.2). Header is a value type (copied by assignment).
func (p Packet) Clone() Packet

// Marshal serializes the packet (header reflects p.Payload's length). Thin wrapper
// over wire.Marshal so origin code can hold a Packet and emit bytes symmetrically.
func (p Packet) Marshal() ([]byte, error)

// --- registry.go : name<->id mapping (README §6.5, 07 §6, R4(a)) ---

// CodecID is the integer wire codec id (README §6.3). It exists ONLY on the wire
// (header offset 6); JSON/ConfigDoc/API always use the string name (07 §6).
type CodecID uint8

const (
    CodecPCM  CodecID = 0 // S16LE, mandatory baseline (05 §5.4.1, A.10 m5)
    CodecOpus CodecID = 1 // optional, capability-gated (05 §5.4.2)
)

// FECID is the integer wire FEC id (README §6.3). Wire-only, like CodecID.
type FECID uint8

const (
    FECNone      FECID = 0 // identity (05 §5.5, §5.9 TCP path)
    FECXORParity FECID = 1 // default (05 §5.5.1)
    FECDuplicate FECID = 2 // MCU-friendly alt (05 §5.5.4)
)

// CodecName maps a CodecID to its canonical JSON string ("pcm"|"opus"). Returns
// ("", false) for an unknown id (a malformed/foreign packet — caller drops it).
func CodecName(id CodecID) (string, bool)

// ParseCodec maps a JSON/ConfigDoc string ("pcm"|"opus") to its CodecID. Returns
// (0, false) on an unknown name (negotiation/profile bug — caller surfaces an error).
func ParseCodec(name string) (CodecID, bool)

// FECName / ParseFEC: the same bidirectional mapping for FEC
// ("none"|"xorParity"|"duplicate" <-> FECNone/FECXORParity/FECDuplicate).
func FECName(id FECID) (string, bool)
func ParseFEC(name string) (FECID, bool)
```

> **Naming fidelity.** Every exported name above is taken verbatim from the canonical
> contracts: `CodecID`/`FECID` and their values from README §6.3 + 05 §5.4/§5.5;
> `Packet` and `Header` field names (`StreamGen`, `Seq`, `SampleIndex`, `MasterMono`,
> `PayloadLen`, the `rate/100` field as `Rate100`) from README §6.4 / 05 §5.2.2; the
> string enums `"pcm"|"opus"`, `"none"|"xorParity"|"duplicate"` from README §6.5 / 07 §6.
> No alternatives invented.

## 5. Behavior & algorithms

### 5.1 The header layout (README §6.4 — verbatim, the only source of truth)

```
offset size field
0      4    magic 'ESND'                  (uint32 BE = 0x45534E44)
4      1    version (=1)
5      1    flags (bit0: repair packet, bit1: keyframe)
6      1    codecID
7      1    fecID
8      8    streamGen   (group stream generation; reset on media/seek change)
16     8    seq         (monotonic source packet sequence)
24     8    sampleIndex (first sample's canonical-rate frame index on the group timeline)
32     8    masterMono  (master monotonic ns when this chunk was sourced)
40     2    payloadLen
42     2    rate/100    (e.g. 480 => 48000 Hz; sanity/redundancy)
44     ...  payload (codec frame, or FEC repair data)
```

All multi-byte fields are **big-endian** (README §6.4). `Marshal` uses
`binary.BigEndian.PutUint{16,32,64}` at the fixed offsets above; `sampleIndex`/`masterMono`
are written as the two's-complement bit pattern via `PutUint64(uint64(v))` and read back
with `int64(BigEndian.Uint64(...))`. `version` and `magic` are written from the package
constants on every `Marshal`.

### 5.2 `Marshal` / `MarshalInto` — origin side (05 §5.2.2, §5.2.3)

The origin builds the header once per chunk (05 §5.2.2) and fans the bytes out to every
listener (05 §5.2.3 — "encode once, copy/send per listener"). To honor the D5 CPU claim
and the Pi budget (README §8), the hot path must not allocate per listener:

- `MarshalInto(dst, h)` writes the 44 header bytes into a caller-owned buffer; the origin
  builds `[header || payload]` once and reuses one backing array across the per-listener
  `WriteToUDP` fan-out. No per-packet allocation in steady state.
- `Marshal(h, payload)` is the convenience form (allocates once) for cold paths and tests.
- `PayloadLen` is set from the actual payload length and validated to fit `uint16`
  (max 65535). For the canonical S16LE PCM chunk that is `480 frames × 2 ch × 2 B = 1920 B`
  (05 §5.10, well under the limit); Opus is tens of bytes (05 §5.11).

### 5.3 `Unmarshal` — receiver side, untrusted input (05 §5.6.2 step 3, 03 §6.3)

Per 05 §5.6.2: `wire.Unmarshal(buf) → (hdr, payload); validate magic 'ESND', version,
payloadLen`. The datagram has already passed the **allowlist** source-IP gate (P2.4) but
its *contents* are still untrusted (03 §6.3), so `Unmarshal` strictly validates and never
panics on hostile bytes:

1. `len(buf) >= HeaderSize` else `ErrShort`.
2. `BigEndian.Uint32(buf) == Magic` else `ErrMagic`.
3. `buf[offVersion] == Version` else `ErrVersion` (mixed-epoch / future header — drop;
   protocol epoch is locked to 1 this build, README §6.6 / D-policy).
4. Read `PayloadLen`; require `HeaderSize+int(PayloadLen) <= len(buf)` else `ErrPayloadLen`
   (truncated/oversized claim). A trailing-bytes case (`len(buf) > HeaderSize+PayloadLen`,
   e.g. UDP padding) is tolerated — payload is exactly `buf[HeaderSize:HeaderSize+PayloadLen]`.
5. `payload` is returned as a **subslice** of `buf` (zero-copy); the caller (receiver
   reorder window) `Clone()`s before retaining (05 §5.6.2 "release to decode in seq order"
   needs ownership). `CodecID`/`FECID` are **not** range-validated here — an unknown id is
   the codec/FEC layer's concern (it maps via the registry and drops on `false`); `wire`
   only guarantees structural integrity, keeping it codec-agnostic.

The TCP-fallback transport (05 §5.9) frames the *same* marshaled bytes behind a 2-byte
length prefix; that prefix framing lives in the origin/receiver TCP path, **not** in
`wire` — `wire` produces/consumes one packet's bytes regardless of transport.

### 5.4 The name↔id registry (README §6.5, 07 §6, R4(a))

README §6.5 and 07 §6 are explicit: **"String enums in JSON; integer ids only on the
wire … mapped to/from these names via a name↔id registry."** This package owns that
registry because it owns the wire boundary. The mapping is a fixed, total table:

| String (JSON/ConfigDoc/API) | Wire id | Const |
|---|---|---|
| `"pcm"` | 0 | `CodecPCM` |
| `"opus"` | 1 | `CodecOpus` |
| `"none"` | 0 | `FECNone` |
| `"xorParity"` | 1 | `FECXORParity` |
| `"duplicate"` | 2 | `FECDuplicate` |

- Lookups are O(1) (tiny `map`/`switch`); both directions are pure functions with a
  `bool` "known?" return — never a panic, never a default-to-PCM guess (an unknown name
  is a negotiation/config bug the caller must surface; an unknown id is a foreign packet
  the receiver drops). This matches the "drop on unknown" posture of 05 §5.6.3.
- The registry is the **only** translation point: the group engine negotiates in strings
  (04 §4.3.1 `Profile{Codec, FEC string}`), `cmd`/`origin` calls `ParseCodec`/`ParseFEC`
  once when constructing the codec/FEC for a `streamGen`, and `Unmarshal` consumers call
  `CodecName`/`FECName` only for logging/metrics. The integer ids never appear in JSON
  (07 §6).

### 5.5 Quoted tunables (A.12 verbatim — never invent)

`wire` hard-codes no audio tunables; it only *carries* them. The values it must round-trip
faithfully, quoted from **A.12**:

| Domain | Parameter | Start value |
|---|---|---|
| Audio canonical | rate / channels / FramesPerChunk | `48000 / 2 / 480 (10 ms)` |

⇒ the `Rate100` field carries `48000/100 = 480` (README §6.4 example "480 => 48000 Hz");
the canonical S16LE PCM payload is `480 × 2 × 2 = 1920` bytes (05 §5.10). `wire` validates
that `PayloadLen` fits `uint16` but imposes no `FramesPerChunk`-specific size — a future
profile may renegotiate it (05 §5.1), and the field is in the header precisely so it can.
`version`/`magic` are fixed constants (README §6.4); `protocolEpoch = 1` is enforced at
adoption (03 §2.7), not in `wire`, but the `version=1` check is the wire-level reflection
of it.

## 6. Import constraints (01 §2 layering)

`internal/stream/wire` is the **bottom** of the `stream/*` stack — it must import *nothing*
from inside the engine, so both the master (origin) and follower (receiver) sides, and the
codec/fec packages (P3.2), can depend on it without cycles.

**MAY import:** Go stdlib only — `encoding/binary`, `errors` (and `fmt` for error
wrapping). Nothing else.

**MUST NOT import:**
- `internal/stream/codec`, `internal/stream/fec` (P3.2) — the dependency runs the *other*
  way: they import `wire` for `CodecID`/`FECID`/`Packet`. `wire` knowing about concrete
  codecs would be a cycle and a layering inversion.
- `internal/group`, `internal/state`, `internal/clock`, `internal/audio/*`,
  `internal/allowlist`, `internal/web` — all are higher layers or unrelated planes.
- Any third-party module — `wire` is pure stdlib so it cross-compiles trivially and has no
  supply-chain surface on the hot path.

> This keeps `wire` a leaf the way 01 §2 intends: the realtime audio plane's serialization
> primitive, testable in complete isolation, importable by everything above it.

## 7. Test plan

All tests are `package wire` (white-box, to assert exact byte offsets), Go 1.26 `testing`,
table-driven, **no network, no allocation on the asserted hot paths** (use `testing.B` +
`-benchmem` / `testing.AllocsPerRun` to prove `MarshalInto`/`Unmarshal` are zero-alloc).
Acceptance ties to **A.13 P4** (a PCM stream plays in sync — its packets must round-trip
bit-exactly) and **05 §5.6.2 / §5.10** (validation + the golden worked packet).

### 7.1 `wire_test.go` — Marshal/Unmarshal (unit)

| Case | Asserts |
|---|---|
| round-trip identity | `Unmarshal(Marshal(h, payload))` yields a `Header` equal to `h` and payload bytes equal to the input, across a table of headers (varied flags, codec/fec ids, large `Seq`/`SampleIndex`/`MasterMono`, max `PayloadLen`) |
| golden bytes (05 §5.10) | the worked-diagram packet (`version=1`, flags `0x02` keyframe, codec=PCM, fec=XORParity, `streamGen=0x2A3F`, `seq=123456`, `sampleIndex=58982400`, `payloadLen=1920`, `rate100=480`) marshals to the **exact** 44 header bytes from §5.10; lock the byte string as a golden constant |
| big-endian-ness | each multi-byte field's bytes appear MSB-first at its README §6.4 offset (assert specific bytes, not just round-trip) |
| signed fields | negative `SampleIndex`/`MasterMono` (defensive) round-trip via two's-complement |
| `MarshalInto` == `Marshal` | header bytes from `MarshalInto(dst,h)` are byte-identical to `Marshal(h,payload)[:44]`; `MarshalInto` does not touch `dst[44:]` |
| zero-alloc hot path | `testing.AllocsPerRun(MarshalInto)` == 0 and `Unmarshal` == 0 allocs (subslice payload, no copy) |
| payloadLen overflow | payload > 65535 B ⇒ `Marshal`/`MarshalInto` return an error, no panic |

### 7.2 `wire_test.go` — Unmarshal validation / hostile input (unit + fuzz)

| Case | Asserts |
|---|---|
| short buffer | `len(buf) < 44` ⇒ `ErrShort`, zero Header, nil payload |
| bad magic | first 4 bytes != 'ESND' ⇒ `ErrMagic` |
| bad version | `buf[4] != 1` ⇒ `ErrVersion` (drop future/foreign header) |
| payloadLen > available | `44 + PayloadLen > len(buf)` ⇒ `ErrPayloadLen` |
| exact fit | `len(buf) == 44 + PayloadLen` ⇒ ok, payload len matches |
| trailing bytes tolerated | `len(buf) > 44 + PayloadLen` (UDP padding) ⇒ ok, payload == `buf[44:44+PayloadLen]` |
| unknown codec/fec id | codec=`9`, fec=`7` ⇒ **no error** from `Unmarshal` (structural only); registry lookup returns `false` (see 7.3) — proves the §5.3 codec-agnostic boundary |
| `FuzzUnmarshal` | `go test -fuzz` over arbitrary `[]byte` ⇒ never panics; on success, `44+PayloadLen <= len(buf)` invariant holds; subslice within bounds |

### 7.3 `packet_test.go` & `registry_test.go` — Packet + registry (unit)

| Case | Asserts |
|---|---|
| `Packet.Repair/Keyframe` | reflect `Flags` bit0/bit1 exactly for all 4 flag combos |
| `Packet.Clone` aliasing | mutating the source buffer/payload after `Clone()` does **not** change the clone (deep copy of `Payload`); under `-race` |
| `Packet.Marshal` symmetry | `Unmarshal(p.Marshal())` reproduces `p.Header` + `p.Payload` |
| codec name↔id | `CodecName(CodecPCM)=="pcm"`, `..(CodecOpus)=="opus"`; `ParseCodec` inverse; `ParseCodec("flac")` ⇒ `(_, false)` (FLAC is **not** a wire codec, R2/05 §5.4); `CodecName(9)` ⇒ `("", false)` |
| fec name↔id | full round-trip for `none/xorParity/duplicate`; `ParseFEC("reedSolomon")` ⇒ `(_, false)` (D4 excludes RS); `FECName(7)` ⇒ `("", false)` |
| totality | every defined `CodecID`/`FECID` const has a name and parses back to itself (table over the const set) |

## 8. Dependencies

**Consumes (upstream):**
- **None at runtime** — `wire` is stdlib-only (§6). It is *logically* downstream of the
  **README §6.4** byte-layout contract and **README §6.5 / 07 §6** registry contract,
  which it implements.
- **P3.2 (`internal/stream/codec`, `internal/stream/fec`) — ownership boundary, see Q1.**
  The spine (README §6.3) names `CodecID`, `FECID`, and `Packet` as the codec/FEC
  interfaces' types. To avoid a duplicate/ambiguous definition, **P4.1 declares the
  canonical `CodecID`/`FECID`/`Packet` types here** (the wire boundary) and **P3.2 imports
  `wire` for them** (`wire.CodecID`, `wire.FECID`, `wire.Packet`) rather than re-declaring.
  This is the dependency direction (codec/fec → wire), matching §6's "they import wire."

**Exposes (downstream consumers):**
- `wire.Header`, `wire.Marshal`, `wire.MarshalInto` → the **origin** (master, 05 §5.2,
  piece P4.x `internal/stream/origin`) builds and fans out packets.
- `wire.Unmarshal`, `wire.Packet`, `wire.Packet.Clone` → the **receiver** (follower,
  05 §5.6, `internal/stream/sink_net`) parses datagrams and retains packets in its reorder
  window.
- `wire.Packet`, `wire.CodecID`, `wire.FECID` → the **codec** (`internal/stream/codec`,
  P3.2) and **FEC** (`internal/stream/fec`, P3.2) packages: `Codec.ID() CodecID`,
  `FEC.ID() FECID`, `FEC.Protect/Recover` trade `Packet` (README §6.3).
- `wire.ParseCodec`/`ParseFEC`/`CodecName`/`FECName` → the **group** engine (04 §4.3.1)
  and `cmd` wiring: map negotiated `Profile{Codec, FEC string}` to the ids used when
  building the per-`streamGen` codec/FEC, and back to strings for status/metrics.
- `wire.Flags`, `FlagRepair`, `FlagKeyframe` → origin (sets keyframe on PCM always / Opus
  keyframes, 05 §5.4; sets repair on XOR parity packets, 05 §5.5.1) and receiver (routes
  repair vs source, 05 §5.6.2).

## 9. Risks / open questions

- **Q1 (BLOCKING — orchestrator must rule): who declares `CodecID`/`FECID`/`Packet`?**
  README §6.3 attaches these types to the **codec/FEC** interfaces (P3.2); README §6.5 /
  07 §6 and this piece's stated scope ("name↔id registry mapping codec/fec string enums to
  wire ints") attach the *registry* to **P4.1/wire**. A registry that maps to `CodecID`
  must reference the `CodecID` type. To keep **one** definition, this spec declares
  `CodecID`/`FECID`/`Packet` **in `wire`** and has P3.2 consume `wire.CodecID` etc. The
  alternative — declaring them in a shared parent or in `codec`/`fec` and importing *up*
  into `wire` — would invert the layering (§6) or create a third package. **Confirm P3.2's
  spec imports `wire` for these three names and does NOT re-declare them.** If P3.2 already
  fixed them in `codec`/`fec`, P4.1 must instead import P3.2 (reversing §6's import
  direction) — a contradiction to resolve before either piece is built.

- **Q2 (non-blocking): `MasterMono` signedness vs README "8 masterMono".** README §6.4
  lists `masterMono` as an 8-byte field without sign; A.2/05 §5.2.2 treat it as a
  monotonic-ns timestamp that is **diffed** (`elapsed = now_master_mono − b.masterMono`),
  which wants signed arithmetic on the *difference*. Storing it as `int64` (this spec) is
  bit-identical to `uint64` on the wire (two's complement) and makes the diff natural.
  Same for `sampleIndex` (a frame index that can be subtracted). Flagged only so a reader
  of README §6.4 is not surprised by the `int64` field types; no wire-format change.

- **Q3 (non-blocking): repair-packet `sampleIndex`/`rate100` semantics.** 05 §5.5.1 says a
  repair packet carries `seq = repair seq` and a payload that is the XOR of the group's
  source payloads; it does not pin what `sampleIndex`/`masterMono`/`codecID` a repair
  packet carries. `wire` is agnostic (it round-trips whatever the FEC layer sets), so this
  is **P3.2's** decision, not P4.1's — noted here only so the FEC piece owns it and `wire`
  imposes no constraint beyond "repair sets `FlagRepair`."

- **R (low): `Rate100` range.** `rate/100` as `uint16` caps the canonical rate at
  6,553,500 Hz — vastly above any audio rate (A.12 default 48000 ⇒ 480). No risk at
  current/foreseeable rates; documented so a future >6.5 MHz "rate" (impossible for audio)
  would be the only overflow.
