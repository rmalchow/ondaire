## 1. Purpose & scope

This piece builds the **two LIVE-status-driven operational screens** of the Ensemble UI:
the **Dashboard** (09 ¬ß3 ‚Äî the landing screen: one card per group with play state,
master, selected media, negotiated profile/transport, an offline-nodes strip, quick
**Play/Stop**, and a per-member table whose **LIVE sync error / render state** is fed by
`GET /api/v1/groups/{id}/status`, 08 G.2), and the **Media** screen (09 ¬ß7 ‚Äî a scope
switcher selecting a group (media plays on the group's **master**), a browse of the
master's `data/` mp3s, per-row **Select & play** / **Stop**, a **loop** toggle, and a
**now-playing** bar with a position/length readout from live status). Both screens are
**pure read + command** surfaces over the existing `/api/v1` HTTP contract: they compose
the P0.4 scaffold (router, API client, state machine, tokens) and the P4.9 live-status
helper, add no new client infrastructure beyond two reusable live sub-components, and
emit assets to `internal/web/dist` for the Go `web` package to embed. **No Go code is
added** ‚Äî the only backend dependency is that P3.3 populates G.2 and that F.1‚ÄìF.4 / E.1 /
C.1 / D.1 exist (they do, per 08).

## 5. Behavior & algorithms

All cited endpoints are authoritative in **08**; all screen behavior in **09**; every
tunable below is quoted verbatim from **A.12** (the single source of truth).

### 5.1 Dashboard (09 ¬ß3)
- **Static structure** (groups, members, roles, selected media, profile, transport, the
  stored `Playing` bool): composed from **`GET /api/v1/cluster/info`** (C.1) +
  **`GET /api/v1/nodes`** (D.1) + **`GET /api/v1/groups`** (E.1) via `loadDashboard()`.
  The ConfigDoc `version` (ETag) returned by these reads **seeds `If-Match`** for Play/Stop
  (08 ¬ß0.5). One card per `Groups[]` entry.
- **LIVE per-member metrics**: **`GET /api/v1/groups/{id}/status`** (G.2), **polled at
  ~1 Hz per *visible* group** via P0.4 `live.ts` (09 ¬ß0 / ¬ß3 "polled ~1 Hz"; WS is optional
  and the MVP path is polling ‚Äî see ¬ß9 risk 1). Each visible `GroupCard` owns one poll
  subscription, started `onMount` / stopped `onDestroy`; off-screen groups are not polled.
  The poll loop reuses the capped-backoff reconnect shape from P0.4 (base 500 ms ‚Üí max
  10 s) on transient failure.
- **Sync-error display** (`MemberRow` + `SyncErrorBadge`): `member.syncErrorUs` is the
  **content-domain sync error** (06; G.2 field `syncErrorUs`). Render as **signed
  milliseconds** (`+0.21 ms`) via `format.ms()`. The **‚öÝ over-threshold flag and its
  numeric threshold are owned by [06](../06-audio-output-scheduling.md)** (the drift-loop
  tolerance) ‚Äî **referenced, not invented here**; the badge takes the threshold from a
  single 06-sourced constant exported by P4.9's live module (or, if absent, a clearly
  TODO-tagged placeholder ‚Äî see ¬ß9 risk 3). The **master row shows "‚Äî"** (no per-listener
  error). A **sink-less node** (`node.caps.render === false`, D17) shows **"‚Äî"** with a
  **"‚äò no sink"** state (no channel role, no sync error); when that sink-less node is the
  group **master/origin** it carries a **"master (no local audio)"** badge on both the card
  header and its member row (09 ¬ß3). These are **content states, not faults** ‚Äî visually
  distinct from offline (normal weight + badge, vs. dimmed + offline chip).
- **Render-state dot** (`StateDot`): `‚óè rendering` (online + live data + render true),
  `‚óã idle` (group stopped), `‚äò no sink (origin)` (sink-less master), `‚åÅ offline`
  (member unreachable ‚Äî row dimmed, sync-error "‚Äî", offline chip).
- **Offline strip** (`OfflineStrip`): cluster members currently unreachable, with their
  last-known group, derived in `dashboard.ts` from `nodes` (D.1) minus the online set
  (`GET /api/v1/discovery` C.2 `members[].online`, or gossip via C.1). Shows the
  last-known ConfigDoc value dimmed with a "last known" tag (09 ¬ß0 offline state).
- **Quick Play/Stop** (per card): **`POST /api/v1/groups/{id}/play`** (F.3) /
  **`POST /api/v1/groups/{id}/stop`** (F.4), both **`If-Match`-required** (they flip
  `GroupRecord.Playing`, part of ConfigDoc). The serving node **proxies to the master**
  (08 F.3/F.4 "fan-out to master") ‚Äî the card shows the "via `<servingNode>` ‚Üí
  `<master>`" proxy hint (09 ¬ß0). Button spins until the next `/status` poll reflects the
  new play state. **`502 proxy_failed`** ‚Üí offline treatment + error **toast** (09 ¬ß3).
  Play is **disabled** when no media is selected (09 ¬ß3 "select media ‚Üí" affordance ‚Üí
  jumps to Media screen).
- **Per-card error isolation** (09 ¬ß3): if one group's `/status` fails, that card shows an
  inline error banner (ErrorBanner) while the **rest still render**. A `409
  version_conflict` on Play/Stop ‚Üí the global **Reload & reapply** affordance (re-`GET` the
  ConfigDoc reads, re-seed `version`, ask to reapply ‚Äî never silently overwrite, 08 ¬ß0.5 /
  09 ¬ß0).
- **Navigation**: click a group ‚Üí `/groups` for that group (P4.9); click a member ‚Üí
  `/nodes/{id}` (P4.9, contextual route); "select media" on a media-less group ‚Üí
  `/media?group=<id>`.

### 5.2 Media (09 ¬ß7)
- **Scope switcher**: choose a **group** (media plays on the group's **master** ‚Äî D5
  master-side decode) or a **node** scope. The browsed `data/` belongs to the **master**
  of the scoped group; the header surfaces it ("data/ on N1 (Kitchen)"). Scope persists in
  the route query (`/media?group=<id>` or `/media?node=<id>`) so Dashboard/Groups can deep-
  link into a scoped Media view.
- **Folder listing**: **`GET /api/v1/media?nodeId=<masterId>`** (F.1) ‚Äî proxied to the
  master node (its `data/` is local to its disk). Per-row `length` from
  `MediaFile.durationMs` via `mmss()`. **Refresh** re-issues F.1.
- **Select & play**: per-row ‚Üí **`POST /api/v1/groups/{id}/play`** with body
  `{file, loop}` (F.3 select-and-play in one shot), `If-Match` required; writes
  `Groups[].media` and flips `Playing=true`. **Stop** ‚Üí **`POST /api/v1/groups/{id}/stop`**
  (F.4). Selection persists in the ConfigDoc so Dashboard/Groups reflect it. Inline **loop
  toggle**: when playing it is part of the next play call; when stopped it persists via
  **`POST /api/v1/groups/{id}/media`** (F.2) `{file, loop}` without starting playback.
  Default **loop is on** (D14 "play looped").
- **Now-playing bar**: current file + loop state + target group + **position/length**.
  `length` from the selected `MediaFile.durationMs`. **`position` is derived** from
  `GroupStatus` ‚Äî there is **no `positionSec` field in 08 G.2** (it exposes `streamGen`,
  `playing`, per-member `syncErrorUs`/`offsetUs`/`driftRatio`/`underruns`, not a play
  cursor). The UI therefore derives position from the master member's progress where
  obtainable and otherwise shows file + loop without a live cursor (see ¬ß9 risk 2). All
  media commands **proxy to the master** (08 F-table).
- **States** (09 ¬ß7): loading (list skeleton, commands disabled); empty (`data/` has no
  mp3s ‚Üí "No media in data/ on `<master>`"); error (scan/playback failure ‚Üí envelope
  `code`/`message` + Retry); offline (scoped master offline ‚Üí show last-selected media from
  ConfigDoc dimmed, browse/play disabled + offline banner); uninitialized (router-gated).

### 5.3 A.12 tunables surfaced (quoted verbatim, never redefined)
The Media/Dashboard UI **displays** but does not redefine these:
- canonical **rate `48000`** / **channels `2`** / **FramesPerChunk `480` (10 ms)** ‚Äî shown
  in the profile sub-line as "48k" (09 ¬ß3 wireframe `48k`).
- the negotiated **codec** (`"pcm"|"opus"`), **FEC** (`"none"|"xorParity"|"duplicate"`),
  and **transport** (`"udp"|"tcp"`) string enums (08 ¬ß0.7 / D3/D4) ‚Äî read-only here.
- The sync-error **‚öÝ threshold and units are owned by 06** (the drift PI loop tolerance:
  A.12 lists `MaxPPM 200 ppm`, `HardErrSamp 2400 samples (50 ms)`, `tick 20 ms` for that
  loop) ‚Äî the badge references 06's display threshold, it does **not** invent one.
The **~1 Hz poll cadence** for `/status` is a 09 ¬ß0 / ¬ß3 UI convention (not an A.12
number); it is implemented in P0.4 `live.ts`.

## 6. Import constraints

This piece is **entirely the `web/` frontend tree** (TypeScript + Svelte) ‚Äî **no Go
package-layering rule applies**, and it creates/modifies **no `internal/web` Go file**.
The frontend talks to the node **exclusively over HTTP `/api/v1`** (08), through P0.4's
`apiFetch` ‚Äî never any other surface. Per README ¬ß5/¬ß6.6 + doc 01 ¬ß2, the `internal/web`
server (a separate piece) reaches the engine only via the `web.Deps` function-value seam
and **must not import `group`, `stream/*`, or `audio/*`** (A.14.3) ‚Äî P4.10 introduces no
engine imports and so trivially honors this; it only consumes the embedded asset FS that
P0.4 produces. **TS import rule for this piece:** screens/components import only from
`web/src/lib/*` (P0.4 scaffold + this piece's `media.ts`/`dashboard.ts`) and
`web/src/components/{state,ui,live}/*`; they must **not** duplicate `apiFetch`/envelope/
`If-Match` logic (all mutating calls route through `apiFetch`, per P0.4 ¬ß4).

## 9. Risks / open questions

1. **Live feed transport (carried over from P0.4 ¬ß9.1).** 09 ¬ß0/¬ß3 and 08 G.2 say live
   status is "polled ~1 Hz, **or SSE/WebSocket if 08 defines one**" ‚Äî but **08 defines no
   streaming endpoint** (only `GET ‚Ä¶/groups/{id}/status`). P4.10 implements **~1 Hz
   polling** as the primary path (per-visible-group). *Orchestrator: confirm the MVP
   backend (P3.3) will NOT expose a `/ws` group-status stream; if it will, its path + frame
   shape must be specified so `live.ts` (P0.4) and the vite `/ws` proxy target it.*
   **Non-blocking for build** (polling works), **blocking for the final transport choice.**
2. **Now-playing position has no source field (BLOCKING for the now-playing cursor).**
   09 ¬ß7 wireframe shows `‚ñ∂ 00:48 / 03:12`, but **08 G.2 exposes no play-position /
   `sampleIndex` / cursor** (only `streamGen`, `playing`, and per-member
   sync/offset/drift/underruns). The group timeline sample index (A.2 `NowSample`) *is* the
   natural source. *Orchestrator: either (a) add a `positionSamples`/`sampleIndex` field to
   G.2 (preferred ‚Äî P3.3 owns it), or (b) accept that the now-playing bar shows file + loop
   + length only, with no live cursor in the MVP.* P4.10 ships **(b) by default** and lights
   up the cursor automatically if the field appears.
3. **Sync-error ‚öÝ threshold value.** 09 ¬ß3/¬ß6 defer the threshold + units to
   [06](../06-audio-output-scheduling.md) (the drift-loop tolerance); A.12 pins the loop's
   `HardErrSamp 2400 samples (50 ms)` and `MaxPPM 200 ppm` but not a *UI warning* threshold.
   `SyncErrorBadge` must import a single 06-sourced constant (owned by P4.9 / 06), not
   invent one. *Orchestrator: confirm 06 (or P4.9) exports the display threshold (a sub-ms
   value, given the sample images `+1.8 ms ‚öÝ` vs `+0.21 ms` OK); until then the badge uses a
   clearly TODO-tagged placeholder.*
4. **`SyncErrorBadge`/`StateDot` ownership.** Both are needed by P4.9 (Node detail / Groups)
   **and** P4.10 (Dashboard). To avoid a duplicate, the orchestrator should assign these two
   `components/live/*` primitives to **whichever of P4.9/P4.10 lands first**; the other
   imports them. P4.10's spec assumes P4.9 lands first and imports, but declares them so it
   can build standalone.
5. **Online-set source.** The offline strip needs node liveness. 08 exposes it via C.2
   `discovery.members[].online` and via gossip behind C.1 counts, but there is **no single
   `GET /api/v1/nodes` liveness flag** (D.1 returns static `NodeRecord`s). P4.10 uses C.2
   `members[].online`; *confirm C.2 is the intended liveness source for the Dashboard
   offline strip* (it is the only endpoint that returns a per-node `online` bool).
