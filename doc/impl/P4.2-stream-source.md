## 1. Purpose & scope

`internal/stream/source` is the **master's input decode stage**: it turns a media input ‚Äî
an mp3, FLAC, or WAV/PCM file in the node's `data/` folder, **or** an HTTP(S) stream URL
(D14) ‚Äî into a single continuous, interleaved `float32` PCM stream at the group's
**canonical rate** (48000 Hz, 2 channels; A.12), and **loops seamlessly at EOF** so the
group `Timeline` is a single unbroken sample counter (05 ¬ß5.2.1). It exposes the canonical
`Reader` interface (`Read`/`Rate`/`Channels`), the `Open(path, canonicalRate, channels)`
constructor, and `List(dataDir)` for the Media UI (05 ¬ß5.3, 09). It is **wire-codec
agnostic** ‚Äî it knows nothing about PCM/Opus framing, `streamGen`, FEC, or the wire header;
it only produces rate-uniform PCM that `internal/stream/origin` (P4.3) chunks and encodes.
Source formats (mp3/FLAC/WAV) live **entirely** here and never reach the wire (05 ¬ß5.3,
¬ß5.4; the FLAC source decoder is unrelated to the `{PCM, Opus}` wire codec set, R2).

## 4. Public API (canonical names ‚Äî README ¬ß6 / 05 ¬ß5.3)

```go
package source

// Reader is the master's decoded-PCM input stream. Read fills dst with up to
// len(dst) interleaved float32 frames at the canonical rate; it LOOPS at EOF and
// NEVER returns io.EOF while looping (05 ¬ß5.3). n is always a multiple of Channels().
type Reader interface {
    Read(dst []float32) (frames int, err error) // "frames" = interleaved float32 samples count
    Rate() int                                  // canonical rate, e.g. 48000
    Channels() int                              // 2
    Close() error
}

// Open decodes a source (mp3/FLAC/WAV) from the data/ folder ‚Äî or an HTTP(S) stream
// URL ‚Äî and returns a looping Reader emitting interleaved float32 at canonicalRate /
// channels (05 ¬ß5.3). path is either a data/-relative filename or an http(s):// URL.
func Open(path string, canonicalRate, channels int) (Reader, error)

// List enumerates playable media in the data/ folder (drives the Media UI, 09 / 08).
func List(dataDir string) ([]MediaInfo, error)

// MediaInfo describes one playable item in data/ (fields drive the Media screen, 09).
type MediaInfo struct {
    Name       string // base filename (the value stored in ConfigDoc.Groups[].Media.File, 07)
    Format     string // "mp3" | "flac" | "wav"
    SizeBytes  int64
    SampleRate int    // native rate as declared by the file header (0 if unknown w/o full decode)
    Channels   int    // native channel count (0 if unknown)
}
```

Internal (unexported) seam used by the dispatch/loop layers:

```go
// frameSource is a finite, native-rate decoder; loopReader/resample wrap it.
type frameSource interface {
    read(dst []float32) (n int, err error) // native rate/channels; io.EOF at true end
    rate() int
    channels() int
    seekStart() error                      // reposition to frame 0 for looping (loop.go)
    close() error
}
```

`Reader` adds `Close()` to the README/05 ¬ß5.3 listing (the doc's snippet omits it but the
FLAC source already has `Close`, and `Open` allocates a file handle / HTTP body that must
be released) ‚Äî this is an additive, non-conflicting clarification, not a renamed contract.

## 5. Behavior & algorithms (cited)

- **Canonical rate / channels / chunk (A.12, 05 ¬ß5.1).** The pipeline downstream is
  rate-uniform; `Open` always emits **`48000` Hz / `2` ch** interleaved `float32` (the
  caller passes these explicitly; A.12 row *Audio canonical ‚Üí 48000 / 2 / 480 (10 ms)*).
  `FramesPerChunk` (**480**) is **not** this package's concern ‚Äî chunking happens in
  `origin` (P4.3); the source just yields a continuous stream. Quote A.12 verbatim:
  *"rate / channels / FramesPerChunk = 48000 / 2 / 480 (10 ms)."*
- **Decode ‚Üí PCM (05 ¬ß5.3).** mp3 (`go-mp3`) and WAV decode to S16LE/native then widen to
  `float32` in `[-1, 1)`; FLAC uses the `scale = 1/2^(bps-1)` per-subframe conversion
  copied from `../media`. All emit interleaved `float32`.
- **Content resample (05 ¬ß5.3).** Sources are *typically 44.1 kHz*; resample native‚Üícanonical
  here. This is **content resampling at the source**, explicitly *distinct from the per-node
  drift resampler in 06* (A.3, ¬±200 ppm). If native == canonical the resampler is bypassed.
- **Seamless loop (D14, 05 ¬ß5.2.1, ¬ß5.3).** At EOF `loopReader` calls `seekStart()` and
  continues; **looping does not reset `sampleIndex` and does not bump `streamGen`** (that is
  enforced in `origin`, but the source must make looping invisible by never emitting EOF and
  never emitting a short/partial slice at the boundary). Leftover frames straddling the
  boundary are carried so *"every emitted chunk is exactly `FramesPerChunk` (no short frames
  on the wire)"* (05 ¬ß5.3) ‚Äî the source guarantees its `Read` returns whole frames and an
  uninterrupted stream so the chunker upstream never sees a runt.
- **`data/` browse (05 ¬ß5.3, 09).** `List` walks `dataDir` (non-recursive), filters by
  extension/magic to `{mp3, flac, wav}`, returns `MediaInfo` sorted by `Name`. The selected
  `Name` + loop flag persist in `ConfigDoc.Groups[].Media{File, Loop}` (07).
- **HTTP(S) input (D14).** A `path` that is an `http(s)://` URL streams via `net/http`;
  looping a non-seekable HTTP body re-opens the request (¬ß9 open question on live/infinite
  streams).
- **Performance (hardware budget, README ¬ß8).** One decode + one resample per master, on the
  reference quad-A53 ‚Äî *"audio is orders of magnitude lighter than a video pipeline."* Reuse
  the residual-buffer pattern (no per-`Read` allocation), preallocate the resampler scratch,
  and keep the conversion a single tight pass; do not allocate in the steady-state `Read`.

## 6. Import constraints (01 ¬ß2.1 / ¬ß2.2)

Per the ¬ß2.1 package table, **`stream/source` may import internal package `config` only**
(for `Paths.Data` / data-dir resolution; though `List`/`Open` take an explicit `dataDir`/
`path` string, so even `config` may not be strictly required). It may **not** import
`stream/codec`, `stream/fec`, `stream/wire`, `stream/origin`, `group`, `audio/*`, `clock`,
`state`, `cluster`, `allowlist`, `web`, `pki`, or `auth`. It freely imports the stdlib and
the vetted decode deps `github.com/hajimehoshi/go-mp3` and `github.com/mewkiz/flac` (01
¬ß2.1 explicitly whitelists "source-decode libs `go-mp3`/`mewkiz/flac`"). It is a **leaf**
of `stream/*`: `origin` (P4.3) imports *it*, never the reverse ‚Äî this keeps the graph
acyclic and the decoder independently testable with no engine wiring.

## 9. Risks / open questions

1. **HTTP(S) looping of non-seekable / live streams (D14).** `Open` accepts an HTTP(S) URL,
   but D14's looping semantics (05 ¬ß5.2.1) assume a finite, seekable source (seek to frame
   0). For a finite HTTP object we can re-issue the GET to "loop"; for a **live/infinite**
   stream (e.g. internet radio) looping is meaningless and EOF may never come. **Resolve:**
   does P4.2 must support live streams, or only finite downloadable files? Proposed default:
   treat HTTP input as finite (loop by re-request); document that live streams are out of
   MVP scope. **Orchestrator should confirm.**
2. **`go-mp3` is fixed-stereo, fixed-S16.** It always outputs 2-channel S16LE. That matches
   the canonical `channels=2`, but a mono mp3 is upmixed by the library ‚Äî acceptable for
   MVP. WAV/FLAC mono sources need an explicit **mono‚Üístereo duplication** step before the
   resampler to honor `channels=2`; that channel-count adaptation lives here (not in the
   per-node channel-role logic of 06, which is about L/R *selection* at playout).
3. **Resampler quality vs. CPU (README ¬ß8).** A linear resampler is cheapest but audibly
   poorer on 44100‚Üí48000; a 4-tap cubic (Catmull-Rom, as in A.3) is a good middle ground and
   shares math with the drift resampler ‚Äî but A.3's resampler is range-limited (near-unity).
   This package needs an arbitrary-ratio variant. **Resolve:** is a 4-tap cubic acceptable
   quality for source resampling, or is a longer polyphase FIR wanted? Proposed: 4-tap cubic
   for MVP (cheap on A53), revisit if artifacts surface in the P4 analyzer pass.
4. **`Reader.Close()` not in the 05 ¬ß5.3 snippet.** Added here (¬ß4) to release file/HTTP
   handles; flagged as an additive clarification, not a contract change. If a strict reading
   of ¬ß5.3 is required, the handle could be closed via a separate mechanism ‚Äî but `Close` on
   the interface is the clean Go idiom and matches the existing FLAC `Source.Close`.
