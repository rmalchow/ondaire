## 1. Purpose & scope

This piece delivers the two **wire-layer transform interfaces** that the streaming
pipeline plugs codecs and loss-recovery schemes into, plus their mandatory-baseline
implementations: the `Codec` interface (README §6.3) with the **PCM S16LE** codec
(doc 05 §5.4.1 — `Encode` narrows interleaved `float32` to little-endian `int16` at the
wire boundary, `Decode` widens back; no inter-frame state, every chunk an implicit
keyframe), and the `FEC` interface (README §6.3) with **`None` = identity** (doc 05
§5.5 — `Protect` returns `[pkt]`, `Recover` returns the source packet as-is). It also
defines the canonical `CodecID`/`FECID` integer enums and a **name↔id registry** so the
JSON/ConfigDoc string enums (`"pcm"`, `"none"`, README §6.5) map to/from the integer IDs
that live only at the §6.4 wire layer. Opus (`CodecID=1`), XOR-parity (`FECID=1`) and
Duplicate (`FECID=2`) are **out of scope** for P4.3 — only their enum constants and
registry name slots are reserved here; their implementations land in P4.4/P4.5. These
packages are pure **leaf** packages (no internal imports except `stream/wire` for the
`Packet` type the `FEC` interface signature requires), zero allocation on the hot path
where practical, and contain no goroutines, no I/O, and no clock/state coupling.

## 4. Public API (canonical names — README §6.3 / §6.5, doc 05 §5.4 / §5.5)

### 4.1 `internal/stream/codec`

```go
// Package codec is the wire-codec layer (README §6.3, doc 05 §5.4). A Codec turns one
// chunk of canonical-rate interleaved float32 PCM into a wire payload and back. The
// internal pipeline is float32; PCM narrows to S16LE only at this boundary (A.10 m5).
package codec

// CodecID is the integer codec identifier carried at the §6.4 wire layer (header off 6).
// JSON/ConfigDoc use the string names (README §6.5); these integers exist ONLY on the wire.
type CodecID uint8

const (
    PCM  CodecID = 0 // S16LE, mandatory baseline (doc 05 §5.4.1)
    OPUS CodecID = 1 // lossy, optional/capability-gated (doc 05 §5.4.2) — P4.4, not here
)

// Codec is the spine interface (README §6.3), implemented verbatim.
type Codec interface {
    ID() CodecID                              // PCM=0, OPUS=1
    Encode(pcm []float32) ([]byte, error)     // master side; one chunk in, one frame out
    Decode(payload []byte) ([]float32, error) // follower side; one frame in, one chunk out
}

// New returns the codec for id, or an error for an unknown/unimplemented id.
// In P4.3 only PCM is constructible; OPUS returns ErrUnsupportedCodec.
func New(id CodecID) (Codec, error)

// NewPCM returns the S16LE baseline codec configured for `channels` interleaved
// channels (always 2 for this build, doc 05 §5.1) so Decode can size its output and
// Encode/Decode can validate frame alignment.
func NewPCM(channels int) Codec

// --- name↔id registry (README §6.5: string enums in JSON, integer IDs on the wire) ---

// NameOf maps a CodecID to its canonical JSON/ConfigDoc string ("pcm"|"opus").
func NameOf(id CodecID) (string, bool)

// FromName maps a canonical string ("pcm"|"opus") to its CodecID.
func FromName(name string) (CodecID, bool)

// Errors (sentinel, wrapped with %w):
//   ErrUnsupportedCodec — id/name not implemented in this build (e.g. opus in P4.3)
//   ErrShortPayload     — Decode got a payload that is not a whole number of S16LE frames
//   ErrChunkAlloc       — Encode got a pcm slice not a whole number of frames (len%channels!=0)
var (
    ErrUnsupportedCodec = errors.New("codec: unsupported codec id")
    ErrShortPayload     = errors.New("codec: payload not a whole number of S16LE frames")
    ErrChunkAlloc       = errors.New("codec: pcm length not a multiple of channel count")
)
```

### 4.2 `internal/stream/fec`

```go
// Package fec is the wire forward-error-correction layer (README §6.3, doc 05 §5.5).
// A FEC protects source packets on the master side and recovers them on the follower
// side. P4.3 ships only the None (identity) scheme; XORParity/Duplicate are P4.5.
package fec

import "gitlab.rand0m.me/ruben/go/ensemble/internal/stream/wire" // for wire.Packet

// FECID is the integer FEC identifier carried at the §6.4 wire layer (header off 7).
type FECID uint8

const (
    None      FECID = 0 // identity (doc 05 §5.5); also forced over TCP fallback (§5.9)
    XORParity FECID = 1 // XOR of k payloads (doc 05 §5.5.1) — P4.5, not here
    Duplicate FECID = 2 // each packet sent twice (doc 05 §5.5.4) — P4.5, not here
)

// FEC is the spine interface (README §6.3), implemented verbatim.
type FEC interface {
    ID() FECID
    // Protect groups source packets and emits source+repair packets to send.
    // None returns exactly [pkt].
    Protect(seq uint64, pkt []byte) (out [][]byte)
    // Recover ingests a received packet; returns any newly-recoverable source packets.
    // None returns exactly [p].
    Recover(p wire.Packet) (recovered []wire.Packet)
}

// New returns the FEC for id, or ErrUnsupportedFEC for an unimplemented one.
// In P4.3 only None is constructible.
func New(id FECID) (FEC, error)

// NewNone returns the identity FEC.
func NewNone() FEC

// --- name↔id registry (README §6.5: "none"|"xorParity"|"duplicate" in JSON) ---
func NameOf(id FECID) (string, bool)
func FromName(name string) (FECID, bool)

var ErrUnsupportedFEC = errors.New("fec: unsupported fec id")
```

> **`wire.Packet` (consumed, from P4.1).** Per doc 05 §5.6.2 the receiver builds
> `p := Packet{hdr, payload}` and per README §6.3 `Recover(p Packet)`. P4.3 imports the
> `wire.Packet` type as defined by P4.1. The expected shape (assert against P4.1's spec):
> `type Packet struct { Header Header; Payload []byte }` carrying at least `Seq uint64`,
> `SampleIndex int64`, `StreamGen uint64`, `Flags` (repair/keyframe bits), `PayloadLen`.
> The `None` scheme touches none of these fields — it passes the value through — so P4.3
> needs only that the type *exists and is copyable*; see §9 Q1.

## 5. Behavior & algorithms

### 5.1 PCM codec (doc 05 §5.4.1, A.10 "Wire PCM format (m5)")

- **Framing is fixed S16LE** — *"PCM on the wire is `S16LE` (16-bit signed little-endian),
  fixed … `Encode` writes the 480 stereo frames as little-endian interleaved `int16`
  samples. There is no profile field for bit depth and no f32-vs-int16 selection."*
  (doc 05 §5.4.1). The internal pipeline stays `float32`; **the f32↔s16 conversion happens
  at the wire boundary** — `Encode` narrows f32→s16, `Decode` widens s16→f32 (A.10 m5).
- **`Encode(pcm []float32) ([]byte, error)`**: `len(pcm)` must be a whole number of frames
  (`len % channels == 0`, else `ErrChunkAlloc`). Output is `2 * len(pcm)` bytes, written
  by the copied `f32ToS16LE` helper (clamp ±1, round-to-nearest, scale 32767, LE). At the
  canonical profile (480 frames × 2 ch, A.12) the payload is **1920 bytes** — matching the
  worked diagram (doc 05 §5.10: `payloadLen = 1920 bytes (480 × 2ch × 2B S16LE)`).
  *P4.3 does not enforce `FramesPerChunk` itself* — the origin chunker (P4.6) feeds exactly
  `FramesPerChunk` frames; the codec is chunk-size-agnostic and only requires frame
  alignment, keeping it reusable for the join/keyframe path.
- **`Decode(payload []byte) ([]float32, error)`**: `len(payload)` must be even and a whole
  number of S16LE frames (`len % (2*channels) == 0`, else `ErrShortPayload`). Widen each LE
  `int16` to `float32` via `s16LEToF32` (scale `1/32768`).
- **No inter-frame state, every chunk a keyframe.** *"Every chunk is self-contained: every
  packet is implicitly a keyframe (the keyframe flag is set on every PCM packet so the join
  path §5.6.4 is uniform)."* (doc 05 §5.4.1). The codec carries **no state** between calls
  — `pcmCodec` is immutable after construction and therefore **safe for concurrent
  `Encode`/`Decode`** (the origin encodes once, doc 05 §5.2.3; setting the keyframe header
  flag is the **wire layer's** job, not the codec's — the codec only produces the payload).
- **CPU budget (README §8).** PCM is *"~none"* encode/decode cost — a clamp + multiply +
  two byte writes per sample. No allocation beyond the one output slice per call; tests
  assert ≤1 alloc/op via `testing.AllocsPerRun`. Suitable for the Cortex-A53 floor.

### 5.2 None FEC (doc 05 §5.5, §5.9)

- `ID()` returns `None` (0).
- `Protect(seq, pkt)` returns `[][]byte{pkt}` — the identity (doc 05 §5.5:
  *"`FECID = None` (0) is the identity: `Protect` returns `[pkt]`"*). It must **not** copy
  `pkt` (zero-cost passthrough) and must **not** retain it; ownership stays with the caller.
- `Recover(p)` returns `[]wire.Packet{p}` — *"`Recover` returns the source packet as-is."*
- **TCP fallback (doc 05 §5.9).** Over TCP, FEC is forced to `None` (*"FEC is disabled
  (`FECID = None`) over TCP … `Protect` is the identity in this mode."*). The same `None`
  implementation serves that path — no separate code.
- No state, no goroutines, no allocation beyond the single-element slice header per call.

### 5.3 Enums & the name↔id registry (README §6.5)

- README §6.5 is explicit: *"JSON (ConfigDoc + API) uses string enums for codec/fec/transport
  (`"pcm"|"opus"`, `"none"|"xorParity"|"duplicate"` …). Integer `CodecID`/`FECID` exist ONLY
  at the §6.4 wire layer, mapped to/from these names via a name↔id registry."* P4.3 owns
  that registry for codec and fec.
- Canonical name tables (verbatim from README §6.5 / doc 05 §6.3):
  - codec: `PCM=0 ↔ "pcm"`, `OPUS=1 ↔ "opus"`.
  - fec: `None=0 ↔ "none"`, `XORParity=1 ↔ "xorParity"`, `Duplicate=2 ↔ "duplicate"`
    (note the camelCase `"xorParity"` — quote it exactly).
- The registry must list **all** enum names (so `FromName("opus")`/`FromName("xorParity")`
  succeed at the *naming* layer used by profile negotiation in P3/P4.2), while the
  **constructors** `New(id)` return `ErrUnsupportedCodec`/`ErrUnsupportedFEC` for the
  not-yet-implemented ones. This separation lets group profile negotiation (doc 04) and
  `state` (P2.1) round-trip string↔id for *any* advertised capability without P4.3 having
  to implement the codec body. Implement as a small fixed slice/array indexed by id (no
  `map`, no `sync` — these tables are immutable; cheapest on the Pi).

### 5.4 Tunables quoted from A.12 (single source of truth)

P4.3 hard-codes **no** tunables; it consumes them via the chunk size the caller passes and
the channel count at construction. The relevant A.12 rows the caller honors:

- *"Audio canonical | rate / channels / FramesPerChunk | 48000 / 2 / 480 (10 ms)"* — so the
  steady-state PCM payload is `480 × 2 × 2 = 1920 B` (doc 05 §5.10 / §5.11).
- *"FEC XOR | k / interleave depth D | 8 / 4"* and *"FEC dup | offset `Ddup` | 5 packets"* —
  **not used by `None`**; reserved for P4.5. Quoted here only to fix that P4.3 introduces
  no FEC parameters of its own.

## 6. Import constraints (doc 01 §2.1 / §2.2)

Per the doc 01 §2.1 dependency column:

- **`stream/codec` → imports: *(none)* internal.** Stdlib only (`encoding/binary` or manual
  LE byte ops, `errors`, `math`). It is a strict leaf. Must **not** import `wire`,
  `source`, `state`, `clock`, `group`, or `audio/*`.
- **`stream/fec` → imports: `stream/wire` only** (for the `wire.Packet` type in the §6.3
  signature; doc 01 lists fec deps as *(none)* but the canonical interface signature itself
  references `Packet`, which is owned by `wire` — this is the one unavoidable leaf-to-leaf
  edge, see §9 Q1). Must **not** import `codec`, `source`, `origin`, `sink_net`, `state`,
  `clock`, `group`, or `audio/*`.
- Both are imported **by** `stream/origin` and `stream/sink_net` (doc 01 §2.1 rows) and
  referenced **by name** (string) from `group`/`state`/`web` — never the reverse.
- `internal/web` must not import either (doc 01 §2.2 rule 1: web reaches the engine only
  through the `Deps` seam) — satisfied automatically since web never imports `stream/*`.

## 7. Test plan

All tests are **table-driven**, pure, fast (no sockets, no sleeps), and run under
`CGO_ENABLED=0`. Acceptance ties to **A.13 P5** (*"wire PCM(S16LE)+optional Opus + XOR/dup
+ profile negotiation"*) — P4.3 is the PCM+None foundation of that row — and to doc 05
§5.4.1 / §5.5 acceptance behavior.

### 7.1 `codec/convert_test.go` (the copied helpers)

| Case | Asserts |
|---|---|
| Endianness | `f32ToS16LE([1 sample])` writes `[lo, hi]` little-endian; `0.5 → int16(16384) → bytes {0x00,0x40}`. |
| Clamp high/low | `+1.5 → +32767`, `-1.5 → -32768` (no rail wrap to opposite sign — the bug the 32767 scale prevents). |
| Round-to-nearest | a value between two codes rounds to nearest, not truncates (asserts the ±0.5 bias split). |
| Full-scale | `+1.0 → +32767` (never `+32768`); `-1.0 → -32768`. |
| Inverse scale | `s16LEToF32(-32768) == -1.0`; `s16LEToF32(0) == 0.0`; `s16LEToF32(32767) ≈ 0.99997`. |
| Round-trip | for a grid of f32 values in [-1,1], `s16LEToF32(f32ToS16LE(v))` is within `±1/32768` (1 LSB) of `v`. |
| Alloc | `f32ToS16LE`/`s16LEToF32` into caller-provided buffers do 0 allocs (`AllocsPerRun`). |

### 7.2 `codec/codec_test.go`

| Case | Asserts |
|---|---|
| `ID()` | `NewPCM(2).ID() == PCM` (== 0). |
| Registry round-trip | `FromName("pcm")==PCM,true`; `NameOf(PCM)=="pcm",true`; `FromName("opus")==OPUS,true` (name layer knows opus); `NameOf(OPUS)=="opus",true`; unknown name → `false`; unknown id → `false`. |
| `New` gating | `New(PCM)` ok; `New(OPUS)` returns `ErrUnsupportedCodec` (P4.3 scope); `New(99)` → `ErrUnsupportedCodec`. |
| Encode size | 480-frame stereo input → 1920-byte payload (doc 05 §5.10). |
| Encode alignment error | odd-length / non-frame-multiple input → `ErrChunkAlloc`. |
| Decode size | 1920-byte payload → 960-float32 (480 stereo frames). |
| Decode alignment error | payload length not a multiple of `2*channels` → `ErrShortPayload`; empty payload → 0 frames, no error. |
| Round-trip fidelity | random 480-frame f32 in [-1,1] → `Decode(Encode(x))` within ±1 LSB per sample. |
| Statelessness / concurrency | two interleaved `Encode` calls on the same `pcmCodec` produce identical output to two separate codecs; `go test -race` over concurrent `Encode`+`Decode`. |
| Alloc budget | `Encode` ≤ 1 alloc/op, `Decode` ≤ 1 alloc/op (`AllocsPerRun`). |

| Case | Asserts |

## 9. Risks / open questions

- **Q1 (blocking-ish) — exact shape of `wire.Packet` (P4.1).** `FEC.Recover` takes a
  `wire.Packet` by value (README §6.3, doc 05 §5.6.2 `Packet{hdr, payload}`). P4.3 needs
  P4.1 to export a value-copyable `wire.Packet` with (at minimum) a `Header` and a
  `Payload []byte`. The `None` scheme uses no fields, so any reasonable shape works, **but
  the import edge `fec → wire` must be agreed**: doc 01 §2.1 lists `fec` deps as *(none)*
  yet the canonical §6.3 signature references `Packet`. Resolution options the orchestrator
  must pick: (a) accept the single `fec → wire` leaf edge (recommended — matches §6.3
  verbatim, keeps the interface canonical); or (b) have P4.1 define `Packet` in a tiny
  shared leaf that both `wire` and `fec` import. **Recommend (a).** Until P4.1's spec
  pins `Packet`, `fec.go` cannot compile against a concrete type.
- **Q2 — channel count source of truth.** `NewPCM(channels)` takes channels explicitly;
  the canonical build is always 2 (doc 05 §5.1 *"master always sources stereo"*, A.12). The
  codec stays generic so the dumb-node/mono future (doc 10) needs no rewrite. Confirm the
  origin/sink_net pieces construct it with the group profile's channel count rather than a
  hard-coded 2.
- **Q3 — keyframe flag ownership.** doc 05 §5.4.1 says *"the keyframe flag is set on every
  PCM packet."* P4.3 deliberately leaves **header flag setting to the wire/origin layer**
  (the codec produces only the payload bytes, statelessly). Confirm P4.6 origin sets
  `flags` bit1 on every PCM chunk (it must, per doc 05 §5.6.4 join path) — P4.3 exposes no
  hook for it, which is correct but worth flagging so the bit is not dropped.
- **Q4 — encode vs. decode scale asymmetry (32767 vs 1/32768).** Intentional and standard
  (encoder avoids the +32768 overflow rail; decoder uses the full negative rail so -1.0
  maps exactly). Documented in `convert.go`; the round-trip test bounds error to 1 LSB.
  No action needed, noted to pre-empt a "bug" report in review.
