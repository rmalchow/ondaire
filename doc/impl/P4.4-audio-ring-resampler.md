## 1. Purpose & scope

P4.4 lands the two **pure, dependency-free, hot-path** primitives the follower render
loop (doc 06) is built on: the **SPSC interleaved-`float32` jitter `Ring`** (A.7) — the
playout buffer sized to `LeadMs` of audio that absorbs producer-scheduling jitter ahead
of the sink — and the **near-unity 4-tap cubic/Hermite `Resampler`** (A.3) — the drift
*actuator* that carries fractional phase + 3-frame tap history across `Process` calls so
block boundaries are seamless, with `SetRatio` clamped to `1 ± MaxPPM·1e-6`
(`MaxPPM = 200`, ±0.02 %, inaudible). Both are copied essentially verbatim from the
proven `mpvsync` (`../media/internal/audio/{ring,resampler}.go`), re-homed from the
single media `package audio` into the two split leaf packages `internal/audio/ring` and
`internal/audio/resampler` per README §5. P4.4 ships **no** rate-conversion, drift PI
logic, channel/gain, or sink I/O — those live in `audio/drift`, `audio/render`,
`audio/sink` (separate pieces). It is two files + two test files; no third-party deps.

## 4. Public API

Exact signatures — these are the canonical names doc 06 §0/§2.2/§3.3 and the `audio/drift`
+ `audio/render` pieces consume; do not rename.

### `internal/audio/ring` (A.7, doc 06 §0 "playout Ring")
```go
package ring

// Ring is a single-producer/single-consumer interleaved-float32 jitter buffer.
type Ring struct { /* mu sync.Mutex; buf []float32; head, tail, n int — unexported */ }

func NewRing(capSamples int) *Ring   // capSamples<0 => empty ring (Write always 0)
func (r *Ring) Write(p []float32) (n int)  // copies up to free space; short/0 when full
func (r *Ring) Read(p []float32) (n int)   // copies up to available; short/0 when empty
func (r *Ring) Len() int             // samples currently buffered
func (r *Ring) Cap() int             // capacity in samples (== len(backing))
func (r *Ring) Reset()               // drop all buffered samples (used on hard reseek)
```
Units are **interleaved samples**, not frames. Callers convert:
`capSamples = LeadMs/1000 · Rate · Channels` (A.7; renderer uses `Ring.Cap()=2×LeadMs`
of slack per doc 06 §2.1), and `ringFrames = Ring.Len()/Channels` (doc 06 §3.3).

### `internal/audio/resampler` (A.3, doc 06 §2.2 / §3 actuator)
```go
package resampler

const MaxPPM = 200 // ±0.02% ratio clamp (A.12 Drift PI MaxPPM); exported, see note below

// Resampler is a near-unity variable-ratio 4-tap cubic-Hermite (Catmull-Rom) interpolator.
type Resampler struct { /* channels int; step float64; hist []float32; nhist int; phase float64 */ }

func NewResampler(channels int) *Resampler  // channels<1 => clamped to 1; ratio starts 1.0
func (r *Resampler) SetRatio(ratio float64) // clamps to [1-MaxPPM*1e-6, 1+MaxPPM*1e-6]
func (r *Resampler) Reset()                 // clears phase + tap history (hard reseek)
// Process appends interleaved output to dst, returns extended slice + INPUT samples
// consumed (always a multiple of channels). Carries phase+history across calls.
func (r *Resampler) Process(dst, in []float32) (out []float32, consumed int)
```
`consumed` is **input samples**; the renderer derives source frames as `consumed/channels`
to advance `sourceConsumed` (doc 06 §3.2/§3.3, §7). `SetRatio`'s independent re-clamp is
the "belt-and-braces" guard doc 06 §3.4 relies on so a mistuned drift gain can never
produce an audible pitch shift.

**`MaxPPM` ownership note.** `DriftParams.MaxPPM` (doc 06 §3.4, A.12 = 200) and this
resampler clamp are the **same number** with two consumers. Keep the literal `const
MaxPPM = 200` **in this package** (matches media; the resampler must self-clamp even if
called directly). The `audio/drift` piece defines its own `DriftParams.MaxPPM` field
(defaulting to `200`) rather than importing this — `drift` must not depend on
`resampler`. Both trace to **A.12 "Drift PI … MaxPPM = 200 ppm"** (single source of
truth). Flag R3 if the orchestrator prefers a shared `audio/internal` const.

## 5. Behavior & algorithms

**Ring — A.7 + doc 06 §0/§2.2/§2.3.** Fixed `[]float32` backing, `head`/`tail`/`n`
guarded by one `sync.Mutex` (A.7: "mutex … sufficient because there is no live hardware
in the hot path"). `Write` copies up to free space and **short-counts (down to 0) when
full** → the producer backs off ~2 ms and retries (doc 06 §2.2). `Read` copies up to
available and **short-counts when empty** → on underrun the consumer does **not** pad
silence (doc 06 §2.3/§6.4: padding would advance `framesWritten` and fight the reseek);
it bumps `underruns` and lets the next tick observe the gross error. `Reset` zeroes the
indices for the §6.1 hard reseek. Wrap-around is handled by the two-segment copy loop.
SPSC only — exactly one producer goroutine (the resampler/channel/gain stage) and one
consumer goroutine (the sink drain), per doc 06 §0.

**Resampler — A.3 + doc 06 §3.2.** Per `Process`, for each input frame: prime the 3-frame
`hist` window first (`nhist < 3`), then while `phase < 1.0` emit one cubic-Hermite output
frame per channel and advance `phase += step` (`step = 1/ratio`), then `phase -= 1.0`,
slide history left, fold the pending frame in. The 4 taps are `s0,s1,s2 = hist[0..2]` and
`s3 =` the pending input frame; `phase` interpolates between `s1` (integer cursor) and
`s2`. `cubicHermite` is Catmull-Rom (`m1=(s2-s0)/2`, `m2=(s3-s1)/2`): **DC- and
linear-exact** (verified by `TestDCPassthrough`), C1-continuous across block boundaries
(verified by `TestBlockBoundarySeam`). `SetRatio` clamps to `[1-MaxPPM·1e-6,
1+MaxPPM·1e-6]` then stores `step = 1/ratio`.

**Quoted A.12 tunables (verbatim, single source of truth):**
- `MaxPPM` — A.12 *Drift PI* row: **"Kp / Ki / MaxPPM / IntegralClamp … 0.05 / 0.005 /
  200 ppm / 200"** → `MaxPPM = 200`. A.3 restates: **"MaxPPM=200"**, "±200 ppm ≈
  ±0.02 % → inaudible".
- `LeadMs` (ring sizing, set by the renderer that owns `NewRing`, not by P4.4) — A.12
  *Buffer* row: **"playout lead `LeadMs` … 300 ms"**.
- Canonical `rate / channels / FramesPerChunk` — A.12 *Audio canonical* row: **"48000 / 2
  / 480 (10 ms)"** (these parameterize callers; the Ring/Resampler are rate-agnostic and
  channel-parametric).

P4.4 introduces **no new tunable**; every number is quoted above from A.12.

## 6. Import constraints

Both packages are **leaves** under `internal/audio/` (doc 01 §2.2 layering). Allowed
imports: **stdlib only** — `ring` imports `sync`; `resampler` imports nothing (tests add
`math`, `sync`, `testing`). They MUST NOT import any other `internal/*` package (not
`audio/drift`, `audio/render`, `audio/sink`, not `group`/`stream/*`/`clock`/`state`), and
no third-party module. This keeps them trivially unit-testable and reusable, and upholds
the README hard invariant that `internal/web` reaches the engine only via the `web.Deps`
seam (these packages are well below that seam and never touch `web`). The dependency
arrow points the other way: `audio/render` (and the producer) import `ring` + `resampler`.

## 9. Risks / open questions

- **R1 (non-blocking) — Ring has no `cond`, A.7 says "mutex+cond".** The proven media impl
  is mutex-only with caller-side back-off, and doc 06 §2.2/§2.3 explicitly specify that
  back-off (producer retries on short write; consumer never pads on underrun). Keeping
  mutex-only is correct and lighter for the Pi (no goroutine wakeups in the hot path);
  treating A.7's "+cond" as descriptive intent, not a literal requirement. Confirm the
  orchestrator agrees, else add a `sync.Cond` (would change the back-off contract doc 06
  relies on — not recommended).
- **R2 (non-blocking) — A.3 vs media cubic coefficients.** A.3 gives the cubic as
  `a0=y3-y2-y0+y1; a1=y0-y1-a0; a2=y2-y0; a3=y1` (a Fritsch/4-point form); media's
  `cubicHermite` uses Catmull-Rom `c0=y1; c1=0.5(y2-y0); c2=y0-2.5y1+2y2-0.5y3;
  c3=0.5(y3-y0)+1.5(y1-y2)`. Both are 4-tap interpolators interpolating between `y1` and
  `y2`, but they are **not bit-identical** (Catmull-Rom is C1/DC-exact and is the one the
  media tests pin: `TestDCPassthrough`, `TestBlockBoundarySeam`). **Decision: keep the
  media Catmull-Rom** (tested, proven, A.3's pseudocode is "normative-by-intent … not
  exact Go" per the appendix preamble). Flag for orchestrator awareness only.
- **R3 (non-blocking) — `MaxPPM` duplicated across `resampler` and `drift`.** Both must
  equal A.12's 200 ppm. Plan keeps two independent constants (no cross-package import, per
  §6). If a single source-of-truth const is preferred, the orchestrator should designate a
  shared `internal/audio` (non-leaf) location; otherwise the A.12 citation is the binding
  contract and a one-line test in each package can assert `== 200`.
- **R4 (cosmetic) — `doc.go` removal.** F0's `internal/audio/{ring,resampler}/doc.go`
  placeholders must be deleted (or their one-line comment folded into the new files) so
  each package is one source + one test file; leaving a bare-package `doc.go` alongside is
  harmless but untidy.
