## 1. Purpose & scope

Provide the stateful PI controller that the renderer (P4.4) drives once per control tick to
hold this node's playout in sample-lock with the group timeline. The controller maps an
error **in canonical-rate samples** to a near-unity resample **ratio** (the actuator) or a
`DriftReseek` signal on gross error. The package owns *only* the control law and its params
(`DriftLoop`, `DriftParams`, `DriftAction`) plus its own faithful convergence test — it does
**not** own the renderer, the resampler, the ring, the sink, or the computation of
`errSamples` (that error is assembled by the P4.4 renderer in the content domain, per 06 §3.3,
and fed in). The single correction over the reused `mpvsync` code is *not* in the controller
math (which is already content-agnostic — `error = played − want`); it is (a) re-homing the
loop into its own package and (b) **replacing the convergence test with one that models the
real plant** (fixed-crystal DAC drain), because the original `mpvsync` test modeled the
actuator as if it drove the regulated variable directly and thereby hid the controllability
bug that 06 §3.1 describes.

## 4. Public API (exact signatures — canonical per 06 §3.4 / A.4)

```go
package drift

// DriftParams tunes the audio drift PI controller. Actuator is resample ratio,
// never playback speed. Defaults are canonical in Appendix A.12.
type DriftParams struct {
    Kp            float64 // proportional gain, samples->ppm.        Default 0.05
    Ki            float64 // integral gain, samples->ppm per tick.   Default 0.005
    MaxPPM        float64 // ratio-trim clamp, ppm.                  Default 200
    HardErrSamp   int     // |error| above this triggers reseek.     Default 2400 (=50ms@48k)
    IntegralClamp float64 // anti-windup clamp on Ki*integral, ppm.  Default 200
}

func DefaultDriftParams() DriftParams

// DriftAction is what the control law selects for an error.
type DriftAction int
const (
    DriftHold   DriftAction = iota // apply ratio trim, keep integrating
    DriftReseek                    // gross error: caller must Seek + refill + Reset
)
func (a DriftAction) String() string

// DriftLoop is the stateful PI controller (holds the integral term). One per renderer.
type DriftLoop struct { /* unexported: p DriftParams; integral float64 */ }

func NewDriftLoop(p DriftParams) *DriftLoop

// Update takes the current error in SAMPLES (error = playedContent - wantContent;
// positive => ahead => slow => ratio < 1) and returns the action plus the resample
// ratio to apply (1 ± clamp). On DriftReseek ratio is 1.0 and the loop does NOT
// integrate the gross error.
func (d *DriftLoop) Update(errSamples int) (DriftAction, float64)

// Reset zeroes the integral term (after a reseek / underrun).
func (d *DriftLoop) Reset()

// Clamp limits a ppm trim to ±MaxPPM.
func (p DriftParams) Clamp(ppm float64) float64
```

Note `DriftParams` is **also** named in `RendererParams.Drift` (06 §2.1). To avoid an import
cycle and keep one definition, the canonical home is **this package**; the P4.4 renderer
imports `internal/audio/drift` and references `drift.DriftParams` / `drift.DefaultDriftParams`
in `RendererParams`. (See §9 — confirm 06 §2.1's bare `DriftParams`/`DriftErr` field name maps
to `drift.DriftParams`.)

## 5. Behavior & algorithms

### 5.1 The control law (06 §3.4, A.4 — unchanged)

`Update(errSamples)`:
1. **Gross error first.** `if abs(errSamples) > HardErrSamp → return (DriftReseek, 1.0)` and
   do **not** integrate. The caller performs the 06 §6.1 reseek (which calls `Reset()`).
2. **Integrate + anti-windup.** `integral += errSamples`; then clamp `integral` so
   `Ki*integral ∈ [−IntegralClamp, +IntegralClamp]` ppm (skip the clamp math when `Ki == 0`
   to avoid divide-by-zero — preserved from the source).
3. **PI law (negative sign).** `ppm = -(Kp*errSamples + Ki*integral)`; `ppm = Clamp(ppm)`
   (±`MaxPPM`); `ratio = 1 + ppm*1e-6`; `return (DriftHold, ratio)`.

`NewDriftLoop` repairs a partially-filled struct: `MaxPPM`, `IntegralClamp`, `HardErrSamp` each
fall back to their default when `<= 0`, so the actuator can never be unbounded and the
anti-windup divide is safe.

Sign convention (06 §3.3, identical to mpvsync): `error = played − want`; **positive ⇒ ahead
⇒ slow ⇒ ratio < 1**; negative ⇒ behind ⇒ ratio > 1; zero ⇒ ratio 1.0.

### 5.2 Tunables — verbatim from A.12 (single source of truth)

> Drift PI — **Kp / Ki / MaxPPM / IntegralClamp = 0.05 / 0.005 / 200 ppm / 200**
> Drift PI — **tick / HardErrSamp = 20 ms / 2400 samples (50 ms)**
> Audio canonical — **rate / channels / FramesPerChunk = 48000 / 2 / 480 (10 ms)**

`HardErrSamp = 2400` is exactly 50 ms at 48 kHz. `MaxPPM = 200` is ±0.02 % → inaudible
(A.3). These values are quoted, never invented; if A.12 changes, `DefaultDriftParams` tracks it.

### 5.3 What this package does **not** do (consumed from P4.4, per 06 §3.3)

The renderer assembles `errSamples` in the **content domain** before calling `Update`:
`playedContent = baseSourceConsumed + sourceConsumed − round((ringFrames + devFrames) * ratio)`
and `wantContent = sampleIndex + hwDelayOffset`. None of that lives here — this package only
sees the final `int` error. That separation is exactly why the controller stays unchanged and
the correction is isolated to (a) the package home and (b) the convergence *test* (06 §3.6).

### 7.2 Corrected convergence tests (the central P4.5 deliverable — REWRITE)

Replace `TestConvergence` / `TestConvergenceNegative` with a sim that obeys 06 §3.6 / A.4's
**"convergence test MUST model the real plant"**. The model MUST:

- **DAC drains output at a fixed crystal rate** `f_out = Rate*(1 + crystalPPM*1e-6)`,
  **independent of the commanded ratio**. The crystal offset is the disturbance to reject.
- **Ratio sets the source→output coupling:** `sourceConsumed` advances at `ratio * f_out` per
  unit time (the resampler consumes `ratio` content frames per output frame; output is
  produced/drained at `f_out`).
- **Buffer backlog evolves from that drain**, not as a free input: the modeled
  `(ringFrames + devFrames)` backlog responds to the producer/consumer rate difference, and
  is fed back into `playedContent` exactly as the renderer does (06 §3.3), closing the loop.
- `wantContent` (the group timeline) advances at the **nominal** `Rate` per tick
  (`Tick = 20 ms ⇒ 960 frames @48k`).

Per-tick sim sketch (content domain, per-channel frames; this is the harness, not prod code):

```
const ( rate = 48000.0; tickSec = 0.020; nominal = rate*tickSec /*960*/ )
crystalPPM := +40.0                       // disturbance; also run -40, 0, +100, -100
fOut := rate * (1 + crystalPPM*1e-6)      // DAC drain rate (frames/sec), FIXED
backlog := lead                            // source-equiv buffered, start ~LeadMs*rate
sourceConsumed, want := 0.0, 0.0
ratio := 1.0
for i := 0; i < N; i++ {                    // N ~ 3000 ticks = 60 s sim
    outThisTick := fOut * tickSec           // output frames the DAC pulled this tick (ratio-independent)
    srcThisTick := ratio * outThisTick      // content frames the resampler consumed to feed them
    sourceConsumed += srcThisTick
    backlog += srcThisTick - outThisTick    // producer in (content) vs consumer out (output@unity≈content)
    played := sourceConsumed - backlog      // == 06 §3.3 with ratio≈1 conversion
    want   += nominal
    err := int(math.Round(played - want))
    act, r := d.Update(err)
    if act == DriftReseek { t.Fatalf("tick %d: unexpected reseek err=%d", i, err) }
    ratio = r
}
```

**Assertions (per 06 §3.6 / A.4 "drives errSamples to a sub-ms steady state"):**
1. **Converges:** `|finalErr| < 48` samples (sub-ms @48k) for each `crystalPPM ∈ {±40, ±100, 0}`.
2. **Ratio settles to oppose the crystal:** final commanded `ratio ≈ 1 − crystalPPM*1e-6`
   (within a small tolerance, e.g. `|ratio − (1 − crystalPPM*1e-6)| < 5e-6`) — i.e. it nulls
   the drift **at a non-clamped value**, *not* by riding `±MaxPPM`.
3. **Integrator is finite and non-clamped** at steady state (`|Ki*integral| < IntegralClamp`),
   the diagnostic that 06 §3.4 calls out as proof the bug is fixed.
4. **No reseek after settling** (the sim asserts no `DriftReseek` once converged; a recurring
   steady-state reseek is the regression signal of 06 §3.5).
5. **Sign sanity:** `crystalPPM>0 ⇒ steady ratio<1`; `crystalPPM<0 ⇒ steady ratio>1`.

Add a **negative-control / guard test** (`TestRejectOutputDomainModel`, optional but
recommended by 06 §3.6): assert that the *buggy* `played += tick*ratio*(1+driftPPM)` model
would converge for *any* ratio sign (so it cannot distinguish a correct from a sign-flipped
loop), documenting in a comment why that model is rejected. This makes the "test must model
the real plant" requirement executable, not just prose.

A faithful sim with a **sign error** in the loop, or one using an **output-domain `played`**,
MUST fail assertions 1–3 — that is the acceptance bar for "the bug is fixed" (A.4 note).

## 8. Dependencies

**Consumes (upstream):**
- **P4.4 `internal/audio/render`** — the *caller*. P4.4 owns the renderer that computes the
  content-domain `errSamples` (06 §3.3) and the `Resampler` actuator it applies the returned
  ratio to via `Resampler.SetRatio` (06 §2.2). P4.5 depends on **no symbols** from P4.4 at
  compile time (the arrow is P4.4 → P4.5); the relationship is "P4.4 drives this loop." Listed
  as the upstream piece per the task framing, but the code dependency is one-directional the
  other way.
- **stdlib only** at the package level (`math`).

**Exposes (downstream consumers):**
- `drift.DriftParams`, `drift.DefaultDriftParams()` — embedded by P4.4's `RendererParams.Drift`
  (06 §2.1) and re-tuned live via the renderer's `SetParams`.
- `drift.DriftLoop` + `NewDriftLoop`, `(*DriftLoop).Update`, `(*DriftLoop).Reset` — the P4.4
  control tick (06 §7) calls `Update(errSamples)` every `Tick`, maps `DriftHold` → `SetRatio`
  and `DriftReseek` → the 06 §6.1 `reseek` (which calls `Reset`).
- `drift.DriftAction` (`DriftHold`/`DriftReseek`) — branched on in the control tick; surfaced
  into `RenderTick.Action` (06 §2.4) for UI/logging.
- `drift.DriftParams.Clamp` — exported helper (table-tested; also usable by the renderer's
  belt-and-braces ratio guard, though `Resampler.SetRatio` re-clamps independently per 06 §3.4).

   extra case; not required by A.4.)

   A.4 + A.12 and can be built independently of P4.4. The only cross-piece coupling (item 1/2)
