## 1. Purpose & scope

This piece implements the **output device seam** for an Ensemble node: the canonical
`AudioSink` interface (README §6.1), the **runtime** backend registry (`Probe`/`Open`/`Backend`,
06 §1.1 — no build tags, every backend compiled into the single pure‑Go binary and selected at
startup), the coarse **exec** sink (`aplay`/`pw-play`, pure‑Go pipe of S16_LE PCM), and the
precise **ALSA** sink driven **directly through kernel ioctls** on `/dev/snd/pcmC<card>D<dev>p`
via `golang.org/x/sys/unix` (06 §1.3 — **no libasound, no dlopen/purego, no cgo**). It exposes the
probed‑and‑enabled backend set so an upstream wiring piece can populate `Caps.Sinks`/`Caps.Render`
(06 §1.5) and skip the render loop when `Render=false`. It does **not** implement the render loop,
drift, ring, resampler, or capability gossip — those are sibling pieces; this piece only produces
the sink seam and the sink half of capability detection. The ALSA backend is a **verification
spike** (06 §1.3) requiring real‑hardware acceptance (Pi + DAC HAT).

## 4. Public API (canonical names — README §6.1, 06 §1.1)

```go
package audio // import "gitlab.rand0m.me/ruben/go/ensemble/internal/audio/sink"

// AudioSink — verbatim from README §6.1 / 06 §1. Interleaved float32 in [-1,1].
type AudioSink interface {
    Start(rate, channels int) error            // open device at rate Hz / channels; idempotent-error if started
    Write(frames []float32) (n int, err error) // interleaved; returns float32 SAMPLES consumed; blocks for backpressure
    Delay() (samples int, ok bool)             // samples-PER-CHANNEL outstanding in device; ok=false => coarse
    Close() error                              // idempotent
}

// errUnderrun is returned by alsaSink.Write on -EPIPE (device underrun) after it
// re-PREPAREs the PCM, so the renderer treats it as a gross error and reseeks.
// Comparable via errors.Is. (06 §1.3, §6.4)
var errUnderrun = errors.New("alsa: underrun (-EPIPE)")

// --- runtime backend registry (06 §1.1) — NOT a build-time switch ---

type Backend struct {
    Name    string // "alsa" | "exec:aplay" | "exec:pw-play"
    Precise bool   // true => Delay() returns a precise kernel figure (alsa only)
}

// ProbeConfig is the per-node config input to the probe: the device string and the
// set of backend names the operator has DISABLED (subtracted from the probe result),
// plus the preference order. (06 §1.1, §1.5)
type ProbeConfig struct {
    Device   string   // per-node device string from NodeRecord (""=each backend's "default")
    Disabled []string // backend Names turned off in config (e.g. ["alsa"] to force coarse)
    Prefer   []string // preference order; missing names default to {"alsa","exec:pw-play","exec:aplay"}
}

// Probe tries every compiled-in backend, returns those that ACTUALLY WORK on this
// machine, minus config-disabled, in preference order. Cheap; run once at startup.
// A backend whose /dev/snd node exists but cannot be opened/configured is NOT returned.
func Probe(cfg ProbeConfig) []Backend

// Open opens the first backend in `preferred` that is present in the probe result.
// `device` is the per-node device string. Returns the opened-but-not-yet-Started sink.
// Returns an error if none of `preferred` is usable.
func Open(preferred []string, device string) (AudioSink, error)

// NewExecSink builds a coarse exec sink for `device` using the default aplay command.
func NewExecSink(device string) *ExecSink
type ExecSink struct{ /* unexported */ }

// f32ToS16LE converts interleaved float32 [-1,1] -> S16_LE (clamp, round-to-nearest,
// scale 32767). Pure function, table-tested. (A.10 wire-PCM convention reused here.)
func f32ToS16LE(src []float32, dst []byte)
```

Internal (unexported, not part of the seam): `newALSASink(device string) *alsaSink`, the
ALSA uapi structs/constants in `alsa_uapi.go`, `defaultPlayerCommand`/`pwPlayCommand`/
`expandPlayerCommand`.

**Consumers of this API.** The wiring/`node.Run` piece calls `Probe(cfg)` once at startup to
derive `Caps.Sinks` (= the returned `Backend.Name`s) and `Caps.Render` (= `len>0`), then calls
`Open(prefer, device)` and hands the sink to the render loop's `Start(rate, channels)` (06 §1.5).
`errUnderrun` is consumed by the render loop (sibling) via `errors.Is`. The `fakeSink` double is
consumed by the render‑loop tests.

## 5. Behavior & algorithms

### 5.1 Registry `Probe`/`Open` (06 §1.1, §1.5)

Selection is **runtime**, never build‑time (D12, 06 §1.1: "no `-tags alsa`, no build‑tagged
twin files, no cgo variant, no compile‑time `NewSink`"). The compiled‑in backend table, in
default preference order:

| Backend `Name` | `Precise` | Probe (must succeed) | `Delay()` |
|---|---|---|---|
| `alsa` | **true** | enumerate `/dev/snd/pcmC*D*p`, `open(O_RDWR)`, complete `HW_REFINE`/`HW_PARAMS` + `PREPARE` for `FORMAT_FLOAT_LE`, then `close` — succeed ⇒ usable | `SNDRV_PCM_IOCTL_DELAY` (precise) |
| `exec:pw-play` | false | `exec.LookPath("pw-play")` resolves | `(0,false)` |
| `exec:aplay` | false | `exec.LookPath("aplay")` resolves | `(0,false)` |

Probe semantics (06 §1.1): presence of the `/dev/snd` node is **necessary but not sufficient** —
the `alsa` probe must actually `HW_PARAMS`/`PREPARE` once (a card owned by PipeWire/Pulse, busy,
or format‑incapable fails and is dropped). `Probe`:

1. For each compiled‑in backend in preference order, run its liveness probe; keep the ones that
   pass.
2. Subtract any `cfg.Disabled` names.
3. Reorder per `cfg.Prefer` (entries not in `Prefer` keep the default order after the prefixed
   ones).
4. Return the surviving `[]Backend`.

`Open(preferred, device)` walks `preferred`, and for the first name also present in a fresh
`Probe`‑style usability check, constructs the matching sink (`newALSASink(device)` or
`NewExecSink(device)` with the right command template) **without** calling `Start` (the renderer
commits `rate`/`channels` later via `Start`). `device==""` maps to each backend's `"default"`.

**Capability advertisement (06 §1.5).** The probe result *is* the sink half of capabilities:
`Caps.Sinks = [Backend.Name…]` and `Caps.Render = len(Caps.Sinks) > 0`. This piece only
**produces** that set from the probe; the wiring piece writes it into `NodeRecord.Caps` and
gossips it. **`Render=false` ⇒ the render loop is never started** (06 §1.5) — enforced by the
wiring piece, not here.

### 5.2 Exec sink (coarse) — 06 §1.2

Verbatim media behavior: `Start` launches `aplay -q -t raw -f S16_LE -r {rate} -c {channels}
-D {device} -` (or the `pw-play` template) and grabs stdin; `Write` clamps to `[-1,1]`, converts
f32→S16_LE (round‑to‑nearest, scale **32767** symmetric so `+1.0` never wraps the rail) and writes
to the pipe (the block *is* the playout pacing); `Write` returns `len(frames)`; a broken pipe
surfaces as an error; `Delay()→(0,false)` (no readback ⇒ renderer uses the coarse content model,
06 §4.2); `Close` closes stdin, waits, idempotent. Do **not** inflate the player's internal
buffer — the jitter buffer is the sibling `Ring`, not aplay's (06 §1.2).

### 5.3 ALSA sink (precise, pure‑Go direct ioctl) — 06 §1.3 — verification spike

Drive the kernel PCM character device directly via `unix.Syscall(SYS_IOCTL, fd, req, arg)` on
`/dev/snd/pcmC<card>D<dev>p`, using `golang.org/x/sys/unix` only (A.11). The ioctl set (the same
ALSA PCM uapi ABI `libasound` sits on; struct layouts mirror `<sound/asound.h>`):

- **`Start(rate, channels)`** — `open(O_RDWR)` the playback node; run `SNDRV_PCM_IOCTL_HW_REFINE`
  then `SNDRV_PCM_IOCTL_HW_PARAMS` to fix `SNDRV_PCM_FORMAT_FLOAT_LE`,
  `SNDRV_PCM_ACCESS_RW_INTERLEAVED`, `channels`, `rate`, and period/buffer sizes for a **≈100 ms**
  device buffer (the `Ring` is the real jitter buffer, 06 §1.3); then `SNDRV_PCM_IOCTL_SW_PARAMS`
  and `SNDRV_PCM_IOCTL_PREPARE`. Blocking mode is the backpressure.
- **`Write(frames)`** — submit interleaved float frames via `SNDRV_PCM_IOCTL_WRITEI_FRAMES`
  (`snd_xferi{result, buf, frames}`) in a loop until all accepted; return consumed **samples**
  (= frames×channels). On `-EPIPE` (underrun) issue `SNDRV_PCM_IOCTL_PREPARE` to recover and return
  `errUnderrun` (renderer reseeks). On `-ESTRPIPE` (suspend) recover (`RESUME`/`PREPARE`) and retry
  silently. `len(frames)` must be a multiple of `channels`.
- **`Delay() → (frames, true)`** — precise outstanding sample count via
  `SNDRV_PCM_IOCTL_DELAY` (or `SNDRV_PCM_IOCTL_SYNC_PTR` then `appl_ptr − hw_ptr`). A negative or
  failed query degrades to `(0,false)` — never lies (06 §1.3).
- **`Close()`** — `SNDRV_PCM_IOCTL_DRAIN` then `close(fd)`; idempotent.

All ioctls serialised by `mu` (a single PCM fd is not concurrency‑safe; the renderer's consumer
goroutine calls `Write`/`Delay` while `Close` may race). `alsa_uapi.go` carries the request
constants (`_IOWR('A', …)` encodings) and the `snd_pcm_hw_params`/`sw_params`/`sync_ptr`/`xferi`
struct layouts, `//go:build linux`. `alsa_stub.go` (`//go:build !linux`) makes `newALSASink`
return an error and `Probe` skip `alsa`, so the package compiles on a non‑linux dev host while the
real path is linux/arm64. **No cgo, no dlopen** — the whole binary stays pure‑Go and
cross‑compiles to arm64 with no C toolchain (D24, 06 §1.1, doc 11 §3).

### 5.4 Tunables (Appendix A.12 — verbatim, single source of truth)

This piece quotes A.12, never invents:

- **rate / channels / FramesPerChunk = `48000 / 2 / 480 (10 ms)`** — the canonical format the
  renderer commits via `Start(rate, channels)`.
- ALSA device buffer ≈ **100 ms** (06 §1.3; "the `Ring` is the real jitter buffer"). Not an A.12
  knob; quoted from 06 §1.3.

This piece does **not** own `LeadMs`, drift PI gains (`Kp 0.05 / Ki 0.005 / MaxPPM 200 ppm /
IntegralClamp 200`, `tick 20 ms / HardErrSamp 2400`) or `Delay()`‑consuming math — those are the
render/drift siblings. `HardErrSamp = 2400` is relevant only insofar as `errUnderrun` is the gross
error that triggers a reseek (06 §3.5, §6.4).

## 6. Import constraints (doc 01 §2 layering)

- This package is `internal/audio/sink`. Allowed imports: **Go stdlib** (`os`, `os/exec`, `io`,
  `errors`, `fmt`, `sync`, `strconv`, `path/filepath`, `unsafe` for ioctl arg pointers) and
  **`golang.org/x/sys/unix`** (A.11). No third‑party audio library.
- It must **NOT** import `group`, `stream/*`, `clock`, `state`, `web`, or sibling
  `internal/audio/{ring,resampler,drift,render}` — the seam is a leaf the renderer depends on, not
  vice versa. It defines `AudioSink`; the render piece imports *this*.
- It must **NOT** import `internal/web` (the `web.Deps` seam rule, doc 01 §2 / README §6 — web
  reaches the engine only through `Deps`). Capability values flow *out* via the return of `Probe`,
  consumed by the wiring piece, never by a back‑import.
- No build tags select backends; the only build tags are the OS split (`alsa_uapi.go`/
  `sink_alsa.go` = `//go:build linux`, `alsa_stub.go` = `//go:build !linux`) — an OS portability
  guard, **not** a capability switch (D12 still holds: on linux/arm64 every backend is compiled in).

## 9. Risks / open questions

1. **BLOCKING (orchestrator must resolve): ALSA uapi struct layout source.** `alsa_uapi.go`
   transcribes `<sound/asound.h>` (`snd_pcm_hw_params` with its `masks[]`/`intervals[]` arrays,
   `snd_pcm_sync_ptr`, the `_IOWR('A', …)` request numbers). `golang.org/x/sys/unix` does **not**
   ship these PCM structs/constants, so they must be hand‑authored and pinned to a kernel uapi
   version. **Decision needed:** which kernel uapi version to target (Raspberry Pi OS arm64 ships a
   specific `SNDRV_PCM_VERSION`), and whether to gate `HW_PARAMS` on a `SNDRV_PCM_IOCTL_PVERSION`
   check + fall back to coarse on mismatch. Until a Pi+DAC is available to validate the layout, the
   `alsa` probe should be treated as failing‑closed (degrade to `exec:*`) so the rest of the system
   is unblocked.
2. **`HW_PARAMS` refine complexity.** The mask/interval refine dance is the intricate part (06 §1.3
   calls it out). Recommend implementing `HW_REFINE`‑then‑`HW_PARAMS` with explicit interval‑set
   helpers in `alsa_uapi.go` and unit‑testing the helpers against known‑good byte patterns; the
   end‑to‑end correctness is only provable on hardware (the spike).
3. **`pw-play` vs `aplay` default order.** 06 §1.1 lists both coarse backends but does not pin a
   preference between them. Defaulting to `exec:pw-play` ahead of `exec:aplay` (since a PipeWire box
   is exactly the case where direct ALSA is unavailable) is a judgement call — confirm with the
   owner; it is config‑overridable via `ProbeConfig.Prefer` regardless.
4. **`SYNC_PTR` mmap path.** 06 §1.3 mentions `SNDRV_PCM_IOCTL_SYNC_PTR` as an equivalent for
   `DELAY`. MVP uses `SNDRV_PCM_IOCTL_DELAY` (simpler, no mmap); the `SYNC_PTR` route is a
   documented fallback if `DELAY` proves unreliable on a given driver — note, not implement, now.
```
