# P4.7 — `internal/audio/render` — render loop & content-domain control tick

Track: BACKEND (Go) · Module `gitlab.rand0m.me/ruben/go/ensemble` · Go 1.26 · pure-Go, `CGO_ENABLED=0`, must cross-compile to `linux/arm64`.

## 1. Purpose & scope

This piece implements the **output-half render loop** of a player node: the two goroutines (producer + consumer/control) run under `Renderer.Run(ctx)` that map the group's `Timeline.NowSample()` to this node's physical playout, drive the `Resampler` against an `AudioSink`, and hold sub-millisecond inter-node sync via the **corrected content-domain drift PI loop**. It owns the 20 ms control tick, the `wantContent`/`playedContent` computation (including the `HWDelayUs` sample-bias and the `Delay()`-aware backlog subtraction), per-node channel-select / `GainDB`, and the `reseek` primitive that handles startup, `streamGen` seek, and underrun. It is a faithful re-home of `media internal/audio/renderer.go` **with one mandatory correction**: the regulated variable becomes content-domain (doc 06 §3, Appendix A.4) — the media file's `played = framesWritten − Delay()` output-domain formula is the known bug and must NOT be copied. Authoritative specs: doc/06 §2,§4,§5,§6,§7; Appendix A.4 (loop) / A.12 (numbers).

**Itemized rewrite of `renderer.go` (the load-bearing changes):**

1. **Package + imports.** `package render`. Import `internal/audio/ring`, `internal/audio/resampler`, `internal/audio/drift`, `internal/audio/sink`, `internal/group`, `internal/state`. Drop the media `syncpkg` and `media/internal/state` imports.

2. **Timeline seam: `float64 seconds` → `int64 sample`.** Media used `src syncpkg.TimelineSource` with `TimelineNow() (float64, bool, bool)` (T in *seconds*). Ensemble's canonical contract (README §6.2, doc 04 §4.4) is `Timeline.NowSample() (sample int64, playing bool, ok bool)` in **canonical-rate frames (48 kHz)**. Replace the field `src syncpkg.TimelineSource` with `timeline group.Timeline`. This **eliminates** all `clipT*Rate` second→sample conversion: `wantContent` is now `sample + hwDelayOffset` directly (doc 06 §3.2/§7 step 4). The whole `wantSampleFor(clip, tNow float64, ...)` collapses to integer math on `sample`.

3. **CRITICAL — content-domain `played` (doc 06 §3, A.4).** Replace the media `playedSample()` body. Media computed the **output-domain** `played := wantBase + framesWritten − delay` (the bug, doc 06 §3.1). The ensemble formula (doc 06 §3.3, §7 step 5):
   ```
   ratio        := r.lastRatio                       // content-per-output, near 1.0
   ringFrames   := r.ring.Len() / p.Channels         // output-domain backlog
   devFrames, ok := r.sink.Delay()                   // ALSA precise; exec => !ok
   if !ok { devFrames = r.coarseDeviceDelay() }
   bufferedContent := int64(math.Round(float64(ringFrames+devFrames) * ratio))
   played := r.baseSourceConsumed + r.sourceConsumed - bufferedContent
   ```
   Add fields `sourceConsumed int64` (cumulative SOURCE frames the resampler consumed since the reseek baseline) and `baseSourceConsumed int64` (the `want` captured at the last reseek). **Remove** `framesWritten`/`wantBase` from the regulated path; keep `lastWriteAt` (and a vestigial `framesWritten` only if needed for telemetry) ONLY for the coarse `Delay()` wall-time model (doc 06 §4.2, §7 trailing note).

4. **`sourceConsumed` accumulation.** Media's `resampleSelect` discarded the resampler's `consumed` return (`resampled, _ := r.resampler.Process(...)`). Ensemble must accumulate it: `resampled, consumed := r.resampler.Process(nil, in)`; under the resampler lock add `r.sourceConsumed += int64(consumed / srcCh)` (Process returns consumed in interleaved samples; `/srcCh` → per-channel frames; A.4 "srcConsumed"). This counter is advanced by the **producer** and read by the control goroutine under `rmu`.

5. **`reseek(want)` re-baseline (doc 06 §6.1).** Replace the media baseline reset (steps 5–6) with: `r.baseSourceConsumed = want; r.sourceConsumed = 0` (instead of `framesWritten = 0; wantBase = want`). Keep: `src.Seek(want)`, `resampler.Reset()`, `ring.Reset()`, `drift.Reset()`, `SetRatio(1.0)`, `settle = settleTicks`, `lastWriteAt = now`.

6. **`coarseDeviceDelay()` (doc 06 §4.2).** Keep the media coarse model but expose it as the documented signature returning **per-channel frames**: `elapsed := time.Since(r.lastWriteAt).Seconds(); return int(float64(p.Rate) * elapsed)`. Note media folded `ringSamples` into its coarse path; in the corrected loop the ring is subtracted separately in `played` (step 3), so `coarseDeviceDelay()` returns the **device** estimate only (do not double-count the ring).

7. **Channel-select / gain (doc 06 §5).** Keep `pickChannel` and `gainLinear` verbatim. Change the lane snapshot source: media read `state.AudioLane{Channels []int, GainDB, DelayUs}`; ensemble reads `NodeRecord{Channel string ∈ {"stereo","left","right"}, GainDB float64, HWDelayUs int}` (README §6.5). Add a small mapper `channelSel(role string) []int`: `"stereo"→nil` (passthrough), `"left"→[]int{0}`, `"right"→[]int{1}` (doc 06 §5.1 table) — feeding the existing `pickChannel` mono-fan-out logic unchanged.

8. **`HWDelayUs` bias (doc 06 §5.3, §7 step 4).** `hwOff := int64(math.Round(float64(node.HWDelayUs) * 1e-6 * float64(p.Rate))); want := sample + hwOff`. (Media used `lane.DelayUs`; same quantity, renamed per §6.5.)

9. **Drop the offline-harness mode** (`src == nil` branch, `setOfflineSource`). Ensemble always has a `Timeline`; the offline pacing path was a media test crutch. The real-plant convergence harness (§7) replaces it and is a better validator.

10. **`docSource`/asset-resolver seam.** Media used `docs docSource{ Get() state.Doc }` + `AssetResolver(state.Clip)`. Ensemble: a `configDoc func() state.ConfigDoc` snapshot accessor + `nodeID string` to look up its own `NodeRecord`, and an `AssetResolver func(clip) (path string, ready bool)` for the source-decode piece (stream/source). Keep the "not ready ⇒ feed silence, do not reload" behavior (doc 06 §7 step 3). The active-clip lookup (`activeClip(group.media, sample)`) is provided by the group/state seam, not reimplemented here.

## 5. Behavior & algorithms (cite + quote)

**Two goroutines under `Run` (doc 06 §2).** Producer fills the playout `Ring` to ~`LeadMs`: `recv-ring → Resampler.Process(ratio) → channel-select+GainDB → Ring.Write` (doc 06 §2.2). Consumer/control drains `Ring.Read → sink.Write` (blocking write = playout pacing) and runs the control law every `Tick` (doc 06 §2.3). The producer holds the resampler lock across `Process` so a concurrent `SetRatio` from the control goroutine cannot race interpolation (doc 06 §2.2).

**Control tick ordering (doc 06 §2.4 / §7), exactly:**
1. `sample, playing, ok := timeline.NowSample()`; `!ok` ⇒ HOLD (silence, no advance, no reseek), `HaveSync=false`, return.
2. Look up own `NodeRecord` + active clip; not active ⇒ feed silence, return.
3. `streamGen` changed or media changed ⇒ resolve asset; not ready ⇒ silence+return; else `loadSource` + `reseek(want)` + `settle = settleTicks`.
4. `wantContent = sample + round(HWDelayUs·1e-6·Rate)` (doc 06 §3.2/§5.3).
5. `playedContent = baseSourceConsumed + sourceConsumed − round((ringFrames+devFrames)·ratio)` (doc 06 §3.3).
6. settle window ⇒ hold; else `errSamples = int(played − want)` → `drift.Update` → `DriftHold` ⇒ `applyRatio` (skip if `|Δ|<1e-7`) / `DriftReseek` ⇒ `reseek` + settle.
7. fire `OnTick`.

Sign convention (doc 06 §3.3, identical to mpvsync): `error = played − want`; `+err ⇒ ahead ⇒ slow ⇒ ratio<1`.

**Why content-domain is controllable (doc 06 §3.2):** the resample ratio is content-per-output-frame; the DAC drains output at the fixed crystal rate `f_out`, so `d(playedContent)/dt ≈ ratio·f_out` — the actuator sits directly in the derivative of the regulated variable, giving the PI loop nonzero loop gain. The output-domain media formula had effectively zero gain and rode the clamp (doc 06 §3.1) — that is the bug being fixed here.

**Tunables — A.12 verbatim (single source of truth; do not invent):**
- rate / channels / FramesPerChunk = **48000 / 2 / 480 (10 ms)**.
- playout lead `LeadMs` = **300 ms**.
- Drift PI: `Kp / Ki / MaxPPM / IntegralClamp` = **0.05 / 0.005 / 200 ppm / 200**.
- Drift PI: `tick / HardErrSamp` = **20 ms / 2400 samples (50 ms)**.
- Resampler near-unity clamp `MaxPPM` = **200** (±0.02 %, inaudible — A.3/doc 06 §3.4 belt-and-braces re-clamp in `SetRatio`).
- `settleTicks` ≈ **2** (doc 06 §6.1).

**`reseek(want)` primitive (doc 06 §6.1):** seek source to `want` → `Resampler.Reset()` → `Ring.Reset()` → `DriftLoop.Reset()` → `baseSourceConsumed=want; sourceConsumed=0` → `framesWritten=0; lastWriteAt=now` → `SetRatio(1.0)` → `settle=settleTicks`. Used by **startup** (first `ok=true` with active clip — doc 06 §6.2), **seek/streamGen change** (doc 06 §6.3), and **underrun** (both producer-starve and ALSA `-EPIPE`/`errUnderrun` — doc 06 §6.4). On a ring underrun the consumer does **not** push silence (that would advance counters and fight the reseek); it bumps `underruns` and lets the next tick observe the gross error (doc 06 §2.3, §6.4).

**Channel role / gain / HWDelay (doc 06 §5):** `"stereo"`=passthrough, `"left"`=src ch0→all outputs, `"right"`=src ch1→all outputs (§5.1). `g = 10^(GainDB/20)`, 0 dB short-circuits to 1.0 (§5.2). `HWDelayUs` applied **once** as an integer sample offset on the *want* side, not in the audio data (§5.3) — drift steers `playedContent` to meet it, so the trim holds exactly in steady state.

**Integration / acceptance — the real-plant convergence harness (doc 06 §3.6, A.4 "MUST model the real plant"):** a fake sink + fake timeline that model:
- DAC drains output frames at `f_out = Rate·(1 + crystalPPM·1e-6)` **independent of commanded ratio**;
- the ratio sets source→output coupling so `sourceConsumed` advances at `ratio·f_out`;
- `Delay()`/ring backlog **evolve from that drain**, fed back into `playedContent` (not free inputs).

Assertions (doc 06 §3.6): with constant `crystalPPM` (e.g. ±40 ppm), `errSamples → 0`, the **commanded ratio settles to `1 − crystalPPM·1e-6`** (NOT at the ±200 clamp), the integrator holds a finite non-clamped value, and **no reseek occurs after settling**. A test that nulls against an output-domain model or only checks "ratio<1 when error>0" MUST be rejected in review (doc 06 §3.6) — this is the regression guard for the mpvsync bug.

**Acceptance (A.13):** this piece is the core of **P4 sync (PCM)** — "one group plays a PCM stream in sync; drift loop holds" / acceptance: "measured inter-node offset stays sub-ms over 10 min." Per-node trim feeds **P6** ("per-node delay trim works"). Unit + harness tests must pass under `CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go vet ./...` and `go build`.

## 9. Risks / open questions

1. **`errUnderrun` sentinel ownership (P3.2).** Doc 06 §1.3/§6.4 says ALSA `-EPIPE` surfaces as `errUnderrun` (comparable via `errors.Is`) so the renderer reseeks immediately. The renderer must import that sentinel from `internal/audio/sink`. **Confirm P3.2 exports `sink.ErrUnderrun`** (exact name) — otherwise the renderer cannot distinguish a gross device underrun from a transient write error.
2. **Producer input seam.** Doc 06 scopes the recv-ring as upstream (P05). The producer needs a concrete reader for decoded canonical-rate frames. This spec keeps it behind a reader interface so `render` does not hard-import `stream/sink_net`; **the exact reader contract (blocking `Read([]float32)`? per-streamGen reset?) must be pinned with the P05 owner.** Provisional: a `FrameReader interface{ Read([]float32) (int, error); StreamGen() uint64 }` passed into `NewRenderer`.
3. **`Timeline` package location.** README §5 lists `internal/group` as the `Timeline` owner (doc 04 §4.4), but P4.4 is named in the dependency list. **Confirm whether `Timeline` lives in `internal/group` or a dedicated piece** so the import in §4 is correct. (No behavior impact; import path only.)
4. **`ratio_content_per_output` approximation.** Doc 06 §3.2 permits using `1.0` for the backlog→content conversion (sub-sample error at ≤200 ppm over a few-hundred-ms backlog). This spec uses the live `lastRatio`; using `1.0` is an acceptable simplification — pick one and keep it consistent between `played` and the harness model.
5. **`framesWritten` vestige.** The corrected loop no longer regulates `framesWritten`; it survives only for the coarse `Delay()` wall-time model via `lastWriteAt`. Confirm whether to drop the `framesWritten` counter entirely (keep `lastWriteAt` only) to avoid a maintainer mistaking it for the regulated quantity.
