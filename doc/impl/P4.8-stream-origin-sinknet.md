# P4.8 — `internal/stream/origin` + `internal/stream/sink_net` (master origin + follower receiver)

> Track: **BACKEND (Go)**. Status: **NEW code** (no equivalent realtime audio pipeline exists in
> `../media`; the mpvsync beacon/publisher idioms are consumed as *shapes only*, not copied).
> Owning doc: **05** (whole document; specifically §5.2 / §5.6 / §5.6.4 / §5.8 for the two halves).
> Pinned numbers: **Appendix A.12**; algorithms: **A.2** (timeline projection), **A.10** (FEC + receiver pipeline).
> Module path `gitlab.rand0m.me/ruben/go/ensemble`, Go 1.26, `CGO_ENABLED=0`, cross-compiles to
> linux/arm64 with **no** C toolchain (D24). `protocolEpoch = 1`.

## 1. Purpose & scope

This piece is the **realtime audio data plane** of a group: the two ends of the
decode-once / unicast-per-listener model (**D5**). The **origin** (`internal/stream/origin`)
runs on the group **master** and turns one looping `source.Reader` PCM stream into
timestamped, FEC-protected wire packets that it **paces** out, one unicast UDP send per
registered listener, ahead of each chunk's playout instant by the buffer lead (05 §5.2,
§5.6.1). It owns the listener registry (`AddListener`/`RemoveListener`), the `streamGen`
bump on media-change / seek / failover (05 §5.8), and the forced late-join keyframe
(05 §5.6.4). The **receiver** (`internal/stream/sink_net`) runs on every follower and is the
inverse: UDP recv → allowlist gate → `wire.Unmarshal` → `FEC.Recover` → reorder/dedupe by
`seq` → `Codec.Decode` → `ring.PushAt(sampleIndex, pcm)`, including `streamGen`-change
flush/re-prime and the late/duplicate/missing-packet concealment policy (05 §5.6.2/§5.6.3/§5.8).
This piece **hands off** to P3.1's ring at `PushAt` and never touches the `AudioSink`, the
drift loop, or channel/gain (those are doc 06 / other pieces). It **consumes** `Timeline`,
`ClockSource`, `Codec`, `FEC`, `wire`, `source.Reader`, and `allowlist.Set` — it implements
none of them.

## 3. Copy-from-`../media` plan

**NEW — copy no `.go` files.** A repo-wide search of `../media`
(`grep -riE "fec|streamgen|sampleindex|recover|reorder|unicast"`) finds nothing: mpvsync is a
*video-wall* that syncs an external `mpv` process by nudging its playback **speed** — it never
moves audio samples over the wire, has no codec/FEC/wire layer, and its "beacon" carries only a
float time-position, not PCM. The realtime audio data plane is entirely net-new (README §5 lists
all of `stream/*` under **New**).

Three upstream **shapes** are consumed (cited, not copied) so the new code matches proven
mpvsync idioms:

- **Subscribe/keepalive unicast push** — `../media/internal/sync/beaconnet.go`
  (`BeaconServer.Publish` fans one payload out to a `map[string]subscriber`, lines 72–87;
  `BeaconClient.Run` recv loop, lines 127–166). Reproduce the *fan-out-to-a-destination-map*
  pattern in `origin/sender.go`, but: destinations come from `AddListener`/`RemoveListener`
  (the group engine, not self-subscribe keepalives), there is **one socket per listener**
  (D5), and the payload is a marshaled `wire` packet, not a Beacon. Do **not** copy the file.
- **Master "publisher" loop driven by a clock-paired instant** —
  `../media/internal/sync/publisher.go` (`publishOnce` pairs the sampled position with a
  monotonic instant, lines 67–93). The origin's per-chunk loop is the analogue: each chunk
  stamps `MasterMono = clock.NowMono()` *at source time* (05 §5.2.2). Pattern only.
- **`Timeline.Now()` projection** — `../media/internal/sync/timeline.go` (lines 89–94) is the
  shape behind the canonical `Timeline.NowSample()` (§6.2 / A.2) that this piece **consumes**
  from P3.1; do not re-implement it here.

Everything the receiver decodes/recovers/pushes (`Codec`, `FEC`, `wire`, `ring.PushAt`,
`allowlist.Set`, `source.Reader`) is **provided by upstream pieces** (§8) and only consumed.

## 4. Public API (canonical names from README §6 / doc 05)

### 4.1 `internal/stream/origin`

```go
package origin

import (
	"context"
	"net"
	"time"

	"gitlab.rand0m.me/ruben/go/ensemble/internal/clock"
	"gitlab.rand0m.me/ruben/go/ensemble/internal/stream/codec"
	"gitlab.rand0m.me/ruben/go/ensemble/internal/stream/fec"
	"gitlab.rand0m.me/ruben/go/ensemble/internal/stream/source"
)

// Origin is the master-side stream origin (05 §5.2). One per playing group while this
// node is master. Not safe for concurrent Run; AddListener/RemoveListener/ResumeAt are
// safe to call concurrently with Run.
type Origin struct {
	tl    clock.Timeline // §6.2 NowSample — group playout timeline (P3.1)
	codec codec.Codec    // §6.3 — negotiated for the group (P4.1)
	fec   fec.FEC        // §6.3 — negotiated for the group (P4.2)
	src   source.Reader  // §5.3 — source decode (mp3/FLAC/WAV)→PCM, looped (P4.4)
	// gen, lead, framesPerChunk, rate, channels, listener registry, pacer: unexported.
}

// Config carries the per-group, per-run parameters (all from A.12 / the group profile).
type Config struct {
	Rate           int           // canonical rate, 48000 (A.12)
	Channels       int           // 2 (A.12)
	FramesPerChunk int           // 480 (A.12)
	Lead           time.Duration // buffer lead, 300ms (A.12 LeadMs)
	StreamGen      uint64        // starting generation (set by group engine on (re)start)
}

func New(tl clock.Timeline, c codec.Codec, f fec.FEC, src source.Reader, cfg Config) *Origin

// Run drives decode→chunk→encode→wire→FEC.Protect→paced unicast until ctx is cancelled
// or the node loses mastership (the group engine cancels ctx). 05 §5.2.
func (o *Origin) Run(ctx context.Context) error

// AddListener registers a unicast destination; on a join into a playing group the origin
// forces a keyframe on the next chunk for that generation (05 §5.6.4). Idempotent by id.
func (o *Origin) AddListener(id string, addr *net.UDPAddr) error

// RemoveListener drops a destination and closes its socket. Idempotent.
func (o *Origin) RemoveListener(id string) error

// ResumeAt is the failover/seek/media-change entry (A.14.4, 05 §5.8): it bumps streamGen,
// resets seq=0 + FEC state, sets the timeline anchor sampleIndex, preserves Playing, and
// forces a keyframe on the first chunk of the new generation. Returns the new streamGen.
func (o *Origin) ResumeAt(sampleIndex int64, playing bool) uint64
```

func (r *Receiver) Run(ctx context.Context, conn *net.UDPConn) error

> **Naming note (matches §6.3 / 05).** The receiver consumes `Packet` (the §6.3 type that
> `FEC.Recover` ingests/returns) and `wire.Header` / `wire.Unmarshal` / `wire.Marshal` from

> pcm []float32)` is the index-addressed push **owned by P3.1** (the plain SPSC `Ring` in

## 5. Behavior & algorithms (cite docs; A.12 numbers verbatim)

### 5.1 Canonical units (A.12, 05 §5.1)

Quoted verbatim from **A.12**: rate / channels / `FramesPerChunk` = **48000 / 2 / 480 (10 ms)**;
buffer playout lead `LeadMs` = **300 ms**; FEC XOR `k` / interleave depth `D` = **8 / 4
(~40 ms burst)**; FEC dup offset `Ddup` = **5 packets (~50 ms)**. Packet cadence is therefore
**100 packets/s/listener** (05 §5.1, 1000 ms ÷ 10 ms). `FramesPerChunk` is fixed for the
lifetime of a `streamGen` (05 §5.1). PCM payload is **S16LE, 1920 B/chunk** (480 × 2ch × 2B,
05 §5.11 / A.10 m5).

### 5.2 Origin loop (05 §5.2 / §5.2.2 / §5.2.3)

Per successive `FramesPerChunk`-frame PCM slice from `src.Read` (05 §5.2.1 — looping at EOF
does **not** reset `sampleIndex` nor bump `streamGen`; partial final chunk is padded with the
loop's leading frames so every wire chunk is exactly 480 frames):

```
payload, _ := codec.Encode(pcm)                 // §6.3; PCM narrows f32→s16 at the wire boundary (A.10 m5)
hdr := wire.Header{
    Magic:'ESND', Version:1,
    Flags: keyframe?,                            // bit1 (§6.4); see §5.4 below
    CodecID: codec.ID(), FECID: fec.ID(),
    StreamGen: o.gen, Seq: seq,
    SampleIndex: idx,                            // canonical-rate frame index (the playout anchor)
    MasterMono: clock.NowMono(),                 // master mono ns AT SOURCE TIME (05 §5.2.2)
    PayloadLen: len(payload), Rate100: rate/100, // 480 ⇒ 48000
}
pkt := wire.Marshal(hdr, payload)
out := fec.Protect(seq, pkt)                     // §6.3 → [source, (repair…)] — ONCE per chunk (D5)
schedule(out, playoutInstant(idx) - o.lead)      // pace per §5.3; fan out to every listener (D5)
seq++; idx += FramesPerChunk
```

`codec.Encode` + `fec.Protect` run **once per chunk**; only the per-socket `WriteToUDP` is
O(listeners) (D5, 05 §5.2.3). `playoutInstant(idx)` is derived from the consumed `Timeline`
(A.2): the origin owns `sampleIndex(0)` at start and advances `idx` by `FramesPerChunk`, so the
send time is `playout(idx) − Lead`. The first `idx` and the master-mono anchor come from
`Timeline.NowSample()` at `Run`/`ResumeAt` (05 §5.6.4 step 1).

### 5.3 Pacing (05 §5.6.1)

Packets are **not** blasted. A monotonic ticker drives ~one chunk every 10 ms; a **min-heap
keyed by send-time** holds the interleaved repair packets (whose send instant trails their
group's source packets per the §5.5.3 send order). Each due bundle is written to **every**
listener address (fan-out copy). Pacing keeps each follower's ring near its target fill and
avoids bursting WiFi. Send order for `k=8, D=4` is the round-robin A/B/C/D layout of 05 §5.10
with repairs `RA RB RC RD` interleaved after each group completes.

### 5.4 Keyframes & `streamGen` (05 §5.4.2 / §5.6.4 / §5.8)

- **PCM:** every chunk is independent ⇒ keyframe flag set on **every** packet (join path is
  uniform, 05 §5.4.1).
- **Opus:** keyframe (encoder reset) **every 50 chunks (~500 ms)** (05 §5.4.2), and **forced
  immediately** for a late joiner (§5.6.4) and on the **first chunk of a new generation** (§5.8).
- **`ResumeAt` / generation change** (05 §5.8, A.14.4): bump `gen++`; reset `seq=0` and FEC
  state; set `sampleIndex` to the supplied anchor (seek target / continuing position / on
  failover the current timeline `sampleIndex` with `Playing` preserved); force a keyframe on
  the first chunk. **Master failover always bumps `streamGen`** (D22 / R11) — there is no
  "continue same generation if reconstructable" path. A **loop boundary never bumps** it.
- **Late join** (05 §5.6.4): on `AddListener` into a playing group, begin unicasting to the new
  listener at the next chunk boundary at/after the current `sampleIndex` (no rewind); for Opus
  reset the encoder and emit the next chunk as a keyframe immediately.

### 5.5 Receiver pipeline (05 §5.6.2, A.10)

Per datagram, in order (05 §5.6.2):

```
1. recvfrom(conn) → (buf, srcIP)
2. allow.Allowed(srcIP)?  ──no──► DROP (no reply)            // P2.4 / 03 / A.8
3. wire.Unmarshal(buf) → (hdr, payload); validate magic 'ESND', version==1, payloadLen
4. hdr.StreamGen vs r.gen:
     >  : flush window+FEC+stale ring tail beyond new idx; r.gen = hdr.StreamGen; resume    // §5.8
     <  : DROP (stale prior-generation straggler)                                            // §5.8
     == : proceed
5. p := Packet{hdr, payload}
6. recovered := fec.Recover(p)                              // §6.3 — newly-decodable source pkts
7. for each recovered source packet:
     a. dedupe by seq (drop if already delivered)            // window
     b. reorder: insert into seq-ordered window
     c. when in-order / window slides past it:
          pcm := codec.Decode(payload)
          ring.PushAt(sampleIndex, pcm)                      // hand off to 06 (P3.1 ring)
```

**Reorder/dedupe window** (05 §5.6.2): bounded, keyed by `seq`, default **~32 packets ≈ 320 ms**
(`Config.WindowPackets`, default 32). It must exceed `D × k` (= 32 at defaults) so interleaved
parity groups can complete before aging out. Released in `seq` order; delivered `seq`s recorded
so duplicates (Dup mode or accidental) are dropped.

**Late / duplicate / missing policy** (05 §5.6.3 table — the cardinal rule: *the ring is always
filled at the correct `sampleIndex`*):

| Condition | Detection | Policy |
|---|---|---|
| Duplicate | `seq` already delivered/in window | Drop silently. |
| Reordered (early) | `seq` > expected, in window | Hold; release on gap-fill / slide. |
| Late | `sampleIndex` already played, or `seq` aged out | Drop — never push behind the play cursor. |
| Missing / unrecoverable | gap survives FEC, window slides past | **Conceal:** one chunk at that `sampleIndex` — Opus PLC if available, else short-fade **silence** for PCM (`conceal.go`). |

**`NowSample` is not implemented here** — projection lives in P3.1 (A.2). The receiver reads
`sampleIndex` from the header (05 §5.1: it never *computes* it from `seq`).

### 5.6 Buffer-lead relationship (05 §5.7)

The 300 ms lead (A.12) covers FEC recovery span (~40 ms at `k=8,D=4`) + WiFi jitter (~100 ms) +
decode/push slack (~20 ms) + drift headroom. The origin streams each chunk at
`playout(idx) − 300 ms`; the receiver's ring sits ~300 ms full at steady state. Underrun
handling (silence at `sampleIndex`, no catch-up) is the renderer's job (doc 06) — out of scope.

### 5.7 TCP fallback (05 §5.9) — minimal support, gated

When `ConfigDoc.Groups[].transport == "tcp"`, the origin writes length-prefixed (2-byte) `wire`
packets over a TCP conn per listener and `FECID = None` (Protect is identity, 05 §5.9); the
receiver runs the same pipeline minus FEC/UDP-reorder, still keying the ring by `sampleIndex`
and honoring `streamGen`. Pacing still applies. **Scope flag:** implement the writer/reader
plumbing behind the same interfaces; if the orchestrator scopes P4.8 to UDP-only for P4 (A.13
P4 is PCM/UDP), leave TCP behind a build-tested but un-wired path — see §9.

## 6. Import constraints (doc 01 §2 layering)

`internal/stream/origin` and `internal/stream/sink_net` are **leaf data-plane** packages. They
**may import**:

- `internal/stream/codec` (P4.1), `internal/stream/fec` (P4.2), `internal/stream/wire` (P4.3),
  `internal/stream/source` (P4.4) — sibling stream packages.
- `internal/clock` (P3.1, for `Timeline` + `NowMono`), `internal/audio/ring` (P3.1, for `*Ring`
  + `PushAt`).
- `internal/allowlist` (P2.4, `*Set` + `Allowed`) — receiver only.
- stdlib (`context`, `net`, `time`, `container/heap`, `math`).

They **must NOT** import: `internal/web` (the seam is one-directional; web reaches the engine
only via `web.Deps`, 01 §2), `internal/group` (the group engine imports *this*, not the
reverse — wiring flows down), `internal/state`, `internal/cluster`, `internal/pki`,
`internal/auth`, `internal/discovery`, or any `internal/audio/{render,sink,drift,resampler}`
package (the hand-off to render is via the ring only, 05 §5.0 boundary). No cgo, no dlopen
(D24) — this whole piece is pure-Go stdlib + sibling pure-Go packages.

## 7. Test plan

All tests pure-Go, no hardware, `go test ./internal/stream/origin/... ./internal/stream/sink_net/...`.
Codec/FEC/wire/source/ring/allowlist are exercised via the real upstream packages (or thin fakes
implementing their interfaces) so the contracts stay honest.

### 7.1 `origin` — table-driven unit tests (`origin_test.go`, `pacer_test.go`)

| Test | Asserts |
|---|---|
| chunking | a synthetic `source.Reader` of N+partial frames yields ⌈N/480⌉ chunks, **each exactly 480 frames** (partial padded with loop-leading frames, 05 §5.2.1); `sampleIndex` advances by 480/chunk; loop boundary does **not** reset idx nor bump gen. |
| header stamping | every emitted `wire.Header` carries `Magic='ESND'`, `Version=1`, `Rate100=480`, monotonic `Seq` from 0, `MasterMono` strictly increasing, correct `CodecID`/`FECID`. |
| keyframe (PCM) | with the PCM codec, **every** packet has the keyframe flag (05 §5.4.1). |
| keyframe-on-join | `AddListener` into a running origin forces the keyframe flag on the next chunk to that listener (05 §5.6.4); PCM = next chunk, Opus = encoder-reset keyframe. |
| `ResumeAt` / gen bump | `ResumeAt(idx,playing)` returns `gen+1`, resets `Seq` to 0, sets `SampleIndex=idx`, forces keyframe on first new-gen chunk (05 §5.8). Failover always bumps even for identical media. |
| fan-out count (D5) | with M listeners, `codec.Encode`/`fec.Protect` call counts are **independent of M** (counted via fake), while `WriteToUDP` count = M × packets (D5 §5.2.3). |
| Add/Remove idempotency | duplicate `AddListener(id)` / removing an unknown id are no-ops; removed listener stops receiving. |
| pacing order (pacer_test) | min-heap releases bundles in send-time order; send time = `playout(idx) − Lead`; repair packets ordered after their group's source packets per §5.5.3. |

### 7.2 `sink_net` — table-driven unit tests (`receiver_test.go`, `window_test.go`)

| Test | Asserts |
|---|---|
| in-order happy path | a stream of source packets → `ring.PushAt` called once per chunk with correct `sampleIndex` and decoded PCM length = 480×channels. |
| reorder | packets fed out of `seq` order are released to decode in `seq` order (window_test). |
| dedupe | a `seq` delivered twice (Dup mode) decodes/pushes **once**; second dropped (05 §5.6.3). |
| FEC recover | drop 1 source packet per parity group (k=8) → `fec.Recover` reconstructs it → ring gets the right chunk at the right index (uses real `fec` XOR, P4.2). |
| unrecoverable gap → conceal | a gap that survives FEC and ages out → exactly one **concealment** chunk pushed at that `sampleIndex` (silence/short-fade for PCM); subsequent audio is **not** shifted (cardinal rule, 05 §5.6.3). |
| late drop | a packet whose `sampleIndex` is behind the play cursor / aged past the window is dropped, never pushed (05 §5.6.3). |
| gen change (>) | `hdr.StreamGen > r.gen` flushes window+FEC+stale ring tail, adopts new gen, resumes from the keyframe packet (05 §5.8). |
| gen change (<) | `hdr.StreamGen < r.gen` packets dropped (stale straggler). |
| allowlist gate | a datagram from a disallowed source IP is dropped **before** unwrap/decode (use a `*allowlist.Set` that denies; assert no `ring.PushAt`). |
| bad packet | wrong magic / version / truncated payloadLen → dropped without panic. |
| `FlushAndReprime` | drops window+FEC+ring tail; next stream re-primes cleanly. |

### 7.3 Integration / acceptance

- **Loopback round-trip** (`origin` → real UDP `127.0.0.1` → `sink_net`): wire a real
  `codec.PCM` + `fec.XORParity` + a `*ring.Ring`; run the origin against a deterministic
  `source.Reader` (a generated test clip / sine), capture `ring.PushAt` calls, and assert the
  reassembled sample sequence is **bit-identical** to the source PCM with **zero gaps** under no
  loss, and **single-loss-per-group recovered** under induced 1/k drop.
- **Acceptance (A.13 P4 — sync PCM):** "one group plays a PCM stream in sync; drift loop holds."
  This piece's contribution: the receiver delivers correctly-indexed, gap-free PCM to the ring so
  the (separately tested) drift loop can hold sub-ms. The loopback round-trip + the
  no-gap/correct-index assertions are the unit-level proxy for the P4 acceptance.
- **Acceptance (A.13 P5 — codec+FEC):** "induced **2 % packet loss → no audible dropout**." The
  FEC-recover + conceal tests at 2 % uniform and bursty (≤ D) loss assert that no chunk is missing
  from the ring (recovered or concealed) and none is mis-indexed.

## 9. Risks / open questions

1. **`ring.PushAt` contract is owned by P3.1, not yet pinned.** This spec assumes
   `PushAt(sampleIndex int64, pcm []float32)` that (a) is idempotent/last-write-wins for a given
   index window and (b) tolerates pushes behind the play cursor by dropping them (so the receiver
   need not know the cursor). **Blocking if P3.1 instead expects the receiver to track the play
   cursor** — confirm whether "late drop" (05 §5.6.3) is enforced by the ring or by `sink_net`.
   *Orchestrator must resolve the `PushAt` signature + late-drop ownership with the P3.1 owner.*
2. **TCP-fallback scope for the P4 milestone.** A.13 P4 is PCM/UDP only; 05 §5.9 specifies TCP
   fallback. Recommend implementing the UDP path fully and stubbing TCP behind the same
   sender/receiver seam (compiles + unit-tested marshaling) but wiring it in a later phase. Needs
   a yes/no from the orchestrator so the `sender.go`/`receiver.go` interface is shaped right now.
3. **Opus on the critical path.** A.11 defers the libopus binding (purego/dlopen, off the default
   build). Origin/receiver must compile and pass all PCM tests with **no** Opus codec present;
   the keyframe-interval / encoder-reset logic is exercised via a fake codec advertising
   inter-frame state. Confirm P4.1 provides such a fake or a build-tagged Opus stub.
4. **Pacer clock source.** Pacing uses `time` ticker + send-time heap against the *master's*
   monotonic clock (origin is the reference, so no offset needed). Confirm the group engine never
   runs an origin on a node that is not the elected master (A.14.4 generation fence) — otherwise
   `playoutInstant` could be computed against a non-reference clock.
5. **Window size vs. deep interleave.** `WindowPackets=32` must always exceed `D×k`. If a future
   profile raises `D` or `k`, the window (and the 300 ms lead, 05 §5.7) must grow with it; the
   `Config` exposes `WindowPackets` so the group profile can drive it — flagged, not a blocker.
