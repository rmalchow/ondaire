## 1. Purpose & scope

P4.9 is the **convergence/wiring** piece: it turns the inert lifecycle skeleton (`internal/daemon`
from P0.3, whose `applyRole` had `TODO(P3/P4)` placeholders and whose `web.Deps` media/calibrate
closures returned `errNotImplemented`) into a running per-group audio engine. It (a) implements
the role-loop body that, per `04 §4.2` transitions and `A.14.4`, starts the **origin** (decode →
encode → FEC → unicast + local render) when this node is master/solo, or the **receiver + clock
follower + render chain** when it is a follower, fenced by `generation`/`streamGen`; (b) implements
the four media/transport HTTP handlers (`media`, `play`, `stop`, `calibrate/play`) and the
`groups/{id}/status` fan-out read in `internal/web`, all going through `Deps` closures; and (c)
fills the `cmd`/`daemon` `buildDeps` closures (`State`, `Transcodes`, `Discovery`, `Adopt`,
`Forget`, `Leave`, `SetNodeConfig`, `CalibratePlay`, plus the new media/play/stop/status fields)
binding them to the constructed `group.Engine`, `state.Store`, `cluster`, and `pki`. It owns **no
new realtime algorithm** — those live in P3.2/P4.7/P4.8; it owns the *orchestration glue and the
HTTP surface*.

## 5. Behavior & algorithms (cite docs; A.12 verbatim)

**B1 — `applyRole` is generation-fenced (A.14.4, 01 §4.2).** On every membership/election/config
tick the role loop calls `applyRole(electedMaster, generation)`. If `!changed && !force` it returns
(P0.3 shape). On change it **cancels the old role ctx** (old goroutines unwind — origin send loop or
follower recv/clock/render stop), then starts the new role under a fresh `rctx` bound to
`election.Generation()`. A stale role goroutine that wakes after a master change is dropped because
its generation no longer matches (01 §4.2). This is the audio re-point of mpvsync's identical guard.

**B2 — master/solo construction (04 §4.2 T1/T2/T4/T8/T10; A.14.4).** When `electedMaster == self`:
stop any follower parts; start `clock.Server` (clock plane, default UDP **`:9000`** — A.12 Ports);
have `group.Engine` own/seed the `Timeline`; start `stream.Origin` which decodes the selected media
(via P4.7 `stream/source`, mp3→PCM, loop), encodes per the negotiated `Profile` codec, applies FEC,
and unicasts one chunk stream per `Render=true` listener (audio plane, default UDP **`:9100`**).
**Local render runs only if `Caps.Render`** (D17, 04 §4.2 "master, no local render"): a sink-less
master origins+clocks but starts no `audio.Renderer`. On **promotion** (was follower, T4/T8) call
`Timeline.Seed(lastNowSample, GroupRecord.Playing)` for continuity (04 §4.4.4) and **bump
`streamGen`** so followers discard stale chunks (04 §4.3.3); `origin.ResumeAt(sampleIndex, Playing)`
(A.14.4). A `solo` group is a `master` with an empty listener set (04 §4.2) — identical construction,
no network listeners.

**B3 — follower construction (04 §4.2 T3/T5/T7; A.14.4).** When `electedMaster != self` **and**
`Caps.Render=true`: stop any master parts; start `clock.Follower.Run(masterAddr)` pinging the elected
master (A.1; per-environment estimator params — wired **`window=8, alpha=0.15`**, WiFi **`window=16,
alpha=0.10`**, ping **`1 s` (`500 ms` initial lock)**, RPC **`timeout=200 ms`** — A.12 Clock, set by
P3.2, not P4.9); start `stream.Receiver` (recv→allowlist→`FEC.Recover`→`Codec.Decode`→ring) and
`receiver.FlushAndReprime()` (~one buffer on (re)start — R11/A.14.4); build the `ring`/`drift`/
`render` chain (P4.8). The render loop calls `Timeline.NowSample()` (04 §4.4.3 derivation A) to pick
the sample due now. A `Caps.Render=false` non-master member takes the **control-only `member`
posture** (04 §4.2.4): `clock.Follower` for liveness only, **no** receiver/decode/ring/render. Sync
degraded ⇒ `orphan`: render held/muted (04 §4.2.3) — P4.9 just doesn't start render; the hold/fade is
P4.8's.

**B4 — buffer/FEC/drift tunables are *consumed*, never reinvented (A.12).** P4.9 passes through the
canonical numbers when constructing the planes: audio canonical **rate 48000 / channels 2 /
FramesPerChunk 480 (10 ms)**; playout lead **`LeadMs = 300 ms`**; drift PI **Kp 0.05 / Ki 0.005 /
MaxPPM 200 / IntegralClamp 200**, **tick 20 ms / HardErrSamp 2400 samples (50 ms)**; FEC XOR **k 8 /
interleave D 4**, dup **`Ddup 5`**. These are owned by P3.2/P4.7/P4.8 defaults; P4.9 only threads
operator overrides (e.g. `--device`) and the group `Profile`.

**B5 — F.2 select-media (08 §F.2).** Writes `GroupRecord.media = {file, loop}` via `state.Apply`
(requires `If-Match`; **412** if absent, **409 version_conflict** on mismatch — 08 §0.5). The file
must exist on the group's **master** (master-side decode, D14): the existence check is **proxied to
the master** over mTLS; absent ⇒ **404 not_found** with the node named (08 §F.2). Non-`.mp3` ⇒ **422
unprocessable**. Returns the new doc → `version` + `ETag`.

**B6 — F.3 play / F.4 stop (08 §F.3/§F.4, R4b).** Both flip the replicated **`GroupRecord.Playing`**
bool (separate from `Media{File,Loop}` — README §6.5) via `state.Apply` (**If-Match required** — even
a pure play, since it mutates `Playing`). Play with no media selected ⇒ **409 conflict**. After the
config write+gossip, **fan out** the start/stop to the group master over mTLS so audio begins/halts
without waiting for gossip convergence (08 §F.3/§F.4 "+ fan-out to master"); master unreachable ⇒
**502 proxy_failed**. The master's role loop, on seeing `Playing` change (gossip *or* fan-out), runs
B2: play begins decode→encode→FEC→unicast on a **fresh `streamGen`** (08 §F.3, §6.4); stop halts the
origin loop. F.3's optional `{file,loop}` body = F.2-then-play (select then flip) in one shot.

**B7 — F2.1 calibrate-play (08 §F2.1, A.10b).** Body carries **exactly one** of `groupId`/`nodeIds`
plus `durationSec` (neither/both ⇒ **400**). `Deps.CalibratePlay` **fans out** synchronous playback
to the selected nodes over mTLS. The signal is **generated in-process at the canonical rate** (no
external file) — A.10b: per 1-second period, **`[0, ~1 ms)` full-scale click + `[~1 ms, ~201 ms)`
1 kHz sine + `[~201 ms, 1 s)` silence**, bit-identical across nodes. A `Render=false` node cannot
play and is reported in `warnings[]` (not fatal). **No `If-Match`** (transient; no ConfigDoc write).
The waveform generator itself lives in `audio/render` or a small `audio` helper (P4.8); P4.9 wires the
fan-out + per-node start and aggregates `playedOn`/`warnings`.

**B8 — G.2 group status (08 §G.2).** **Fan-out read**: the receiving node aggregates each member's
live state over mTLS (each member knows its own `offsetUs`/`driftRatio`/`underruns`/`syncErrorUs`
relative to the master); the master is authoritative for `masterNodeId`/`profile`/`streamGen`.
`syncErrorUs` is the content-domain sync error (06), `offsetUs` the clock offset (04), `driftRatio`
the resampler ppm ratio (06 drift PI), `underruns` the ring underrun count (06), `clockQuality` ∈
`good|fair|poor` keyed off `MinDelay()` (04 §4.1). Group not yet synced/playing ⇒ **503 not_ready**
(members may report `online:false`/omit live fields); a single member unreachable ⇒ reported
per-member, **not** a top-level error, unless the master is unreachable (**502**). Read-only, no
`If-Match`.

**B9 — Deps seam discipline (01 §2 r1, A.14.3).** All five web handlers reach the engine **only**
through `Deps` closures. `internal/web` imports just `auth`/`pki`/`cluster`/`state`/`config` — never
`group`/`stream/*`/`audio/*`. `internal/daemon` (the bridge, `cmd`-level) is the only constructor of
`web.Deps` and the only place that imports both `web` and `group`/`stream`/`audio`. This is the
mpvsync `Deps` pattern verbatim, re-pointed from `mpv`/transport to the audio engine.

## 6. Import constraints (01 §2 layering)

- **`internal/web/*` (api_media.go, api_calibrate.go, api_groupstatus.go):** MAY import `state`
  (view types only, as P0.2 already does), `auth`, `pki`, `cluster`, `config`, stdlib, `coder/
  websocket`. **MUST NOT** import `group`, `stream/*`, `audio/*` (01 §2 r1 — the load-bearing
  invariant). All engine access is via `Deps` function values.
- **`internal/daemon/*` (role.go, media.go, deps.go):** the orchestrator — MAY import *everything*
  (01 §2 r6): `web`, `group`, `clock`, `stream/origin`, `stream/sink_net`, `stream/source`,
  `audio/render`, `audio/sink`, `cluster`, `state`, `pki`, `config`, `allowlist`. It is imported by
  **nothing** below it.
- No realtime package (`stream/*`, `audio/*`) imports `web`/`daemon`/`cmd` (01 §2 r7).
- `group` remains the only package importing both the realtime stack and `state` (01 §2 r5); P4.9
  drives it from `daemon`, it does not add edges into `group`.
