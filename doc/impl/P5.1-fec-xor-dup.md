## 1. Purpose & scope

`internal/stream/fec` is the group's **forward-error-correction layer**: it implements the
two locked schemes behind the single spine `FEC` interface (README ¬ß6.3, 05 ¬ß5.5) ‚Äî **XOR-parity**
(`FECID = XORParity = 1`, the default; `k = 8` source packets per parity group,
**time-interleaved** across `D = 4` concurrent groups so a burst hits ‚â§1 packet per group,
with per-group zero-pad-to-max-len on protect and length-trim on recover) and **packet
duplication** (`FECID = Duplicate = 2`; each source packet re-emitted `Ddup = 5` packets
later, receiver dedupes by `seq`) ‚Äî plus the trivial **identity** passthrough
(`FECID = None = 0`) used for the TCP fallback (05 ¬ß5.9) and `fec=none` profiles. The master
calls `Protect(seq, pkt)` once per source packet (single encode, fan-out copy ‚Äî D5/05 ¬ß5.2.3)
and unicasts every returned packet; the follower calls `Recover(p)` per received packet and
gets back any source packets that just became decodable (XOR reconstruction or the
just-arrived source/duplicate, deduped). It is **pure compute** ‚Äî no I/O, no goroutines, no
clock ‚Äî so it is a leaf of `stream/*`, unit-testable in isolation, and cheap on the
reference quad-A53 (one XOR pass to protect/recover; a hash-set membership test to dedupe ‚Äî
A.10, 05 ¬ß5.5.2). **No Reed‚ÄìSolomon** anywhere (D4, 05 ¬ß5.5.5).

## 4. Public API (canonical names ‚Äî README ¬ß6.3 / 05 ¬ß5.5)

```go
package fec

// FECID is the on-wire FEC scheme id (README ¬ß6.4 header offset 7; ¬ß6.3 enum).
type FECID uint8

const (
    None      FECID = 0 // identity passthrough (TCP fallback 05 ¬ß5.9 / fec="none")
    XORParity FECID = 1 // XOR-parity, k=8, interleave D=4 (default, D4)
    Duplicate FECID = 2 // packet duplication, Ddup=5 (MCU-friendly alt)
)

// Packet is the FEC unit of work: a parsed wire packet (header fields + payload).
// It is OWNED BY internal/stream/wire (P4.3, 01 ¬ß2.1) and consumed here verbatim so the
// FEC.Recover signature in README ¬ß6.3 type-checks. fec imports wire (both are leaves;
// the edge fec -> wire is acyclic ‚Äî see ¬ß6). See ¬ß9 open-question #1 if P4.3 instead
// places Packet in fec.
//   type Packet = wire.Packet
// wire.Packet exposes (consumed read-only here):
//   Seq()        uint64   // header offset 16 ‚Äî monotonic source seq
//   IsRepair()   bool     // header flags bit0 (¬ß6.4) ‚Äî repair vs source packet
//   PayloadLen() int      // header offset 40 ‚Äî true (un-padded) payload length
//   Bytes()      []byte   // the full marshaled packet (header+payload), for fan-out
//   Payload()    []byte   // the payload slice (len == PayloadLen for source pkts)

// FEC is the spine interface (README ¬ß6.3) ‚Äî implemented verbatim by every scheme.
type FEC interface {
    ID() FECID
    // Protect groups source packets and emits source+repair packets to send.
    // pkt is one fully-marshaled source wire packet (header+payload). The returned
    // slice is [source, (repair‚Ä¶)] in SEND order; produced ONCE per chunk and fanned
    // out to every listener by the origin (D5, 05 ¬ß5.2.3). Caller owns/sends the bytes.
    Protect(seq uint64, pkt []byte) (out [][]byte)
    // Recover ingests one received packet; returns any newly-recoverable SOURCE packets
    // (incl. the just-received one if it is itself a source packet), each as a Packet
    // with its payload trimmed to PayloadLen. Returns nil if nothing newly decodable.
    Recover(p Packet) (recovered []Packet)
}

// New constructs the scheme for an id (used by cmd/group when resolving the negotiated
// profile string -> concrete FEC, 01 ¬ß2.2 note). Unknown id -> None (safe floor).
func New(id FECID) FEC

// NewXOR / NewDup / NewNone are the explicit constructors (New dispatches to these).
func NewXOR(cfg XORConfig) FEC
func NewDup(cfg DupConfig) FEC
func NewNone() FEC

// Name<->id registry (README ¬ß6.5: JSON/profile uses string enums "none"|"xorParity"
// |"duplicate"; integer FECID exists ONLY at the ¬ß6.4 wire layer). group/state store the
// STRING; cmd maps to FECID here at wiring time.
func NameToID(name string) (FECID, bool) // "none"/"xorParity"/"duplicate" -> id
func IDToName(id FECID) (string, bool)
```

## 5. Behavior & algorithms (cited; A.12 numbers quoted verbatim)

### 5.1 Identity / `None` (05 ¬ß5.5, ¬ß5.9)
`Protect` returns `[]byte{pkt}` (the single source packet, unchanged). `Recover` returns the
one source packet as-is (payload trimmed to `PayloadLen`). This is the TCP-fallback mode
(05 ¬ß5.9: *"FEC is disabled (`FECID = None`) over TCP ‚Ä¶ `Protect` is the identity"*) and the
`fec="none"` profile.

### 5.2 XOR-parity (`FECID = 1`, default ‚Äî 05 ¬ß5.5.1 / ¬ß5.5.3, A.10)
Pinned tunables, **A.12 verbatim**: *"FEC XOR | k / interleave depth D | 8 / 4 (~40 ms burst)."*

**Protect (master).** Source packets are assigned **round-robin** to `D = 4` interleaved
parity groups by sequence: `group(seq) = seq mod D` (05 ¬ß5.5.3 diagram ‚Äî
`seq 0‚ÜíA,1‚ÜíB,2‚ÜíC,3‚ÜíD,4‚ÜíA,‚Ä¶`). For each group the encoder accumulates the **XOR of the k=8
covered source payloads**, **zero-padding each shorter payload to the group's running max
payload length** before XOR (05 ¬ß5.5.1: *"the encoder zero-pads the shorter payloads within a
group to the group's max payload length and stores `payloadLen` per source packet so the
receiver can trim after reconstruction"*). For PCM every chunk is a fixed 1920 B so padding is
a no-op; padding matters only for Opus/variable payloads ‚Äî but it is always implemented.

```
state per scheme: D parity accumulators; each accumulates {xor []byte, count int, maxLen int,
                  firstSeq uint64} for the currently-open group on that interleave lane.
Protect(seq, pkt):
    out := [pkt]                                  // always emit the source packet first
    g   := seq mod D                              // 05 ¬ß5.5.3 interleave assignment
    acc := lane[g]
    if acc.count == 0 { acc.firstSeq = seq }
    xorInto(acc.xor, payloadOf(pkt), growTo=max(acc.maxLen, len(payload)))  // zero-pad both
    acc.maxLen = max(acc.maxLen, len(payload)); acc.count++
    if acc.count == K {                           // group complete -> emit repair
        repair := wire.MarshalRepair(streamGen, repairSeq(acc.firstSeq), acc.maxLen, acc.xor)
        out = append(out, repair)                 // flags bit0=1; payloadLen=acc.maxLen
        reset(acc)                                 // restart this lane cleanly (05 ¬ß5.8 on gen change)
    }
    return out
```
The repair packet's covered seqs are **contiguous within the lane** and **derivable**
(05 ¬ß5.5.1: *"the repair carries the list of covered seqs implicitly (contiguous:
`[firstSeq ‚Ä¶ firstSeq+k-1]`)"*); concretely with interleave the lane's covered source seqs are
`{firstSeq, firstSeq+D, firstSeq+2D, ‚Ä¶, firstSeq+(k-1)D}`. The repair's own header carries
`payloadLen = maxLen` and `flags bit0 = 1` (05 ¬ß6.4, ¬ß5.10 repair-packet diagram). **Marshaling
the repair header is `wire`'s job** (P4.3 exposes a `MarshalRepair`/equivalent helper);
`fec` supplies the XOR payload, the covered-range anchor, and the lengths ‚Äî see ¬ß9 open-q #2.

**Recover (follower).** Per packet:
- **Source packet** (`!IsRepair`): trim payload to `PayloadLen`, return it immediately
  (it is itself decodable), and **register it** in its lane's group slot keyed by
  `(firstSeq-of-its-group, position)`. If that registration completes a group that was
  waiting on this packet *and* a repair already arrived, no extra recovery is needed (the
  source itself is the data).
- **Repair packet** (`IsRepair`): store it against its lane group `[firstSeq ‚Ä¶ firstSeq+(k-1)D]`.
- **On any arrival, attempt reconstruction** of a lane group that now has **exactly one
  missing member among its k source packets + 1 repair** (k of k+1 present): the missing
  payload = XOR of all present members (05 ¬ß5.5.1: *"recover S2 = S0‚äï‚Ä¶‚äïS7‚äïR"*), then **trim
  the reconstructed payload to the missing source's stored `payloadLen`** and emit it as a
  recovered `Packet` (synthesizing its header from the known `seq`/`sampleIndex`/`streamGen` ‚Äî
  see ¬ß9 open-q #2 for whether the receiver or fec reattaches the header). **Two losses in
  one group are unrecoverable** (05 ¬ß5.5.1) ‚Äî emit nothing for that group; the receiver
  conceals (05 ¬ß5.6.3). A group ages out of the recovery window once the play cursor passes it.

**Burst math (A.10 / 05 ¬ß5.5.3, quoted):** *"recoverable burst ‚âà D packets ‚âà D √ó
FramesPerChunk/rate = 4√ó10ms = 40 ms"*, and *"buffer lead must exceed the recovery span:
LeadMs (300) >> 40 ms ‚úì."* A contiguous burst of up to `D = 4` lost packets touches each of
the 4 lanes at most once ‚áí all recoverable; ‚â•2 losses in one lane ‚áí that lane's group is lost.
**Overhead = 1/k = +12.5%** (05 ¬ß5.5.1 table, A.10).

### 5.3 Duplication (`FECID = 2`, MCU-friendly ‚Äî 05 ¬ß5.5.4, A.10)
Pinned tunable, **A.12 verbatim**: *"FEC dup | offset `Ddup` | 5 packets (~50 ms)."*
(Note: A.12 pins `Ddup = 5`; 05 ¬ß5.5.4's prose default of `D=4` is an illustrative example ‚Äî
**use the A.12 value 5**; Appendix A is the single source of truth per the hard invariants.
Flagged in ¬ß9 open-q #3.)

**Protect (master).** Emit the source packet now, and schedule its **duplicate to be emitted
`Ddup = 5` packets later** in the send order (05 ¬ß5.5.4: *"the duplicate offset by D packets
in the send order"*; A.10: *"the copy offset by `Ddup` packets"*). Implemented with a small
FIFO delay line of depth `Ddup`: `Protect` returns `[source]` plus, if the delay line is
full, the duplicate of the packet that entered `Ddup` calls ago. Zero decode math; cost =
**+100% bandwidth** (05 ¬ß5.5.4 table ‚Äî *"each packet sent twice"*).

```
state: ring of last Ddup source packets (queue).
Protect(seq, pkt):
    out := [pkt]
    if len(queue) == Ddup { out = append(out, queue.pop()) }  // emit the displaced duplicate
    queue.push(pkt)
    return out
// (At end-of-stream/gen change the tail of `queue` is flushed; 05 ¬ß5.8.)
```

**Recover (follower).** Keep a bounded **seen-`seq` set** (the dedupe window, 05 ¬ß5.6.2 ‚Äî
*"a small seen-`seq` window ‚Ä¶ drops duplicates; the first copy to arrive wins"*). On arrival:
if `seq` already seen ‚áí return `nil` (drop the duplicate); else mark seen, return the source
packet (trimmed to `PayloadLen`). A burst shorter than the `Ddup` offset is fully covered
because the duplicate is displaced ~`Ddup √ó 10 ms` in time (A.10: *"Ddup = 5 packets (~50 ms
burst tolerance)"*; 05 ¬ß5.5.4). The window is bounded (see ¬ß5.4) and ages out by `seq` so it
is O(1) memory.

### 5.4 Window sizing & `streamGen`/reset
- The XOR recovery window and the Dup dedupe set must both **exceed `D √ó k`** so interleaved
  groups complete before aging out (05 ¬ß5.6.2: *"window size must exceed `D √ó k`"*; default
  *"~32 packets ‚âà 320 ms"*). With `D=4, k=8` that is `‚â• 32`; size the internal lane buffers to
  cover at least `D √ó k = 32` source packets plus the `Ddup=5` dup displacement.
- **`streamGen` change / reset is owned by the receiver** (P4.8, 05 ¬ß5.8): on a new generation
  the receiver discards its `fec.FEC` recovery state and the origin restarts `seq` at 0 with
  fresh FEC state (05 ¬ß5.8 step 2: *"Reset the `seq` space ‚Ä¶ and the FEC state (parity groups /
  dedupe offsets restart cleanly)"*). This package therefore exposes construction-cheap state
  and **a `Reset()` is desirable** (see ¬ß9 open-q #4) so the receiver can flush without
  reallocating; absent that, the receiver simply re-`New`s the scheme.

### 5.5 Performance (README ¬ß8, A.10)
- **Protect/recover are a single XOR pass** over ‚â§k payloads (XOR) or a hash-set membership
  test (Dup) ‚Äî *"trivial compute, dumb-node-friendly"* (05 ¬ß5.5.2). Reuse preallocated
  per-lane accumulator buffers; **no per-packet allocation** in the steady state (XOR into a
  reused `[]byte`, copy out only when emitting a repair/recovered packet). The Dup FIFO holds
  `Ddup` slices by reference. This keeps the master's per-listener cost to the kernel send +
  the copy (D5) and the follower's cost negligible on a quad-A53.
- XOR is implemented with `subtle.XORBytes` (Go 1.20+ stdlib `crypto/subtle`) for a
  word-at-a-time loop, or a hand-rolled 8-byte-stride loop ‚Äî both pure-Go, arm64-clean.

## 6. Import constraints (01 ¬ß2.1 / ¬ß2.2)

Per the ¬ß2.1 package table, **`stream/fec` lists internal-imports `(none)`** ‚Äî it is a pure
leaf. The one allowed edge is `fec ‚Üí wire` **iff** `Packet` is owned by `wire` (P4.3),
because README ¬ß6.3's `FEC.Recover(p Packet)` references the `Packet` type and 01 ¬ß2.1 assigns
`Packet` to `stream/wire/`. `wire` is itself a `(none)`-internal-import leaf, so `fec ‚Üí wire`
is acyclic and consistent with the ¬ß2.3 DAG (both sit under `stream/*`, below `group`). It may
**not** import `stream/codec`, `stream/source`, `stream/origin`, `stream/sink_net`, `group`,
`audio/*`, `clock`, `state`, `cluster`, `allowlist`, `web`, `pki`, `auth`, or `config`. It
uses only the stdlib (`crypto/subtle` for XOR). It is **driven by** `origin` (P4.8, master
`Protect`) and `sink_net` (P4.8, follower `Recover`), and **configured by** `cmd`/`group`
(profile string ‚Üí `New(NameToID(...))`); never the reverse ‚Äî keeping the graph acyclic and
the package unit-testable with zero engine wiring (rule 7: *"`stream/*` ‚Ä¶ are the lowest
layer; ‚Ä¶ driven by `group` and tested in isolation"*).

## 9. Risks / open questions

1. **`Packet` ownership (P4.3 vs P5.1).** README ¬ß6.3 puts `Packet` in the `FEC` signature;
   01 ¬ß2.1 assigns `Packet` to `stream/wire/`. This spec assumes **`wire` owns `Packet`** and
   `fec` imports it (`type Packet = wire.Packet`). If P4.3 instead defines `Packet` in `fec`
   (so `wire` stays a pure-leaf with no `Packet`), the alias flips and `wire` would need to
   import `fec` ‚Äî which would invert the edge. **Resolve with P4.3:** confirm `Packet` lives in
   `wire`. Proposed: `wire` owns it (it is the parsed-header product), `fec` imports `wire`.
   **Orchestrator should confirm before P4.3 and P5.1 are coded.**
2. **Repair-packet marshaling & recovered-header reattachment.** XOR `Protect` must emit a
   repair packet with a valid ¬ß6.4 header (`flags bit0=1`, `payloadLen=maxLen`, the lane's
   anchor `seq`), and `Recover` must hand back a reconstructed **source** packet with its
   original header (`seq`/`sampleIndex`/`streamGen`/`payloadLen`). Whether `fec` marshals
   these itself (needing a `wire.MarshalRepair` + `wire.SynthSource` helper) or whether the
   reconstructed *payload* is returned and `sink_net` reattaches the header is a **P4.3/P4.8
   contract decision**. Proposed: `wire` exposes `MarshalRepair(...)` and a way to clone a
   header with a substituted payload; `fec` returns full `Packet`s so the receiver path is
   uniform. **Confirm the `wire` helper surface with P4.3.**
3. **`Ddup` value conflict: A.12 says 5, 05 ¬ß5.5.4 prose says D=4.** A.12 (the single source
   of truth, hard invariant) pins `Ddup = 5 packets (~50 ms)`; 05 ¬ß5.5.4's `DupConfig` "default
   4" is an illustrative example reusing the XOR `D`. This spec uses **`Ddup = 5`** per A.12.
   Flagging so the doc prose can be reconciled ‚Äî no code impact (we follow A.12).
4. **`Reset()` on the `FEC` interface for `streamGen` changes.** 05 ¬ß5.8 requires FEC state to
   "restart cleanly" on a generation change. The README ¬ß6.3 interface has **no `Reset`**.
   Options: (a) the receiver re-`New`s the scheme on each `streamGen` bump (simple, a small
   alloc per seek/failover ‚Äî acceptable, these are rare), or (b) add a `Reset()` method
   (faster, but extends the canonical interface). Proposed: **(a) re-`New`** for MVP to keep
   the README ¬ß6.3 interface untouched; revisit if seek churn shows up in profiling.
5. **Repair send-ordering / pacing.** 05 ¬ß5.10 shows repairs interleaved into the send order
   *after* each group completes; `Protect` here returns the repair appended to the source
   packet's bundle at group-completion time, which the origin's paced sender (P4.8, 05 ¬ß5.6.1)
   schedules. If a stricter "repair must be paced to a specific send-time, not bundled with
   the k-th source" ordering is wanted, that pacing decision lives in `origin` (P4.8), not
   `fec` ‚Äî `fec` only decides *which* packets exist and in what relative order. Confirm with
   P4.8 that bundling-at-completion is acceptable.
