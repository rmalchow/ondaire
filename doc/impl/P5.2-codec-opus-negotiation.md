## 1. Purpose & scope

This piece fills the `OPUS` (`CodecID = 1`) slot that **P4.3 reserved** (where
`codec.New(OPUS)` currently returns `ErrUnsupportedCodec`), and closes the **full profile
negotiation** loop so Opus can actually be selected, originated, decoded, key-framed on
profile/generation change, and PLC-concealed on loss. The Opus codec is **optional and
capability-gated** (D3): it is compiled **behind a build tag** and bound to `libopus` via
**`purego`/dlopen** (A.11) so it is **completely absent from the default `CGO_ENABLED=0`
build** — when libopus is not loadable (the default), `codec.New(OPUS)` keeps returning
`ErrUnsupportedCodec`, the capability prober reports no `"opus"`, and negotiation cleanly
falls back to the PCM baseline (doc 04 §4.3.2). The piece also supplies the **keyframe /
encoder-reset extension interface** the origin type-asserts (the canonical
`Encode(pcm)` signature carries no keyframe arg) and the **PLC hook** the receiver's
concealment path type-asserts, plus the **codec capability prober** that P2.6 consumes to
self-write `EncodeCodecs`/`DecodeCodecs`. It does **not** redefine `NegotiateProfile`
(owned by P3.2), the `Codec` interface, `CodecID`, the wire format, or the engine state
machine.

> (the package P4.3 created) so it fills P4.3's reserved `OPUS` constructor slot through an
> in-package registration hook (§3/§4); it is **not** a separate `internal/stream/codec/opus`
> package (that would force the leaf `codec` package to import a child, inverting the

## 4. Public API (canonical names — README §6.3 / §6.5, doc 05 §5.4.2)

### 4.1 The codec itself (implements the P4.3 `codec.Codec` verbatim)

The Opus codec satisfies the **spine `Codec` interface unchanged** (README §6.3, P4.3
`codec.go`); no new method is added to `Codec`:

```go
// opus.go (build tag `opus`), package codec.

// opusCodec is the libopus-backed wire codec (CodecID = OPUS = 1). Unexported: callers
// obtain it only via codec.New(OPUS) / NewOpus, so the registry/build-tag gate is the
// single construction path. Each instance owns one libopus encoder + one decoder handle
// (NOT safe for concurrent Encode/Decode on the same instance — it carries encoder state;
// the origin encodes once per chunk on a single goroutine, doc 05 §5.2.3, so this is fine).
type opusCodec struct { /* enc, dec handles (uintptr from dlopen); rate, channels, frameSize */ }

func (c *opusCodec) ID() codec.CodecID                          // returns OPUS (1)
func (c *opusCodec) Encode(pcm []float32) ([]byte, error)       // one 10 ms chunk → one Opus frame
func (c *opusCodec) Decode(payload []byte) ([]float32, error)   // one Opus frame → FramesPerChunk PCM

// NewOpus constructs the Opus codec for the canonical rate/channels/frame size and the
// negotiated bitrate. Returns ErrUnsupportedCodec (P4.3 sentinel) if libopus is not
// loadable at runtime (dlopen failed) — so callers treat "no Opus" uniformly with the
// default build. Only compiled under the `opus` tag; the `!opus` stub defines no symbol.
func NewOpus(rate, channels, frameSamples, bitrate int) (codec.Codec, error)
```

### 4.2 Keyframe / encoder-reset extension (NEW, always compiled — `opus_iface.go`)

The canonical `Codec.Encode(pcm)` has **no keyframe argument**, yet doc 05 §5.4.2/§5.8
require the origin to *reset the Opus encoder* at a keyframe (periodic, on join, on new
generation) and the wire layer to *set the keyframe flag*. P5.2 introduces an **optional
extension interface** the origin **type-asserts** (PCM does not implement it; the origin's
keyframe logic stays uniform — P4.8 §5.4):

```go
// opus_iface.go (NO build tag — always compiled so origin/receiver can type-assert
// regardless of whether Opus is present; PCM simply does not implement these).
package codec

// KeyframeEncoder is implemented by codecs with inter-frame encoder state (Opus). The
// origin (P4.8) type-asserts the negotiated Codec to this; if absent (PCM), every chunk is
// already independent and no reset is needed (doc 05 §5.4.1).
type KeyframeEncoder interface {
    // ResetEncoder discards inter-frame prediction state so the NEXT Encode produces a
    // frame decodable cold. Called by the origin before encoding the first chunk of a new
    // streamGen, on a late-join, and on the periodic keyframe (doc 05 §5.4.2/§5.6.4/§5.8).
    ResetEncoder()
}

// PLCDecoder is implemented by codecs with native packet-loss concealment (Opus). The
// receiver's conceal path (P4.8 conceal.go) type-asserts the negotiated Codec to this; if
// absent (PCM), concealment is a short-fade silence chunk instead (doc 05 §5.6.3).
type PLCDecoder interface {
    // ConcealLoss synthesizes ONE chunk of FramesPerChunk concealment PCM for a single
    // unrecoverable lost packet, advancing the decoder's PLC state (Opus opus_decode with a
    // NULL frame). Returns exactly FramesPerChunk*channels interleaved float32.
    ConcealLoss() ([]float32, error)
}
```

`opusCodec` (under the `opus` tag) implements **both** extension interfaces:
`ResetEncoder()` calls `opus_encoder_ctl(OPUS_RESET_STATE)`; `ConcealLoss()` calls
`opus_decode_float` with a `nil`/zero-length frame (libopus PLC, doc 05 §5.4.2).

### 4.3 Registry / construction gate (fills P4.3's reserved slot — `opus_register.go`)

P4.3 builds the `OPUS` slot as "reserved, returns `ErrUnsupportedCodec`". P5.2 wires the
real constructor **only when present**, via a small in-package registration hook P4.3 must
expose (see §9 Q1):

```go
// In P4.3's codec.go (added/confirmed by P4.3 per §9 Q1):
//   var opusFactory func(rate, channels, frameSamples, bitrate int) (Codec, error)
//   // codec.New(OPUS) calls opusFactory if non-nil, else returns ErrUnsupportedCodec.

// opus_register.go (build tag `opus`), package codec:
func init() { opusFactory = func(r, ch, fs, br int) (Codec, error) { return NewOpus(r, ch, fs, br) } }
```

Under the default (`!opus`) build `opusFactory` stays `nil`, so `codec.New(OPUS)` returns
`ErrUnsupportedCodec` exactly as P4.3 specifies — **zero behavior change to the default
build**. `NameOf(OPUS)=="opus"` / `FromName("opus")==OPUS` remain true in *both* builds (the
*name* layer always knows Opus so negotiation/state can round-trip the string; P4.3 §5.3).

### 4.4 Codec capability prober (feeds P2.6 — `detect_codec_opus.go`)

P2.6's `detectCodecs()` reports `"pcm"` always and `"opus"` **iff the encoder/decoder are
available** (P2.6 §5.2, currently "deferred → absent for MVP"). P5.2 provides the runtime
detector P2.6 consumes through its existing `CodecProber` seam:

```go
// detect_codec_opus.go (build tag `opus`), package caps:
//   func init() { opusAvailable = probeOpus }   // P2.6 detectCodecs calls opusAvailable() if non-nil
//   func probeOpus() bool { return codec.OpusRuntimeAvailable() }  // dlopen libopus once, cache result

// In package codec (opus.go, build tag `opus`): exported probe used by caps.
func OpusRuntimeAvailable() bool   // true iff libopus dlopens AND a test encoder/decoder constructs
```

Under `!opus`, `opusAvailable` stays `nil` and `detectCodecs()` reports `"pcm"` only —
identical to P2.6's documented MVP default. **`EncodeCodecs` and `DecodeCodecs` get the same
answer** (this build links one shared libopus that does both encode and decode; there is no
encode-only/decode-only split for Opus here).

### 4.5 What is NOT (re)defined here

`codec.Codec`, `codec.CodecID`, `codec.PCM`/`OPUS`, `codec.New`, `NameOf`/`FromName`, the
sentinel errors → **P4.3**. `group.Profile`, `group.NegotiateProfile`, the `streamGen` bump
on profile change, the engine `applyRole`/role machine → **P3.2**. The wire header,
keyframe flag bytes, `wire.Packet` → **P4.1**. The origin's chunk loop / keyframe-flag
setting / per-listener fan-out and the receiver's reorder/conceal driver → **P5.1 (origin +
sink_net, the file P4.8 in this tree)**. P5.2 only provides the Opus `Codec` body, the two
extension interfaces, the registration/prober hooks, and the negotiation *consequences*
(§5).

## 5. Behavior & algorithms

### 5.1 Opus framing (doc 05 §5.4.2, A.12)

- **One Opus frame per chunk at 10 ms.** `FramesPerChunk = 480` (A.12: *"Audio canonical |
  rate / channels / FramesPerChunk | 48000 / 2 / 480 (10 ms)"*) at the canonical rate
  **48000 Hz**, **2 channels**. The Opus encoder frame size = `frameSamples = 480` per
  channel. The codec is configured for **`OPUS_APPLICATION_AUDIO`** (doc 05 §5.4.2 verbatim).
- **`Encode(pcm []float32)`**: `len(pcm)` must equal `frameSamples*channels` (= 960); else
  return `codec.ErrChunkAlloc` (reuse P4.3's sentinel — the chunker feeds exactly
  `FramesPerChunk`, doc 05 §5.1). Calls `opus_encode_float` into a reusable scratch buffer;
  output is the content-dependent Opus frame (tens to ~160 B at typical bitrates, doc 05
  §5.11). The keyframe *flag* is set by the wire/origin layer (P4.8 §5.4), **not** by
  `Encode` — `Encode` only resets prediction when the origin called `ResetEncoder()` first.
- **`Decode(payload []byte)`**: calls `opus_decode_float` for exactly `frameSamples` per
  channel; returns `frameSamples*channels` interleaved `float32`. Bad/short Opus frames →
  `codec.ErrShortPayload`.
- **Bitrate** is the negotiated profile bitrate (doc 05 §5.11 table shows @128k and @64k as
  typical music estimates); set via `opus_encoder_ctl(OPUS_SET_BITRATE)`. The default
  starting bitrate is a profile/UI value (doc 05 does not pin one number — §9 Q2); the codec
  takes it as a `NewOpus` argument so the policy lives upstream, not in the codec.

### 5.2 Keyframes & encoder reset (doc 05 §5.4.2 / §5.6.4 / §5.8)

- Opus carries **inter-frame state** (prediction, PLC warm-up). To bound loss cost and
  enable cold join, the origin marks a **keyframe** (wire flag bit1) and the Opus codec
  **resets the encoder** so that frame is decodable cold. Per doc 05 §5.4.2: *"Default
  keyframe interval **every 50 chunks (~500 ms)**; forced immediately for a late joiner."*
  The **50-chunk / 500 ms cadence and the join/new-generation forcing live in the origin**
  (P4.8 §5.4); P5.2's codec only exposes `ResetEncoder()` (§4.2), which the origin calls
  **before** the keyframe `Encode`.
- On a **`streamGen` change** (media change, seek, or **failover — always bumps**, doc 05
  §5.8 / D22): the origin bumps `gen`, resets `seq`, forces a keyframe on the first chunk →
  calls `ResetEncoder()` then `Encode`. The Opus codec needs no `streamGen` awareness itself;
  it just resets when told.
- A late joiner experiences **at most one keyframe-interval of startup delay** for Opus
  (until the forced reset frame), versus none for PCM (doc 05 §5.6.4) — this is a property of
  the origin's forced-keyframe-on-`AddListener`, satisfied because the codec resets on demand.

### 5.3 PLC concealment on loss (doc 05 §5.4.2 / §5.6.3)

- Per doc 05 §5.6.3, on an **unrecoverable loss** the receiver inserts **one chunk** of
  concealment at the missing `sampleIndex` (never shifting subsequent audio): *"Opus PLC if
  available (§5.4.2), else short-fade silence for PCM."* P5.2's `PLCDecoder.ConcealLoss()`
  (§4.2) produces that one chunk via libopus's native PLC (`opus_decode_float` with a NULL
  frame). The receiver's `conceal.go` (P4.8) type-asserts `PLCDecoder`; if absent (PCM) it
  uses the silence fade. The ring is always filled at the correct `sampleIndex` (the cardinal
  rule, doc 05 §5.6.3) — `ConcealLoss()` returns exactly one chunk regardless of codec.

### 5.4 Negotiation consequences — what "full negotiation" means here (doc 04 §4.3)

`NegotiateProfile` itself is **P3.2** (pure function over `Caps`). P5.2 makes Opus a *real*
negotiation outcome by satisfying every precondition doc 04 §4.3 assumes:

1. **Capability truth (doc 04 §4.3.1, §6.5).** Negotiation reads each `Render=true`
   listener's `Caps.DecodeCodecs` and the master's `Caps.EncodeCodecs`. Those lists are
   written by P2.6 from the codec prober (§4.4): Opus appears **iff** `OpusRuntimeAvailable()`
   on that node. So a group selects Opus **only when every listener can decode it and the
   master can encode it** (doc 04 §4.3.2 worked example: dropping dumb N4 flips
   `{"pcm","duplicate",…}` → `{"opus","xorParity",…}`). On the default build, no node
   advertises `"opus"` ⇒ negotiation always yields the PCM floor (doc 04 §4.3.2: *"PCM is
   always in every full node's DecodeCodecs/EncodeCodecs … the PCM floor admits future dumb
   nodes"*). **Graceful absence is automatic — no negotiation code special-cases it.**
2. **`codec.New(OPUS)` succeeds on the master/follower** that selected Opus, because the
   `opus`-tagged registration (§4.3) populated `opusFactory`. If a node *advertised* Opus but
   `New(OPUS)` later fails (e.g. libopus unloaded), the engine/origin treats it as an origin
   error and the next renegotiation drops Opus — but this cannot happen on a default build
   (the slot is never advertised).
3. **Profile change ⇒ `streamGen` bump ⇒ keyframe (doc 04 §4.3.3, doc 05 §5.8).** When
   membership/capability change flips the negotiated codec (e.g. `opus → pcm` because a dumb
   node joined, doc 04 §4.3.3 diagram), the **master publishes the new profile and bumps
   `streamGen`** (P3.2), reconfigures its `Codec` (constructs the new one via `codec.New`),
   and the origin forces a keyframe on the first chunk of the new generation. For
   `pcm → opus` the new Opus codec is reset (its first frame is a keyframe by construction).
   The **`sampleIndex` timeline is continuous across the bump** (doc 04 §4.3.3: *"renegotiation
   does not reset playout position — only the encoding changes"*) — at most one ring-refill of
   re-buffer. P5.2's contribution is that the Opus codec **constructs cleanly mid-stream and
   produces a cold-decodable first frame after `ResetEncoder()`**, which is what makes the
   generation switch graceful.
4. **Follower side (doc 04 §4.3.3).** A follower reads the published `profile`, configures its
   `Codec`/`FEC` decoders **before admitting the next `streamGen`**, drains its ring to the
   generation boundary, and resumes decoding the new generation's first (keyframe) chunk. The
   Opus decoder is freshly constructed for the new profile, so its first decode is the cold
   keyframe — no warm-up artifact.

### 5.5 `purego`/dlopen binding (A.11 / R7 — `opus_binding.go`, build tag `opus`)

- Binds `libopus.so.0` at **runtime** via `github.com/ebitengine/purego` (pure-Go; loads with
  `purego.Dlopen` and registers C functions with `purego.RegisterLibFunc`). **No cgo, no C
  toolchain at build time** — `purego` calls the dynamic linker at runtime. This is the
  *candidate* binding A.11 names (*"purego/dlopen is the candidate binding when pursued"*).
- Symbols bound (minimal set): `opus_encoder_create`, `opus_encode_float`,
  `opus_encoder_ctl` (for `OPUS_SET_BITRATE`, `OPUS_SET_VBR`, `OPUS_RESET_STATE`,
  `OPUS_SET_APPLICATION`/created with `OPUS_APPLICATION_AUDIO`), `opus_encoder_destroy`;
  `opus_decoder_create`, `opus_decode_float`, `opus_decoder_destroy`. The Opus C constants
  (`OPUS_APPLICATION_AUDIO=2049`, `OPUS_RESET_STATE=4028`, `OPUS_SET_BITRATE_REQUEST=4002`,
  …) are inlined as Go consts in `opus_binding.go` (they are stable ABI values).
- **Graceful absence:** `purego.Dlopen` failing (no `libopus.so.0` on the box), or any symbol
  missing, makes `OpusRuntimeAvailable()` return `false` and `NewOpus` return
  `codec.ErrUnsupportedCodec`. dlopen happens **once**, cached (a `sync.Once`), so per-codec
  construction is cheap.
- **Lifecycle:** `opusCodec` owns the C encoder/decoder handles (opaque `uintptr`). A `Close`
  (or finalizer-free explicit destroy when the origin/receiver tears down a generation) calls
  `opus_*_destroy`; the codec is constructed once per `streamGen` and destroyed when the
  generation ends, so there is no per-chunk allocation of C handles. **No cgo `runtime.Pinner`
  gymnastics on the hot path** — encode/decode pass Go `[]float32`/`[]byte` whose backing
  arrays are handed to `purego` as pointers for the duration of one call only (the slices are
  caller-owned and not retained by libopus).

### 5.6 Tunables quoted from A.12 (single source of truth)

- *"Audio canonical | rate / channels / FramesPerChunk | **48000 / 2 / 480 (10 ms)**"* — the
  Opus frame size is **480 samples/channel** (10 ms), matching the chunk.
- Keyframe interval: doc 05 §5.4.2 *"**every 50 chunks (~500 ms)**"* — enforced by the origin
  (P4.8), not hard-coded in the codec. (A.12 does not list a separate Opus-keyframe row; the
  500 ms figure is doc 05 §5.4.2's, quoted verbatim.)
- Bitrate: **not pinned in A.12** — doc 05 §5.11 gives @128k / @64k as *typical* estimates,
  not a locked default. Taken as a `NewOpus` argument; see §9 Q2.
- *"Buffer | playout lead `LeadMs` | **300 ms**"* — unchanged by Opus; the lead already
  contains the FEC span + jitter (doc 05 §5.7). Opus does not change the lead budget.

## 6. Import constraints (doc 01 §2.1 / §2.2)

- **`internal/stream/codec` (the package these files join) → imports:** stdlib only on the
  default build. Under the `opus` tag it additionally imports
  **`github.com/ebitengine/purego`** (pure-Go, A.11 candidate) and stdlib `sync`. It must
  **not** import `wire`, `fec`, `source`, `origin`, `sink_net`, `state`, `clock`, `group`,
  or `audio/*` — it stays a **leaf** (P4.3 §6). The extension interfaces in `opus_iface.go`
  reference only `codec`'s own types.
- **`internal/caps` → imports `internal/stream/codec`** for `OpusRuntimeAvailable()` (the
  `opus`-tagged `detect_codec_opus.go` only). This edge already exists in spirit in P2.6
  (caps detects wire codecs); P2.6 §6 lists `caps`'s deps including the codec name layer.
  Under `!opus` the stub adds no edge. `caps` must **not** import `group`/`stream/origin`.
- **`internal/web` imports neither** (doc 01 §2.2 rule 1: web reaches the engine only through
  the `Deps` seam) — satisfied automatically (web never imports `stream/*` or `caps`).
- Consumed **by** `internal/stream/origin` + `internal/stream/sink_net` (type-assert the
  extension interfaces) and referenced **by name** from `group`/`state` — never the reverse.

### 7.4 Negotiation integration (P3.2-owned, P5.2-exercised) — `internal/group/profile_test.go` row

P5.2 does not own `NegotiateProfile`, but adds the codec-availability fixtures so the doc 04
§4.3.2 examples are exercised with *real* `Caps` produced by the prober: the
`{pcm,opus}`-decoding listener set selecting `"opus"`, and the dumb-node-joins case
flipping back to `"pcm"` with a `streamGen` bump. These assert the **negotiation outcome**
(string codec name) and that the master's published `streamGen` increments on the codec
change — the integration realization of doc 04 §4.3.3 / A.13 P5.

> **End-to-end 2 % packet-loss acceptance (A.13 P5)** is owned by P5.1 (origin + sink_net)
> driving the real Opus codec + XOR FEC; P5.2 guarantees the Opus `Encode`/`Decode`/PLC the
> loss test relies on. Run that acceptance under `-tags opus` on a box with libopus + a DAC.

## 8. Dependencies

**Consumes (upstream):**
- **P4.3 `internal/stream/codec`** — the `Codec` interface, `CodecID`, `OPUS` constant,
  `New`, `NameOf`/`FromName`, the sentinel errors (`ErrUnsupportedCodec`, `ErrChunkAlloc`,
  `ErrShortPayload`), **and the in-package `opusFactory` registration hook** (§9 Q1 — P4.3
  must add it; without it the `OPUS` slot cannot be filled in-package).
- **P3.2 `internal/group`** — `Profile`, `NegotiateProfile`, the `streamGen`-bump-on-profile-
  change behavior, the `applyRole` loop that (re)constructs `codec.New(profile.Codec)` per
  generation. P5.2 supplies the codec that negotiation can now actually choose.
- **P2.6 `internal/caps`** — the `detectCodecs()` codec detector + `Mask.DisableCodecs`
  masking; P5.2 wires the runtime Opus prober into it (`opusAvailable` hook, §4.4).
- **P5.1 `internal/stream/origin` + `internal/stream/sink_net`** (the **P4.8** file in this
  tree — see §9 Q3 on the ID mismatch) — the origin type-asserts `KeyframeEncoder` and the
  receiver's `conceal.go` type-asserts `PLCDecoder`; the origin owns the keyframe cadence /
  forced-keyframe-on-join / new-generation logic. P5.2 supplies the two extension interfaces
  these assert and the Opus implementations of them.
- **A.11** — `github.com/ebitengine/purego` (the candidate Opus binding), added to `go.mod`
  **only as an indirect/optional dep guarded by the `opus` tag** (pure-Go; does not enter the
  default `CGO_ENABLED=0` link unless `-tags opus`). See §9 Q4.

**Exposes (for downstream pieces):**
- `codec.KeyframeEncoder`, `codec.PLCDecoder` (always compiled) — type-asserted by
  **P5.1** origin/receiver. These are the canonical hooks that make the origin's
  keyframe/concealment logic codec-agnostic.
- `codec.NewOpus`, `codec.OpusRuntimeAvailable` (under `-tags opus`) — the real Opus codec
  + the runtime prober, consumed by P4.3's `New(OPUS)` (via `opusFactory`) and by P2.6's
  `detectCodecs` (via `opusAvailable`).
- The behavioral guarantee that **the default build is byte-for-byte unchanged**: no Opus,
  no `purego` in the link, `New(OPUS)==ErrUnsupportedCodec`, negotiation falls to PCM.

## 9. Risks / open questions

- **Q1 (BLOCKING) — P4.3 must expose an in-package `opusFactory` registration hook.** P4.3 as
  written hard-codes `New(OPUS) → ErrUnsupportedCodec` with no extension point. P5.2 cannot
  fill the slot without either (a) P4.3 adding a package-level
  `var opusFactory func(rate,channels,frameSamples,bitrate int)(Codec,error)` that `New`
  consults (recommended — keeps the leaf `codec` package the single construction authority,
  set by the `opus`-tagged `init()`), or (b) moving the `New` switch into a build-tagged file.
  **Recommend (a).** The orchestrator must confirm P4.3's spec is amended to include the hook;
  otherwise P5.2 has no compile-time seam.
- **Q2 (non-blocking) — Opus default bitrate is not pinned in A.12.** doc 05 §5.11 lists
  @128k / @64k as *typical* estimates only. P5.2 takes bitrate as a `NewOpus` argument so the
  policy lives in the profile/UI (doc 04/09), not the codec. Orchestrator/UI track must decide
  the default (suggest **128000** for music on 5 GHz, lower on constrained links) and where it
  is stored (a `GroupRecord.Profile` field is **not** currently in P3.2's `Profile` struct —
  it has only Codec/FEC/Rate/FramesPerChunk; adding `OpusBitrate int` to `Profile` is the
  cleanest home but is a P3.2/P2.1 schema change, not P5.2's to make).
- **Q3 (naming) — "P5.1" in the dependency list = the origin/sink_net piece, which exists in
  this tree as `P4.8-stream-origin-sinknet.md`.** P5.2 consumes its `KeyframeEncoder`/
  `PLCDecoder` type-asserts and its origin keyframe cadence. The orchestrator must reconcile
  the ID (P5.1 vs P4.8) so the dependency graph is consistent; the *contract* (extension
  interfaces asserted by origin/receiver) is unambiguous regardless of the number.
- **Q4 (non-blocking) — `purego` enters `go.mod` even though the default build never links
  it.** Go records a build-tag-guarded import in `go.mod` as a normal `require` (the source is
  still compiled-checked by `go vet`/`go mod tidy` under all tags). F0 pinned only A.11 deps +
  `go-mp3`. Adding `github.com/ebitengine/purego` is the minimal addition; confirm it is
  acceptable to list it (it is pure-Go, MIT, no transitive C). The default release artifact is
  unaffected because the `opus` files are excluded from the default build.
- **Q5 — libopus ABI constants are inlined.** `OPUS_APPLICATION_AUDIO=2049`,
  `OPUS_RESET_STATE=4028`, the CTL request macros, etc. are hard-coded Go consts (stable ABI).
  Low risk (libopus ABI is frozen) but worth a one-time check against the target's
  `opus_defines.h` during the verification spike. The whole Opus path is, per A.11, a
  **verification spike** to prove on a Pi + DAC before relying on it in production.
- **Q6 — concurrency.** `opusCodec` is **not** safe for concurrent `Encode`/`Decode` on one
  instance (libopus encoder/decoder handles are stateful). This is correct for the pipeline
  (origin encodes once on one goroutine, doc 05 §5.2.3; receiver decodes on one goroutine) but
  differs from PCM's stateless concurrency-safe codec (P4.3 §5.1). Documented on the type; the
  origin/receiver must not share a single Opus codec across goroutines (they don't).
