## 1. Purpose & scope

This piece is the **generation-correctness layer** of the realtime audio path: the small,
load-bearing logic that keeps the stream sample-aligned across discontinuities. On the
**master/origin** it owns the `streamGen` *controller* — the rules for **when** to bump
(operator media change, operator seek, profile change, and **every** master change /
failover — D22/R11), and **never** on a loop boundary (05 §5.2.1) — and the
generation-reset effects (restart `seq`=0, reset FEC state, set `sampleIndex` to the seek /
continuing / failover-resume position, force a **keyframe** on the first chunk). On the
**follower/receiver** it owns the generation gate (`hdr.StreamGen` vs accepted `gen`):
**adopt-newer** (flush reorder window + FEC state + stale ring tail, resume at the new
keyframe `sampleIndex`), **drop-older** (stale stragglers), and the **late-join /
flush-and-reprime** startup (keyframe-first decode, accumulate one buffer lead, then play
sample-aligned from `Timeline`). It does **not** implement the origin send loop (P4.8), the
receiver recv/decode loop (P4.9), the wire codec (P4.3), FEC schemes (P4.5), the ring
(P4.4), or the Timeline/election (P3.2) — it implements the *correctness invariants those
loops call into*, as a focused unit-testable package plus the two integration hooks
(`OriginResumeAt`, `ReceiverFlushReprime`) the `applyRole` loop (A.14.4) already names.

## 4. Public API (canonical names — README §6.3/§6.4/§6.2; 05 §5.6.4/§5.8; 04 §4.4.4)

```go
package streamgen

// Reason classifies why the master bumps streamGen. Per 05 §5.8 + D22/R11 the bumping
// events are media change, seek, profile change, and EVERY master change/failover; a loop
// boundary is deliberately NOT a Reason (05 §5.2.1 — the timeline is continuous, just more
// PCM). There is intentionally no "loop" or "reconnect" reason.
type Reason int

const (
    ReasonMediaChange  Reason = iota // operator selected new media (05 §5.8; Groups[].media, 07)
    ReasonSeek                       // operator seek (05 §5.8) — FirstSampleIndex = seek target
    ReasonProfileChange              // codec/FEC/rate renegotiation (04 §4.3.3)
    ReasonMasterChange               // failover / takeover / rejoin-flip (D22/R11/A.5)
)

// Generation is the result of a bump: the values the origin stamps on the FIRST chunk of
// the new generation (README §6.4 header fields) plus the reset directives.
type Generation struct {
    Gen              uint64 // the new streamGen value (header offset 8)
    FirstSampleIndex int64  // sampleIndex of the first chunk of this generation (header offset 24)
    Keyframe         bool   // ALWAYS true: first chunk of any new generation is a keyframe (05 §5.8 step 4)
    Playing          bool   // carried/preserved play-state (failover reads replicated Playing — 04 §4.4.4 / R4)
    ResetSeq         bool   // ALWAYS true: new generation restarts seq at 0 (05 §5.8 step 2)
    ResetFEC         bool   // ALWAYS true: parity groups / dedupe offsets restart cleanly (05 §5.8 step 2)
}

// Controller is the master-side streamGen state machine (one per group origin). Not safe
// for concurrent use; the origin serializes Bump/Current with its send loop.
type Controller struct { /* gen uint64 — unexported */ }

func NewController(initialGen uint64) *Controller // initialGen=0 for a fresh group (lifecycle: Create => streamGen=0, 04 §4.6)

// Current returns the live generation counter (the value the origin stamps on chunks).
func (c *Controller) Current() uint64

// Bump advances the generation for `reason` and returns the directives for the first chunk
// of the new generation. atSample is the timeline position the new generation starts at:
//   - ReasonSeek         => the seek target sampleIndex
//   - ReasonMediaChange  => the continuing timeline position (or 0 on a fresh media start)
//   - ReasonProfileChange=> the CONTINUING sampleIndex (timeline is continuous — 04 §4.3.3)
//   - ReasonMasterChange => the current timeline sampleIndex at promotion (04 §4.4.4)
// playing is the play-state to preserve (failover reads the replicated GroupRecord.Playing).
// Bump always sets Keyframe=ResetSeq=ResetFEC=true (05 §5.8 steps 2 & 4).
func (c *Controller) Bump(reason Reason, atSample int64, playing bool) Generation

// ── Receiver-side generation gate ───────────────────────────────────────────

// Action is the receiver's decision for an inbound packet's StreamGen vs the accepted gen.
type Action int

const (
    Pass  Action = iota // hdr.StreamGen == gate.gen  => normal decode path (05 §5.8)
    Adopt               // hdr.StreamGen >  gate.gen  => flush + reprime, adopt new gen
    Drop                // hdr.StreamGen <  gate.gen  => stale straggler from a prior gen, drop
)

// Gate is the receiver-side generation gate (one per Receiver). Not concurrency-safe; the
// recv loop is single-goroutine (05 §5.6.2).
type Gate struct { /* gen uint64 — unexported */ }

func NewGate() *Gate // starts at gen 0; first Adopt locks onto the live generation

// Accept compares an inbound StreamGen to the accepted generation and returns the action.
// On Adopt the caller MUST flush (reorder window + FEC state + stale ring tail beyond the
// new sampleIndex) BEFORE decoding the packet, then the gate's gen is advanced. Accept
// itself only decides + (on Adopt) advances gen; the flush side effects are the caller's
// (Receiver.FlushAndReprime, §reprime.go) so this stays pure/testable.
func (g *Gate) Accept(gen uint64) Action

// Current returns the accepted generation (for the FollowerTimeline streamGen gate, 04 §4.4.3).
func (g *Gate) Current() uint64
```

Wiring methods added to the consumed packages (same-package files, P4.8/P4.9 own the types):

## 5. Behavior & algorithms

**When `streamGen` bumps (05 §5.8, D22/R11, 04 §4.3.3).** The master bumps on **media
change**, **seek**, **profile change** (04 §4.3.3), and on **every master change /
failover / takeover / rejoin-flip** (D22, R11, A.5 — "if master changed: generation++"). It
does **not** bump on a **loop** boundary (05 §5.2.1: the timeline is continuous, just more
PCM, so a loop is invisible to listeners). There is **no "continue the same generation if
reconstructable" path** (05 §5.8): a new master, even one resuming identical media, starts a
fresh generation — keeping failover unconditional and split-brain-safe rather than depending
on whether the new master can reconstruct the old `seq`/FEC state.

**Master generation-change effects (05 §5.8 steps 1–4, applied by `Origin.Bump`).**
1. `gen++` (`Controller.Bump`).
2. Reset the `seq` space (new generation starts at `seq = 0`) and FEC state (parity groups /
   dedupe offsets restart cleanly) — `ResetSeq`/`ResetFEC` directives.
3. Set `sampleIndex` to the operator-selected seek target (`ReasonSeek`), the continuing
   position (`ReasonMediaChange`/`ReasonProfileChange` — timeline continuous, 04 §4.3.3), or,
   on failover (`ReasonMasterChange`), the **current timeline `sampleIndex` with `Playing`
   preserved** (04 §4.4.4 — the new master reads replicated `GroupRecord.Playing`, calls
   `MasterTimeline.Seed(lastSample, Playing)`).
4. Force a **keyframe** (header flags bit1, P4.1) on the **first chunk** of the new
   generation so all listeners — including Opus, which carries encoder state — resync cold
   (05 §5.4.2 resets the Opus encoder at a keyframe; PCM is always a keyframe, 05 §5.4.1).

**Late-join keyframe-first (05 §5.6.4, A.10 "Late join").** When the group engine registers
a new listener mid-stream (`Origin.AddListener`, P4.8), the origin: (1) reads the current
play position from `Timeline.NowSample()` (does **not** rewind — joins the live timeline);
(2) begins unicasting at the **next chunk boundary at/after the current `sampleIndex`**; (3)
**forces a keyframe** for codecs with inter-frame state — PCM's next chunk already works
(every PCM packet is a keyframe), Opus resets its encoder and emits the keyframe
**immediately** rather than waiting for the periodic one (05 §5.4.2: default keyframe interval
**every 50 chunks (~500 ms)**). A late joiner therefore experiences at most one
keyframe-interval of startup delay (Opus) or none (PCM), plus the buffer-lead fill. (Per-join
keyframe forcing does **not** bump `streamGen` — it is a single keyframe flag for that
listener's cold start, not a generation change.)

**Receiver generation gate (05 §5.8, A.2 / 04 §4.4.3).** Per inbound datagram, after
`wire.Unmarshal` (05 §5.6.2 step 4): `Gate.Accept(hdr.StreamGen)`:
- `hdr.StreamGen == gen` ⇒ **Pass**: normal FEC.Recover → reorder/dedupe → Decode → ring.
- `hdr.StreamGen > gen` ⇒ **Adopt**: `FlushAndReprime()` (flush reorder window + FEC state +
  the **stale ring tail beyond the new `sampleIndex`**), set `gen = hdr.StreamGen`, then
  resume decoding from this (keyframe) packet at its `sampleIndex`. A newer generation
  **invalidates** buffered audio from the old one (a seek means the old queued samples are
  wrong), so flushing keeps a seek crisp instead of playing out ~300 ms of stale audio first.
- `hdr.StreamGen < gen` ⇒ **Drop**: stale stragglers reordered late from a previous
  generation; `streamGen` monotonicity disambiguates them.

The `FollowerTimeline` (P3.2) `streamGen` gate keys off `Gate.Current()`: `ok=false` until a
chunk of the current generation arrives (A.2: "`streamGen` mismatch ⇒ treat as not-synced
until a chunk of the current generation arrives") so render holds (orphan/silence) across a
bump rather than playing wrong audio.

**Flush + reprime (R11, D22, A.14.4).** `FlushAndReprime` is called (a) on `applyRole`
(re)starting the follower receiver (one buffer on (re)start, A.14.4) and (b) internally on a
`Gate.Adopt`. It: clears the reorder/dedupe window; resets FEC `Recover` state; drops the
ring tail that is past/ahead of the new `sampleIndex` (`ring.Reset()` on a full flush, or a
bounded trim); and re-enters the **prime** state — playout is withheld (FollowerTimeline
`ok=false` until a current-gen keyframe lands, then ring fills) until **one buffer lead**
(A.12 `LeadMs = 300 ms`) of keyframe-anchored audio is queued, matching 05 §5.6.4 step 4 and
§5.7. The **cardinal rule is preserved throughout**: the ring is always filled at the correct
`sampleIndex` (05 §5.6.3) — a generation change re-anchors the index, it never shifts
subsequent audio.

**Continuous `sampleIndex` (05 §5.1, 04 §4.3.3/§4.4.4).** The sample index is **authoritative
on the wire** (05 §5.1: "the receiver never *computes* `sampleIndex` from `seq`; it reads it
from the header") and **continuous across** a loop boundary, a profile change, and a master
failover — only an explicit operator **seek** (or a fresh media start) resets it. The send
loop's frame arithmetic (`sampleIndex(seq) = sampleIndex(0) + seq·FramesPerChunk`, 05 §5.1)
is re-based at each `Generation.FirstSampleIndex`; `seq` restarts at 0, `sampleIndex` does
not (except on seek/fresh-start).

**Tunables (A.12 — quoted verbatim; never invented here).** Buffer / playout lead
`LeadMs` = **300 ms**; canonical rate / channels / `FramesPerChunk` = **48000 / 2 / 480
(10 ms)**; Opus keyframe interval (05 §5.4.2) = **every 50 chunks (~500 ms)**. FEC span that
the reprime lead must contain: `D × FramesPerChunk / rate` = `4 × 10 ms` ≈ **40 ms** (A.12 FEC
XOR `k = 8`, `D = 4`). Reorder/dedupe window (05 §5.6.2) ≈ **32 packets** (must exceed
`D × k = 32`). `protocolEpoch = 1` (fixed this build).

## 6. Import constraints (doc 01 §2 layering)

`internal/stream/streamgen` (the pure decision package) MAY import: **stdlib only**
(`errors` at most). It must NOT import `wire`, `codec`, `fec`, `ring`, `group`, `clock`,
`state`, or `web` — it is leaf decision logic over plain `uint64`/`int64`/`bool`, so it is
trivially unit-testable and reusable by both origin and receiver without a cycle.

`internal/stream/origin/streamgen.go` (same package as P4.8 `Origin`) MAY import:
`internal/stream/streamgen`, `internal/stream/wire` (P4.1), `internal/stream/codec` (P4.3),
`internal/stream/fec` (P4.3/P4.5), and `internal/group` Timeline interface **only via the
field P4.8 already holds** (`clock.Timeline`/`group.Timeline`) — it must not import
`internal/group`'s engine or `internal/cluster`. It MUST NOT import `internal/web` or
`internal/audio/*` (the origin is master-side; it never touches a sink).

`internal/stream/sink_net/reprime.go` (same package as P4.9 `Receiver`) MAY import:
`internal/stream/streamgen`, `internal/stream/wire` (P4.1), `internal/stream/codec`,
`internal/stream/fec`, `internal/audio/ring` (P4.4), and `internal/allowlist` (only as P4.9
already does). It MUST NOT import `internal/group` engine, `internal/cluster`, or
`internal/web`. The `streamGen`/flush behaviors reach the group engine **only** through the
`Hooks.OriginResumeAt` / `Hooks.ReceiverFlushReprime` function-value seam (A.14.4, wired in
`cmd/ensemble`), mirroring the `web.Deps` discipline — `group` must not import `stream/*`,
and `stream/*` must not import `group`'s engine, so the graph stays acyclic.

## 9. Risks / open questions

1. **Package boundary of the wiring files.** `origin/streamgen.go` and
   `sink_net/reprime.go` are specified to live **inside** the P4.8/P4.9 packages (to reach
   unexported `gen`/`seq`/reorder-window/ring state). This requires P4.8/P4.9 to either
   leave those fields package-visible or expose minimal mutators. **Confirm with the
   orchestrator** that P5.3 may add files to those packages rather than forcing a wider
   public API; alternative is exported setters (`SetGen`, `ResetSeqFEC`, `FlushWindow`),
   which leak generation mechanics into the public surface. Recommend same-package files.
2. **Prime-lead "one buffer" exactness.** 05 §5.6.4/§5.7 says "accumulate the buffer lead"
   (A.12 `LeadMs = 300 ms`) before playout, but does not pin the *keyframe-anchored* prime
   threshold precisely (e.g. exactly `LeadMs`, or `LeadMs + FEC span ≈ 340 ms`). A.12 pins
   `LeadMs = 300 ms` and the FEC span ≈ 40 ms separately. **Proposed:** prime to `LeadMs`
   (300 ms) of contiguous keyframe-anchored audio, since the lead already contains the FEC
   span (§5.7 table). Needs orchestrator ratification (no new tunable invented).
3. **Profile-change `streamGen` ownership overlap.** 04 §4.3.3 (P3.2's territory) and 05 §5.8
   (this piece) both describe the profile-change bump. P5.3 owns the *mechanism*
   (`ReasonProfileChange` → bump + keyframe + continuous `sampleIndex`); P3.2 owns *deciding*
   the profile changed (`NegotiateProfile` result differs). Confirm the bump is triggered by
   cmd on P3.2's "profile changed" signal, not duplicated in both — flagged to avoid two
   writers.
4. **Opus keyframe reset hook.** Forcing an Opus keyframe (05 §5.4.2 / §5.6.4) requires the
   `codec.Codec` to expose an encoder-reset / force-keyframe entry. P4.3 ships PCM only;
   the Opus codec (deferred, A.11) must add a `ForceKeyframe()`/reset method behind the
   `Codec` interface or a narrow optional interface. P5.3 assumes such a hook exists for
   Opus; for PCM the keyframe flag is unconditional and needs no reset. **Confirm the codec
   keyframe-forcing contract** so origin late-join/bump can call it uniformly.
5. **`MasterMono` re-baseline on resume.** On `ResumeAt`/failover the new master's
   `masterMono` clock differs from the old master's; the first new-gen chunk's
   `(sampleIndex, masterMono)` re-anchors the FollowerTimeline line (04 §4.4.3 self-corrects).
   No action needed if `MasterTimeline.Seed(base, playing)` stamps `baseMono = NowMono()` as
   P3.2 specifies — noted to confirm the seed stamps the *new* master's mono.
