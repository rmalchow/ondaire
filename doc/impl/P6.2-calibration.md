## 1. Purpose & scope

P6.2 lands the **built-in, in-process calibration signal generator** (A.10b: per-1 s period =
~1 ms full-scale click + ~200 ms 1 kHz tone + silence, generated deterministically at the
canonical rate so every node emits a **bit-identical** waveform), and the **`POST /api/v1/calibrate/play`**
control path that plays that signal **synchronously on the shared group timeline** so all
selected nodes emit the same click at the same group sample instant (08 §F2.1, 06 §5.3). It
owns (a) the pure signal generator, (b) a `CalibratePlayer` that — on the nodes asked to play —
injects the generated frames into the local output path at a scheduled group sample so playout
is sample-aligned across nodes, and (c) the `cmd`-side fan-out implementation bound to
`web.Deps.CalibratePlay` (A.14.3) reaching peers over mTLS. It explicitly **does not** implement
`/calibrate/measure` (D21 / 08 §F2.2 — documented future enhancement only), does not own the
`HWDelayUs` trim application (that is the renderer, 06 §5.3 / P4.9), does not own the HTTP route
table or auth (P1.3/web owns routing; this piece supplies the `Deps` function value), and does
not generate any external clip.

## 4. Public API (canonical names — A.10b, A.14.3, 08 §F2.1, README §6.2)

### 4.1 Signal generator — `internal/audio/calib/signal.go`

```go
package calib

// SignalParams describes the built-in calibration waveform (A.10b). All durations are
// derived from the canonical Rate so the produced samples are deterministic and identical
// across nodes. The defaults below are the A.10b/A.12 values; do not invent alternatives.
type SignalParams struct {
    Rate     int     // canonical sample rate, Hz. A.12 = 48000.
    Channels int     // output channels. A.12 = 2 (the click/tone are mono, fanned to all channels).
    ClickMs  float64 // full-scale click length. A.10b "~1 ms".
    ToneMs   float64 // 1 kHz tone length. A.10b "~200 ms".
    ToneHz   float64 // tone frequency. A.10b "1 kHz".
    Amp      float32 // tone amplitude (the click is full-scale ±1.0 regardless). 0<Amp<=1.
}

// DefaultSignalParams returns the A.10b signal at the canonical rate (A.12):
// {Rate:48000, Channels:2, ClickMs:1, ToneMs:200, ToneHz:1000, Amp:0.5}.
func DefaultSignalParams() SignalParams

// Signal is one fully-precomputed 1-second period of the calibration waveform, interleaved
// float32 in [-1,1]. It is immutable after construction and safe for concurrent reads.
type Signal struct { /* unexported: period []float32; rate, channels int; periodFrames int */ }

// NewSignal builds the 1 s period from p (filling unset fields from DefaultSignalParams).
// periodFrames = Rate (exactly 1 s). Layout per period (A.10b), all fanned to every channel:
//   [0, clickFrames)             : a single full-scale impulse (+1.0 on the first sample,
//                                  then the remainder of the ~1 ms window at 0 — see §5.1)
//   [clickFrames, clickFrames+toneFrames) : sin(2π·ToneHz·t)·Amp
//   [clickFrames+toneFrames, Rate)        : silence (0)
func NewSignal(p SignalParams) *Signal

func (s *Signal) Rate() int
func (s *Signal) Channels() int
func (s *Signal) PeriodFrames() int           // == Rate (1 s)
func (s *Signal) Period() []float32            // the interleaved 1 s period (read-only)

// Fill writes `len(dst)` interleaved samples into dst starting at absolute frame index
// fromSample, wrapping the 1 s period modulo PeriodFrames. dst length must be a multiple of
// Channels. fromSample is the GROUP timeline sample so the same group instant maps to the
// same point in the period on every node (this is what makes the click coincident). Pure.
func (s *Signal) Fill(dst []float32, fromSample int64)
```

### 4.2 Local player — `internal/audio/calib/player.go`

```go
package calib

import (
    "context"
    "time"

    "gitlab.rand0m.me/ruben/go/ensemble/internal/audio/sink"   // AudioSink (P4.6)
    "gitlab.rand0m.me/ruben/go/ensemble/internal/group"        // Timeline (P3.2 / P4.9 dep)
)

// CalibratePlayer plays the built-in Signal on THIS node, sample-aligned to the group
// timeline. It is the local executor invoked by calibctl (directly on the receiving node,
// or via the proxied per-node handler on a remote node).
type CalibratePlayer struct { /* unexported: sig *Signal; sink sink.AudioSink; tl group.Timeline; mu; running */ }

// NewCalibratePlayer binds the (already-Started) sink and the group Timeline used for
// alignment. tl may be nil for a solo/no-timeline node, in which case Play aligns to the
// node's own immediate now (still deterministic from the period, just not cross-node-aligned).
func NewCalibratePlayer(sig *Signal, snk sink.AudioSink, tl group.Timeline) *CalibratePlayer

// Play emits the signal for durationSec, beginning at group sample `startSample`. It busy-
// fills the sink (sink.Write blocks = playout pacing, exactly the render-loop drain idiom)
// from `startSample` for durationSec*Rate frames, wrapping the period via Signal.Fill. It
// returns when the duration elapses, ctx is cancelled, or Stop() is called. Concurrent Play
// is rejected (ErrBusy). A node whose sink is nil (Render=false) returns ErrNoSink.
func (c *CalibratePlayer) Play(ctx context.Context, startSample int64, durationSec int) error

// Stop ends an in-flight Play early (idempotent).
func (c *CalibratePlayer) Stop()

var (
    ErrBusy   = errors.New("calibrate: already playing")
    ErrNoSink = errors.New("calibrate: node has no audio sink (Render=false)")
)
```

> **Render coexistence (06 §5.3 / §2).** Calibration is a *diagnostic* and is mutually
> exclusive with normal rendering on the same sink. The player does **not** mix with the render
> loop; `calibctl`/`cmd` pauses/ stops the group's render-drain for the calibration window and
> resumes after (see §5.3, Risk R1). For the MVP the simplest correct wiring is: the node is
> typically **not playing media** while calibrating (operator stops playback first); the player
> takes the sink exclusively for `durationSec`.

## 5. Behavior & algorithms

### 5.1 Signal generation (A.10b, A.12)

Quoting A.10b verbatim — per 1-second period at the canonical rate (A.12 **48000**):

> `[0, ~1 ms)` full-scale **click** (single full-amplitude sample / ~1 ms impulse) ·
> `[~1 ms, ~201 ms)` **1 kHz sine tone** (~200 ms) · `[~201 ms, 1 s)` **silence** ·
> identical across nodes (same generator, deterministic samples).

Concrete construction in `NewSignal`:

- `periodFrames = Rate` (1 s exactly).
- `clickFrames  = round(ClickMs * 1e-3 * Rate)` → at 48000/1 ms = **48 frames**.
- `toneFrames   = round(ToneMs * 1e-3 * Rate)` → at 48000/200 ms = **9600 frames**.
- **Click:** A.10b says "single full-amplitude sample / ~1 ms impulse" — implement as a single
  `+1.0` sample at frame 0 of the period (the sharp transient that gives the ear/phone-mic a
  crisp coincidence cue), with the rest of the `clickFrames` window at 0. (A single-sample
  impulse is the maximally sharp transient; the ~1 ms is the *reserved window* before the tone,
  not 1 ms of full-scale energy — that keeps it ear-safe and maximally localizable. See R2.)
- **Tone:** for frame `f` in `[clickFrames, clickFrames+toneFrames)`,
  `v = Amp * sin(2π * ToneHz * (f-clickFrames) / Rate)`. Phase starts at 0 at the tone onset so
  every node's tone is phase-identical.
- **Silence:** 0 for the remainder.
- **Channels:** the mono value `v` is written to **all** `Channels` (fan-out), so a stereo DAC
  plays the identical signal on both pins (mirrors the `"left"`/`"right"` mono-fan convention in
  06 §5.1 — but calibration always fans to all, irrespective of `NodeRecord.Channel`, so a
  left/right node still emits a full-energy click; see R4).

`Fill(dst, fromSample)` copies `len(dst)` interleaved samples by indexing the precomputed period
at `(fromSample + i) mod periodFrames` per frame `i` — `O(len(dst))`, no allocation, no per-call
`Sin` (the period is computed once in `NewSignal`). This is the hot path during playback and is
trivially cheap on the Pi-class target (one period = `48000*2*4 B` ≈ 375 KB precomputed once).

**Determinism / identical-across-nodes (A.10b core requirement):** the period depends only on
`SignalParams` (which is the same default on every node) — no seed, no time, no per-node state.
Two nodes calling `Fill(dst, S)` with the same group sample `S` produce **byte-identical** `dst`.
This is the property the test asserts and the property that makes the click coincident.

### 5.2 Cross-node synchronous playout (06 §5.3, 08 §F2.1, README §6.2/§6.4)

The phrase "synchronous on the group timeline" (08 §F2.1 / 06 §5.3) means: every selected node
emits the **same period offset at the same group sample instant**. Mechanism:

1. The controller (`calibctl`, on the receiving node) reads the group's current playout sample
   `S0 = NowSample()` for the target group (README §6.2; for a `nodeIds` selection that spans a
   group, use that group's Timeline; for nodes not in a common group, see R5).
2. It picks a **common future start** `startSample = S0 + leadFrames`, where `leadFrames` is one
   buffer lead so every node can be primed before the instant arrives — reuse the playout
   `LeadMs` (A.12 **300 ms** → `0.300 * Rate = 14400 frames @48k`). This is the same `LeadMs`
   the renderer uses (06 §2.1 / A.12); P6.2 quotes it, does not invent.
3. `startSample` is sent identically to every target (local + proxied).
4. Each node's `CalibratePlayer.Play(ctx, startSample, durationSec)`:
   - Waits until its **local** `Timeline.NowSample()` reaches `startSample` (the Timeline already
     folds in the clock offset, A.2 — so "local now == group sample" is cross-node aligned to the
     clock sync accuracy, sub-ms wired / few-ms WiFi per A.13 P3).
   - Then fills the sink from `Signal.Fill(buf, currentGroupSample)` each drain, advancing the
     group sample by the frames written, for `durationSec*Rate` frames.
   - Because `Fill` indexes the **group** sample (not a local counter), even a small priming
     skew self-corrects to the period position the group timeline dictates — the click lands on
     the period boundary the same group instant on every node.
5. `HWDelayUs` is **not** applied here (it is the very thing being calibrated; 06 §5.3). The
   operator judges the residual by ear and enters `HWDelayUs` via `PATCH /nodes/{id}` (D.3).

> A solo / not-yet-synced node (`Timeline==nil` or `NowSample` `ok=false`) plays the signal
> immediately from its own now — still a valid single-node click track, just not cross-node
> aligned (there is no group to align to). Reported normally in `playedOn`.

### 5.3 `/calibrate/play` request handling + fan-out (08 §F2.1)

**Validation (08 §F2.1, §0.4):**
- Exactly one of `groupId` / `nodeIds` must be set → else `400 invalid_request` ("neither/both").
- `durationSec` in a sane range (e.g. `1..120`) → else `400 invalid_request`.
- Unknown `groupId` / any unknown `nodeId` → `404 not_found` (resolved against `Deps.State()`).

**Target resolution:**
- `groupId` → `GroupRecord.MemberNodeIDs` (08 §F2.1).
- `nodeIds` → those ids (each must be a current `NodeRecord`).

**Fan-out (08 §F2.1 proxy = "fan-out"):** the receiving node drives synchronous playback across
the selected nodes over mTLS. Concretely:
- Compute the single `startSample` once (§5.2).
- For **self** (if a target): run `LocalPlayer(ctx, startSample, durationSec)`.
- For **each remote target**: `Proxy(ctx, addr, nodeID, startSample, durationSec)` in parallel —
  this hits the peer's own `POST /api/v1/calibrate/play` with `nodeIds:[thatNode]` **and** the
  shared `startSample` (carried as an internal field so the peer does **not** recompute it — all
  nodes must agree on the instant; see R6 on the wire field for the proxied start sample).
- A `Render=false` target → not fatal, added to `warnings[]` ("node X cannot play: Render=false")
  per 08 §F2.1 ("a `Render=false` node cannot play and is reported, not fatal").
- An unreachable target → `warnings[]` with a `502`-style note per-node; the call still returns
  `200` with the nodes that did play (08 §F2.1: "reported per-node where the master is
  reachable"). Only a wholly-failed selection (bad input, no target, group unsynced) is a
  top-level error.

**Response (08 §F2.1):** `200` `{ "playedOn":[ids], "durationSec":N, "warnings":[...] }`.
**If-Match:** not used — transient playback, no `ConfigDoc` write (08 §F2.1).

### 5.4 Tunables referenced (verbatim from A.12 / A.10b)

- **Rate / Channels = 48000 / 2** (A.12 "Audio canonical").
- **Tone = 1 kHz, ~200 ms; click ~1 ms; period 1 s** (A.10b, quoted verbatim).
- **Lead = `LeadMs` 300 ms** for the common-start lead (A.12 "Buffer playout lead LeadMs 300 ms").
- No other tunable is invented. `Amp` (tone amplitude, 0.5) and `durationSec` bounds (1..120) are
  **not** pinned in A.12 — see Risks R2/R7 (proposed, flagged for ratification).

## 6. Import constraints (doc 01 §2 layering)

- **`internal/audio/calib`** MAY import: `internal/audio/sink` (the `AudioSink` it writes to,
  P4.6), `internal/group` (the `Timeline` interface for alignment, P3.2/P4.9), and stdlib
  (`math`, `context`, `time`, `errors`, `sync`). It is a leaf-ish audio package and MUST NOT
  import `internal/web`, `internal/stream/*`, `internal/state`, `internal/cluster`, or
  `internal/calibctl` (the arrow points calibctl → calib via a seam, not the reverse).
- **`internal/calibctl`** MAY import: `internal/state` (read `ConfigDoc` for target resolution +
  peer addrs) and `internal/web` **only for the `CalibrateSel` type** (it implements the
  `Deps.CalibratePlay` signature). It MUST NOT import `internal/audio/*` or `internal/group`
  directly — it reaches the local player and the timeline through the `LocalPlayer` / `Proxy` /
  `NowSampleFunc` function-value seams, wired in `cmd` (mirrors A.14.3 / P3.2 `Hooks`
  discipline). It MUST NOT import `internal/pki` (the mTLS client is injected via `Proxy`).
  > If importing `internal/web` for `CalibrateSel` is judged a layering smell (calibctl is below
  > web), the cleaner option is to define `CalibrateSel` in a lower shared package (or have
  > calibctl take `groupID string, nodeIDs []string` and let the *handler* construct the call).
  > **Recommendation:** the handler passes scalars; `Deps.CalibratePlay(CalibrateSel, int)` keeps
  > `CalibrateSel` in `web` and calibctl accepts scalars, so calibctl does **not** import web.
  > See R3. (Adjust §4.3 signatures to scalars accordingly when wiring.)
- **`internal/web/api_calibrate.go`** MAY import: stdlib + the in-package `Deps`/`CalibrateSel`
  types. It reaches the engine **only** through `s.deps` (A.14.3) — it MUST NOT import
  `calibctl`, `group`, `audio/*`, `cluster`, or `stream/*` (doc 01 §2: web→engine only via Deps).
- **`cmd/ensemble`** wires it all: builds the `Signal`, constructs the per-group
  `CalibratePlayer`, binds `calibctl.Controller` with the local-play closure (picks `startSample`
  from the group Timeline, runs the player), the mTLS `Proxy` (from `pki`), and the `NowSampleFunc`
  (from `group`), then assigns `deps.CalibratePlay = controller.CalibratePlay`.

## 8. Dependencies

**Consumes (upstream pieces):**
- **P4.9 (`internal/audio/render` — the render loop)** — the named upstream dependency. P6.2
  consumes the **`group.Timeline`** contract P4.9 chases (README §6.2 `NowSample()`) for
  cross-node alignment (§5.2), and the **sink/render-coexistence** model (06 §2/§5.3): the
  calibration player takes the sink for `durationSec`, and `cmd` pauses the render drain for the
  window (R1). It also reuses the `LeadMs`/`Rate`/`Channels` canonical params P4.9 defines
  (`RendererParams`, 06 §2.1 / A.12) for the lead and the sink format. No compile-time symbol of
  P4.9's internals is imported by `calib` beyond the shared `group.Timeline` + `sink.AudioSink`
  interfaces.
- **P4.6 (`internal/audio/sink`)** — `sink.AudioSink` (`Start`/`Write`/`Delay`/`Close`); the
  player writes the signal to an already-`Open`ed+`Start`ed sink.
- **P3.2 (`internal/group`)** — `group.Timeline` interface for `NowSample()`.
- **P2.1 (`internal/state`)** — `ConfigDoc`/`GroupRecord`/`NodeRecord`/`Addrs` for target +
  peer-address resolution in `calibctl`.
- **P1.1 (`internal/pki`)** — supplies the mTLS client behind the `calibctl.Proxy` seam (wired
  in cmd; not imported by calibctl).
- **P0.2 (`internal/web`)** — the `Deps`/`CalibrateSel`/`Server` types + the route registration
  the handler plugs into.

**Exposes (for downstream pieces):**
- `calib.Signal` / `NewSignal` / `DefaultSignalParams` / `Fill` — the reusable deterministic
  generator; consumed by the future `/calibrate/measure` (08 §F2.2) as the **reference signal**
  to cross-correlate against (so the generator must stay stable). Also usable by a test-clip
  generator (`scripts/make-test-clip` could embed it) if desired.
- `calib.CalibratePlayer` — wired by `cmd` per node/group.
- `calibctl.Controller.CalibratePlay` (and `PlayDetailed`) — the implementation `cmd` binds to
  `web.Deps.CalibratePlay`; the UI (09 Node-detail calibration flow) drives it via the HTTP
  endpoint.
- `web.handleCalibratePlay` — the `POST /api/v1/calibrate/play` route (08 §F2.1).

## 9. Risks / open questions

1. **Render/calibration coexistence on one sink (needs P4.9 confirmation).** A node can only
   drive its sink from one writer. The spec (06 §5.3) implies the operator stops media before
   calibrating, but the exact handoff — does `CalibratePlayer` *pause* the render loop's drain,
   *take over* the open sink, then *resume*, or does it require `GroupRecord.Playing=false`
   first? — is **not** pinned. **Recommendation:** for the MVP, `calibctl`/`cmd` stops the render
   drain (or asserts `!Playing`) for the window and resumes after; P4.9 should expose a
   `Pause()/Resume()` (or the player borrows the same sink under a mutex). **Confirm the P4.9
   render loop exposes a pause/sink-handoff hook**; otherwise calibration and playback race the
   sink. Blocking-ish for wiring (not for this package's unit tests, which use a dedicated fake
   sink).
2. **Click shape + tone amplitude are under-specified by A.10b.** A.10b says "single
   full-amplitude sample / ~1 ms impulse" (ambiguous: one sample, or 1 ms of full-scale?) and
   does not pin the tone amplitude. This plan uses a **single +1.0 sample** (sharpest transient,
   ear-safest) and `Amp=0.5` for the tone. **Neither the click-width interpretation nor `Amp` is
   in A.12** — flag for ratification (proposed: single-sample click, `Amp=0.5`). A 1 ms full-scale
   square would be louder/harsher but still valid; orchestrator to confirm.
3. **`Deps.CalibratePlay` returns only `error`, but 08 §F2.1 needs `playedOn`/`warnings`.** The
   A.14.3 seam is `CalibratePlay func(sel CalibrateSel, durationSec int) error`, which cannot
   carry the per-node `playedOn`/`warnings` body. **Options:** (a) widen the `Deps` field to
   return `(CalibratePlayResult, error)` — a small A.14.3 amendment; (b) keep `error` and have the
   handler treat any non-error as "all selected played" with empty warnings (loses per-node
   reporting — contradicts 08 §F2.1 `warnings[]`). **Recommendation:** amend the seam to
   `func(CalibrateSel, int) (CalibratePlayResult, error)` (or add a sibling `Deps.CalibratePlayDetailed`).
   **Needs orchestrator decision** (affects P0.2 `Deps` shape). Until resolved, `calibctl` exposes
   both `CalibratePlay` (error-only, satisfies the current seam) and `PlayDetailed`.
4. **Calibration ignores `NodeRecord.Channel`.** A left/right node fans the click to all output
   channels (full energy) so it is audible for offset judgement, rather than honoring its
   stereo-pair role (06 §5.1). This is the right behavior for *ear* calibration but worth
   confirming against the 09 UI flow (an operator calibrating a `"left"` node still wants to hear
   it). **Recommendation:** always fan to all channels during calibration; documented in §5.1.
5. **`nodeIds` spanning multiple groups / ungrouped nodes.** §5.2 aligns to one group's Timeline.
   If `nodeIds` cross groups (or include solo nodes), there is no single shared timeline, so
   cross-node coincidence is undefined. **Recommendation:** when `nodeIds` are not all in one
   group, the controller still plays per-node from each node's own now (best-effort, no
   cross-group alignment) and adds an informational `warnings[]` note; cross-node-aligned
   calibration requires a `groupId` (or a single group's members). Confirm this matches the 09 UI
   expectation.
6. **Proxied `startSample` wire field.** The fan-out must send the *same* `startSample` to peers
   so they do not each recompute a different instant. The public 08 §F2.1 body has only
   `{groupId|nodeIds, durationSec}`. **Recommendation:** add an **internal/optional** body field
   (e.g. `startSampleHint int64`, omitted in the operator-facing call, set on the node→node
   proxied call) so the peer aligns to the controller's instant; document it as proxy-internal in
   08 (a tiny addendum). Without it, peers cannot agree on the instant and the clicks will not
   coincide. **Needs an 08 addendum.**
7. **`durationSec` bounds not in A.12.** Proposed `1..120 s`; not a pinned tunable. Flag for
   ratification (low risk; the UI will offer a small fixed set like 5/10/30 s).
8. **`/calibrate/measure` is explicitly out of scope** (D21 / 08 §F2.2) — the generator is built
   so the future measure path can cross-correlate against `calib.Signal`, but no measure code,
   route, or upload handling is in P6.2.
