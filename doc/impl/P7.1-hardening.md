## 1. Purpose & scope

This is the **P7 hardening piece**: it does not introduce a new subsystem but makes the already-built failover,
adoption, allowlist, and streaming machinery *robust and provably correct under adversarial conditions*, then
proves it with soak + chaos integration tests (A.13 P7). Concretely it (a) **polishes master failover** so the
audible gap is bounded and the new master resumes the timeline continuously (D22/R11, 01 §6d, 05 §5.8);
(b) implements the **TCP fallback audio transport** end-to-end (D2/05 §5.9) behind the existing
`origin`/`sink_net` seam; (c) makes **takeover-under-contention deterministic** (two controllers racing to
re-adopt the same node, or a node revoked mid-handshake — 03 §4/§5); (d) hardens **allowlist churn under
membership flap** so a forgotten/flapping node's realtime packets are dropped without dropping a live member's
(03 §6, A.8); (e) verifies **partition recovery** converges to one master per group on heal (02 §6.2/§6.3);
(f) verifies **revoked-cert rejection** at the mTLS verify hook (03 §5.4); and (g) supplies the **soak + chaos
harness** that runs a multi-node in-process cluster under induced loss/churn/partition and asserts sync holds
over a long run. The new production code is small and surgical; the bulk is tests and the TCP transport.

## 5. Behavior & algorithms (cite docs; A.12 verbatim)

### 5.1 Bounded-gap master failover (01 §6d, 02 §6.1, 05 §5.8, D22/R11)

On master death: SWIM marks the master `suspect`→`dead`; **detection time ≈ the suspicion window**, which from
`DefaultLANConfig()` is `SuspicionMult × log(N) × ProbeInterval` with `ProbeInterval` ≈ **1 s** (02 §3.4, A.14.2),
i.e. **≈ 4–5 s for small N**. `Changed()` fires; every surviving member re-runs the per-group election
(A.5): `master = (MasterHint in alive∩group) ? MasterHint : min(alive∩group)`; `Generation++` on change.
The audible gap is bounded by **suspicion window + clock re-baseline + buffer refill** (02 §6.1). This piece's job
is to make the *second and third* terms minimal and the resume continuous:

1. The new master **seeds its `MasterTimeline` from the last sample index it was rendering as a follower**
   (`SeedAndResume`, 01 §4.2 step 2 / 04 §4.4.4) — playout continues, no reset to 0.
2. It bumps `streamGen` (`origin.ResumeAt`, 05 §5.8) — **master failover ALWAYS bumps** (R11); there is no
   "continue same generation" path. `seq` resets to 0, FEC state resets, the first chunk is a **keyframe**.
3. It preserves the replicated `GroupRecord.Playing` flag (R4) and resumes at the current timeline `sampleIndex`.
4. Followers see the new `streamGen > r.gen` → **flush + re-prime ~one buffer** (`FlushAndReprime`, P4.8); the
   buffer refill is bounded by the lead `LeadMs` = **300 ms** (A.12). They re-point their `ClockSource` at the
   new master's `clock.Server` and re-lock (the clock plane is already allowlisted to the member, 02 §6.1).

The **generation fence** (A.14.4): a stale role goroutine from the deposed master is torn down before the new
role starts; any clock/audio packet it emits carries the old `generation`/`streamGen` and is dropped by receivers
(05 §5.8 `hdr.StreamGen < r.gen → DROP`) and by the allowlist if the node is gone. **Graceful** master leave
(`Leave()`) skips the suspicion wait → re-elect in ~1 gossip round, sub-second (02 §6.4).

### 5.2 TCP fallback transport (05 §5.9, D2)

Selected per group via `ConfigDoc.Groups[].Transport == "tcp"`. Same chunks, same `wire` packets (§6.4), same
`seq`/`sampleIndex`/`streamGen` — over a reliable stream:
- Origin opens **one TCP connection per listener** and writes **2-byte big-endian length-prefixed** marshaled
  `wire` packets back-to-back. TCP frames the stream; the prefix delimits packets (the header `payloadLen` is
  present but not relied on for framing) (05 §5.9).
- **FEC is forced `None`** (`FECID = 0`) — TCP retransmission already guarantees delivery, so XOR/dup is pure
  waste; `Protect` is identity in this mode (05 §5.9).
- The receiver runs the **same pipeline minus FEC and minus UDP reordering** (TCP delivers in order); it still
  keys the ring by `sampleIndex` and **still honors `streamGen`** (flush + re-prime on a newer gen) (05 §5.9).
- **Pacing still applies**: the origin writes each chunk paced to `playout(idx) − Lead` (`Lead = 300 ms`, A.12)
  so it does not flood the TCP socket and inflate the queue (05 §5.9). Cadence stays **100 packets/s/listener**
  (10 ms chunks, A.12 `FramesPerChunk=480`).
- **Tradeoff (documented, not a bug):** HOL blocking + retransmit can increase tail latency; the lead may need
  to grow. It is a fallback, not the default (05 §5.9). It is still **allowlisted by peer IP** (the TCP listener
  checks `RemoteAddr().IP` against `allowlist.Set` before accepting/serving, 03 §6.1 "and the TCP-fallback audio
  listener").

### 5.3 Takeover-under-contention determinism (03 §4, §2.7)

Takeover is adoption-with-supersede, still PIN-gated (03 §4). Two failure modes this piece nails down:
- **Two controllers race the same target.** `TakeoverGuard.Do(nodeID, …)` **single-flights** per target id:
  the first caller runs the A.9 handshake + `CA.Sign` + `If-Match` ConfigDoc write; concurrent callers for the
  same id block and then observe the committed result rather than signing a *second* competing leaf. If both
  somehow sign (cross-node, no shared guard), **LWW on the ConfigDoc resolves it** (A.6): higher `Version` wins,
  equal version → higher `UpdatedBy` node id; the superseded leaf's fingerprint is added to the grow-only
  `RevokedSet` (union-merged, never lost — A.6) so the loser's cert is rejected at the verify hook (03 §5.4).
- **Node revoked / epoch-mismatched mid-handshake.** `precheck` aborts **before signing** if the target's
  fingerprint is already in `RevokedSet` or its `protocolEpoch` differs (03 §2.7 "abort before signing";
  mixed-epoch clusters unsupported). The atomic identity swap on the adoptee (03 §4 "must not end up half in two
  clusters") means a failure before `complete` leaves it in its old cluster.

### 5.4 Allowlist churn under membership flap (03 §6, A.8)

`allowed = ∪ NodeRecord.Addrs (ConfigDoc) ∪ live-member addrs (membership)` (A.8, 03 §6.2). `Set.Update` swaps an
**atomic snapshot** on every `state.Changed()` or `cluster.Changed()` (P2.4 / 01 §3.4); the per-packet
`Allowed(ip)` is lock-free. Invariants this piece locks with tests:
- **Forget closes the IP and keeps it closed**: a forgotten node is in **neither** half (dropped from `Nodes[]`
  *and* gossip-rekeyed so it cannot re-announce to re-add its live IP, 03 §5.3) → its clock/audio packets drop.
- **Flap does not drop a live member**: a member that goes `suspect`→`alive` (transient packet loss) keeps its
  durable `NodeRecord.Addrs` half, so its IP stays allowed across the flap (the durable half survives even if the
  live half momentarily lacks it).
- **DHCP address change**: the live half (from `Members()`) admits the new IP before `NodeRecord.Addrs` is
  written back (01 §3.4 / 03 §6.2).
- **No torn reads**: `Update` is a single atomic pointer swap; a concurrent `Allowed` sees either the old or new
  complete set, never a partial union (race-tested with `-race`).

### 5.5 Partition recovery (02 §6.2 / §6.3)

Each side of a partition independently elects `min(alive∩group)` on its side → possibly **two masters for one
group** (accepted; no quorum, LWW not Raft — 02 §6.2). Each side keeps internally in sync; the two may drift apart
relative to each other. **On heal**: memberlist re-merges in ~1 gossip round (peer-cache + mDNS accelerate
rediscovery); each group's stateless election now sees the **union** of alive members and converges to the single
global `min` id; the losing-side master observes it is no longer lowest, **demotes to follower**, `Generation++`
(02 §6.3). The ConfigDoc reconciles via LWW (higher `Version`, id-tiebreak — A.6). Convergence is **structural and
automatic** (no human action): the moment both sides share `Members()`, they compute the same `min` (02 §6.3). The
deposed master's lingering realtime packets self-heal — receivers accept clock/audio only from the *currently
elected* master and the allowlist drops non-members (02 §6.3, 03 §6.4).

### 5.6 Revoked-cert rejection (03 §5.4, §5.2)

`PeerVerifier.Verify` runs on **both** client and server side after CA-chain validation: it rejects a peer whose
`SHA-256(leaf DER)` ∈ `ConfigDoc.RevokedSet` (live snapshot) and optionally cross-checks CN/URI-SAN against a
current `NodeRecord` (03 §5.4). Revocation = `RevokedSet` entry + 30-day cert expiry backstop (A.12 leaf lifetime
**30 days**, renew at **⅓ life / day 10**); no CRL/OCSP (03 §5.2). The set is **grow-only union-merged** so a
stale replica cannot resurrect a forgotten cert (A.6). This piece adds the tests that prove rejection fires on
both directions and that a `caPool`/`PeerVerifier` rebuilt on `ConfigDoc` change takes effect without restart
(03 §8 last paragraph).

### 5.7 Soak + chaos (A.13 P7)

A `test/soak` harness (build tag `soak`, opt-in — not in the default `go test ./...`) spins **N in-process nodes**
on loopback, forms a group, plays a deterministic synthetic source, and runs **chaos injectors** while sampling
each node's `Timeline.NowSample()` against the group reference:
- injectors: uniform + bursty **packet loss** (≤ D, with FEC; > D to force conceal), **master kill** (ungraceful)
  + graceful `Leave`, **partition**/heal, **allowlist flap** (suspect↔alive), **forget**, **CA-revoke**.
- assertions (per A.13 P7 "takeover supersedes old cert; 24 h soak holds sync"):
  - measured inter-node sync error stays **sub-ms** at steady state and after each failover (proxy for the 24 h
    soak; the long run is gated behind an env var / longer duration so CI can run a compressed version).
  - **failover gap bounded** by suspicion-window + 300 ms refill (§5.1); no permanent desync.
  - takeover **supersedes** the old cert (old fingerprint in `RevokedSet`, old leaf rejected — §5.6).
  - partition heal converges to **one master per group** (§5.5).
  - no goroutine/FD leak across thousands of failover cycles (leak check in `metrics.go`).

> CI note: the full **24 h** soak (A.13 P7) is a manual/scheduled job; the unit-and-chaos suite runs a compressed
> deterministic version (simulated clock where possible) so a regression is caught in seconds. The 24 h run on a
> Pi + DAC HAT is the real acceptance gate (and overlaps the ALSA precise-sink verification spike, A.11).

## 6. Import constraints (doc 01 §2 layering)

- **`internal/stream/origin` + `internal/stream/sink_net`** (TCP files): same constraints as P4.8 §6 — may import
  `stream/{codec,fec,wire,source}`, `internal/clock`, `internal/audio/ring`, `internal/allowlist` (receiver only),

  `cluster`, `pki`, `auth`, or any `audio/{render,sink,drift,resampler}`. No cgo/dlopen (D24).
- **`internal/group`** (`failover.go`, `contention.go`): `group` is the orchestrator below `cmd` (01 §2 rule 5).
  May import `clock`, `state`, `stream/*`, `audio/*`, `allowlist`, `config`, and (for the contention guard's
  precheck) read `pki`-provided verification via the hooks/closures it is given — but it is **not** imported by
  `web` (rule 1) and not by `state`/`clock` (leaves). The `OriginResumer` interface keeps the `group→origin` edge
  one-directional (group depends on the narrow interface, origin never imports group).
- **`internal/pki`, `internal/allowlist`, `internal/cluster`** test files: tests live in the same package (or
  `_test` package) and import only what production code already allows (`pki`→`config`; `allowlist`→`state`;
  `cluster`→`state,config,discovery`). They add **no** new import edges.
- **`test/soak`**: an external test package (`package soak_test`) under `test/` — it is allowed to import the full
  set (it is test-only wiring analogous to `cmd`), but it is **not** part of the production import graph and is
  excluded from the default build by the `soak` build tag.

| expiry backstop | a leaf past `NotAfter` (30 days, A.12) fails chain validation even if not in `RevokedSet`. |

### 7.7 Soak + chaos integration (`test/soak`, tag `soak`) — acceptance per A.13 P7

| Scenario | Asserts |
|---|---|
| steady-state sync | N-node group, no faults: inter-node sync error sub-ms over the run (A.13 P4/P7). |
| master kill → re-elect | kill master; new master (next-lowest id) resumes continuously; gap bounded (§5.1); followers re-lock; sync sub-ms after (A.13 P3/P7). |
| 2% + bursty loss | induced 2% uniform and ≤ D burst loss → no missing ring chunk (FEC-recovered or concealed), no mis-index (A.13 P5). |
| partition / heal | two masters during split; one master per group on heal; no permanent desync (02 §6). |
| forget + revoke | forgotten node's realtime packets dropped (allowlist) **and** mTLS rejected (RevokedSet); cannot rejoin (gossip rekey) (03 §5). |
| takeover supersede | re-adopted node's old cert in `RevokedSet`, old leaf rejected, new identity active (A.13 P7 "takeover supersedes old cert"). |
| churn cycles (leak) | thousands of failover/flap cycles → no goroutine/FD growth (`metrics.go` leak check). |
| (manual) 24 h soak | env-gated long run holds sync for 24 h (A.13 P7) — manual/scheduled; the compressed deterministic run is the CI proxy. |

## 9. Risks / open questions

1. **`GroupRecord.Transport` field shape.** This spec assumes P2.1 already carries `Transport string` ("udp"|"tcp")
   on `GroupRecord` (README §6.5 / 07 names it). **Blocking if P2.1's GroupRecord lacks the field** — the
   orchestrator must confirm P2.1 owns `Transport` (and `Playing bool`, R4) so this piece only *reads* them.
2. **TCP-listener allowlist hook ownership.** 03 §6.1 says the allowlist guards "the TCP-fallback audio listener,"
   but P2.4's `allowlist.GateUDP` is UDP-only. This piece adds a peer-IP check at TCP `Accept`; confirm the
   allowlist owner is OK with `sink_net` calling `Set.Allowed(remoteIP)` directly at accept time (vs. a new
   `GateTCP` helper in P2.4). Recommend the direct call to keep P2.4 unchanged.
3. **`lastSample` source on promotion.** `SeedAndResume` needs the last rendered/seen sample index of the
   demoting follower. P3.2's `FollowerTimeline.NowSample()` returns it, but if the follower had no lock
   (`ok=false`) at the instant of promotion there is no continuity anchor → it must fall back to the replicated
   `GroupRecord` position (or sample 0 if never played). Confirm with the P3.2 owner which source is authoritative
   on a cold-promotion (no prior lock).
4. **Compressed-soak fidelity (the mpvsync bug class).** A.4 warns the *test plant* must model reality (the
   mpvsync drift bug was a test that didn't). The compressed soak must drive a **faithful** clock/loss model
   (real `time` or a deterministic simulated clock that advances the DAC at the crystal rate), or it will pass
   while the field fails. The 24 h Pi+DAC run (A.13 P7) is the only true acceptance — flag as the gating risk.
5. **Suspicion-window tunability.** The ~4–5 s failover detection is a `DefaultLANConfig()` property (A.14.2), not
   an A.12 number this piece may change. If a tighter bounded gap is required, that is a P2.3 memberlist-config
   change, out of scope here — flagged, not owned.
