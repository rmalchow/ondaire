# S — skeleton & shared contracts

Source of truth: [docs/README.md](../README.md). This piece owns the build
scaffolding and the cross-piece contracts every other piece codes against.
After this piece lands, `go build ./...` and `go vet ./...` are green over a
tree of stubs, and no later piece may edit any file listed here.

Design rule for S: **contracts only, plus the two pieces of real machinery
that have no other natural owner** — the wire codec (`internal/stream/wire.go`)
and the UDP mux (`internal/stream/mux.go`) — because clock (F) and transport
(G) both depend on them and neither can own them. Everything else in S is a
small, dependency-free type or function plus its test.

---

the contracts are §1's `internal/contracts` plus the concrete types in
`internal/id`, `internal/stream/wire.go`, `internal/stream/mux.go`,
`internal/netx`. Stub list (one empty `package` file each, plus a
`cmd/ensemble/main.go` that prints version and exits 0):


Rationale for `internal/contracts`: the three interfaces (`Sink`, `Clock`,
`StateStore`) and the snapshot DTOs are referenced by **multiple** pieces
(E↔H, F↔E, C↔I↔H). Go style says "define interfaces where consumed", but here
they are consumed in three packages each and an import cycle results if any one
of them owns the type. A leaf package with zero imports beyond stdlib + id is
the simplest cycle-free home. It imports only `internal/id` and stdlib.

## 2. Concrete Go API

### 2.1 `internal/id`

16 random bytes, lowercase hex (32 chars). Group ID = XOR of member IDs.

```go
package id

import (
	"crypto/rand"
	"encoding/hex"
	"errors"
)

// ID is a node or group identifier: 16 bytes, rendered as 32 lowercase hex chars.
type ID [16]byte

// Zero is the all-zero ID (XOR identity; also the "no group" sentinel).
var Zero ID

// New returns a cryptographically-random ID. Panics only if crypto/rand fails.
func New() ID

// Parse decodes exactly 32 lowercase-or-uppercase hex chars into an ID.
func Parse(s string) (ID, error)

// MustParse panics on error; for tests and constants.
func MustParse(s string) ID

// String renders the ID as 32 lowercase hex chars.
func (i ID) String() string

// IsZero reports whether i == Zero.
func (i ID) IsZero() bool

// XOR returns the bytewise XOR of all arguments (commutative, associative).
// XOR() == Zero; XOR(x) == x; XOR(x,x) == Zero. This is the group-ID rule (§5).
func XOR(ids ...ID) ID

// MarshalText / UnmarshalText make ID a hex string in JSON and map keys.
func (i ID) MarshalText() ([]byte, error)
func (i *ID) UnmarshalText(b []byte) error

var ErrBadLength = errors.New("id: want 32 hex chars")
```

JSON note: implementing `encoding.TextMarshaler/TextUnmarshaler` (not
`json.Marshaler`) makes `ID` work as a hex string in struct fields **and** as a
`map[ID]…` key. The replicated doc and API DTOs rely on both.

### 2.2 `internal/stream/wire.go`

Canonical PCM (§8.1) and the common frame header (§8.4). **All multi-byte
header fields are big-endian.**

```go
package stream

import (
	"encoding/binary"
	"errors"
)

// Canonical PCM format — every frame on the wire is exactly this (§8.1).
const (
	SampleRate    = 48000 // Hz
	Channels      = 2     // stereo
	BytesPerSmpl  = 2     // s16le, per channel
	FrameDuration = 20    // ms
	FrameSamples  = 960   // samples per channel per frame (48000*20/1000)
	FrameBytes    = 3840  // FrameSamples * Channels * BytesPerSmpl
	FrameNanos    = int64(FrameDuration) * 1_000_000 // pts step, ns
)

// Packet types multiplexed on the STREAM_PORT UDP socket (§8.4).
const (
	TypeAudio    byte = 0x01 // audio frame:  header + PCM/Opus payload
	TypeFEC      byte = 0x02 // XOR parity:   header + parity payload
	TypeClockReq byte = 0x10 // clock request (F)
	TypeClockRsp byte = 0x11 // clock reply   (F)
)

// Magic byte starting every framed packet (audio/fec). Clock packets carry it too.
const Magic byte = 0xE5

// HeaderSize is the fixed on-wire size of Header, in bytes.
const HeaderSize = 24

// Header is the common frame header that precedes every payload.
//
// Byte layout (big-endian, offsets in bytes):
//
//	off size field        meaning
//	  0    1  Magic        0xE5, sanity / framing
//	  1    1  Type         packet type (TypeAudio | TypeFEC | TypeClock*)
//	  2    4  Gen          session generation (uint32); receivers drop stale gens
//	  6    8  Seq          frame sequence number (uint64), 0-based per session
//	 14    8  PTS          presentation timestamp, master-clock nanoseconds (int64)
//	 22    2  PayloadLen   payload byte count following the header (uint16)
//	 ----      total 24 bytes ----
type Header struct {
	Magic      byte
	Type       byte
	Gen        uint32
	Seq        uint64
	PTS        int64
	PayloadLen uint16
}

// Encode writes the 24-byte header into dst[:HeaderSize] (big-endian).
// Panics if len(dst) < HeaderSize. Returns HeaderSize for chaining.
func (h Header) Encode(dst []byte) int

// AppendFrame appends header+payload to dst and returns the grown slice.
// It sets PayloadLen = len(payload); caller need not pre-set it.
func (h Header) AppendFrame(dst, payload []byte) []byte

// Decode parses a 24-byte header from src. Does not validate Magic/Type;
// callers decide policy. Returns ErrShort if len(src) < HeaderSize.
func Decode(src []byte) (Header, error)

// DecodeFrame parses header + exactly PayloadLen payload bytes from a single
// datagram (or a TCP length-framed chunk). Returns the header and a sub-slice
// of buf aliasing the payload (copy if retained). ErrShort / ErrBadMagic on
// malformed input.
func DecodeFrame(buf []byte) (Header, []byte, error)

// XORInto computes parity for FEC: dst[i] ^= src[i] for i < len(src); shorter
// src is zero-padded (no-op past its end). Used by sender (parity build) and
// receiver (recovery). dst must be FrameBytes for audio payloads.
func XORInto(dst, src []byte)

var (
	ErrShort    = errors.New("wire: buffer shorter than header")
	ErrBadMagic = errors.New("wire: bad magic")
)
```

Encoding is `binary.BigEndian.PutUint32/64/16` at the offsets above. This is
the **load-bearing byte contract**: F (clock) and G (transport) serialize and
parse against these exact offsets; do not reorder fields.

Clock packets (`TypeClockReq/Rsp`) reuse `Header` for framing — Magic+Type are
set, and the clock piece (F) carries its t1/t2/t3 timestamps in the **payload**
(three int64 big-endian = 24 payload bytes), not in header fields. S only pins
that clock packets start with a valid `Header` whose `Type` selects them on the
mux; the payload layout is F's to define, documented here so F and the mux
agree: `payload = t1(8) | t2(8) | t3(8)`, big-endian, t2/t3 zero on a request.

### 2.3 `internal/stream/mux.go`

Owns the single STREAM_PORT **UDP** socket and fans inbound datagrams out to
handlers by `Type` (byte at offset 1). Audio TCP is a *separate* listener owned
by G (the UDP socket and TCP socket share the port number via netx, but are
distinct `*net.UDPConn` / `*net.TCPListener`).

```go
package stream

import (
	"log/slog"
	"net"
	"net/netip"
	"sync"
)

// Handler receives the full datagram (header+payload) and the sender's address.
// pkt aliases the read buffer and is only valid for the duration of the call;
// handlers that retain bytes must copy.
type Handler func(pkt []byte, from netip.AddrPort)

// Mux multiplexes the STREAM_PORT UDP socket by packet type (§8.4).
// One Mux per node. Handlers are registered before Run and run on the single
// read goroutine — they must not block (hand off to a channel if needed).
type Mux struct {
	conn     *net.UDPConn
	mu       sync.RWMutex          // guards handlers only
	handlers map[byte]Handler
	log      *slog.Logger
	done     chan struct{}
	wg       sync.WaitGroup
}

// NewMux wraps an already-bound UDP socket (from netx.BindTCPUDP).
func NewMux(conn *net.UDPConn, log *slog.Logger) *Mux

// Register installs the handler for a packet type, replacing any prior one.
// Safe to call before or after Run.
func (m *Mux) Register(typ byte, h Handler)

// WriteTo sends a pre-encoded packet to addr. Safe for concurrent use.
func (m *Mux) WriteTo(pkt []byte, addr netip.AddrPort) (int, error)

// LocalAddr returns the bound UDP address (for clock follower self-dial).
func (m *Mux) LocalAddr() netip.AddrPort

// Run starts the read loop (one goroutine). Non-blocking. Idempotent-safe to
// call once.
func (m *Mux) Run()

// Close stops the read loop and closes the socket. Safe to call once.
func (m *Mux) Close() error
```

Dispatch: read into a reusable buffer sized `64*1024`; if `n < HeaderSize` or
`pkt[0] != Magic`, drop+count; else look up `pkt[1]` and call the handler with
`pkt[:n]`. No allocation per packet on the read path except handler copies.

### 2.4 `internal/netx`

```go
package netx

import "net"

// BindTCPUDP implements bind-or-increment with all-or-nothing semantics (§2).
// It tries ports base, base+1, … base+tries-1. A port is accepted only if BOTH
// the TCP listener and the UDP socket bind on that exact number; on partial
// failure both are closed and the next port is tried. host is the bind address
// ("" / "0.0.0.0" for all interfaces). Returns the bound listeners and the
// port actually used.
func BindTCPUDP(host string, base, tries int) (
	tcpLn *net.TCPListener, udpConn *net.UDPConn, port int, err error)

// BindTCP is the HTTP-port variant: TCP only, same increment policy.
func BindTCP(host string, base, tries int) (*net.TCPListener, int, error)

// InterfaceCIDRs returns the node's own non-loopback, up interface addresses
// in CIDR notation (e.g. "192.168.1.17/24", "fd00::5/64") for the node record
// (§3.1). Link-local and down interfaces are skipped. Order is stable-ish but
// callers must not depend on it.
func InterfaceCIDRs() []string

// DefaultTries is the spec's 64-attempt cap (§2).
const DefaultTries = 64
```

### 2.5 `internal/contracts` — cross-piece interfaces & snapshot DTOs

These are the three interfaces named in IMPLEMENTATION.md S ("output backend,
frame sink, state store") plus the replicated-state snapshot the API layer
reads. Pinned here so E, F, H, I share one definition with no import cycle.

```go
package contracts

import (
	"context"
	"ensemble/internal/id"
)

// ---- Output backend (sink piece E owns the implementations) -----------------

// Backend is a PCM output device. The default build provides an exec-pipe
// backend (pw-play/pw-cat/aplay/paplay, §8.5) and a null backend (timed
// discard). A second implementation exists in v1, so this is an interface.
//
// Audio is raw canonical PCM: 48 kHz stereo s16le, written as 20 ms frames
// (stream.FrameBytes each). Write must consume the whole frame or error.
type Backend interface {
	// Write plays one canonical PCM frame. Blocks at most until buffered.
	Write(frame []byte) error
	// Close stops the device and releases resources.
	Close() error
}

// ---- Frame sink (playout; sink piece E owns it, group H feeds it) -----------

// Sink is what the receiver/group hands decoded frames to for scheduled
// playout. It is the consumer end of the pipeline on every member (§8.5).
// One Sink per active session; Reset on generation change.
type Sink interface {
	// Push enqueues a frame for playout. seq/pts/gen come from the wire header;
	// payload is canonical PCM (stream.FrameBytes). Non-blocking: late or
	// stale-gen frames are dropped+counted inside the sink, never block.
	Push(gen uint32, seq uint64, pts int64, payload []byte)
	// Reset arms the sink for a new session generation, discarding queued
	// frames from older generations and re-zeroing stats counters that are
	// per-session.
	Reset(gen uint32)
	// Stats returns a snapshot of playout counters for /api/status (§9.1).
	Stats() SinkStats
	// Close stops the playout loop and the underlying Backend.
	Close() error
}

// SinkStats is surfaced via /api/status and used by the e2e smoke test (K).
type SinkStats struct {
	Played   uint64 // frames written to the backend
	Silence  uint64 // silent frames inserted for gaps
	LateDrop uint64 // frames dropped for arriving past their deadline
	StaleGen uint64 // frames dropped for an old generation
	Synced   bool   // clock follower has a usable offset (gates playout)
}

// ---- Clock (clock piece F owns it; playout E + source H consume it) ---------

// Clock translates between local and master time. Playout schedules frame PTS
// (master-clock ns) against this. Until the follower has a sample, Synced is
// false and playout must not start (§7).
type Clock interface {
	// MasterNow returns the current master-clock time in nanoseconds and
	// whether the estimate is usable (>=1 good sample). When ok is false the
	// returned time is meaningless.
	MasterNow() (masterNanos int64, ok bool)
	// LocalToMaster / MasterToLocal convert a specific instant using the
	// current offset. ok mirrors MasterNow.
	MasterToLocal(masterNanos int64) (localNanos int64, ok bool)
	LocalToMaster(localNanos int64) (masterNanos int64, ok bool) // DECISIONS.md D10
}

// ---- Replicated state snapshot (cluster piece C produces; API I reads) ------

// StateStore is the read side of the replicated cluster doc (§4) that the API
// and group pieces query. The cluster piece (C) implements it; writes go
// through C's own setters (SetName/SetFollowing/…), NOT through this interface,
// so the read contract stays small and side-effect-free.
type StateStore interface {
	// Self returns this node's own ID.
	Self() id.ID
	// Snapshot returns an immutable, resolved view of the whole cluster:
	// node records + liveness + derived groups. Safe to read concurrently;
	// the returned value is a deep copy / read-only.
	Snapshot() Snapshot
	// Subscribe returns a channel signaled (non-blocking, coalesced) on every
	// state change, for the WebSocket push (§9.2).
	Subscribe() <-chan struct{}
}

// Snapshot is the resolved cluster view behind GET /api/cluster and the WS
// "cluster" event (§9.1/§9.2). It is plain data: JSON-serializable, no methods,
// no pointers into live state.
type Snapshot struct {
	Nodes  []NodeView  `json:"nodes"`
	Groups []GroupView `json:"groups"`
}

// NodeView is one node record resolved with liveness and observed addrs (§4, §3.1).
type NodeView struct {
	ID           id.ID             `json:"id"`
	Name         string            `json:"name"`
	Addrs        []string          `json:"addrs"`        // self-reported CIDRs
	HTTPPort     int               `json:"httpPort"`
	StreamPort   int               `json:"streamPort"`
	GossipPort   int               `json:"gossipPort"`
	Capabilities Capabilities      `json:"capabilities"`
	Following    id.ID             `json:"following"`    // Zero == solo master
	Observed     map[id.ID]Observed `json:"observed"`    // peerID -> last observation
	Alive        bool              `json:"alive"`        // from memberlist liveness
	LastSeenUnix int64             `json:"lastSeen"`
	Stale        bool              `json:"stale"`        // not updated recently (UI hint)
	UpdatedAt    int64             `json:"updatedAt"`
	Version      uint64            `json:"version"`
}

// Capabilities mirror §1.
type Capabilities struct {
	Playback bool     `json:"playback"`
	Codecs   []string `json:"codecs"`  // ["pcm"] (+ "opus" if built with it)
	Formats  []string `json:"formats"` // ["wav","mp3","flac"]
}

// Observed is one observed-address entry (§3.1).
type Observed struct {
	IP           string `json:"ip"`
	LastSeenUnix int64  `json:"lastSeen"`
}

// GroupView is one derived group (§5) with its name + playback status.
type GroupView struct {
	ID       id.ID       `json:"id"`     // XOR of member IDs
	Name     string      `json:"name"`   // from group-names map, "" if unnamed
	Master   id.ID       `json:"master"`
	Members  []id.ID     `json:"members"`
	Playback Playback    `json:"playback"`
	Settings GroupSettings `json:"settings"`
}

// Playback mirrors the replicated playback-status record (§4).
type Playback struct {
	State       string  `json:"state"`       // "idle" | "playing"
	File        string  `json:"file"`
	StartedUnix int64   `json:"startedAt"`
	PositionSec float64 `json:"positionSec"`
	Codec       string  `json:"codec"`       // "pcm" | "opus"
	Transport   string  `json:"transport"`   // "udp" | "tcp"
}

// GroupSettings mirrors the per-group settings record (§8.3/§8.4/§9.1).
type GroupSettings struct {
	Codec     string `json:"codec"`     // default "pcm"
	Transport string `json:"transport"` // default "udp"
	BufferMs  int    `json:"bufferMs"`  // default 150
}

// Defaults for group settings (single source of truth, §8.5).
const (
	DefaultCodec     = "pcm"
	DefaultTransport = "udp"
	DefaultBufferMs  = 150
	DefaultLeadMs    = 50  // source release lead over the clock (§8.2)
)

// FollowClient is the small HTTP client func the group piece (H) uses to drive
// takeover (§5.2): it calls POST /api/follow or /api/unfollow on a peer. The
// API piece (I) injects a concrete implementation. Defined here to avoid an
// H→I import cycle.
type FollowClient interface {
	Follow(ctx context.Context, peer id.ID, target id.ID) error
	Unfollow(ctx context.Context, peer id.ID) error
}
```

These DTOs are deliberately the **same shape** the SPA renders (§9, §10): the
API piece (I) serializes `Snapshot` almost verbatim for `GET /api/cluster` and
the WS `cluster` event, so J codes against this JSON. Group settings, playback,
and capabilities all live in the snapshot so the UI needs no extra round-trips.

## 3. Control flow, goroutines, locking

S is mostly leaf code; only the mux has a goroutine and shared state.

### `internal/stream/mux.go`
- **Startup**: `NewMux(conn, log)` (no goroutine). Pieces call `Register` for
  their types: F registers `0x10` (server) / receives `0x11` (follower), G
  registers `0x01` + `0x02`. Then `main` (K) calls `Run()` once.
- **Steady state**: one read goroutine loops `ReadFromUDPAddrPort` into a fixed
  64 KiB buffer; validates `Magic` + length; `RLock` the handler map; calls the
  matched `Handler` synchronously. Handlers are **non-blocking** by contract —
  clock answers inline (cheap), audio hands the packet to its receiver's
  channel. `WriteTo` calls `WriteToUDPAddrPort` directly (the `*net.UDPConn` is
  safe for concurrent writers); no lock needed for writes.
- **Locking**: one `sync.RWMutex` guarding the `handlers` map only. Read path
  takes `RLock`; `Register` takes `Lock`. The socket itself needs no mutex.
- **Shutdown**: `Close()` closes `done`, closes the `*net.UDPConn` (unblocking
  the read), and `wg.Wait()`s the read goroutine. Idempotent via the closed
  socket erroring out the read loop.

### `internal/netx`
- Pure synchronous functions, no goroutines, no shared state. `BindTCPUDP`
  loops up to `tries`; on each candidate it `net.ListenTCP` then
  `net.ListenUDP`; on UDP failure it closes the TCP listener and continues; on
  success returns both. All-or-nothing is local to the loop.

### `internal/id`, `internal/stream/wire.go`, `internal/contracts`
- Pure value types and functions; no goroutines, no locks, no shared mutable
  state. `id.New` uses `crypto/rand`.

The cross-piece **locking convention** S sets (enforced by review, not code):
every stateful component (cluster, group, sink, clock follower) keeps **one
mutex** guarding its own fields; components communicate across goroutine
boundaries by channels (mux→receiver, cluster→subscribers) or by calling each
other's already-locked methods — never by sharing a struct across two mutexes.

## 4. Edge cases & failure handling

- **Port exhaustion (§2)**: `BindTCPUDP` exhausts `tries` (default 64) → returns
  a wrapped error naming base port and range; `main` (K) treats it as fatal.
- **TCP-binds-UDP-doesn't (§2)**: partial bind must not leak the TCP listener.
  `BindTCPUDP` closes it before trying the next port; tested explicitly.
- **Same number for TCP+UDP (§2)**: the loop only advances together, so the two
  returned sockets always share a port number. A port where TCP binds but UDP
  is held by another process is skipped entirely (not split across numbers).
- **Short / malformed datagrams (§8.4)**: mux drops `n < HeaderSize` and bad
  magic without dispatch (counter). `Decode`/`DecodeFrame` return `ErrShort` /
  `ErrBadMagic` rather than panicking — receivers must tolerate garbage on an
  open UDP port.
- **PayloadLen vs datagram size**: `DecodeFrame` requires
  `len(buf) >= HeaderSize+PayloadLen`; a truncated frame is `ErrShort` and
  dropped (FEC may recover it, §8.4).
- **PTS sign / monotonic ns (§7, §8.2)**: `PTS` is `int64` ns and may exceed
  2^53; never round-trip through float. Master clock is monotonic-derived, so
  PTS is large but positive within a session; `int64` is deliberate.
- **Unknown packet type on the mux**: dropped silently (counter), not an error
  — forward-compat with the reserved Opus/codec space (§8.3).
- **Handler registered after Run**: allowed (RWMutex); a packet arriving in the
  gap before registration is dropped as unknown-type, which is acceptable at
  startup.
- **`XOR()` with no args / `XORInto` length mismatch**: `XOR()` returns `Zero`
  (identity, = a solo node's "no members beyond self" never happens since self
  is always a member; but the identity is correct). `XORInto` clamps to
  `len(src)` and zero-pads, matching FEC's padded-parity rule (§8.4).
- **ID case on parse (§1)**: `Parse` accepts upper/lower hex but `String`
  always emits lowercase, so map keys and the canonical node.json form are
  stable.
- **InterfaceCIDRs on a container/loopback-only host**: returns an empty slice,
  not an error; the node still functions via observed-IP reporting (§3.1).
- **go:embed of placeholder (§10, IMPLEMENTATION.md J/I)**: `web/dist/index.html`
  is committed so the embed always compiles even before the UI is built.
  The embed directive lives in `web/embed.go` (`package web`,
  `//go:embed all:dist`, exports `DistFS`) because `go:embed` cannot reference
  parent directories from `internal/api`; the API piece takes the FS via its
  config (DECISIONS.md D15). `.gitignore` excludes `web/dist/*` **except**
  that placeholder.

## 5. Test plan

`internal/id/id_test.go`
- `TestNewRandomDistinct` — two `New()` differ; not Zero.
- `TestStringParseRoundTrip` — `Parse(i.String()) == i` for random IDs.
- `TestParseRejectsBadLength` — wrong length / non-hex → `ErrBadLength`.
- `TestParseAcceptsUppercase` — uppercase hex parses; `String` re-emits lower.
- `TestXORIdentityAndInvolution` — `XOR()==Zero`, `XOR(x)==x`, `XOR(x,x)==Zero`.
- `TestXORCommutativeAssociative` — order-independent over a random member set.
- `TestJSONTextMarshal` — struct field and `map[ID]int` key round-trip via JSON.

`internal/stream/wire_test.go`
- `TestHeaderRoundTrip` — `Decode(h.Encode(buf)) == h` across field extremes.
- `TestHeaderBigEndianLayout` — hand-checked bytes at offsets 0,1,2,6,14,22.
- `TestHeaderSizeConstant` — `HeaderSize == 24`; `Encode` writes exactly 24.
- `TestEncodeShortBufferPanics` — `Encode` into a 23-byte buffer panics.
- `TestDecodeShortBuffer` — `< HeaderSize` → `ErrShort`.
- `TestDecodeFrameBadMagic` — wrong magic → `ErrBadMagic`.
- `TestDecodeFrameTruncatedPayload` — `len < HeaderSize+PayloadLen` → `ErrShort`.
- `TestAppendFrameSetsPayloadLen` — `AppendFrame` fills `PayloadLen`; round-trips.
- `TestXORIntoPadsShortSrc` — parity of unequal-length payloads zero-pads.
- `TestPCMConstants` — `FrameBytes == FrameSamples*Channels*2 == 3840`,
  `FrameNanos == 20e6`.

`internal/stream/mux_test.go`
- `TestMuxDispatchByType` — register `0x01`+`0x10`; loopback `WriteTo` each;
  correct handler receives correct bytes + `from`.
- `TestMuxDropsShortAndBadMagic` — short / bad-magic datagrams never dispatch.
